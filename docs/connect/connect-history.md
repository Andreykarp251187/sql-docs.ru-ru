# <a name="driver-history-for-microsoft-sql-server"></a>Журнал драйверов для Microsoft SQL Server

На этой странице описываются технологии подключения к данным с предысторией Майкрософт для подключения к SQL Server.

## <a name="odbc"></a>интерфейс ODBC

Есть три поколения Microsoft ODBC Driver for SQL Server. Первый драйвер ODBC "SQL Server" по-прежнему поставляется в составе [компонентов доступа к данным Windows](#microsoft-or-windows-data-access-components). Не рекомендуется использовать этот драйвер для новой разработки. Начиная с SQL Server 2005, [SQL Server Native Client](#sql-server-native-client) включает интерфейс ODBC и драйвер ODBC, поставляемый с SQL Server 2005 через SQL Server 2012. Не рекомендуется использовать этот драйвер для новой разработки. После SQL Server 2012 [Microsoft ODBC Driver for SQL Server](#microsoft-odbc-driver-for-sql-server) — это драйвер, который обновляется самыми последними серверными компонентами.

### <a name="sql-server-native-client"></a>собственный клиент SQL Server

SQL Server Native Client — это изолированная Библиотека, которая используется как для OLE DB, так и для ODBC. SQL Server Native Client (часто сокращение SNAC) включалось в SQL Server 2005 до 2012. SQL Server Native Client можно использовать для приложений, которые должны использовать преимущества новых функций, появившихся в SQL Server 2005, до SQL Server 2012. (Компоненты доступа к данным Microsoft/Windows не обновляются для этих новых функций в SQL Server.) Для новых функций, которые выходят за пределы SQL Server 2012, SQL Server Native Client обновляться не будет. Переключитесь на Microsoft ODBC Driver for SQL Server или драйвер Microsoft OLE DB для SQL Server, если хотите воспользоваться новыми возможностями SQL Server.

Полную документацию по SQL Server Native Client см. в [документации по SQL Server Native Client](../relational-databases/native-client/sql-server-native-client-programming.md).

### <a name="microsoft-odbc-driver-for-sql-server"></a>Microsoft ODBC Driver for SQL Server

После SQL Server 2012 основной драйвер ODBC для SQL Server разрабатывается и выпускается как Microsoft ODBC Driver for SQL Server. Дополнительные сведения см. в [документации по Microsoft ODBC Driver for SQL Server](./odbc/microsoft-odbc-driver-for-sql-server.md).

## <a name="ole-db"></a>OLE DB

Существует три поколения поставщиков Microsoft OLE DB для SQL Server. Microsoft OLE DB Provider for SQL Server (SQLOLEDB) по-прежнему поставляется в составе [компонентов доступа к данным Windows](#microsoft-or-windows-data-access-components). Этот поставщик не будет обновлен с помощью новых функций, и его не рекомендуется использовать для новой разработки. Начиная с SQL Server 2005, [SQL Server Native Client](#sql-server-native-client) включает в себя интерфейс OLE DB поставщика (SQLNCLI) и является поставщиком OLE DB, который поставляется с SQL Server 2005 через SQL Server 2017. [С 2011 года он считается устаревшими](https://blogs.msdn.microsoft.com/sqlnativeclient/2011/08/29/microsoft-is-aligning-with-odbc-for-native-relational-data-access/) и мы не рекомендуем использовать этот драйвер для разработки новых приложений. В 2017 OLE DB технология доступа к данным была впоследствии нерекомендуемой, [а новая запланированная версия была объявлена](https://blogs.msdn.microsoft.com/sqlnativeclient/2017/10/06/announcing-the-new-release-of-ole-db-driver-for-sql-server/) для 2018. Новый поставщик OLE DB называется "драйвер Microsoft OLE DB для SQL Server" (МСОЛЕДБСКЛ) и поддерживается в настоящее время.

## <a name="adonet"></a>ADO.NET

ADO.NET был впервые появился в составе платформы Microsoft .NET, и его поддержка будет улучшена и поддерживаться. Это основной компонент платформы Microsoft .NET. Дополнительные сведения см. в [статье Microsoft ADO.NET for SQL Server](ado-net/microsoft-ado-net-for-sql-server.md).

## <a name="jdbc"></a>JDBC

### <a name="microsoft-jdbc-driver-for-sql-server"></a>Драйвер Microsoft JDBC для SQL Server

Выпущенная в 2000, драйвер Microsoft JDBC для SQL Server по мере необходимости будет улучшен и поддерживаться. Он был открыт с открытым кодом в 2016. Последние сведения о том, как загрузить драйвер, см. в статье [Обзор драйвера JDBC](./jdbc/overview-of-the-jdbc-driver.md).

## <a name="php"></a>PHP

### <a name="microsoft-drivers-for-php-for-sql-server"></a>Драйверы Microsoft SQL Server для PHP

В 2009 в качестве проекта с открытым исходным кодом драйверы Майкрософт для PHP для SQL Server продолжат улучшаться и поддерживаться. Последние сведения, в том числе о загрузке драйвера PHP, см. в [статье драйверы Майкрософт для PHP для SQL Server](./php/microsoft-php-driver-for-sql-server.md).

## <a name="nodejs"></a>Node.js

### <a name="microsoft-driver-for-nodejs-for-sql-server"></a>Драйвер Microsoft для Node. js для SQL Server

Драйвер Майкрософт для Node. js для SQL Server позволяет приложениям Node. js в Microsoft Windows и Microsoft Azure получать доступ к Microsoft SQL Server и База данных SQL Microsoft Azure. Процесс разработки больше не фокусируется на этом драйвере. Не рекомендуется создавать новые приложения с помощью драйвера Майкрософт для Node. js для SQL Server.

Дополнительные сведения о драйвере Microsoft для Node. js для SQL Server см. в разделе [WindowsAzure/node-SQLServer](https://github.com/Azure/node-sqlserver).

### <a name="tedious"></a>Усилий

В настоящее время корпорация Майкрософт предоставляет и поддерживает трудоемкий модуль с открытым кодом в Node. js для подключения к SQL Server с помощью JavaScript. Дополнительные сведения см. в статье [драйвер Node. js для SQL Server](./node-js/node-js-driver-for-sql-server.md).

## <a name="microsoft-or-windows-data-access-components"></a>Компоненты доступа к данным Microsoft или Windows

Компоненты доступа к данным Microsoft/Windows (MDAC/WDAC) поставляются вместе и поддерживаются в Windows для обеспечения обратной совместимости приложений и не входят в текущий стек технологий SQL Server. Новые функции не будут добавляться в компоненты MDAC/WDAC, и их не рекомендуется использовать для разработки новых приложений.

В рамках этого документа стек MDAC/WDAC можно разделить на следующие компоненты на основе технологий и продуктов:

* **ADO** (включая ADOMD и ADOX)
* **OLE DB** (включая OLE DB Core Services, поставщик SQL Server OLE DB, поставщик OLE DB Oracle, поставщик OLE DB для драйверов ODBC, поставщик данных формы и удаленный поставщик данных).
* **ODBC** (включая диспетчер драйверов ODBC, драйвер ODBC для SQL и драйвер Oracle ODBC)

### <a name="mdacwdac-components"></a>Компоненты MDAC/WDAC

MDAC/WDAC включает следующие компоненты:

* **ODBC:** Интерфейс Microsoft Open Database Connectivity (ODBC) — это интерфейс языка программирования C, позволяющий приложениям получать доступ к данным из различных систем управления базами данных (СУБД). Приложения, использующие этот API, ограничены только доступом к реляционным источникам данных.
* **OLE DB:** OLE DB — это набор COM-интерфейсы для доступа к данным в различных хранилищ данных. Поставщики OLE DB существуют для доступа к данным в базах данных, файловых системах, хранилищах сообщений, службах каталогов, рабочих процессов и хранилищах документов.
* **ADO:** Объекты данных ActiveX (ADO) предоставляет модель программирования высокого уровня. Несмотря на то, что написание кода для непосредственного OLE DB или ODBC напрямую не требуется, ADO легко освоить и использовать. Его можно использовать на языках сценариев, таких как Microsoft Visual Basic Scripting Edition (VBScript) или Microsoft JScript.
* **ADOMD:** ADO многомерных данных (ADOMD) будет использоваться с поставщиками многомерных данных, такими как поставщик Microsoft OLAP, также известный как поставщик Microsoft Analysis Services. С момента выпуска MDAC 2,0 в него не были внесены существенные улучшения.
* **ADOX:** Расширения ADO для DDL и Security (ADOX) позволяют создавать и изменять определения базы данных, таблицы, индекса или хранимой процедуры. ADOX можно использовать с любым поставщиком. Поставщик OLE DB Microsoft Jet обеспечивает полную поддержку ADOX, а поставщик Microsoft SQL Server OLE DB предоставляет ограниченную поддержку.
* **Microsoft SQL Server сетевые библиотеки:** SQL Server сетевые библиотеки позволяют SQLOLEDB и SQLODBC взаимодействовать с базой данных SQL Server. Следующие SQL Server сетевые библиотеки не рекомендуются в выпусках MDAC/WDAC: Banyan Vines, AppleTalk, Сервернет, IPX/SPX, Гиганет и RPC. TCP/IP и именованные каналы будут по-прежнему поддерживаться и доступны в 64-разрядной операционной системе Windows.
* **MSDASQL:** Поставщик Microsoft OLE DB для ODBC (MSDASQL) позволяет приложениям, созданным в OLE DB и ADO (который внутренне используется OLEDB) обращаться к источникам данных посредством драйвера ODBC. MSDASQL — это поставщик OLEDB, который подключается к ODBC, а не к базе данных. Он предназначен как мост от OLE DB к драйверу ODBC, если для источника данных не существует прямого OLE DB поставщика. MSDASQL поставляется с операционной системой Windows, а Windows Server 2008 и Vista SP1 были первыми выпусками Windows, включающими в себя 64-разрядную версию технологии.

### <a name="deprecated-mdacwdac-components"></a>Устаревшие компоненты MDAC или WDAC

Эти компоненты по-прежнему поддерживаются в текущем выпуске MDAC/WDAC, но в будущих выпусках они могут быть удалены. При разработке новых приложений корпорация Майкрософт рекомендует избегать использования этих компонентов. Кроме того, при обновлении или изменении существующих приложений удалите все зависимости от этих компонентов.

* **SQLOLEDB:** Поставщик Microsoft OLE DB для SQL Server (SQLOLEDB), которая поддерживает доступ к Microsoft SQL Server, является устаревшим. Возможность подключения к будущим версиям SQL Server может не поддерживаться. Возможность подключения к версиям, предшествующим SQL Server 7, будет удалена из операционной системы после Windows 7. Новые приложения должны использовать драйвер Microsoft OLE DB для SQL Server (МСОЛЕДБСКЛ), который поддерживает новые функции SQL Server. Существующие приложения должны переноситься на драйвер Microsoft OLE DB для SQL Server, чтобы повысить производительность, надежность и поддержку. Дополнительные сведения см. [в разделе Обновление приложения до OLE DB драйвера для SQL Server из MDAC](oledb/applications/updating-an-application-to-oledb-driver-for-sql-server-from-mdac.md).
* **SQLODBC:** Драйвер Microsoft SQL Server ODBC (SQLODBC), который поддерживает доступ к Microsoft SQL Server, является устаревшим. Возможность подключения к будущим версиям SQL Server может не поддерживаться. Возможность подключения к версиям, предшествующим SQL Server 7, будет удалена из операционной системы после Windows 7. Новые приложения должны использовать Microsoft ODBC Driver for SQL Server в Windows, который поддерживает новые функции SQL Server. Существующие приложения должны переноситься на Microsoft ODBC Driver for SQL Server, чтобы повысить производительность, надежность и поддержку. Соответствующие сведения см. [в разделе Обновление приложения для SQL Server Native Client из MDAC](../relational-databases/native-client/applications/updating-an-application-to-sql-server-native-client-from-mdac.md).
* **Microsoft Jet ядро СУБД 4,0:** Начиная с версии 2,6, MDAC больше не содержит компонентов Jet. Другими словами, MDAC 2,6, 2,7 и 2,8 не содержат Microsoft Jet, поставщика OLE DB Microsoft Jet, драйверов базы данных ODBC для настольных систем или объектов Jet Data Access Objects (DAO). 

  Отсутствует 64-разрядная версия ядро СУБД Jet, драйвер Jet OLEDB, драйверы Jet ODBC или Jet DAO. Дополнительные сведения см. в [статье базы знаний 957570](https://support.microsoft.com/kb/957570). В 64-разрядных версиях Windows версия 32-bit Jet выполняется в подсистеме Windows WOW64. Дополнительные сведения о WOW64 см. в [документации MSDN WOW64](/windows/desktop/WinProg64/wow64-implementation-details). Собственные 64-разрядные приложения не могут взаимодействовать с 32-разрядными драйверами Jet, работающими в эмуляторе WOW64.

  Вместо Microsoft Jet Корпорация Майкрософт рекомендует использовать [выпуск Microsoft SQL Server Express Edition](https://www.microsoft.com/sql-server/sql-server-editions-express) при разработке новых приложений, не использующих Microsoft Access, которым необходимо реляционное хранилище данных. Эти новые или преобразованные приложения Jet могут продолжать использовать Jet с целью использования файлов Microsoft Office 2003 и более ранних версий (MDB и XLS) для неосновного хранилища данных. Однако для этих приложений следует запланировать переход с Jet на ядро СУБД Microsoft Access. Можно [скачать ядро СУБД Microsoft Access](https://www.microsoft.com/download/details.aspx?id=54920), который позволяет выполнять чтение и запись существующих файлов в файлах Office 2003 (MDB и XLS) или в форматах файлов Office 2007 (*. ACCDB, *. xlsm, *. xlsx и *. xlsb).

  > [!IMPORTANT]
  > Ознакомьтесь с лицензионным соглашением для конечных пользователей системы Office 2007 на предмет особых ограничений использования.

  > [!NOTE]
  > SQL Server приложения также могут получить доступ к системе Office 2007 и более ранним версиям, а также к файлам, SQL Server разнородным возможностям подключения к данным и службам интеграции, с помощью драйвера системы Office 2007. Кроме того, 64-разрядные SQL Server приложения могут получать доступ к 32-разрядным файлам Jet и 2007, используя 32-bit SQL Server Integration Services (SSIS) в 64-разрядной системе Windows.

* **Мсдадс:** С помощью поставщика OLE DB Майкрософт для формирования данных (МСДАДС) можно создавать иерархические связи между ключами, полями или наборами строк в приложении. Начиная с MDAC 2,1, усовершенствования основных компонентов не были внесены. Этот поставщик является устаревшим. Корпорация Майкрософт рекомендует использовать XML вместо МСДАДС.
* **Oracle ODBC и oracle OLE DB:** Драйвер Microsoft Oracle ODBC (Oracle ODBC) и поставщик OLE DB для Oracle (Майкрософт) (Oracle OLE DB) предоставляют доступ к серверам базы данных Oracle. Они создаются с помощью интерфейса Oracle (OCI) версии 7 и обеспечивают полную поддержку Oracle 7. Кроме того, он использует эмуляцию Oracle 7, чтобы предоставить ограниченную поддержку для баз данных Oracle 8. Oracle больше не поддерживает приложения, использующие вызовы OCI версии 7. Эти технологии являются устаревшими. При использовании источников данных Oracle следует выполнить миграцию на предоставляемый Oracle драйвер и поставщик.
* **RDS:** Remote Data Services (RDS) — это собственный механизм Microsoft, для доступа к удаленной объекты ADO Recordset через Интернет или интрасеть. RDS устарела; в RDS 2,1 не были внесены существенные улучшения в компоненты. Корпорация Майкрософт выпустила .NET Framework, которая обладает широкими возможностями SOAP и заменяет компоненты RDS. После Windows 7 все компоненты сервера RDS будут удалены из операционной системы.
* **JRO:** Объекты репликации Jet (JRO) являются устаревшими. JRO используется в ADO с базами данных Jet *(. mdb) для создания и сжатия баз данных Jet (. mdb) и выполнения управления репликацией Jet. MDAC 2,7 будет иметь последний выпуск. JRO не будет доступен в 64-разрядной операционной системе Windows. JRO не поддерживается в формате файлов Microsoft Access 2007 (* . accdb).
* **Поддержка 16-разрядных ODBC:** При использовании 16-разрядных приложений следует выполнить миграцию в 32-разрядное приложение. 16-разрядные функции являются устаревшими и удаляются из 64-разрядных операционных систем. Дополнительные сведения см. в [статье базы знаний 896458](https://support.microsoft.com/kb/896458).
* **Простой поставщик OLEDB (мсдаосп):** Простой поставщик OLEDB предлагает платформу для быстрого создания OLE DB поставщиков по сравнению с простыми данными. МСДАОСП является устаревшим.
* **Библиотека курсоров ODBC:** Библиотека курсоров ODBC (ODBCCR32. dll) предоставляет ограниченные курсоры данных на стороне клиента. Библиотека курсоров ODBC устарела; приложение может использовать реализации курсоров на стороне сервера в качестве замены.
* **OLE DB необработанного удаленного взаимодействия интерфейса:** Удаленное взаимодействие с интерфейсом OLEDB (мсдапс. dll) попыталось разрешить поставщикам OLE DB невыполнение процессов. Удаленное взаимодействие OLEDB необработанного интерфейса не рекомендуется.
* **Сетевые библиотеки SQL для сетей AppleTalk и Banyan Vines:** Сетевые библиотеки Banyan Vines, AppleTalk, Сервернет, IPX/SPX, Гиганет и RPC SQL являются устаревшими. Если вы используете любую из этих технологий, следует изменить приложения так, чтобы они использовали одну из других сетевых библиотек, например TCP/IP и именованный канал.

### <a name="mdacwdac-releases"></a>Выпуски MDAC/WDAC

Ниже приведен список сценариев поддержки для прошлых выпусков MDAC/WDAC, начиная с самого раннего.

* **Mdac 1,5, mdac 2,0 и mdac 2,1:** Эти версии MDAC были независимыми выпусками, которые были выпущены с помощью пакета дополнительных параметров Microsoft Windows NT, пакета SDK для платформы Microsoft Windows или веб-узла MDAC. Эти версии MDAC больше не поддерживаются.
* **MDAC 2.5:** Эта версия MDAC вошло в состав операционной системы Windows 2000. Пакеты обновления MDAC 2,5 были добавлены в соответствующие пакеты обновления для Windows 2000.
* **Компоненты MDAC 2.6:** MDAC 2.6 RTM, SP1 и SP2 входили в состав Microsoft SQL Server 2000 RTM, SP1 и SP2, соответственно. Кроме того, эти пакеты обновления MDAC были выпущены на веб-узле MDAC в соответствии с расписанием выпуска пакета Service-Pack Microsoft SQL Server 2000. Эту версию MDAC и ее пакеты обновления можно установить на платформах Windows 2000, Windows Millennium Edition, Windows NT, Windows 95 и Windows 98. Эта версия MDAC больше не поддерживается.
* **MDAC 2.7:** Эта версия MDAC вошло в состав операционных систем Microsoft Windows XP RTM и SP1. Эту версию MDAC и ее пакеты обновления можно установить на платформах Windows 2000, Windows Millennium, Windows NT и Windows 98. Эту версию можно установить на платформе Windows XP только с помощью операционной системы или ее пакетов служб. Эта версия MDAC больше не поддерживается.
* **MDAC 2.8:** Эта версия MDAC был вместе с Windows Server 2003 и Windows XP SP2 и более поздних версий. Вы также можете установить эту версию MDAC и ее пакеты обновления в Windows 2000.

  * 32-разрядная версия MDAC 2,8 также была выпущена на веб-узле MDAC в то же время, когда Windows Server 2003 была выпущена для клиента.
  * 64-разрядная версия MDAC 2,8 была выпущена с 64-разрядной версией Windows Server 2003 и Windows XP.

* **Компоненты доступа к данным Windows (WDAC):** Имя MDAC изменилось на WDAC-"компоненты доступа к данным Windows", начиная с Windows Vista и Windows Server 2008. WDAC входит в состав операционной системы и недоступен отдельно для повторного распространения. Удобство обслуживания для WDAC регулируется жизненным циклом операционной системы.

  32-разрядная и 64-разрядная версии WDAC выпускаются с 32-разрядными и 64-разрядными версиями операционных систем Windows соответственно.

## <a name="obsolete-data-access-technologies"></a>Устаревшие технологии доступа к данным

Устаревшие технологии — это технологии, которые не были расширены или обновлены в нескольких выпусках продукта и будут исключены из будущих выпусков продукта. Не используйте эти технологии при написании новых приложений. При изменении существующих приложений, написанных с помощью этих технологий, рассмотрите возможность переноса этих приложений в ADO.NET или другую текущую технологию.

Следующие компоненты считаются устаревшими:

* **DB-Library:** DB-Library — это модель программирования связанные с SQL Server, который включает API C. С момента SQL Server 6,5 в библиотеку DB-Library не было улучшений функций. Его окончательный выпуск — SQL Server 2000, который не будет перенесен в 64-разрядную операционную систему Windows.
* **Внедренный SQL (E-SQL):** E-SQL — это SQL Serverная модель программирования, которая позволяет внедрять инструкции Transact-SQL в код Visual C. Начиная с SQL Server 6,5, в E-SQL не были внесены улучшения в функции. Его окончательный выпуск — SQL Server 2000, который не будет перенесен в 64-разрядную операционную систему Windows.
* **Объекты доступа к данным (DAO):** DAO предоставляет доступ к базам данных JET (Access). Этот API можно использовать на языках Microsoft Visual Basic, Microsoft Visual C++, а также для языков сценариев. Он входит в состав Microsoft Office 2000 и Office XP. DAO 3,6 — это окончательная версия этой технологии. Он не будет доступен в 64-разрядной операционной системе Windows.
* **Удаленные объекты данных (RDO):** RDO разрабатывался специально для доступа к удаленным реляционным источникам данных ODBC и упрощает использование ODBC без сложного кода приложения. Она была включена в состав Microsoft Visual Basic версий 4, 5 и 6. Технология RDO версии 2,0 была окончательной версией этой технологии.
