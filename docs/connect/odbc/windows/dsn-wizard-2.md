# <a name="data-source-wizard-screen-2"></a>Экран 2 мастера источников данных

Укажите метод проверки подлинности, настройте записи расширенного клиента Microsoft SQL Server, а также имя входа и пароль, которые драйвер ODBC будет использовать для подключения к SQL Server при настройке источника данных.

## <a name="options"></a>Параметры

### <a name="with-integrated-windows-authentication"></a>Со встроенной проверкой подлинности Windows

Указывает, что драйвер запрашивает безопасное (или надежное) подключение к SQL Server. Если выбран данный режим, при установке соединений с использованием данного источника данных SQL Server будет использовать встроенную систему безопасности входа в систему, вне зависимости от того, какой режим безопасности установлен на сервере. Не учитываются любые предоставляемые идентификаторы входа и пароли. Системный администратор SQL Server должен связать имя входа Windows с SQL Serverным ИДЕНТИФИКАТОРом входа (например, с помощью SQL Server Management Studio).

Дополнительно можно указать для сервера имя участника-службы.

### <a name="with-active-directory-integrated-authentication"></a>Со встроенной проверкой подлинности Active Directory

Указывает, что драйвер проверяет подлинность для SQL Server с помощью Azure Active Directory. Если выбран данный режим, то при установке подключений с использованием данного источника данных SQL Server будет использовать встроенную систему безопасности входа в систему Azure Active Directory, вне зависимости от того, какой режим безопасности установлен на сервере.

### <a name="with-sql-server-authentication"></a>С проверкой подлинности SQL Server

Указывает, что драйвер проверяет подлинность для SQL Server с использованием идентификатора входа и пароля.

### <a name="with-active-directory-password-authentication"></a>С проверкой подлинности с помощью пароля Active Directory

Указывает, что драйверу требуется проверка подлинности для SQL Server с использованием идентификатора входа Azure Active Directory и пароля.

### <a name="with-active-directory-interactive-authentication"></a>С интерактивной проверкой подлинности Active Directory

Указывает, что драйвер проверяет подлинность для SQL Server использования Azure Active Directory интерактивного режима, предоставляя идентификатор входа. Это приведет к запуску диалогового окна запроса проверки подлинности Azure.

### <a name="login-id"></a>Идентификатор входа

Указывает идентификатор входа, используемый драйвером при подключении к SQL Server при условии, что при **проверке Подлинности SQL Server используется идентификатор входа и пароль, указанные пользователем** , или **с проверкой подлинности Active Directory с использованием идентификатора входа и пароля, вводимых пользователем.** или, если выбран параметр **"Интерактивная проверка подлинности Active Directory", используется идентификатор входа, указанный пользователем** . Это относится только к соединениям, установленным для определения настроек по умолчанию сервера; это не относится к последующим соединениям, установленным с использованием данного источника данных, после того как он был создан.

### <a name="password"></a>Пароль

Указывает пароль, используемый драйвером при подключении к SQL Server при условии **SQL Server аутентификации с использованием идентификатора входа и пароля, вводимых пользователем** , или с проверкой подлинности Active Directory с помощью пароля, **указанного пользователем.** выбран. Это относится только к соединениям, установленным для определения настроек по умолчанию сервера; это не относится к последующим соединениям, установленным с использованием нового источника данных.

Поля " **идентификатор входа** " и " **пароль** " отключены при **использовании встроенной проверки подлинности Windows** или **при выборе встроенной проверки подлинности Active Directory** .
