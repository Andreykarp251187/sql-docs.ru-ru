# <a name="data-source-wizard-screen-2"></a>Экран 2 мастера источников данных

Укажите метод проверки подлинности, настройте записи расширенного клиента Microsoft SQL Server, а также имя входа и пароль, которые драйвер ODBC будет использовать для подключения к SQL Server при настройке источника данных.

## <a name="options"></a>Параметры

### <a name="with-integrated-windows-authentication"></a>Со встроенной проверкой подлинности Windows

Указывает, что драйвер запрашивать безопасное (или доверительное) соединение с SQL Server. Если выбран данный режим, при установке соединений с использованием данного источника данных SQL Server будет использовать встроенную систему безопасности входа в систему, вне зависимости от того, какой режим безопасности установлен на сервере. Не учитываются любые предоставляемые идентификаторы входа и пароли. Системный администратор SQL Server должен связать имя входа Windows с Идентификатором имени входа SQL Server (например, с помощью SQL Server Management Studio).

Дополнительно можно указать для сервера имя участника-службы.

### <a name="with-active-directory-integrated-authentication"></a>Со встроенной проверкой подлинности Active Directory

Указывает, что драйвер пройти проверку подлинности SQL Server с помощью Azure Active Directory. Если выбран данный режим, то при установке подключений с использованием данного источника данных SQL Server будет использовать встроенную систему безопасности входа в систему Azure Active Directory, вне зависимости от того, какой режим безопасности установлен на сервере.

### <a name="with-sql-server-authentication"></a>С проверкой подлинности SQL Server

Указывает, что драйвер пройти проверку подлинности SQL Server с помощью идентификатора входа и пароль.

### <a name="with-active-directory-password-authentication"></a>С проверкой подлинности с помощью пароля Active Directory

Указывает, что драйвер пройти проверку подлинности SQL Server с помощью Azure Active Directory идентификатор входа и пароль.

### <a name="with-active-directory-interactive-authentication"></a>С интерактивной проверкой подлинности Active Directory

Указывает, что драйвер пройти проверку подлинности SQL Server с помощью Azure Active Directory интерактивный режим, указав идентификатор входа. Это активирует диалогового окна запроса проверки подлинности Windows Azure.

### <a name="login-id"></a>Идентификатор входа

Указывает идентификатор входа, драйвер использует при соединении с SQL Server, если **проверка подлинности SQL Server с помощью идентификатора входа и пароль, введенный пользователем** или **проверки подлинности с помощью пароля Active Directory с именем для входа и пароль, введенный пользователем** или **с Active Directory интерактивную проверку подлинности с именем для входа, введенный пользователем** выбран. Это относится только к соединениям, установленным для определения настроек по умолчанию сервера; это не относится к последующим соединениям, установленным с использованием данного источника данных, после того как он был создан.

### <a name="password"></a>Пароль

Указывает пароль, драйвер использует при соединении с SQL Server, если **проверка подлинности SQL Server с помощью идентификатора входа и пароль, введенный пользователем** или **проверки подлинности с помощью пароля Active Directory с именем для входа и пароль, введенный пользователем** выбран. Это относится только к соединениям, установленным для определения настроек по умолчанию сервера; это не относится к последующим соединениям, установленным с использованием нового источника данных.

Оба **идентификатор входа** и **пароль** будут отключены, если **проверки подлинности Windows, интегрированных с** или **с интегрированные службы Active Directory Проверка подлинности** выбран.
