# <a name="sql-server-security"></a>Безопасность SQL Server

![Download-DownArrow-Circled](../../../ssdt/media/download.png)[Скачать ADO.NET](../../sql-connection-libraries.md#anchor-20-drivers-relational-access)

SQL Server предоставляет много функций для создания безопасных приложений баз данных.  
  
Такие общие угрозы безопасности, как кража данных или вандализм, существуют независимо от используемой версии SQL Server. Целостность данных также следует рассматривать как проблемы безопасности. Если данные не защищены, возможно, они станут бесполезными, если разрешена нерегламентированная обработка данных, а данные случайно или злонамеренно были изменены с неправильными значениями или полностью удалены. Кроме того, часто предъявляются юридические требования, которые необходимо соблюдать, например правильное хранение конфиденциальной информации. Хранение некоторых видов персональных данных полностью минимума в зависимости от законов, применяемых в определенной юрисдикции.  
  
В каждой версии SQL Server есть свои средства безопасности, как и в каждой версии Windows, при этом возможности более поздних версий шире, чем возможности более ранних. Важно понимать, что только функции безопасности не могут гарантировать безопасное приложение базы данных. Каждое приложение базы данных уникально в требованиях, средах выполнения, модели развертывания, физическом расположении и заполнении пользователей. Некоторые приложения, локальные в области, могут нуждаться в минимальной безопасности, а другие локальные приложения или приложения, развернутые через Интернет, могут потребовать строгих мер безопасности и постоянного мониторинга и оценки.  
  
Требования безопасности для приложения базы данных SQL Server следует рассматривать во время разработки, а не впоследствии. Оценка угроз на раннем этапе разработки дает возможность снизить вероятность повреждения при обнаружении уязвимости.  
  
Даже если первоначальный дизайн приложения является звуком, в процессе развития системы могут возникнуть новые угрозы. Создавая несколько строк защиты базы данных, можно снизить ущерб, пострадает нарушением безопасности. Первая линия защиты заключается в том, чтобы уменьшить контактную зону для атак, чтобы предоставить больше разрешений, чем абсолютно необходимо.  
  
В этом разделе кратко описываются средства безопасности в SQL Server, которые нужны разработчикам. Здесь также приведены ссылки на соответствующие разделы электронной документации на SQL Server и другие ресурсы, где эти возможности описаны более подробно.
