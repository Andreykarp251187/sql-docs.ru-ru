# <a name="step-1--configure-development-environment-for-nodejs-development"></a>Шаг 1. Настройка среды разработки для разработки на Node.js
Чтобы разработать приложение с помощью драйвера Node. js для SQL Server, необходимо настроить среду разработки с учетом необходимых компонентов.  Наиболее распространенным способом является использование диспетчера пакетов node (NPM) для установки трудоемкого модуля, но вы можете скачать утомительный модуль непосредственно в [GitHub](https://github.com/pekim/tedious) , если хотите.  
  
Обратите внимание, что драйвер Node. js использует протокол TDS, который по умолчанию включен в SQL Server и базу данных SQL Azure.  Дополнительная настройка не требуется.  
  
## <a name="windows"></a>Windows  
  
1. **Установка среды выполнения Node. js и диспетчера пакетов NPM**  
A. Переход к [node. js](https://nodejs.org/en/download/)  
Б. Щелкните соответствующую ссылку MSI установщика Windows.   
в. После скачивания запустите MSI, чтобы установить Node. js.  
  
2. **Открыть CMD. exe**  
  
3. **Создайте каталог проекта** и перейдите к нему.    
```  
> mkdir HelloWorld  
> cd HelloWorld  
```  
4. **Создайте проект узла.**  Чтобы значения по умолчанию сохранялись во время создания проекта, нажмите клавишу ВВОД, чтобы создать проект. В конце этого шага вы увидите файл Package. JSON в каталоге проекта.  
```  
> npm init  
```  
  
5. **Установите утомительный модуль в проект.**  Это реализация протокола TDS, который используется драйвером для взаимодействия с SQL Server.  
```  
> npm install tedious  
```
