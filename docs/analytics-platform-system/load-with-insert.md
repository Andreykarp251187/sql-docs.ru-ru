# <a name="load-data-with-insert-into-parallel-data-warehouse"></a>Загрузка данных с помощью инструкции INSERT в Parallel Data Warehouse

Инструкции INSERT tsql можно использовать для загрузки данных в SQL Server Parallel Data Warehouse (PDW) распределяется или реплицируется таблицы. Дополнительные сведения о ВСТАВКЕ, см. в разделе [вставить](../t-sql/statements/insert-transact-sql.md). Для реплицированных таблиц и все столбцы без распределения в распределенную таблицу SQL Server использует PDW выполнить неявное преобразование значения данных, указанных в инструкции, чтобы тип данных целевого столбца. Дополнительные сведения о правилах преобразования данных SQL Server см. в разделе [преобразования SQL типа данных](../t-sql/data-types/data-type-conversion-database-engine.md). Тем не менее для столбцов распределения PDW поддерживает только подмножество неявные преобразования, поддерживаемые SQL Server. Таким образом при использовании инструкции INSERT для загрузки данных в столбец распределения, источника данных необходимо указать в одном из форматов, определенных в следующих таблицах.

|Тип литерала|Формат|Правила преобразования|  
|----------------|----------|--------------------|  
|Двоичный литерал|0x*hexidecimal_string*<br /><br />Пример 0x12Ef|Двоичные литералы должны иметь префикс 0 x.<br /><br />Длина источника данных не может превышать количество байтов, указанное для типа данных.<br /><br />Если длина источника данных меньше, чем размер **двоичных** тип данных, данные дополняется нулями вправо до размера типа данных.|

Секунд являются необязательными и если он имеется, должно быть присвоено значение 00. Любое другое значение приводит к ошибке.|

Например значение-литерал «12:35:29.123» вставляется как «12:35:29.1230000».<br /><br />Значение, которое имеет большую точность, чем в целевой тип данных отклоняется.|  
  
### <a name="datetimeoffset-data-type"></a>Тип данных DateTimeOffset  
В следующей таблице определены допустимые форматы и правила вставки литеральных значений в столбец распределения, типа **datetimeoffset** (*n*). Формат по умолчанию является "гггг-мм-дд чч:мм:сс.ннннннн {+ |-} чч: мм". Пустая строка ("") преобразуется в значение по умолчанию "12:00:00.0000000 1900-01-01 + 00:00". Строки, содержащие только пробелы ("") выдает ошибку. Число цифр дробной части, зависит от определения столбца. Например, столбец, определенный как **datetimeoffset** (2) будет иметь две цифры дробной части.  
  
|Тип литерала|Формат|Правила преобразования|  
|----------------|----------|--------------------|  
|Строковый литерал в **datetime** формат|«ГГГГ-мм-дд чч: мм: [.Ннн]»<br /><br />Пример ' 2007-05-08 12:35:29.123 "|Отсутствует цифр дробной части и значений смещения имеют значение 0 при вставке значение. Например, литерал "2007-05-08 12:35:29.123" вставляется в качестве "2007-05-08 12:35:29.1230000 + 00:00".|  
|Строковый литерал в **smalldatetime** формат|«ГГГГ-мм-дд чч: мм»<br /><br />Пример ' 2007-05-08 12:35 "|Секунды, оставшиеся цифры дробной части и значений смещения имеют значение 0 при вставке значение.|  
|Строковый литерал в **даты** формат|«ГГГГ ММ ДД»<br /><br />Пример ' 2007-05-08 "|Значения времени (час, минуты, секунды и дроби), присваивается 0, когда вставляется значение. Например, литерал "2007-05-08' вставляется в качестве" 2007-05-08 00:00:00.0000000 + 00:00 ".|  
|Строковый литерал в **datetime2** формат|«ГГГГ-мм-дд чч:мм:сс.ннннннн»<br /><br />Пример ' 2007-05-08 12:35:29.1234567 "|Источник данных не может превышать указанное число долей секунды в столбце datetimeoffset. Если источник данных имеет меньшее или равное количество долей секунды, данные дополняется нулями справа. Например, если тип данных datetimeoffset (5), литеральное значение "2007-05-08 12:35:29.123 + 12:15 ' вставляется в качестве" 12:35:29.12300 + 12:15 '.|  
|Строковый литерал в **datetimeoffset** формат|"Гггг-мм-дд чч:мм:сс.ннннннн {+&#124;-} чч: мм"<br /><br />Пример ' 2007-05-08 12:35:29.1234567 + 12:15 '|Источник данных не может превышать указанное число долей секунды в столбце datetimeoffset. Если источник данных имеет меньшее или равное количество долей секунды, данные дополняется нулями справа. Например, если тип данных datetimeoffset (5), литеральное значение "2007-05-08 12:35:29.123 + 12:15 ' вставляется в качестве" 12:35:29.12300 + 12:15 '.|  
  
### <a name="datetime2-data-type"></a>Тип данных datetime2  
В следующей таблице определены допустимые форматы и правила вставки литеральных значений в столбец распределения, типа **datetime2** (*n*). Формат по умолчанию является «Гггг-мм-дд чч:мм:сс.ннннннн». Пустая строка ("") преобразуется в значение по умолчанию "1900-01-01 12:00:00". Строки, содержащие только пробелы ("") выдает ошибку. Число цифр дробной части, зависит от определения столбца. Например, столбец, определенный как **datetime2** (2) будет иметь две цифры дробной части.  
  
|Тип литерала|Формат|Правила преобразования|  
|----------------|----------|--------------------|  
|Строковый литерал в **datetime** формат|«ГГГГ-мм-дд чч: мм: [.Ннн]»<br /><br />Пример ' 2007-05-08 12:35:29.123 "|Доли секунды, являются необязательными и устанавливаются в значение 0 при вставке значение.<br /><br />Значение, которое имеет больше цифр дробной части, чем у целевого типа данных отклоняется.|  
|Строковый литерал в **smalldatetime** формат|«ГГГГ-мм-дд чч: мм»<br /><br />Пример ' 2007-05-08 12 "|Необязательный секунды и оставшиеся цифры дробной части равны 0, при вставке значение.|  
|Строковый литерал в **даты** формат|«ГГГГ ММ ДД»<br /><br />Пример ' 2007-05-08 "|Значения времени (час, минуты, секунды и дроби), присваивается 0, когда вставляется значение. Например, литерал "2007-05-08' вставляется в качестве" 2007-05-08 12:00:00.0000000 ".|  
|Строковый литерал в **datetime2** формат|«ГГГГ-мм-дд hh:mm:ss:nnnnnnn»<br /><br />Пример ' 2007-05-08 12:35:29.1234567 "|Если источник данных содержит дату и время компонентов, которые меньше или равно значению, указанному в **datetime2**(*n*), вставляются данные; в противном случае формируется ошибка.|

### <a name="money-and-smallmoney-data-types"></a>типов данных Money и smallmoney  
Money литеральные значения отображаются в виде числа с необязательный десятичный разделитель и префикс в виде знака валюты. Источник данных не может превышать диапазона, разрешенного для заданного типа данных. Например, диапазон для **smallmoney** —-214 748,3648 214748,3647 и допустимый диапазон для **деньги** — -922,337,203,685,477.5808 до 922,337,203,685,477.5807. В следующей таблице определены допустимые форматы и правила вставки литеральных значений в столбец распределения, типа **деньги** или **smallmoney**.  
  
|Тип литерала|Формат|Правила преобразования|  
|----------------|----------|--------------------|  
|Строковый литерал в **целое число** формат|«nnnnnnnn»<br /><br />Пример "123433"|При вставке значение отсутствует цифр после десятичной запятой имеют значение 0. Например литерал "12345" вставляется как 12345.0000.|  
|Строковый литерал в **десятичное** формат|«nnnnnn.nnnnn»<br /><br />Пример «123344.34455"|Если количество цифр после десятичной запятой превышает 4, значение округляется до ближайшего значения. Например значение «123344.34455"вставляется как 123344.3446.|

|Целочисленный литерал|nnnnnnnn<br /><br />Пример 123433|При вставке значение отсутствует цифр после десятичной запятой имеют значение 0. Например литерал 12345 вставляется как 12345.0000.|  
|Десятичный литерал|nnnnnn.NNNNN<br /><br />Пример 123344.34455|Если количество цифр после десятичной запятой превышает 4, значение округляется до ближайшего значения. Например значение 123344.34455 вставляется как 123344.3446.|  
|Литерал деньги|$nnnnnn.nnnn<br /><br />Пример: $123456.7890|Символ валюты не вставляется со значением.<br /><br />Если количество цифр после десятичной запятой превышает 4, значение округляется до ближайшего значения.|  
  
## <a name="InsertLiteralsString"></a>Вставка литералов в строковые типы  
Следующие таблицы определяют допустимые форматы и правила преобразования для вставки литеральное значение в столбец SQL Server PDW, который использует строковый тип.  
  
### <a name="char-varchar-nchar-and-nvarchar-data-types"></a>типы данных char, varchar, nchar и nvarchar  
В следующей таблице определены допустимые форматы и правила вставки литеральных значений в столбец распределения, типа **char**, **varchar**, **nchar** и **nvarchar**. Длина источника данных не может превышать размер, указанный для типа данных. Если длина источника данных меньше, чем размер **char** или **nchar** тип данных, данные дополняется пробелами вправо до размера типа данных.  
  
|Тип литерала|Формат|Правила преобразования|  
|----------------|----------|--------------------|  
|Строковый литерал|Формат: «строка символов»<br /><br />Пример: 'abc'| None|  
|Строка литерала в Юникоде|Формат: Строка N'character "<br /><br />Пример N'abc'|  None |  
|Целочисленный литерал|Формат: nnnnnnnnnnn<br /><br />Пример 321312313123| None |  
|Десятичный литерал|Формат: nnnnnn.nnnnnnn<br /><br />Пример 12344.34455| None |  
|Литерал деньги|Формат: $nnnnnn.nnnnn<br /><br />Пример: $123456.99|Символ валюты не вставляется со значением. Чтобы вставить символ валюты, вставьте значение в виде строкового литерала. Это будет соответствовать формату **dwloader** средство, которое обрабатывает каждому литералу как строковый литерал.<br /><br />Не допускаются запятые.<br /><br />Если количество цифр после десятичной запятой превышает 2, значение округляется до ближайшего значения. Например значение 123.946789 вставляется как 123.95.<br /><br />При использовании функции CONVERT для вставки литералы деньги, допускается только стиль по умолчанию 0 (без запятых и двух цифр после десятичной запятой).|
