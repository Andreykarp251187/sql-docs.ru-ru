# <a name="configure-windows-server-update-services-wsus-in-analytics-platform-system"></a>Настройка Windows Server Update Services (WSUS) в Analytics Platform System
Эти инструкции описывают шаги по использованию мастера настройки Windows Server Update Services (WSUS) для настройки служб WSUS для Analytics Platform System. Необходимо настроить службы WSUS, перед применением обновлений программного обеспечения на устройство. На виртуальной машине VMM устройства уже установлена служба WSUS.  
  
Дополнительные сведения о настройке служб WSUS см. в разделе [руководство по установке WSUS пошаговое](https://go.microsoft.com/fwlink/?LinkId=202417) веб-сайта WSUS. После настройки WSUS, см. в разделе [скачивание и применение обновлений Майкрософт &#40;Analytics Platform System&#41; ](download-and-apply-microsoft-updates.md) инициировать обновление.  
  
> [!WARNING]  
> Если возникают ошибки во время этого процесса настройки, остановите и обратитесь за помощью в службу поддержки. Не пропускать ошибки или продолжить в процессе, после получения ошибки.  
  
## <a name="before-you-begin"></a>Перед началом  
Чтобы настроить службы WSUS, вам потребуется:  
  
-   У имени входа данные учетной записи администратора Analytics Platform System appliance домена.  
  
-   Наличие Analytics Platform System имени входа с разрешениями для доступа к **консоли администрирования** и просмотр сведений о состоянии устройства.  
  
-   Знаете IP-адрес вышестоящий сервер WSUS, если вы планируете синхронизировать обновления с вышестоящим сервером WSUS вместо синхронизации обновлений непосредственно из центра обновления Майкрософт. Убедитесь, что вышестоящий сервер WSUS, допускает анонимные подключения и поддерживает SSL.  
  
-   Знаете IP-адрес прокси-сервера, если ваше устройство будет использовать прокси-сервер для доступа к вышестоящему серверу или Центра обновления Майкрософт.  
  
-   В большинстве случаев WSUS требуется доступ к серверов за пределами устройства. Для поддержки этого сценария использования DNS системы платформы аналитики могут быть настроены для поддержки пересылки внешнее имя, позволяющее Analytics Platform System узлов и виртуальных машин (ВМ), использовать внешние DNS-серверы для разрешения имен за пределами устройство. Дополнительные сведения см. в разделе [использовать DNS-сервера пересылки для разрешения имен DNS не являющегося устройством &#40;Analytics Platform System&#41;](use-a-dns-forwarder-to-resolve-non-appliance-dns-names.md).  
  
## <a name="to-configure-windows-server-update-services-wsus"></a>Чтобы настроить Windows Server Update Services (WSUS)  
  
1.  Войдите на **консоли администрирования**. На **состояние устройства** вкладке, убедитесь, что **кластера** и **сети** показывают зеленый (или **NA**) для всех узлов. Проверьте индикаторы состояния для всех узлов на **состояние устройства**.  
  
    -   Можно безопасно продолжать работать с зеленым или индикаторы н/д.  
  
    -   Оцените некритическими ошибками (желтый) предупреждения. В некоторых случаях предупреждающие сообщения не будет блокировать обновлений. Если имеется ошибка некритические диска тома, не на диске C:\, можете перейти к следующему шагу до разрешения тома диска.  
  
    -   Большинство красный индикаторы должны быть разрешены, прежде чем продолжить. При возникновении сбоев диска, используйте **предупреждения консоли администратора** страницу, чтобы проверить, происходит ошибка не более одного диска в пределах каждого сервера или массив SAN. Если происходит ошибка не более одного диска в пределах каждого сервера или массив SAN, можете перейти к следующему шагу до исправления сбоев диска. Убедитесь, что в службу поддержки Майкрософт для устранения сбоев диска, как можно скорее.  
  
2.  Войдите на виртуальную машину VMM как администратор домена устройства.  
  
3.  Запустите мастер настройки.  
  
    #### <a name="to-launch-the-configuration-wizard"></a>Чтобы запустить мастер настройки  
  
    1.  В **панели мониторинга диспетчера серверов**на **средства** меню, щелкните **Windows Server Update Services**.  
  
    2.  В левой части **Update Services** окно, щелкните, чтобы развернуть сервер, на узел управления виртуальными машинами (**_appliance_domain_- VMM**), а затем нажмите кнопку **Параметры**.  
  
    3.  В **параметры** панели щелкните **мастер настройки сервера WSUS** для запуска мастера настройки.  
  
        ![Меню панели мониторинга диспетчера серверов](./media/configure-windows-server-update-services-wsus/WSUS_Wiz0.png "WSUS_Wiz0")  
  
    4.  Если при первом запуске мастера WSUS, возможно, потребуется настроить каталог для хранения обновлений. `C:\wsus` является соответствующее расположение; Тем не менее может предоставить другой путь.  
  
        ![Путь WSUS](./media/configure-windows-server-update-services-wsus/WSUS_Wiz1.png "WSUS_Wiz1")  
  
    5.  Просмотрите **перед началом работы** список элементов для выполнения до завершения работы мастера.  
  
        ![WSUS перед началом](./media/configure-windows-server-update-services-wsus/WSUS_Wiz2.png "WSUS_Wiz2")  
  
    6.  На **участие в программе улучшения Microsoft Update** выберите **Да, я бы хотел присоединиться к программе улучшения Microsoft Update**, а затем нажмите кнопку **Далее**.  
  
        ![Программа улучшения WSUS](./media/configure-windows-server-update-services-wsus/WSUS_Wiz3.png "WSUS_Wiz3")  
  
    Теперь вы увидите **Выбор вышестоящего сервера** страницы. Следующий снимок экрана является отправной точкой в мастере настройки.  
  
    ![Синхронизация с вышестоящим сервером WSUS](./media/configure-windows-server-update-services-wsus/WSUS_Wiz4a.png "WSUS_Wiz4a")  
  
4.  Выбор вышестоящего сервера.  
  
    На **Выбор вышестоящего сервера** страницы мастера настройки WSUS вы сможете выбрать, как WSUS на этом узле управления виртуальной машины будут подключаться к вышестоящему серверу для получения обновлений программного обеспечения. Доступны два варианты для синхронизации с вышестоящим сервером [центра обновления Майкрософт](https://go.microsoft.com/fwlink/?LinkId=133349) или синхронизация обновлений с другим сервером Windows Server Update Services.  
  
    #### <a name="to-update-by-using-microsoft-update"></a>Чтобы обновить с помощью центра обновления Майкрософт  
  
    1.  Если выбран для синхронизации с центром обновления Майкрософт, не нужно внести изменения в **Выбор вышестоящего сервера** страницы. Нажмите кнопку **Далее**.  
  
        ![Синхронизация с вышестоящим сервером WSUS](./media/configure-windows-server-update-services-wsus/WSUS_Wiz4a.png "WSUS_Wiz4a")  
  
    #### <a name="to-update-from-another-wsus-server"></a>Для обновления с другого сервера WSUS  
  
    1.  Если выбран для синхронизации с источника, отличного от центра обновления Майкрософт (сервер), укажите сервер (введите IP-адрес) и номер порта, на котором данный сервер будет связываться с вышестоящим сервером.  
  
        ![Синхронизация с вышестоящим сервером WSUS из WSUS](./media/configure-windows-server-update-services-wsus/WSUS_Wiz4b.png "WSUS_Wiz4b")  
  
    2.  Чтобы использовать протокол Secure Sockets Layer (SSL), выберите **использовать SSL при синхронизации данных об обновлениях** "флажок". В этом случае серверы будут использовать порт 443 для синхронизации.  
  
        ![Синхронизация с вышестоящим сервером WSUS из WSUS SSL](./media/configure-windows-server-update-services-wsus/WSUS_Wiz4c.png "WSUS_Wiz4c")  
  
    3.  Если это сервер-реплику, выберите **это реплика вышестоящего сервера** "флажок". Можно выбрать оба **использовать SSL при синхронизации данных об обновлениях** и **это реплика вышестоящего сервера**.  
  
        ![Реплика вышестоящего сервера WSUS](./media/configure-windows-server-update-services-wsus/WSUS_Wiz4d.png "WSUS_Wiz4d")  
  
    4.  На этом этапе вы закончите с вышестоящим сервером конфигурации. Нажмите кнопку **Далее**, или выберите **прокси-сервера укажите** в левой области навигации.  
  
5.  Укажите прокси-сервера.  
  
    Если для этого сервера требуется прокси-сервер для доступа к центра обновления Майкрософт или другого вышестоящего сервера, можно настроить параметры прокси-сервера в противном случае нажмите **Далее**.  
  
    ![Прокси-сервер WSUS](./media/configure-windows-server-update-services-wsus/WSUS_Wiz5a.png "WSUS_Wiz5a")  
  
    #### <a name="to-configure-proxy-server-settings"></a>Чтобы настроить параметры прокси-сервера  
  
    1.  На **Настройка прокси-сервера** странице мастера настройки, выберите **использовать прокси-сервер при синхронизации** флажок и введите адрес IP-адрес прокси-сервера (не по имени) и номер порта (порт 80 по по умолчанию) в соответствующие поля.  
  
    2.  Если вы хотите подключиться к прокси-сервера с помощью определенных учетных данных пользователя, выберите **использовать учетные данные пользователя для подключения к прокси-сервера** флажок и введите имя пользователя, домен и пароль пользователя в соответствующем поля. Если вы хотите включить обычную проверку подлинности для пользователя, подключающегося к прокси-серверу, установите **разрешить обычную проверку подлинности (пароль передается открытым текстом)** "флажок".  
  
        ![Учетные данные прокси-сервера WSUS](./media/configure-windows-server-update-services-wsus/WSUS_Wiz5b.png "WSUS_Wiz5b")  
  
    3.  На этом этапе вы закончите с конфигурацию прокси-сервера. Нажмите кнопку **Далее** для перехода на следующую страницу, где можно начать для настройки процесса синхронизации.  
  
6.  Начните подключение.  
  
    ![Прокси-сервер WSUS приступить к подключению](./media/configure-windows-server-update-services-wsus/WSUS_Wiz6.png "WSUS_Wiz6")  
  
    Нажмите кнопку **приступить к подключению**.  
  
    После подключения завершилась успешно, нажмите кнопку **Далее** для перехода к следующей странице, где можно выбрать языки интерфейса.  
  
7.  Выбор языков.  
  
    Выберите **загружать обновления только на этих языках**.  
  
    Выберите **Английский**, а затем нажмите кнопку **Далее**.  
  
    ![Выбор языков](./media/configure-windows-server-update-services-wsus/SQL_Server_PDW_WSUSChooseLanguages.png "SQL_Server_PDW_WSUSChooseLanguages")  
  
8.  Выбор продуктов.  
  
    > [!NOTE]  
    > Если вы работаете с вышестоящим сервером, может не появиться возможность выбор продуктов. Если этот параметр недоступен, пропустите этот шаг.  
  
    Снимите флажки для всех выбранных обновлений.  
  
    Выберите **Windows Server 2012 R2**, и **System Center 2012 R2 — Virtual Machine Manager**, а затем нажмите кнопку **Далее**.  
  
9. Выберите классификации.  
  
    > [!NOTE]  
    > Если вы работаете с вышестоящим сервером, может не появиться возможность выбор классов.  Если этот параметр недоступен, пропустите этот шаг.  
  
    Снимите флажки для всех ранее выбранных обновлений.  
  
    Выберите **критические обновления** и **обновления для системы безопасности** обновлений, которые будут синхронизированы для Analytics Platform System appliance и нажмите кнопку **Далее**.  
  
    ![Выберите классификации](./media/configure-windows-server-update-services-wsus/SQL_Server_PDW_WSUSChooseClassifications.png "SQL_Server_PDW_WSUSChooseClassifications")  
  
10. Настройте расписание синхронизации.  
  
    Выберите **вручную синхронизировать**, а затем нажмите кнопку **Далее**.  
  
    ![Задайте расписание синхронизации](./media/configure-windows-server-update-services-wsus/SQL_Server_PDW_WSUSSyncSchedule.png "SQL_Server_PDW_WSUSSyncSchedule")  
  
11. Запустить первоначальную синхронизацию.  
  
    Выберите **запустить первоначальную синхронизацию**, затем нажмите кнопку **Далее**.  
  
12. Готово.  
  
    Нажмите кнопку **Готово**.  
  
## <a name="bkmk_WSUSGroup"></a>Группы серверов модуль в службах WSUS  
После настройки WSUS для Analytics Platform System, следующим шагом является для группирования серверов устройства. Добавив все серверы устройства в группу, службы WSUS будут иметь возможность обновления программного обеспечения на всех серверах в устройстве.  
  
> [!NOTE]  
> Система WSUS предназначен для асинхронного выполнения. Инициирование действия не всегда приводит немедленного обновления. Таким образом может потребоваться подождать некоторое время, пока компьютеры будут видны в диалоговых окнах служб WSUS. Под управлением `setup.exe /action=ReportMicrosoftUpdateClientStatus /DomainAdminPassword="<password>"` команды, описанной в конце этой статьи [скачивание и применение обновлений Майкрософт &#40;Analytics Platform System&#41; ](download-and-apply-microsoft-updates.md) помогут обновить диалоговым окнам.  
  
#### <a name="to-group-the-appliance-servers"></a>Для группирования серверов устройства  
  
1.  Откройте консоль WSUS, щелкните правой кнопкой мыши **все компьютеры** и нажмите кнопку **добавить группу компьютеров**.  
  
    ![Добавьте группу компьютеров. ](./media/configure-windows-server-update-services-wsus/SQL_Server_PDW_WSUSAddComputerGroup.png "SQL_Server_PDW_WSUSAddComputerGroup")  
  
2.  Введите имя «APS» для группы компьютеров и нажмите кнопку **добавить**.  
  
    ![Введите имя для новой группы компьютеров. ](./media/configure-windows-server-update-services-wsus/SQL_Server_PDW_WSUSSpecifyGroupName.png "SQL_Server_PDW_WSUSSpecifyGroupName")  
  
3.  Нажмите кнопку **все компьютеры** снова изменить состояние в **состояние** раскрывающееся меню для **любой**и нажмите кнопку **обновить**. Может потребоваться развернуть **все компьютеры** , щелкнув его в структуре дерева слева, чтобы увидеть новую группу вы только что добавили.  
  
    ![Измените состояние на любой и щелкните "Обновить". ](./media/configure-windows-server-update-services-wsus/SQL_Server_PDW_WSUSChangeStatusToAny.png "SQL_Server_PDW_WSUSChangeStatusToAny")  
  
4.  Выберите все компьютеры, которые являются частью устройство, щелкните правой кнопкой мыши и выберите команду **изменить членство**.  
  
    ![Измените членство всех компьютеров PDW. ](./media/configure-windows-server-update-services-wsus/SQL_Server_PDW_WSUSChangeMembership.png "SQL_Server_PDW_WSUSChangeMembership")  
  
5.  Выберите новую группу компьютеров, созданный, щелкнув флажок и нажмите кнопку **ОК**.  
  
    ![Set Computer Group Membership](./media/configure-windows-server-update-services-wsus/SQL_Server_PDW_WSUSSetComputerGroupMembership.png "SQL_Server_PDW_WSUSSetComputerGroupMembership")  
  
6.  Выберите новую группу компьютеров, изменить его **состояние** для **любой**и нажмите кнопку **обновить**. Теперь все компьютеры должны назначенную этой группе и перечисленных в правой области. Обычно безопасно продолжить, когда узлы Показать предупреждения, например **этот узел не сообщил о состоянии еще**.  
  
    ![Измените состояние на любой и щелкните "Обновить". ](./media/configure-windows-server-update-services-wsus/SQL_Server_PDW_WSUSChangeStatusAnyRefresh.png "SQL_Server_PDW_WSUSChangeStatusAnyRefresh")
