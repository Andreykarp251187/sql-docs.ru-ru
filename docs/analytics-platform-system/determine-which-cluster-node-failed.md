# <a name="determine-which-cluster-node-failed-for-analytics-platform-system"></a>Определите, какой узел кластера не удалось найти в системе платформы аналитики
В этом разделе описывается, как определить имя узла системы аналитики (ТД), который завершился сбоем после отработки отказа кластера и порождено предупреждение отработки отказа кластера. В рамках устранения неполадок при отработке отказа кластера необходимо определить имя узла, который завершился сбоем, перед обращением в корпорацию Майкрософт для помощи в решении проблемы.  
  
## <a name="Background"></a>Историческая справка  
Для обеспечения высокой доступности в SQL Server PDW узел управления и узлы вычислений настраиваются как активные или пассивные компоненты отказоустойчивых кластеров Windows. Когда активный сервер не отвечает на критически важные системные запросы, пассивный сервер отключается и выполняет функции сервера, на котором произошел сбой.  
  
После отработки отказа кластера при SQL Server PDW отчетов о состоянии узла пассивный сервер переходит в состояние сбоя. Однако неочевидно, какой сервер или узел завершился сбоем, особенно если сервер, на котором произошел сбой, остается в сети. Чтобы устранить неполадки кластера, необходимо определить имя узла, для которого выполнена отработка отказа.  
  
## <a name="AdminConsoleSolution"></a>Решение консоли администрирования  
  
#### <a name="to-find-the-name-of-the-node-that-failed"></a>Поиск имени узла, на котором произошел сбой  
  
1.  Откройте консоль администрирования. Дополнительные сведения о консоли администрирования см. в разделе [мониторинг устройства с помощью консоли администрирования &#40;Analytics Platform System&#41;](monitor-the-appliance-by-using-the-admin-console.md). После отработки отказа событие отработки отказа включается в число оповещений на странице **работоспособности** . Существует страница **работоспособности** для региона PDW и для региона структуры устройства. На каждой странице работоспособности есть вкладка **оповещения** . Чтобы получить дополнительные сведения о предупреждении, щелкните страницу работоспособность, вкладку оповещения и щелкните оповещение.  
  
## <a name="SystemView"></a>Решение системного представления  
Следующая инструкция SQL показывает, как использовать Системное представление [sys. dm_pdw_component_health_active_alerts](../relational-databases/system-dynamic-management-views/sys-dm-pdw-component-health-active-alerts-transact-sql.md) для поиска имени сервера, на котором произошел сбой.  
  
```sql  
SELECT  
SUBSTRING( component_instance_id, 2, charindex(' ', component_instance_id, 1)-2) AS failed_node_name,  
create_time AS failover_time  
FROM sys.dm_pdw_component_health_active_alerts  
WHERE alert_id = 500139  
ORDER BY failed_node_name;  
```
