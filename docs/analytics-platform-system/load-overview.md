# <a name="loading-data-into-parallel-data-warehouse"></a>Загрузка данных в Parallel Data Warehouse
Можно загрузить или вставки данных в SQL Server Parallel Data хранилища (PDW) с помощью службы Integration Services, [служебная программа bcp](../tools/bcp-utility.md), **dwloader** загрузчика командной строки или инструкции SQL INSERT.  

## <a name="loading-environment"></a>Идет загрузка среды  
Чтобы загрузить данные, требуется один или несколько серверов загрузки. Вы можете использовать собственные существующие ETL или других серверах, или вы можете приобрести новые серверы. Дополнительные сведения см. в разделе [приобретение и настройка сервера, загрузка](acquire-and-configure-loading-server.md). Эти инструкции включают [загрузки таблица планирования емкости сервера](loading-server-capacity-planning-worksheet.md) помогут вам спланировать эффективное решение для загрузки.  
  
## <a name="load-with-dwloader"></a>Загрузка с помощью dwloader  
С помощью [dwloader командной строки загрузчика](dwloader.md) является самым быстрым способом загрузить данные в PDW.  
  
![Процесс загрузки](media/loading-process.png "процесс загрузки")  
  
dwloader загружает данные непосредственно на вычислительных узлах без передачи данных через узел элемента управления. Чтобы загрузить данные, dwloader сначала взаимодействует с узла управления для получения контактных данных для вычислительных узлов. dwloader настраивает канал связи с каждый вычислительный узел и затем отправляет фрагменты по 256 КБ данных на вычислительных узлах в циклического перебора.  
  
На каждом вычислительном узле служба перемещения данных (DMS) получает и обрабатывает фрагменты данных. Обработка данных включает в себя преобразования каждой строки в формате SQL Server и вычисления хэша распространения для определения вычислительном узле, к которой принадлежит каждой строки.  
  
После обработки строк, DMS использует перемещение в случайном порядке для передачи каждой строки на правильный вычислительного узла и экземпляра SQL Server. Как SQL Server получает строки, он создает пакеты их согласно **-b** параметр размера пакета, установите в dwloader, а затем выполняет массовую загрузку пакета.  

## <a name="load-with-prepared-statements"></a>Загрузка с помощью подготовленных инструкций

Подготовленные инструкции можно использовать для загрузки данных в распределенной и реплицированные таблицы. Если входные данные не соответствует в целевой тип данных, выполняется неявное преобразование. Неявные преобразования, поддерживаемые классом подготовленных инструкций PDW — это подмножество преобразований, поддерживаемые SQL Server. То есть поддерживаются только подмножество преобразований, но также поддерживаемое преобразование соответствует неявные преобразования SQL Server. Независимо от того, определен ли целевой таблицы необходимо загрузить как распределенные или реплицируемые таблицы неявные преобразования применяются (при необходимости) для всех столбцов, которые существуют в целевой таблице.
