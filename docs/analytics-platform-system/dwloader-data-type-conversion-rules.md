# <a name="data-type-conversion-rules-for-dwloader---parallel-data-warehouse"></a>Правила преобразования для dwloader - Parallel Data Warehouse типов данных
В этом разделе описываются форматы входных данных и неявные преобразования типов данных, [dwloader командной строки загрузчика](dwloader.md) поддерживает при загрузке данных в PDW. Неявные преобразования данных возникают, если входные данные не совпадает с типом данных в целевой таблице SQL Server PDW. Используйте эти сведения при разработке процесс загрузки будет поддерживать ваши данные будут успешно загрузить в SQL Server PDW.  
   
  
## <a name="InsertBinaryTypes"></a>Вставка литералов в двоичные типы  
Следующая таблица определяет допустимый тип литерала, формат и правила преобразования для загрузки литеральное значение в столбец типа SQL Server PDW **двоичных** (*n*) или **varbinary** (*n*).  
  
|Тип входных данных|Примеры входных данных|Преобразование в binary или varbinary тип данных|  
|-------------------|-----------------------|-----------------------------------------------|  
|Двоичный литерал|[0x]*hexidecimal_string*<br /><br />Пример 12Ef или 0x12Ef|Префикс 0 x является необязательным.<br /><br />Длина источника данных не может превышать количество байтов, указанное для типа данных.<br /><br />Если длина источника данных меньше, чем размер **двоичных** тип данных, данные дополняется нулями вправо до размера типа данных.|  
  
## <a name="InsertDateTimeTypes"></a>Вставка литералов в типы даты и времени  
Литералы даты и времени представлены с помощью строковых литералов в специального формата, заключенных в одинарные кавычки. В следующих таблицах допустимые типы литералов, формат и правила преобразования для загрузки в столбец типа даты или времени литерала **datetime**, **smalldatetime**, **даты**, **время**, **datetimeoffset**, или **datetime2**. Таблицы определяют формат по умолчанию для заданного типа данных. Другие форматы, которые могут быть определены в разделе [форматов даты и времени](#DateFormats). Литералы даты и времени не может содержать начальные или конечные пробелы. **Дата**, **smalldatetime**, и значения null не удается загрузить в режиме фиксированную ширину.  
  
### <a name="datetime-data-type"></a>Тип данных datetime  
Следующая таблица определяет формат по умолчанию и правила для загрузки литеральных значений в столбец типа **datetime**. Пустая строка ("") преобразуется в значение по умолчанию "12:00:00.000 1900-01-01". Строки, содержащие только пробелы ("") выдает ошибку.  
  
|Тип входных данных|Примеры входных данных|Преобразование в тип данных datetime|  
|-------------------|-----------------------|------------------------------------|  
|Строковый литерал в **datetime** формат|«гггг мм дд чч: мм: [.fff]»<br /><br />Пример '2007-05-08 12:35:29.123'|При вставке значение отсутствует цифр дробной части имеют значение 0. Например, литерал "2007-05-08 12:35 ° вставляется в качестве" 2007-05-08 12:35:00.000 ".|  
|Строковый литерал в **smalldatetime** формат|«гггг мм дд чч: мм»<br /><br />Пример '2007-05-08 12:35'|Секунды и оставшиеся цифры дробной части равны 0, при вставке значение.|  
|Строковый литерал в **даты** формат|«гггг мм дд»<br /><br />Пример '2007-05-08'|Значения времени (час, минуты, секунды и дроби) присваивается 12:00:00.000 при вставке значение.|  
|Строковый литерал в **datetime2** формат|«yyyy-MM-dd hh:mm:ss.fffffff»<br /><br />Пример '2007-05-08 12:35:29.1234567'|Источник данных не может превышать три цифры дробной части. Например, литерал "2007-05-08 12:35:29.123" будет вставлен, но значение "12:35:29.1234567 2007-05-8" приводит к ошибке.|  
  
### <a name="smalldatetime-data-type"></a>Тип данных smalldatetime  
Следующая таблица определяет формат по умолчанию и правила для загрузки литеральных значений в столбец типа **smalldatetime**. Пустая строка ("") преобразуется в значение по умолчанию "1900-01-01 12:00". Строки, содержащие только пробелы ("") выдает ошибку.  
  
|Тип входных данных|Примеры входных данных|Преобразование в тип данных smalldatetime|  
|-------------------|-----------------------|-----------------------------------------|  
|Строковый литерал в **smalldatetime** формат|«гггг мм дд чч: мм» или «гггг мм дд чч»<br /><br />Пример ' 2007-05-08, 12:00 "или" 2007-05-08 12:00:15 "|Исходные данные должны иметь значения для года, месяца, даты, час и минуту. Секунд являются необязательными и если он имеется, должно быть присвоено значение 00. Любое другое значение приводит к ошибке.<br /><br />Секунд является необязательной. При загрузке в столбец smalldatetime dwloader округлит секунды и доли секунды. Например 01 05 20:11 загрузит 20:10:35.123 1999-01-05.|  
|Строковый литерал в **даты** формат|«гггг мм дд»<br /><br />Пример '2007-05-08'|Значения времени (час, минуты, секунды и дроби), присваивается 0, когда вставляется значение.|

### <a name="datetimeoffset-data-type"></a>Тип данных DateTimeOffset  
Следующая таблица определяет формат по умолчанию и правила для загрузки литеральных значений в столбец типа **datetimeoffset** (*n*). Формат по умолчанию — "yyyy-MM-dd hh:mm:ss.fffffff {+ |-} чч: мм". Пустая строка ("") преобразуется в значение по умолчанию "12:00:00.0000000 1900-01-01 + 00:00". Строки, содержащие только пробелы ("") выдает ошибку. Число цифр дробной части, зависит от определения столбца. Например, столбец, определенный как **datetimeoffset** (2) будет иметь две цифры дробной части.  
  
|Тип входных данных|Примеры входных данных|Преобразование в тип данных datetimeoffset|  
|-------------------|-----------------------|------------------------------------------|  
|Строковый литерал в **datetime** формат|«гггг мм дд чч: мм: [.fff]»<br /><br />Пример '2007-05-08 12:35:29.123'|Отсутствует цифр дробной части и значений смещения имеют значение 0 при вставке значение. Например, литерал "2007-05-08 12:35:29.123" вставляется в качестве "2007-05-08 12:35:29.1230000 + 00:00".|  
|Строковый литерал в **smalldatetime** формат|«гггг мм дд чч: мм»<br /><br />Пример '2007-05-08 12:35'|Секунды, оставшиеся цифры дробной части и значений смещения имеют значение 0 при вставке значение.|  
|Строковый литерал в **даты** формат|«гггг мм дд»<br /><br />Пример '2007-05-08'|Значения времени (час, минуты, секунды и дроби), присваивается 0, когда вставляется значение. Например, литерал "2007-05-08' вставляется в качестве" 2007-05-08 00:00:00.0000000 + 00:00 ".|  
|Строковый литерал в **datetime2** формат|«yyyy-MM-dd hh:mm:ss.fffffff»<br /><br />Пример '2007-05-08 12:35:29.1234567'|Источник данных не может превышать указанное число долей секунды в столбце datetimeoffset. Если источник данных имеет меньшее или равное количество долей секунды, данные дополняется нулями справа. Например, если тип данных datetimeoffset (5), литеральное значение "2007-05-08 12:35:29.123 + 12:15 ' вставляется в качестве" 12:35:29.12300 + 12:15 '.|  
|Строковый литерал в **datetimeoffset** формат|"yyyy-MM-dd hh:mm:ss.fffffff {+&#124;-} чч: мм"<br /><br />Пример '2007-05-08 12:35:29.1234567 +12:15'|Источник данных не может превышать указанное число долей секунды в столбце datetimeoffset. Если источник данных имеет меньшее или равное количество долей секунды, данные дополняется нулями справа. Например, если тип данных datetimeoffset (5), литеральное значение "2007-05-08 12:35:29.123 + 12:15 ' вставляется в качестве" 12:35:29.12300 + 12:15 '.|

|[d]d-[M[M]]M-[yy]yy hh:mm:ss[.fff][tt]|[d]d-[M[M]]M-[yy]yy hh:mm[:00][tt]||[d]d-[M[M]]M-[yy]yy hh:mm:ss[.fffffff][tt]|[d]d-[M[M]]M-[yy]yy hh:mm:ss[.fffffff][tt] zzz|

Сведения.  
  
-   Для разделения значений месяца, дня и года, можно использовать '-', '/', или ". '. Для простоты в таблице используется только разделитель "-".  
  
-   Чтобы указать месяца в виде текста используйте три или более символов. Месяцы с 1 или 2 символов будут интерпретироваться как число.  
  
-   Для разделения значений времени используйте ":" символ.  
  
-   Буквы в квадратных скобках необязательны.  
  
-   Буквы tt обозначают время суток [AM|PM|am|pm]. По умолчанию используется AM. Если задано значение tt, значение часа (hh) должно находиться в диапазоне от 0 до 12.  
  
-   Буквы zzz обозначают смещение часового пояса для текущего часового пояса системы в формате {+ |-} HH:ss].  
  
## <a name="InsertNumerictypes"></a>Вставка литералов в числовые типы

Например значение-«123"приводит к ошибке.<br /><br />Значение, превышающее 1 преобразуется в 1. Например "123" преобразуется в значение 1.|  
|Строковый литерал|«TRUE» или «ЛОЖЬ»<br /><br />Пример: «true»|Значение «ИСТИНА» преобразуется в 1; значение «FALSE» преобразуется в 0.|  
|Целочисленный литерал|fffffffn<br /><br />Пример 1 или 321|Значение меньше 0 или больше 1 преобразуется в 1. Например значения 123 и -123 преобразуются в 1.|

### <a name="char-varchar-nchar-and-nvarchar-data-types"></a>Типы данных char, varchar, nchar и nvarchar  
Следующая таблица определяет формат по умолчанию и правила для загрузки литеральных значений в столбец типа **char**, **varchar**, **nchar** и **nvarchar** . Длина источника данных не может превышать размер, указанный для типа данных. Если длина источника данных меньше, чем размер **char** или **nchar** тип данных, данные дополняется пробелами вправо до размера типа данных.  
  
|Тип входных данных|Примеры входных данных|Преобразование в символьные типы данных|

|Строка литерала в Юникоде|Формат: Строка N'character "<br /><br />Пример N'abc'| Н/Д |  
|Целочисленный литерал|Формат: ffffffffffn<br /><br />Пример 321312313123| Н/Д |  
|Десятичный литерал|Формат: ffffff.fffffff<br /><br />Пример 12344.34455| Н/Д |  
|Литерал деньги|Формат: $ffffff.fffnn<br /><br />Пример: $123456.99|Символ валюты не вставляется со значением. Чтобы вставить символ валюты, вставьте значение в виде строкового литерала. Например значение 123.946789 вставляется как 123.95.<br /><br />При использовании функции CONVERT для вставки литералы деньги, допускается только стиль по умолчанию 0 (без запятых и двух цифр после десятичной запятой).|
