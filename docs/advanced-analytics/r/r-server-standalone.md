# <a name="r-server-standalone-and-machine-learning-server-standalone-in-sql-server"></a>R Server (изолированный) и Machine Learning Server (автономный) в SQL Server
[!INCLUDE[appliesto-ss-xxxx-xxxx-xxx-md-winonly](../../includes/appliesto-ss-xxxx-xxxx-xxx-md-winonly.md)]

SQL Server предоставляет поддержку установки для автономного сервера R или Machine Learning Server, который работает независимо от SQL Server. В зависимости от версии SQL Server изолированный сервер имеет основу R с открытым кодом и, возможно, Python, которая перекрываются с высокопроизводительными библиотеками Майкрософт, которые добавляют статистические и прогнозные аналитические средства в масштабе. Библиотеки также включают задачи машинного обучения, внесенные в скрипты R или Python. 

В SQL Server 2016 эта функция называется **r Server (изолированная)** и доступна только для r. В SQL Server 2017 он называется **Machine Learning Server (автономный)** и включает R и Python.  

> [!Note]
> Как установлено SQL Server установки, изолированный сервер функционально эквивалентен версиям [Microsoft Machine Learning Server](https://docs.microsoft.com/machine-learning-server/what-is-machine-learning-server), не относящимся к SQL, и поддерживает те же пользовательские сценарии, включая удаленное выполнение, эксплуатацию и веб-приложения. службы, а также полный набор библиотек R и Python.

## <a name="using-a-standalone-server"></a>Использование изолированного сервера

Разработчики r и Python обычно выбирают отдельный сервер для перехода за пределы памяти и ограничений на обработку с открытым исходным кодом R и Python. Библиотеки R и Python, которые выполняются на изолированном сервере, могут загружать и обрабатывать большие объемы данных на нескольких ядрах и объединять результаты в единый объединенный выход. Высокопроизводительные функции разрабатываются как для масштабирования, так и для служебной программы: предоставление прогнозной аналитики, статистического моделирования, визуализаций данных и алгоритмов машинного обучения в коммерческих серверных продуктах, разработанных и поддерживаемых NNTP.

В качестве независимого сервера, отделенного от SQL Server, среда R и Python настраивается, защищена и доступна с помощью базовой операционной системы и стандартных средств, предоставляемых на отдельном сервере, но не SQL Server. Встроенная поддержка SQL Server реляционных данных отсутствует. Если вы хотите использовать SQL Server данные, можно создать объекты источника данных и соединения так же, как и любой клиент.

Как дополнения SQL Server, изолированный сервер также полезен в качестве мощной среды разработки, если требуются как локальные, так и удаленные вычисления. Пакеты R и Python на изолированном сервере те же, что и при установке ядра СУБД, что позволяет выполнять переносимость кода и [Переключение контекста вычислений](https://docs.microsoft.com/machine-learning-server/r/concept-what-is-compute-context).

## <a name="how-to-get-started"></a>Как приступить к работе

Начните с установки, вложите двоичные файлы в избранное средство разработки и напишите свой первый скрипт.

### <a name="step-1-install-the-software"></a>Шаг 1. Установка программного обеспечения

Установите одну из следующих версий:

+ [Machine Learning Server SQL Server 2017 (автономная)](../install/sql-machine-learning-standalone-windows-install.md)
+ [SQL Server 2016 R Server (изолированный) — только R](../install/sql-r-standalone-windows-install.md)

### <a name="step-2-configure-a-development-tool"></a>Шаг 2. Настройка средства разработки

На отдельном сервере обычно работает локальная работа с использованием среды разработки, установленной на том же компьютере.

+ [Настройка средств R](set-up-a-data-science-client.md)
+ [Настройка средств Python](../python/setup-python-client-tools-sql.md)

### <a name="step-3-write-your-first-script"></a>Шаг 3. Напишите первый скрипт

Напишите сценарий R или Python, используя функции из RevoScaleR, revoscalepy и алгоритмов машинного обучения.
  
  + [Изучите R и RevoScaleR в 25 функциях](https://docs.microsoft.com/machine-learning-server/r/tutorial-r-to-revoscaler): Начните с основных команд R, а затем приступайте к RevoScaleRным аналитическим функциям, которые обеспечивают высокую производительность и масштабирование для решений R. Она содержит параллелизуемые версии многих популярных пакетов моделирования для R, например кластеризацию методом К-средних, деревья и леса принятия решений, а также средства для работы с данными.

  + [QuickStart Пример двоичной классификации с помощью пакета](https://docs.microsoft.com/machine-learning-server/python/quickstart-binary-classification-with-microsoftml)Python для microsoftml: Создайте модель двоичной классификации с помощью функций из microsoftml и хорошо известного набора данных молочной железы рака.

Выберите оптимальный язык для задачи. R лучше подходит для статистических вычислений, которые трудно реализовать с помощью SQL. Для операций на основе наборов данных используйте возможности [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] для достижения максимальной производительности. Используйте ядро СУБД в памяти для очень быстрых вычислений над столбцами.

### <a name="step-4-operationalize-your-solution"></a>Шаг 4. Эксплуатацию свое решение

Изолированные серверы могут использовать функцию [оперативности](https://docs.microsoft.com//machine-learning-server/what-is-operationalization) [Microsoft Machine Learning Server](https://docs.microsoft.com/machine-learning-server/what-is-machine-learning-server), не зависящей от языка SQL. Вы можете настроить изолированный сервер для работы, что дает вам следующие преимущества: разверните и разместите свой код как веб-службы, запустите диагностику, проверьте емкость веб-службы.

### <a name="step-5-maintain-your-server"></a>Шаг 5. Обслуживание сервера

SQL Server выпуски накопительных обновлений на регулярной основе. Применение накопительных обновлений повышает безопасность и функциональные усовершенствования существующей установки. 

Описание новых или измененных функциональных возможностей можно найти в статье [загрузки CAB](../install/sql-ml-cab-downloads.md) -файлов и на страницах для [SQL Server 2016 накопительных обновлений](https://support.microsoft.com/help/3177312/sql-server-2016-build-versions) и [SQL Server 2017 накопительных обновлений](https://support.microsoft.com/help/4047329). 

Дополнительные сведения о том, как применить обновления к существующему экземпляру, см. в разделе [Apply Updates](../install/sql-machine-learning-standalone-windows-install.md#apply-cu) инструкции по установке.
