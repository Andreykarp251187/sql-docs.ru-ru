# <a name="predict-dmx"></a>Predict (расширения интеллектуального анализа данных)
[!INCLUDE[ssas](../includes/applies-to-version/ssas.md)]

  Функция **Predict** Возвращает прогнозируемое значение или набор значений для указанного столбца.  
  
## <a name="syntax"></a>Синтаксис  
  
```  
  
Predict(<scalar column reference>, [option1], [option2], [option n], [INCLUDE_NODE_ID], n)  
Predict(<table column reference>, [option1], [option2], [option n], [INCLUDE_NODE_ID], n)  
```  
  
## <a name="applies-to"></a>Применение  
 Ссылка на скалярный столбец или столбец таблицы.  
  
## <a name="return-type"></a>Тип возвращаемых данных  
 \<scalar column reference>  
  
 or  
  
 \<table column reference>  
  
 Тип возвращаемых данных зависит от типа столбца, к которому применяется функция.  
  
> [!NOTE]  
>  Параметры INCLUSIVE, EXCLUSIVE, INPUT_ONLY и INCLUDE_STATISTICS применяются только к ссылкам на столбцы таблицы, а EXCLUDE_NULL и INCLUDE_NULL — только к ссылкам на скалярные столбцы.  
  
## <a name="remarks"></a>Комментарии  
 Имеются следующие параметры: EXCLUDE_NULL (по умолчанию), INCLUDE_NULL, INCLUSIVE, EXCLUSIVE (по умолчанию), INPUT_ONLY и INCLUDE_STATISTICS.  
  
> [!NOTE]  
>  Для моделей временных рядов функция Predict не поддерживает INCLUDE_STATISTICS.  
  
 Параметр INCLUDE_NODE_ID в качестве результата возвращает столбец $NODEID. NODE_ID является узлом содержимого, на котором прогнозирование осуществляется для определенного объекта. Этот параметр является необязательным при использовании Predict для столбцов таблицы.  
  
 Параметр *n* применяется к столбцам таблицы. В зависимости от типа прогноза данный параметр задает количество возвращаемых строк. Если базовый столбец является последовательностью, он вызывает функцию **PredictSequence** . Если базовый столбец имеет временные ряды, то вызывается функция **PredictTimeSeries** . Для ассоциативных типов прогноза вызывается функция **PredictAssociation** .  
  
 Функция **Predict** поддерживает полиморфизм.  
  
 Часто используются следующие сокращенные формы:  
  
-   [Gender] является альтернативой для **Predict**([Gender], EXCLUDE_NULL).  
  
-   [Покупки продуктов] — это альтернатива для **прогнозирования**([покупки продуктов], EXCLUDE_NULL, исключительные).  
  
    > [!NOTE]  
    >  Результатом выполнения данной функции является ссылка на столбец. Это означает, что функцию **Predict** можно использовать в качестве аргумента в других функциях, которые принимают ссылку на столбец в качестве аргумента (за исключением самой функции **Predict** ).  
  
 При передаче INCLUDE_STATISTICS прогнозу по столбцу, возвращающему табличное значение, в результирующую таблицу добавляются столбцы **$Probability** и **$support** . В данных столбцах содержится сведения о вероятности существования для соответствующей записи вложенной таблицы.  
  
## <a name="examples"></a>Примеры  
 В следующем примере функция Predict используется для возврата четырех продуктов в базе данных Adventure Works, которые, скорее всего, будут продаваться вместе. Поскольку функция прогнозируется для модели интеллектуального анализа правил взаимосвязей, она автоматически использует функцию **PredictAssociation** , как описано выше.  
  
```  
SELECT  
    Predict([Association].[v Assoc Seq Line Items],INCLUDE_STATISTICS,4)  
FROM     [Association]  
```  
  
 Образец результатов.  
  
 Этот запрос возвращает одну строку данных с одним столбцом, `Expression`, но этот столбец содержит следующую вложенную таблицу.  
  
|Модель|$SUPPORT|$PROBABILITY|$ADJUSTEDPROBABILITY|  
|-----------|--------------|------------------|--------------------------|  
|Sport-100|4334|0.291283016331743|0.252695851192499|  
|Фляга для воды|2866|0.192620471805901|0.175205052318795|  
|Ремонтный комплект|2113|0.142012232004839|0.132389356196586|  
|Камера шины для велосипеда Mountain|1992|0.133879965051415|0.125304947722259|
