# <a name="understanding-the-dmx-select-statement"></a>Общие сведения об инструкции расширения интеллектуального анализа данных SELECT
[!INCLUDE[ssas-appliesto-sqlas](../includes/ssas-appliesto-sqlas.md)]

  Инструкция [SELECT](../dmx/select-dmx.md) является основанием для большинства запросов, создаваемых с помощью расширений интеллектуального анализа данных (DMX [!INCLUDE[msCoName](../includes/msconame-md.md)] ) в. [!INCLUDE[ssNoVersion](../includes/ssnoversion-md.md)] [!INCLUDE[ssASnoversion](../includes/ssasnoversion-md.md)] Она служит для выполнения множества различных видов задач, например обзора или прогноза по моделям интеллектуального анализа данных.  
  
 Ниже перечислены задачи, которые можно выполнить с помощью инструкции **SELECT** :  
  
-   Обзор модели интеллектуального анализа данных. Набор строк схемы определяет структуру модели.  
  
-   Поиск возможных значений столбца модели интеллектуального анализа данных.  
  
-   Обзор вариантов, назначенных узлам модели интеллектуального анализа данных, или получение представления варианта.  
  
-   Создание прогнозов с помощью различных входных данных.  
  
-   Копирование моделей интеллектуального анализа данных.  
  
 Каждая из этих задач использует другой набор данных, который мы будем называть *доменом данных*. Домен данных определяется в предложении **from** инструкции.  
  
-   Требуется найти объекты в самой модели интеллектуального анализа данных, например правило, которое определяет набор данных или формулу, которая используется для формирования прогнозов.  
  
     В этом случае необходимо просмотреть метаданные, которые хранятся в самой модели. Поэтому доменом данных являются столбцы в наборе строк схемы интеллектуального анализа данных.  
  
-   Вам необходимо получить подробные сведения от вариантов, которые использовались для построения модели.  
  
     В этом случае необходимо выполнить детализацию структуру интеллектуального анализа данных, которая и является доменом данных, и посмотреть на отдельные строки в таких столбцах как Gender, Bike Buyer и так далее.  
  
 **ВНИМАНИЕ!** Все элементы, включенные в список выражений или в предложение **WHERE** , должны поступать из домена данных, определенного предложением **from** . Смешивать домены данных нельзя.

###  <a name="Predicting"></a>Прогнозирование  
 Прогнозы можно выполнять на основе модели интеллектуального анализа данных, используя следующие типы запросов.  
  
 Можно включить любую инструкцию **выбора** или прогнозирования в предложениях **from** и **WHERE** прогнозного объединения **SELECT** .  
  
|Тип запроса|Описание|  
|----------------|-----------------|  
|ВЫБОР ИЗ [ЕСТЕСТВЕННОГО] ПРОГНОЗИРУЮЩЕГО ПОДКЛЮЧЕНИЯ|Возвращает прогноз, созданный путем соединения столбцов модели интеллектуального анализа данных со столбцами внутреннего источника данных.<br /><br /> Доменом для этого типа запроса являются прогнозируемые столбцы модели и столбцы входного источника данных.<br /><br /> [ВЫБОР из &#60;модели&#62; интеллектуального &#40;анализа данных прогнозного подключения&#41;](../dmx/select-from-model-prediction-join-dmx.md)<br /><br /> [Прогнозирующие запросы (интеллектуальный анализ данных)](https://docs.microsoft.com/analysis-services/data-mining/prediction-queries-data-mining)|  
|Выбор из  *\<> модели*|Возвращает наиболее вероятное состояние прогнозируемого столбца исключительно на основе модели интеллектуального анализа данных. Этот тип запроса представляет собой быстрый способ создания прогноза с пустым прогнозируемым соединением.<br /><br /> Доменом для этого типа запроса являются прогнозируемые столбцы модели.<br /><br /> [Выбор из &#60;расширений&#62; &#40;интеллектуального анализа данных модели&#41;](../dmx/select-from-model-dmx.md)<br /><br /> [Прогнозирующие запросы (интеллектуальный анализ данных)](https://docs.microsoft.com/analysis-services/data-mining/prediction-queries-data-mining)|

###  <a name="Browsing"></a>Поиска  
 Можно выполнять обзор содержимого модели интеллектуального анализа данных, используя следующие типы запросов.  
  
|Тип запроса|Описание|  
|----------------|-----------------|  
|Выберите DISTINCT из  *\<> модели*|Возвращает все значения состояний модели интеллектуального анализа данных для заданного столбца.<br /><br /> Доменом данных для этого типа запроса является модель интеллектуального анализа данных.<br /><br /> [Выберите DISTINCT из &#60;расширений &#62; &#40;интеллектуального анализа данных модели&#41;](../dmx/select-distinct-from-model-dmx.md)<br /><br /> [Запросы содержимого (интеллектуальный анализ данных)](https://docs.microsoft.com/analysis-services/data-mining/content-queries-data-mining)|  
|Выберите из  *\<> модели*. СОДЕРЖАНИ|Возвращает содержимое, описывающее модель интеллектуального анализа данных.<br /><br /> Доменом данных для этого типа запроса является набор строк схемы содержимого.<br /><br /> [Выберите из &#60;модели&#62;. расширения &#40;интеллектуального анализа содержимого&#41;](../dmx/select-from-model-content-dmx.md)<br /><br /> [Запросы содержимого (интеллектуальный анализ данных)](https://docs.microsoft.com/analysis-services/data-mining/content-queries-data-mining)|  
|Выберите из  *\<> модели*. DIMENSION_CONTENT|Возвращает содержимое, описывающее модель интеллектуального анализа данных.<br /><br /> Доменом данных для этого типа запроса является набор строк схемы содержимого.<br /><br /> [Выберите из &#60;модели&#62;. расширения &#40;интеллектуального анализа данных DIMENSION_CONTENT&#41;](../dmx/select-from-model-dimension-content-dmx.md)|  
|Выберите из  *\<> модели*. СПЕЦИФИКАЦИ|Возвращает представление модели интеллектуального анализа данных на языке разметки моделей прогнозов (PMML) для алгоритмов, поддерживающих такую функцию.<br /><br /> Доменом для этого типа запроса является набор строк схемы PMML.<br /><br /> [Набор строк DMSCHEMA_MINING_MODEL_CONTENT_PMML](https://docs.microsoft.com/bi-reference/schema-rowsets/data-mining/dmschema-mining-model-content-pmml-rowset)|

###  <a name="Copying"></a>Идет  
 Можно копировать модель интеллектуального анализа данных и связанную с ней структуру интеллектуального анализа данных в новую модель, а затем переименовать эту модель в рамках инструкции.  
  
|Тип запроса|Описание|  
|----------------|-----------------|  
|Выберите новую  *модель>\<*|Создает копию модели интеллектуального анализа данных.<br /><br /> Доменом для этого типа запроса является модель интеллектуального анализа данных.<br /><br /> [ВЫБОР &#40;РАСШИРЕНИЙ ИНТЕЛЛЕКТУАЛЬНОГО АНАЛИЗА ДАННЫХ&#41;](../dmx/select-into-dmx.md)|
