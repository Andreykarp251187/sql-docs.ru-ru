# <a name="lesson-2-add-data"></a>Урок 2. Добавление данных
[!INCLUDE[ssas-appliesto-sql2016-later-aas](../../includes/ssas-appliesto-sql2016-later-aas.md)]

На этом занятии вы используете мастер импорта таблиц в SSDT для подключения к базе данных AdventureWorksDW SQL, выбора данных, предварительного просмотра и фильтрации данных и затем импортировать данные в рабочую область модели.  
  
С помощью мастера импорта таблиц, можно импортировать данные из различных реляционных источников: Access, SQL, Oracle, Sybase, Informix, DB2, Teradata и многое другое. Шаги по импорту данных из каждого из этих реляционных источников очень похожи на те, которые будут описаны ниже. Можно также выбрать данные с помощью хранимой процедуры. Дополнительные сведения об импорте данных и различные типы источников данных, можно импортировать из см. в разделе [источников данных](../tabular-models/data-sources-supported-ssas-tabular.md).  
  
Предполагаемое время для выполнения этого занятия: **20 минут**  
  
## <a name="prerequisites"></a>предварительные требования  
Этот раздел является частью учебника по табличному моделированию, который необходимо изучать по порядку. Перед выполнением задач на этом занятии, необходимо завершить предыдущее занятие: [Занятие 1. Создание нового проекта табличной модели](lesson-1-create-a-new-tabular-model-project.md).  
  
## <a name="create-a-connection"></a>Создание подключения  
  
#### <a name="to-create-a-connection-to-a-the-adventureworksdw2014-database"></a>Чтобы создать подключение к базе данных AdventureWorksDW2014  
  
1.  В обозревателе табличных моделей, щелкните правой кнопкой мыши **источников данных** > **Импорт из источника данных**.  
  
    Будет запущен мастер импорта таблицы, который поможет установить соединение с источником данных. Если вы не видите обозреватель табличных моделей, дважды щелкните **Model.bim** в **обозревателе решений** для открытия модели в конструкторе. 
    
    ![как табличных — занятие 2 tme](media/as-tabular-lesson2-tme.png) 

    Примечание. При создании модели с уровнем совместимости 1400, вы увидите новый интерфейс получения данных, вместо мастера импорта таблиц. Диалоговые окна будут отображаться некоторые отличия от указанные ниже действия, но по-прежнему смогут выполнения этой процедуры. 
  
2.  В мастере импорта таблиц в разделе **реляционных баз данных**, нажмите кнопку **Microsoft SQL Server** > **Далее**.  
  
3.  На странице **Подключение к базе данных Microsoft SQL Server** в поле **Понятное имя подключения**введите **База данных Adventure Works из SQL**.  
  
4.  В **имя сервера**, введите имя сервера, на котором установлена база данных AdventureWorksDW.  
  
5.  В **имя базы данных** выберите **AdventureWorksDW**, а затем нажмите кнопку **Далее**.  
  
    ![как табличных-занятие 2 tiw-name](media/as-tabular-lesson2-tiw-name.png)
  
6.  На странице **Сведения об олицетворении** нужно указать учетные данные, которые службы Analysis Services будут использовать для подключения к источнику данных во время импорта и обработки данных. Убедитесь, что пункт **Имя и пароль определенного пользователя Windows** выбран, после чего в поле **Имя пользователя** и **Пароль**введите учетные данные для входа в систему Windows и нажмите кнопку **Далее**.  
  
    > [!NOTE]  
    > Использование другой пользовательской учетной записи и пароля Windows обеспечивает наиболее безопасный метод подключения к источнику данных. Дополнительные сведения см. в разделе [олицетворения](../tabular-models/impersonation-ssas-tabular.md).  
  
7.  На странице **Выбор способа импорта данных** убедитесь, что пункт **Выбрать из списка таблиц и представлений данные для импорта** выбран. Поскольку необходимо выбрать из списка таблиц и представлений, нажмите **Далее** , чтобы отобразить список всех исходных таблиц в базе данных-источнике.  
  
8.  В **Выбор таблиц и представлений** странице, установите флажок напротив следующих таблиц: **DimCustomer**, **DimDate**, **DimGeography**, **DimProduct**, **DimProductCategory**,  **DimProductSubcategory**, и **FactInternetSales**.  
  
    **НЕ** нажимайте кнопку **Готово**.  
  
## <a name="FilterData"></a>Filter the table data  
Таблица DimCustomer, импортируемые из образца базы данных содержит подмножество данных из исходной базы данных SQL Server Adventure Works. Вы отфильтруете некоторые новые столбцы из таблицы DimCustomer, которые не являются обязательными при импорте в модели. Если это возможно, необходимо отфильтровать данные, которые не будут использоваться для экономии места в памяти, используемого моделью.  
  
#### <a name="to-filter-the-table-data-prior-to-importing"></a>Фильтрация данных таблицы перед импортом  
  
1.  Выберите строку для **DimCustomer** таблицы, а затем нажмите кнопку **Просмотр и фильтрация**. Откроется окно **Предварительный просмотр выбранной таблицы** , в котором будут показаны все столбцы в таблице DimCustomer.  
  
2.  Снимите флажки над следующими столбцами. **SpanishEducation**, **FrenchEducation**, **SpanishOccupation**, **FrenchOccupation**. 

    ![как табличных — занятие 2 tiw-clear](media/as-tabular-lesson2-tiw-clear.png)
  
    Поскольку значения для этих столбцов не являются актуальными для анализа интернет-продаж, нет необходимости их импортировать. Исключение ненужных столбцов сделает ваша модель становится компактнее и эффективнее.  
  
3.  Убедитесь, что флажки для всех остальных столбцов установлены, и нажмите кнопку **OK**.  
  
    Обратите внимание, что слова **примененные фильтры** теперь отображаются в **сведения о фильтре** столбца в **DimCustomer** строк; Если щелкнуть эту ссылку, вы увидите текстовое описание фильтры, которые были применены.  
    
    ![как табличных — занятие 2 примененные фильтры](media/as-tabular-lesson2-applied-filters.png)
    
  
4.  Отфильтруйте оставшиеся таблицы, сняв флажки для следующих столбцов в каждой таблице:

## <a name="Import"></a>Import the selected tables and column data  
Теперь, когда вы предварительного просмотра и фильтрации ненужных данных, можно импортировать оставшиеся данные, которые требуется. Мастер импортирует данные таблицы вместе со всеми связями между таблицами. Новые таблицы и столбцы создаются в модели и не будут импортированы данные, отфильтрованные.  
  
#### <a name="to-import-the-selected-tables-and-column-data"></a>Импорт данных выбранных таблиц и столбцов  
  
1.  Просмотрите выбранные параметры. Если все выглядит правильно, нажмите кнопку **Готово**.  
  
    При импорте данных мастер отображает количество выбранных строк. После импорта всех данных отображается сообщение, указывающее на успешное завершение процесса.  
    
    ![как — табличных — занятие 2-успешное завершение](media/as-tabular-lesson2-success.png) 
  
    > [!TIP]  
    > Для просмотра связей, которые были автоматически созданы между импортированными таблицами, в строке **Подготовка данных** щелкните **Подробные сведения**. 
  
2.  Щелкните **Закрыть**.  
  
    Мастер закроется, и конструктор моделей отобразит импортированные таблицы.
