# <a name="impersonation"></a>Олицетворение 
[!INCLUDE[ssas-appliesto-sqlas-aas](../../includes/ssas-appliesto-sqlas-aas.md)]
  В этой статье предоставляет разработчикам табличных моделей понимания как входа в учетные данные используются службами Analysis Services при подключении к источнику данных для импорта и обработки данных (Обновить).  

##  <a name="bkmk_conf_imp_info"></a> Настройка олицетворения  
 Где и в каком контексте модель существует определяет настройки сведений об олицетворении. При создании нового проекта модели, олицетворения при подключении к источнику данных для импорта данных настраивается в SQL Server Data Tools (SSDT). После развертывания модели можно настроить олицетворение в свойство строки подключения базы данных модели с помощью SQL Server Management Studio (SSMS). Для табличных моделей в службах Azure Analysis Services, вы можете использовать SSMS или **просмотреть как: Сценарий** режим в конструкторе на основе веб-обозревателя, чтобы изменять файл Model.bim в формате JSON.
  
##  <a name="bkmk_how_imper"></a> Использование олицетворения  
 *Олицетворение* — это способность серверного приложения, например служб Analysis Services, выступать от имени клиентского приложения. Analysis Services работают под учетной записью службы, тем не менее, когда сервер устанавливает подключение к источнику данных, он использует олицетворение, чтобы проверка доступа для импорта и обработки данных может быть выполнена.  
  
 Учетные данные, используемые для олицетворения, отличаются от учетных данных, которые вы вошли на с. Вход в учетные данные пользователя используются для определенных операций на стороне клиента при создании модели.  
  
 Очень важно, чтобы понять, как учетные данные олицетворения указываются и защищаются, а также разницу между контекстами, в которых оба свое подписанное на используются учетные данные пользователя, и если используются другие учетные данные олицетворения.  
  
 **Основные сведения об учетных данных на стороне сервера**  
 
При импорте или обработке данных, учетные данные олицетворения используются для подключения к источнику данных и получения данных. Это подключение является *серверных* выполняется в контексте клиентского приложения, поскольку сервер служб Analysis Services, базой данных рабочей области, подключается к источнику данных и получает данные.  
  
 Если при развертывании модели на сервере служб Analysis Services база данных рабочей области находится в памяти, учетные данные передаются на сервер служб Analysis Services, на котором выполняется развертывание модели. Учетные данные пользователя не сохраняются на диске.  
  
 Когда развернутая модель обрабатывает данные из источника данных, учетные данные олицетворения, сохраняются в базе данных в памяти, используются для подключения к источнику данных и получения данных. Так как этот процесс обрабатывается сервером служб Analysis Services, управление базой данных модели, это подключение еще раз является операцией на стороне сервера.  
  
 **Основные сведения об учетных данных на стороне клиента**  
  
 При создании новой модели или добавлении источника данных к существующей модели, подключения к источнику данных и Выбор таблиц и представлений для импорта в модель. На странице мастера импорта таблиц или получить конструктор Data\Query Предварительный просмотр и фильтр компонентов см. образец данных, которые можно импортировать. Также можно указать фильтры для исключения данных, в модели не требуется.  
  
 Аналогичным образом для существующих моделей, которые уже были созданы, использовать **свойства таблицы** диалогового окна, чтобы просматривать и фильтровать данные, импортируемые в таблицу.  
  
 Функции предварительного просмотра и фильтрации, **свойства таблицы**, и **диспетчер секций** диалоговые окна — это в процесс *клиентского* операции, это означает, что происходит во время это операции, отличаются от как к источнику данных и данные извлекаются из источника данных; серверную операцию. Учетные данные пользователя, выполнившего используются учетные данные для предварительного просмотра и фильтрации данных. На самом деле, учетные данные. 
  
 Разделение учетных данных, используемых во время серверных и клиентских операций может привести к несоответствию того, что вы видите, и какие данные извлекаются во время импорта или обработки (операции на стороне сервера). Если учетные данные сейчас вы вошли в систему и учетные данные олицетворения различаются, данные, вы увидите в компонентах предварительной версии и фильтр или **свойства таблицы** диалоговое окно и данных, полученных в результате импорта или процесс может отличаться в зависимости от учетные данные, необходимые источнику данных.  
  
> [!IMPORTANT]  
>  При создании модели, убедитесь, учетные данные, которые вы вошли в систему и заданным для олицетворения учетные данные имеют достаточные права для получения данных из источника данных.
  
##  <a name="bkmk_imp_info_options"></a> Параметры  
 При настройке олицетворения или при изменении свойств для существующего соединения источника данных, укажите один из следующих вариантов:  
  
**Табличные модели 1400 и более поздних версий**
 
|Параметр|Описание|  
|------------|-----------------|  
|**Выполнять олицетворение учетной записи**|Указывает модели использовать учетную запись пользователя Windows для импорта или обработки данных из источника данных. Домен и имя учетной записи пользователя используют следующий формат: **\<доменное имя >\\< имя учетной записи пользователя\>** .|  
|**Олицетворение текущего пользователя**|Указывает, что данные должны поступать из источника данных, используя удостоверение пользователя, отправившего запрос. Этот параметр применяется только в режиме DirectQuery.|  
|**Олицетворение удостоверения**|Задает имя пользователя для доступа к источнику данных, но не требует указания пароля для учетной записи. Этот параметр применяется только в том случае, если включено делегирование Kerberos и указывает, что следует использовать проверку подлинности S4U.|  
|**Выполнять олицетворение учетной записи службы**|Указывает модели использовать учетные данные безопасности, связанные с экземпляром службы Analysis Services, управляющей моделью.|  
|**Олицетворить автоматическую учетную запись**|Указывает подсистему служб Analysis Services следует использовать предварительно настроенные автоматической учетной записи для доступа к данным.|  

> [!IMPORTANT]
> Олицетворение текущего пользователя не поддерживается в некоторых средах. Олицетворение текущего пользователя не поддерживается для табличных моделей, развертываемых в Azure Analysis Services для подключения к локальным источникам данных. Так как ресурс сервера Azure Analysis Services не подключен к домену организации, учетные данные клиента не может пройти проверку подлинности на сервере источника данных в этот домен. Службы Azure Analysis Services также не поддерживает интеграцию в настоящее время база данных SQL (Azure) поддержка единого входа (SSO). В зависимости от среды другие параметры олицетворения также имеют ограничения. При попытке использовать параметр олицетворения не поддерживается, возвращается ошибка. 


**Табличных моделей 1200**
 
|Параметр|Описание|  
|------------|-----------------|  
|**Конкретное имя пользователя Windows и пароль**|Этот параметр указывает модели использовать учетную запись пользователя Windows для импорта или обработки данных из источника данных. Домен и имя учетной записи пользователя используют следующий формат: **\<доменное имя >\\< имя учетной записи пользователя\>** . При создании новой модели с помощью мастера импорта таблиц, этот параметр является параметром по умолчанию.|  
|**Учетная запись службы**|Этот параметр указывает модели использовать учетные данные безопасности, связанные с экземпляром служб Analysis Services, управляющим моделью.|  
  
##  <a name="bkmk_impers_sec"></a> безопасность  
 Учетные данные, используемые для олицетворения, сохраняются в памяти подсистемой VertiPaq™. Учетные данные никогда не записываются на диск. Если база данных рабочей области не в памяти, при развертывании модели, пользователю предлагается ввести учетные данные, используемые для подключения к источнику данных и получения данных.  
  
> [!NOTE]  
>  В качестве учетных данных для олицетворения рекомендуется указать учетную запись и пароль пользователя Windows. Наименьшие права доступа, необходимые для подключения и чтения данных из источника данных можно настроить учетную запись пользователя Windows.
