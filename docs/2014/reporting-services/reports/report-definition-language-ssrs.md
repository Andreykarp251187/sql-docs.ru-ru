# <a name="report-definition-language-ssrs"></a>Язык определения отчетов (SSRS)
  Язык определения отчетов (RDL) — это XML-представление [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] [!INCLUDE[ssRSnoversion](../../includes/ssrsnoversion-md.md)] определение отчета. Определение отчета содержит данные извлечения и сведения о макете отчета. Язык определения Отчетов состоит из XML-элементов, которые соответствуют грамматике XML, созданной для [!INCLUDE[ssRSnoversion](../../includes/ssrsnoversion-md.md)]. Вы можете добавить собственные функции для управления значениями элементов отчета, стилями и форматированием путем обращения к сборкам кода в файлах определения отчета.  
  
 Язык определения Отчетов способствует функциональной совместимости коммерческих продуктов формирования, определив общую схему, которая обеспечивает обмен определениями отчетов. С помощью языка определения Отчетов можно использовать любой протокол или программный интерфейс, который работает с XML. Язык определения Отчетов является:  
  
-   Схема XML для определений отчетов.  
  
-   Формат обмена для организаций и сторонних разработчиков.  
  
-   Схему расширяемую и открытую, которая поддерживает дополнительные пространства имен и пользовательские элементы.  
  
##  <a name="bkmk_RDL_Specifications"></a> Спецификации RDL  
 Загрузки спецификаций для конкретных версий схемы, см. в разделе [спецификация языка определения отчетов](https://go.microsoft.com/fwlink/?linkid=116865).  
  
##  <a name="bkmk_RDL_XML_Schema_Definition"></a> Определение схемы XML языка определения Отчетов  
 Объект [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] [!INCLUDE[ssRSnoversion](../../includes/ssrsnoversion-md.md)] файл языка определения отчетов (RDL) проверяется с помощью файла определения схемы XML (XSD). Схема определяет правила применения элементов языка определения Отчетов можно в RDL-файл. Элемент содержит его тип данных и количество элементов, то есть количество вхождений, которые разрешены. Элемент могут быть простыми или сложными. Простой элемент не имеет дочерние элементы или атрибуты. Составной элемент иметь дочерние узлы и при необходимости атрибуты.  
  
 Например, следующая схема включает элемент языка определения Отчетов `ReportParameters`, который является сложным типом `ReportParametersType`. По соглашению, сложный тип для элемента является имя элемента, за которым следует слово `Type`. Объект `ReportParameters` могут содержаться элемент `Report` элемент (сложного типа) и может содержать `ReportParameter` элементов. Объект `ReportParameterType` — это простой тип, который может принимать только одно из следующих значений: `Boolean`, `DateTime`, `Integer`, `Float`, или `String`. Дополнительные сведения о типах данных схем XML, см. в разделе [XML Schema Part 2: DataTypes Second Edition](https://go.microsoft.com/fwlink/?linkid=4871).  
  
 XSD-СХЕМЫ языка определения Отчетов доступна в файле ReportDefinition.xsd, расположенном в папке Extras на компакт-диске продукта. Его также можно найти на сервере отчетов по следующему URL-АДРЕСУ: http://servername/reportserver/reportdefinition.xsd.  
  
##  <a name="bkmk_Creating_RDL"></a> Создание RDL  
 Из-за характера открытость и расширяемость языка определения отчетов широкий набор средств и приложений может быть построен, создание на основе его XML-схемы языка определения Отчетов.  
  
 [!INCLUDE[ssRSnoversion](../../includes/ssrsnoversion-md.md)] предоставляет несколько средств для создания RDL-файлов. Дополнительные сведения см. в разделе [средств служб Reporting Services](../tools/reporting-services-tools.md).  
  
 Одним из самых простых способов создания языка определения Отчетов в приложении является использование [!INCLUDE[msCoName](../../includes/msconame-md.md)] [!INCLUDE[dnprdnshort](../../includes/dnprdnshort-md.md)] классы <xref:System.Xml> пространства имен и <xref:System.Linq> пространства имен. Один класс, в частности, **XmlTextWriter** , который может использоваться для записи языка определения Отчетов. С помощью **XmlTextWriter**, можно создать полное определение отчета от начала до конца в любом [!INCLUDE[dnprdnshort](../../includes/dnprdnshort-md.md)] приложения. Кроме того, разработчики могут расширить языка определения Отчетов, добавляя пользовательские элементы отчета с пользовательскими свойствами. Дополнительные сведения о **XmlTextWriter** класс и <xref:System.Xml> пространства имен, см. в разделе [!INCLUDE[msCoName](../../includes/msconame-md.md)] [!INCLUDE[dnprdnshort](../../includes/dnprdnshort-md.md)] руководство разработчика. Дополнительные сведения о Language-Integrated Query (LINQ) найдите «LINQ to XML» на сайте MSDN.  
  
 Стандартное расширение файла для файлов определения отчета является RDL. Можно также разрабатывать файлы определения клиентских отчетов, которые имеют расширения RDLC. Тип MIME для обоих расширений является text/xml. Дополнительные сведения об отчетах см. в разделе [отчетов служб Reporting Services &#40;SSRS&#41;](reporting-services-reports-ssrs.md).  
  
##  <a name="bkmk_RDL_Types"></a> Типы RDL  
 В следующей таблице перечислены типы, используемые в элементах и атрибутах RDL.  
  
|Тип|Описание|  
|----------|-----------------|  
|`Binary`|Свойство с закодированным base-64 в кодировке двоичное значение.|  
|`Boolean`|Свойство с `true` или `false` в качестве значения объекта. Если не указано иное, значение дополнительного объекта Boolean `False`.|  
|`Date`|Свойство с полностью указанным значением даты или даты и времени в формате даты ISO8601: ГГГГ ММ ДД [THH:MM [: SS [. S]]].|  
|`Enum`|Свойство со строковым значением текста, который должен быть одним из списка определенных значений.|  
|`Float`|Свойство со значением с плавающей запятой. Точка (.) используется как необязательный десятичный разделитель.|  
|`Integer`|Свойство со значением целого типа (int32).|  
|`Language`|Свойство с текстовое значение, которое содержит код языка и региональных параметров, таких как «en-us» для английского языка США. Значение должно быть конкретного языка, либо нейтральным языком, для которого определен язык по умолчанию в [!INCLUDE[msCoName](../../includes/msconame-md.md)] [!INCLUDE[dnprdnshort](../../includes/dnprdnshort-md.md)].|  
|`Name`|Свойство со значением текстовой строки. Имена должны быть уникальными в пределах пространства имен элемента. Если не указан, пространство имен для элемента считается самое внутреннее, содержащий объект, который имеет имя.|  
|`NormalizedString`|Свойство со строковым значением, которое было нормализовано.|  
|`Size`|Элемент размера должен содержать число (с символом, используемым в качестве необязательного десятичного разделителя). Номер должен следовать указатель единицы измерения для единицы длины CSS, например: cm, mm, in, pt или pc. Пробел между числом и указателем является необязательным. Дополнительные сведения о единицах измерения см. в разделе [Справочник по единицам длины CSS](https://www.w3schools.com/CSSref/css_units.asp).<br /><br /> В RDL максимальное значение для `Size` равно 160 дюймам. Минимальный размер — 0 дюймов.|  
|`String`|Свойство со значением текстовой строки.|  
|`UnsignedInt`|Свойство со значением целого числа без знака (uint32).|  
|`Variant`|Свойство с любого простого типа XML.|
