# <a name="rowset-properties-and-behaviors"></a>Свойства и поведение наборов строк
  Это [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] свойства наборов строк поставщика OLE DB для собственного клиента.  
  
|Идентификатор свойства|Описание|  
|-----------------|-----------------|  
|DBPROP_ABORTPRESERVE|И ЗАПИСЬ: чтение и запись<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание. Этим свойством определяется поведение набора строк после аварийного завершения операции.<br /><br /> VARIANT_FALSE: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщика OLE DB для собственного клиента делает наборы строк недействительными после аварийного завершения операции. Объект набора строк практически утратил свою функциональность. Он поддерживает только операции **IUnknown**, а также освобождение необработанных строк и обработчики методов доступа.<br /><br /> VARIANT_TRUE: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщик OLE DB для собственного клиента поддерживает допустимый набор строк.|  
|DBPROP_ACCESSORDER|И ЗАПИСЬ: чтение и запись<br /><br /> По умолчанию: DBPROPVAL_AO_RANDOM<br /><br /> Описание. Порядок доступа. Порядок, в котором должен осуществляться доступ к столбцам в наборе строк.<br /><br /> DBPROPVAL_AO_RANDOM: Столбец может осуществляться в любом порядке.<br /><br /> DBPROPVAL_AO_SEQUENTIALSTORAGEOBJECTS: Столбцы привязаны как объекты хранилища доступен только в последовательном порядке, по порядковому номеру столбца.<br /><br /> DBPROPVAL_AO_SEQUENTIAL: Все столбцы должны осуществляться в последовательном порядке, по порядковому номеру столбца.|  
|DBPROP_APPENDONLY|Это свойство набора строк не реализован [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщика OLE DB для собственного клиента. При попытке считать или записать значение свойства возникает ошибка.|  
|DBPROP_BLOCKINGSTORAGEOBJECTS|И ЗАПИСЬ: Только для чтения<br /><br /> По умолчанию: VARIANT_TRUE<br /><br /> Описание. [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] OLE DB для собственного клиента поставщика хранилища объектов блока с помощью других методов набора строк.|  
|DBPROP_BOOKMARKS DBPROP_LITERALBOOKMARKS|И ЗАПИСЬ: чтение и запись<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание. [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщика OLE DB для собственного клиента поддерживает закладки для определения строки набора строк, если свойства DBPROP_BOOKMARKS или dbprop_literalbookmarks имеют значение VARIANT_TRUE.<br /><br /> Присвоение любому из этих свойств значения VARIANT_TRUE не включает позиционирование набора строк по закладке. Чтобы создать набор строк, поддерживающий позиционирование по закладке, присвойте свойству DBPROP_IRowsetLocate или DBPROP_IRowsetScroll значение VARIANT_TRUE.<br /><br /> [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Использует поставщик OLE DB для собственного клиента [!INCLUDE[msCoName](../../includes/msconame-md.md)] [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] курсора для поддержки набора строк, который содержит закладки. Дополнительные сведения см. в статье [Наборы строк и курсоры SQL Server](rowsets-and-sql-server-cursors.md). **Примечание.**  Настройка этих свойств, конфликтует с другой [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] свойства определяющими курсоры поставщика OLE DB для собственного клиента приводит к ошибке. Например, при попытке потребителя открыть набор строк, свойства DBPROP_BOOKMARKS и DBPROP_OTHERINSERT которого имеют значение VARIANT_TRUE, будет выдана ошибка.|  
|DBPROP_BOOKMARKSKIPPED|И ЗАПИСЬ: Только для чтения<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание. [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщика OLE DB для собственного клиента вернет значение DB_E_BADBOOKMARK, если потребитель указал недопустимую закладку при позиционировании или поиске набора строк с закладками.|  
|DBPROP_BOOKMARKTYPE|И ЗАПИСЬ: Только для чтения<br /><br /> По умолчанию: DBPROPVAL_BMK_NUMERIC<br /><br /> Описание. [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщика OLE DB для собственного клиента реализует только числовые закладки. Объект [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] закладкой поставщика OLE DB для собственного клиента является 32-разрядного целого числа без знака, имеющее тип DBTYPE_UI4.|  
|DBPROP_CACHEDEFERRED|Это свойство набора строк не реализован [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщика OLE DB для собственного клиента. При попытке считать или записать значение свойства возникает ошибка.|  
|DBPROP_CANFETCHBACKWARDS DBPROP_CANSCROLLBACKWARDS|И ЗАПИСЬ: чтение и запись<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание. [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщик OLE DB для собственного клиента поддерживает обратную выборку и прокрутку в непоследовательных наборах строк. [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщика OLE DB для собственного клиента создает набор строк, поддерживаемый курсорами, когда либо DBPROP_CANFETCHBACKWARDS и DBPROP_CANSCROLLBACKWARDS имеют значение VARIANT_TRUE. Дополнительные сведения см. в статье [Наборы строк и курсоры SQL Server](rowsets-and-sql-server-cursors.md).|  
|DBPROP_CANHOLDROWS|И ЗАПИСЬ: чтение и запись<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание. По умолчанию [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщика OLE DB для собственного клиента возвращает DB_E_ROWSNOTRELEASED при попытке потребителя получить больше строк для набора строк во время ожидающие изменения, в котором существуют в настоящее время в наборе строк. Это поведение может быть изменено.<br /><br /> Присвоение свойствам DBPROP_CANHOLDROWS и DBPROP_IRowsetChange значения VARIANT_TRUE подразумевает набор строк с закладками. Если оба свойства имеют значение VARIANT_TRUE, то для набора строк доступен интерфейс **IRowsetLocate**, а свойства DBPROP_BOOKMARKS и DBPROP_LITERALBOOKMARKS имеют значение VARIANT_TRUE.<br /><br /> [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Собственные наборы строк поставщика OLE DB для клиента, содержащие закладки, поддерживаются [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] курсоров.|  
|DBPROP_CHANGEINSERTEDROWS|И ЗАПИСЬ: чтение и запись<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание. Это свойство может быть присвоено только значение VARIANT_TRUE, если набор строк использует курсор, управляемый набором ключей.|  
|DBPROP_COLUMNRESTRICT|И ЗАПИСЬ: Только для чтения<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание. [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщика OLE DB для собственного клиента задает свойство значение VARIANT_TRUE, если потребитель не может изменить столбец в наборе строк. Другие столбцы в наборе строк можно изменять, а сами строки — удалять.<br /><br /> Если это свойство имеет значение VARIANT_TRUE, то потребитель по элементу *dwFlags* структуры DBCOLUMNINFO определяет, может ли быть записано значение отдельного столбца. Для изменяемых столбцов элемент *dwFlags* принимает значение DBCOLUMNFLAGS_WRITE.|  
|DBPROP_COMMANDTIMEOUT|И ЗАПИСЬ: чтение и запись<br /><br /> По умолчанию: 0<br /><br /> Описание. По умолчанию [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщика OLE DB для собственного клиента не простаивает на **ICommand::Execute** метод.|  
|DBPROP_COMMITPRESERVE|И ЗАПИСЬ: чтение и запись<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание. Этим свойством определяется поведение набора строк после выполнения фиксации.<br /><br /> VARIANT_TRUE: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщик OLE DB для собственного клиента поддерживает допустимый набор строк.<br /><br /> VARIANT_FALSE: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщика OLE DB для собственного клиента делает наборы строк недействительными после фиксации. Объект набора строк практически утратил свою функциональность. Он поддерживает только операции **IUnknown**, а также освобождение необработанных строк и обработчики методов доступа.|  
|DBPROP_DEFERRED|И ЗАПИСЬ: чтение и запись<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание. При значении VARIANT_TRUE [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщик OLE DB для собственного клиента пытается использовать серверный курсор для набора строк. Столбцы типа **text**, **ntext** и **image** не возвращаются с сервера до тех пор, пока приложение не обратится к ним.|  
|DBPROP_DELAYSTORAGEOBJECTS|И ЗАПИСЬ: Только для чтения<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание. [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщик OLE DB для собственного клиента поддерживает режим немедленного обновления в объектах хранилища.<br /><br /> Изменения, примененные к данным в объекте последовательного потока, немедленно передаются [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]. Фиксация изменений производится в соответствии с режимом транзакций набора строк.|  
|DBPROP_HIDDENCOLUMNS|И ЗАПИСЬ: Только для чтения<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> **Описание:** Число скрытых столбцов.<br /><br /> Если DBPROP_UNIQUEROWS имеет значение VARIANT_TRUE, то свойство DBPROP_HIDDENCOLUMNS возвращает число дополнительных «скрытых» столбцов, добавленных поставщиком для уникальной идентификации строк в наборе. Эти столбцы возвращаются методами **IColumnsInfo::GetColumnInfo** и **IColumnsRowset::GetColumnsRowset**. Однако они не включаются в число строк, которое возвращается в аргументе *pcColumns* метода **IColumnsInfo::GetColumnInfo**.<br /><br /> Чтобы выяснить общее число столбцов, представленных в структуре *prgInfo*, которую возвращает метод **IColumnsInfo::GetColumnInfo** (включая скрытые), потребитель должен добавить значение свойства DBPROP_HIDDENCOLUMNS к числу столбцов, возвращаемых методом **IColumnsInfo::GetColumnInfo** в *pcColumns*. Если свойство DBPROP_UNIQUEROWS имеет значение VARIANT_FALSE, то свойство DBPROP_HIDDENCOLUMNS равно нулю.|  
|DBPROP_IAccessor DBPROP_IColumnsInfo DBPROP_IConvertType DBPROP_IRowset DBPROP_IRowsetInfo|И ЗАПИСЬ: Только для чтения<br /><br /> По умолчанию: VARIANT_TRUE<br /><br /> Описание. [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщика OLE DB для собственного клиента поддерживает эти интерфейсы для всех наборов строк.|  
|DBPROP_IColumnsRowset|И ЗАПИСЬ: чтение и запись<br /><br /> По умолчанию: VARIANT_TRUE<br /><br /> Описание. [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поддерживает поставщик OLE DB для собственного клиента **IColumnsRowset** интерфейс.|  
|DBPROP_IConnectionPointContainer|И ЗАПИСЬ: чтение и запись<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание. IConnectionPointContainer. Если значение равно VARIANT_TRUE, то набор строк поддерживает заданный интерфейс. Если значение равно VARIANT_FALSE, то набор строк не поддерживает заданный интерфейс. Поставщики, поддерживающие тот или иной интерфейс, должны поддерживать связанное с этим интерфейсом свойство, которое возвращает значение VARIANT_TRUE. Эти свойства служат главным образом для запроса интерфейсов с помощью метода ICommandProperties::SetProperties.|  
|DBPROP_IMultipleResults|И ЗАПИСЬ: чтение и запись<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание. [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поддерживает поставщик OLE DB для собственного клиента **IMultipleResults** интерфейс.|  
|DBPROP_IRowsetChange DBPROP_IRowsetUpdate|И ЗАПИСЬ: чтение и запись<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание. [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поддерживает поставщик OLE DB для собственного клиента **IRowsetChange** и **IRowsetUpdate** интерфейсов.<br /><br /> Набор строк, созданный с DBPROP_IRowsetChange, равным VARIANT_TRUE, отражает режимы немедленного обновления.<br /><br /> Если DBPROP_IRowsetUpdate имеет значение VARIANT_TRUE, то DBPROP_IRowsetChange также имеет значение VARIANT_TRUE. Набор строк характеризуется поведением отложенного обновления.<br /><br /> [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Использует поставщик OLE DB для собственного клиента [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] курсора для поддержки набора строк **IRowsetChange** или **IRowsetUpdate**. Дополнительные сведения см. в статье [Наборы строк и курсоры SQL Server](rowsets-and-sql-server-cursors.md).|  
|DBPROP_IRowsetIdentity|И ЗАПИСЬ: чтение и запись<br /><br /> По умолчанию: VARIANT_TRUE<br /><br /> Описание. [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поддерживает поставщик OLE DB для собственного клиента **IRowsetIdentity** интерфейс. Если набор строк поддерживает этот интерфейс, то любые два дескриптора строк, относящихся к одной базовой строке, всегда представляют одни и те же данные и состояние. Потребители могут вызвать **IRowsetIdentity:: IsSameRow** метод для сравнения двух дескрипторов строк, чтобы увидеть, если они ссылаются на один и тот же экземпляр строки.|

[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщика OLE DB для собственного клиента присваивает свойствам DBPROP_BOOKMARKS и DBPROP_LITERALBOOKMARKS значение VARIANT_TRUE, если потребитель запрашивает один из этих интерфейсов.<br /><br /> [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Использует поставщик OLE DB для собственного клиента [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] курсоры для поддержки **IRowsetLocate** и **IRowsetScroll**. Дополнительные сведения см. в статье [Наборы строк и курсоры SQL Server](rowsets-and-sql-server-cursors.md).<br /><br /> Настройка этих свойств, конфликтует с другой [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] свойства определяющими курсоры поставщика OLE DB для собственного клиента приводит к ошибке. Например, при попытке потребителя открыть набор строк, свойства DBPROP_IRowsetScroll и DBPROP_OTHERINSERT которого имеют значение VARIANT_TRUE, будет выдана ошибка.|

[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщика OLE DB для собственного клиента может поддерживать этот интерфейс в любом наборе строк.|  
|DBPROP_ISupportErrorInfo|И ЗАПИСЬ: чтение и запись<br /><br /> По умолчанию: VARIANT_TRUE<br /><br /> Описание. [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщик OLE DB для собственного клиента предоставляет **ISupportErrorInfo** интерфейс наборов строк.|  
|DBPROP_ILockBytes|Этот интерфейс реализуется не [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщика OLE DB для собственного клиента. При попытке считать или записать свойство возникает ошибка.|

|DBPROP_IStorage|Этот интерфейс реализуется не [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщика OLE DB для собственного клиента. При попытке считать или записать свойство возникает ошибка.|

|DBPROP_IMMOBILEROWS|И ЗАПИСЬ: чтение и запись<br /><br /> По умолчанию: VARIANT_TRUE<br /><br /> Описание. Свойство имеет значение VARIANT_TRUE только для [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] курсоров; для всех остальных курсоров оно имеет значение VARIANT_FALSE.<br /><br /> VARIANT_TRUE: Набор строк не будет переупорядочивать вставленные или обновленные строки. Для метода **IRowsetChange::InsertRow** строки появляются в конце набора. Если набор строк не упорядочен, то метод **IRowsetChange::SetData** не меняет положение обновленных строк. Если метод **IRowsetChange::SetData** изменяет столбец, который использовался для упорядочивания набора строк, то строка будет перемещена. Если набор строк не упорядочен, то положение строки не меняется.|
