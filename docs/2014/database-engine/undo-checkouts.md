# <a name="undo-checkouts"></a>Отмена извлечения
  Чтобы отменить существующее извлечение, следует использовать команду **Отмена извлечения** . Она особенно удобна, если необходимо выполнить откат изменений после сохранения их в файле.  
  
 В зависимости от параметров, установленных в диалоговом окне **Дополнительные параметры отмены извлечений** , среда Studio либо оставляет работающую копию элемента на диске, либо заменяет ее последней версией из системы управления версиями. Если изменения элемента были произведены вне контекста системы управления версиями, полученная версия может оказаться не последней.  
  
### <a name="to-undo-a-checkout"></a>Отмена извлечения  
  
1.  В обозревателе решений выберите проект.  
  
2.  В меню **Файл** укажите пункт **Управление версиями**, затем выберите **Отменить извлечение**.  
  
3.  В диалоговом окне **Отмена извлечения** выберите соответствующие параметры и нажмите кнопку **Отменить извлечение** .  
  
     **Столбцы**  
     Определите отображаемые столбцы и порядок, в котором они будут отображаться.  
  
     **Плоское представление**  
     Позволяет отобразить элементы в виде плоских списков внутри соответствующего подключения к системе управления версиями.  
  
     **Имя**  
     Отображает имена элементов, для которых нужно отменить извлечение. Напротив элементов отображаются флажки, которые установлены. Если не требуется отменять извлечение какого-либо элемента, снимите соответствующий флажок.  
  
     **Параметры**  
     При нажатии стрелки справа от кнопки отображаются параметры отмены извлечения, специфические для конкретного модуля управления версиями.  
  
     **Сортировка**  
     Отсортируйте порядок отображаемых столбцов.  
  
     **Структура в виде дерева**  
     Отображает иерархию папок и элементов для элементов, извлечение которых отменяется.  
  
     **Отмена извлечения**  
     Позволяет отменить извлечение, при этом отменяются какие-либо изменения в извлекаемом файле.
