# <a name="project-settings-conversion-db2tosql"></a>Параметры проекта (преобразование) (DB2ToSQL)
На странице преобразования **параметры проекта** диалоговое окно содержит настройки, установленные как SSMA преобразует синтаксис DB2 [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] синтаксис.  
  
Область преобразования доступна в **параметры проекта** и **параметры проекта по умолчанию** диалоговые окна:  
  
-   Для задания параметров для всех проектов SSMA на **средства** меню **параметры проекта по умолчанию**, выберите тип проекта миграции, для которого требуются параметры для просмотра и изменения из  **Целевой версии миграции** раскрывающийся список, а затем щелкните **Общие** в нижней части левой панели, а затем нажмите кнопку **преобразования**.  
  
-   Для задания параметров для текущего проекта на **средства** меню **параметры проекта**, нажмите кнопку **Общие** в нижней части левой панели и нажмите кнопку **Преобразование**.  
  
## <a name="conversion-messages"></a>Преобразование сообщений  
  
### <a name="generate-messages-about-issues-applied"></a>Создают сообщения о проблемах, которые применены  
Разрешение SSMA создает информационных сообщений во время преобразования, отображает их в области вывода и добавляет их в преобразованном коде.  
  
Если выбран режим преобразования в **режим** », вкладка «SSMA применяется следующий параметр:  
  
**По умолчанию/оптимистического режима:** Нет  
  
**Полный режим:** Нет  
  
## <a name="miscellaneous-options"></a>Различные параметры  
  
### <a name="cast-rownum-expressions-as-integers"></a>Выражения приведения ROWNUM как целые числа  
Когда SSMA преобразует ROWNUM выражения, он преобразует их в предложение TOP, а затем с помощью выражения. В следующем примере показано ROWNUM в инструкцию DB2 DELETE:  
  
`DELETE FROM Table1`  
  
`WHERE ROWNUM < expression and Field1 >= 2`  
  
В следующем примере показан итоговый [!INCLUDE[tsql](../../includes/tsql-md.md)]:  
  
`DELETE TOP (expression-1)`  
  
`FROM Table1`  
  
`WHERE Field1>=2`  
  
ВЕРХНЕЙ требует, что предложения TOP выражение принимает значение является целым числом. Если целое число является отрицательным, инструкция вызовет ошибку.  
  
-   При выборе **Да**, SSMA приведение выражения к целому числу.  
  
-   При выборе **нет**, SSMA пометит все выражения, отличный от integer в преобразованный код как ошибка.  
  
Если выбран режим преобразования в **режим** », вкладка «SSMA применяется следующий параметр:  
  
**Режим по умолчанию/Full:** Нет  
  
**Режим оптимистичного:** Да  
  
### <a name="default-schema-mapping"></a>Сопоставление схемы по умолчанию  
Этот параметр указывает сопоставление схем DB2 со схемами SQL Server. В этом параметре, доступны два варианта:  
  
1.  **Схемы в базу данных:** В этой схеме DB2 режим «sch1» будет сопоставляться по умолчанию для схемы «dbo» SQL Server в базе данных SQL Server «sch1».  
  
2.  **Схемы для схемы:** в этом режиме DB2 схемы «sch1» будет сопоставляться по умолчанию для схемы «sch1» SQL Server в базе данных SQL Server по умолчанию, в диалоговом окне соединения.  
  
Если выбран режим преобразования в **режим** », вкладка «SSMA применяется следующий параметр:  
  
**Режим по умолчанию/оптимистичный/Full:** Схемы в базу данных

-   При выборе **инструкции с помощью СЛИЯНИЯ**, SSMA преобразует инструкции СЛИЯНИЯ в инструкции MERGE в [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)].

**Режим по умолчанию/оптимистичный/Full:** С помощью инструкции MERGE  
  
### <a name="convert-calls-to-subprograms-that-use-default-arguments"></a>Преобразования вызовов для подпрограммы, использующие аргументы по умолчанию  
[!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] функции не поддерживают Пропуск аргументов в вызове функции. Кроме того [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] функции и процедуры не поддерживают выражения, как значения параметров по умолчанию.  
  
-   При выборе **Да** и вызов функции опускает параметров, SSMA вставит ключевое слово **по умолчанию** в функцию и вызовите в правильном положении. Затем он помечает вызов с предупреждением.  
  
-   При выборе **нет**, SSMA пометит вызовы функций, как ошибки.  
  
Если выбран режим преобразования в **режим** », вкладка «SSMA применяется следующий параметр:  
  
**Режим по умолчанию/оптимистичный/Full:** Да

Если ваши функции COUNT, скорее всего для возврата значений, размер которых превышает 2 147 483 647, который составляет 2<sup>31</sup>-1, следует преобразовать функции COUNT_BIG.  
  
-   При выборе **Да**, SSMA преобразует все случаи использования число COUNT_BIG.  
  
-   При выборе **нет**, функции будет оставаться COUNT. [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Возвращает ошибку, если функция возвращает значение, размер которых превышает 2<sup>31</sup>-1.  
  
Если выбран режим преобразования в **режим** », вкладка «SSMA применяется следующий параметр:  
  
**Режим по умолчанию/Full:** Да  
  
**Режим оптимистичного:** Нет  
  
### <a name="convert-forall-statement-to-while-statement"></a>Преобразовать FORALL оператор WHILE инструкции  
Определяет, как интерпретировать SSMA FORALL циклы в коллекции элементов PL/SQL.  
  
-   При выборе **Да**, SSMA создает цикл WHILE, где элементы коллекции, полученные по одному.  
  
-   При выборе **нет**, SSMA создает набор строк из коллекции, используя метод узлов () и использует его как одна таблица. Это более эффективно, но становится менее удобочитаемым выходной код.  
  
Если выбран режим преобразования в **режим** », вкладка «SSMA применяется следующий параметр:  
  
**По умолчанию/оптимистического режима:** Нет  
  
**Полный режим:** Да  
  
### <a name="convert-foreign-keys-with-set-null-referential-action-on-column-that-is-not-null"></a>Внешние ключи Convert со ссылочным действием SET NULL для столбца, который является NOT NULL.  
DB2 позволяет создавать ограничения внешнего ключа, где действием SET NULL возможно невозможно, так как значения NULL не разрешены в указываемом столбце. [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] не поддерживает такой конфигурации внешнего ключа.  
  
-   При выборе **Да**, SSMA создаст ссылочные действия, как и в DB2, но необходимо будет внести изменения вручную перед загрузкой, чтобы ограничение [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]. Например можно выбрать NO ACTION, а не значение NULL.  
  
-   При выборе **нет**, ограничение отмечается как ошибка.  
  
Если выбран режим преобразования в **режим** », вкладка «SSMA применяется следующий параметр:  
  
**Режим по умолчанию/оптимистичный/Full:** Нет  
  
### <a name="convert-function-calls-to-procedure-calls"></a>Преобразовать вызовы функций, чтобы вызовы процедур  
Некоторые функции DB2 определяются как автономные транзакции или содержать инструкции, которые не будет действителен в [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]. В этих случаях SSMA создает процедуры и функции, которая является оболочкой для процедуры. Преобразованный функция вызывается процедура реализации.  
  
SSMA можно преобразовать вызовов функции-оболочки в вызова процедуры. Это создает более удобочитаемый код и может повысить производительность. Тем не менее контекст не всегда разрешено; Например невозможно заменить вызов функции в списке SELECT при вызове процедуры. SSMA имеет несколько параметров для распространенных случаев:  
  
-   При выборе **всегда**, SSMA пытается преобразовать вызовы функций программы-оболочки в вызовы процедур. Если текущий контекст не позволяет это преобразование, создается сообщение об ошибке. Таким образом, не вызовы функций, остаются в созданном коде.  
  
-   При выборе **по возможности**, SSMA делает перемещения для вызовов процедур, только в том случае, если функция выходных параметров. Перемещение не поддерживается, выходной атрибут параметра будет удалено. Во всех остальных случаях SSMA оставляет вызовы функций.  
  
-   При выборе **никогда**, SSMA оставит всех вызовов функций, как вызовы функций. Иногда этот выбор может быть неприемлемо из-за повышения производительности.  
  
Если выбран режим преобразования в **режим** », вкладка «SSMA применяется следующий параметр:  
  
**Режим по умолчанию/оптимистичный/Full:** Если это возможно  
  
### <a name="convert-lock-table-statements"></a>Преобразовать инструкции БЛОКИРОВКИ таблицы  
SSMA можно преобразовать много инструкций БЛОКИРОВКИ таблицы в табличных подсказок. SSMA не удается преобразовать операторов БЛОКИРОВКИ таблицы, которая содержит СЕКЦИЮ, ПОДРАЗДЕЛ, @dblinkи предложения NOWAIT и пометит такие инструкции, с сообщениями об ошибках преобразования.  
  
-   При выборе **Да**, SSMA преобразует поддерживаемых инструкций БЛОКИРОВКИ таблицы в табличных подсказок.  
  
-   При выборе **нет**, SSMA пометит все инструкции БЛОКИРОВКИ таблицы с сообщениями об ошибках преобразования.  
  
В следующей таблице показаны как SSMA преобразует режимов блокировки DB2:  
  
|||  
|-|-|  
|DB2 Режим блокировки|SQL Server табличное Указание|  
|СТРОКИ РЕСУРСА|ROWLOCK, HOLDLOCK|  
|ЭКСКЛЮЗИВНЫЕ СТРОКИ|HOLDLOCK ROWLOCK, XLOCK,|  
|ОБНОВЛЕНИЕ ОБЩЕГО РЕСУРСА = СТРОКИ РЕСУРСА|ROWLOCK, HOLDLOCK|  
|ПРЕДОСТАВИТЬ ОБЩИЙ ДОСТУП|TABLOCK, HOLDLOCK|  
|ЭКСКЛЮЗИВНЫЕ СТРОКУ ОБЩЕГО РЕСУРСА|HOLDLOCK TABLOCK, XLOCK,|  
|ЭКСКЛЮЗИВНЫЕ|TABLOCKX, HOLDLOCK|  
  
Если выбран режим преобразования в **режим** », вкладка «SSMA применяется следующий параметр:  
  
**Режим по умолчанию/оптимистичный/Full:** Да  
  
### <a name="convert-open-for-statements-for-ref-cursor-out-parameters"></a>Преобразовать инструкции OPEN FOR для параметров REF CURSOR OUT

-   При выборе **Да**, SSMA преобразует инструкции OPEN FOR в инструкции SELECT, который возвращает результирующий набор клиенту.  
  
-   При выборе **нет**, SSMA сформирует сообщение об ошибке, в преобразованный код и в области вывода.  
  
Если выбран режим преобразования в **режим** », вкладка «SSMA применяется следующий параметр:

### <a name="convert-synonyms"></a>Преобразовать синонимы

-   Представления и представления объектов  
  
-   Хранимые процедуры и функции  
  
-   Материализованные представления  
  
Синонимы для следующих объектов DB2 могут быть заменены прямые ссылки на объекты:

-   При выборе **Да**, создаст SSMA [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] синонимов и прямые объектные ссылки в соответствии с в списках выше.  
  
-   При выборе **нет**, SSMA создаст прямые объектные ссылки для все синонимы, перечисленные здесь.  
  
Если выбран режим преобразования в **режим** », вкладка «SSMA применяется следующий параметр:  
  
**Режим по умолчанию/оптимистичный/Full:** Да

-   При выборе **TO_CHAR_DATE с помощью функции**, SSMA преобразует TO_CHAR (даты, формат) в TO_CHAR_DATE функцию с помощью английского языка для преобразования.  
  
-   При выборе **TO_CHAR_DATE_LS с помощью функции (NLS осторожностью)**, SSMA преобразует TO_CHAR (даты, формат) в TO_CHAR_DATE_LS функцию с помощью языка сеанса для преобразования  
  
Если выбран режим преобразования в **режим** », вкладка «SSMA применяется следующий параметр:

**Полный режим:** С помощью функции TO_CHAR_DATE_LS (NLS осторожностью)

-   При выборе **Да**, SSMA преобразует инструкции обработки транзакций DB2 для [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] инструкций.  
  
-   При выборе **нет**, SSMA помечает транзакцию, обработка инструкций как ошибки преобразования.

Кроме того можно выполнить команду SET IMPLICIT_TRANSACTIONS ON в начале сеанса. SSMA автоматически добавляет SET IMPLICIT_TRANSACTIONS ON, при преобразовании подпрограммы с автономной транзакциями.

### <a name="emulate-db2-null-behavior-in-order-by-clauses"></a>Эмулировать поведение null DB2 в предложениях ORDER BY  
Значения NULL упорядочиваются по-разному в [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] и DB2:  
  
-   В [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)], или значение NULL — это наименьшее значения в упорядоченном списке. Из списка по возрастанию значений NULL будут отображаться первыми.  
  
-   В DB2 значения NULL являются самыми высокими значениями в упорядоченном списке. По умолчанию значения NULL отображаются последними в возрастающем порядке.  
  
-   DB2 имеет значения NULL и пустые значения ФАМИЛИИ предложений, которой можно изменить как DB2 упорядочивает значения NULL.  
  
SSMA можно эмулировать поведение DB2 ORDER BY, проверка значений NULL. Он сначала упорядочивает значениями NULL в указанном порядке, а затем заказы на другие значения.  
  
-   При выборе **Да**, SSMA преобразует инструкции DB2 способом, который эмулирует поведение DB2 ORDER BY.  
  
-   При выборе **нет**, SSMA будет игнорировать правила DB2 и появится сообщение об ошибке при обнаружении первого значения NULL и пустые значения ПОСЛЕДНЕГО предложения.  
  
Если выбран режим преобразования в **режим** », вкладка «SSMA применяется следующий параметр:  
  
**По умолчанию/оптимистического режима:** Нет  
  
**Полный режим:** Да  
  
### <a name="emulate-row-count-exceptions-in-select"></a>Эмуляция исключения число строк в SELECT  
Если инструкция SELECT с предложением INTO не возвращает строк, DB2 вызывает исключение NO_DATA_FOUND. Если инструкция возвращает две или несколько строк, TOO_MANY_ROWS исключение. Инструкцию, преобразованное в [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] не вызывает любое исключение, если число строк не совпадает с одним.  
  
-   При выборе **Да**, SSMA добавляет вызов процедуры db_error_exact_one_row_check sysdb после каждой инструкции SELECT. Эта процедура имитирует NO_DATA_FOUND и TOO_MANY_ROWS исключения. Это значение по умолчанию и позволяет воспроизводить поведение DB2, как можно ближе. Следует всегда выбирать **Да** Если исходный код содержит обработчики исключений, которые обрабатывают эти ошибки. Обратите внимание, что в случае запроса SELECT внутри определяемой пользователем функции этого модуля будут преобразовываться в хранимую процедуру, так как выполнение хранимых процедур и создание исключений не совместим с [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] контексте функции.  
  
-   При выборе **нет**, исключения не создаются. Что может быть полезно, когда SSMA преобразует определяемой пользователем функции, и вы хотите, чтобы он оставался в функцию в [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]  
  
Если выбран режим преобразования в **режим** », вкладка «SSMA применяется следующий параметр:  
  
**Режим по умолчанию/оптимистичный/Full:** Да
