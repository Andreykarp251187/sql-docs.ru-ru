# <a name="global-settings-dialogs-db2tosql"></a>Глобальные параметры (диалоговые окна) (DB2ToSQL)
Используйте страницу диалоговые окна **глобальные параметры** диалоговое окно, чтобы указать действие пользователя по умолчанию и параметры предупреждения для SSMA.  
  
Для доступа к параметрам диалогового окна на **средства** меню, выберите **глобальные параметры**, нажмите кнопку **графического пользовательского интерфейса** в нижней части левой панели, а затем выберите **диалоговые окна**.  
  
## <a name="options"></a>Параметры  
**Предупреждать перед перезаписью объектов**  
Когда SSMA преобразует объекты SQL Server, некоторые объекты могут уже существовать в метаданных проекта на SQL Server. Эти объекты могут уже были преобразованы, или же объекты могут просто совпадает с именем в целевой схеме объекты, которые нужно преобразовать.  
  
Используйте этот параметр для указания ли SSMA должен запросить перезапись определений повторяющихся объектов:  
  
-   При выборе **True**, SSMA отобразит диалоговое окно предупреждения, когда он встречает дубликат объекта. В этом диалоговом окне можно указать для перезаписи отдельных или всех дубликатов объектов или пропуска отдельных или всех повторяющихся объектов.  
  
-   При выборе **False**, **объекта действие по умолчанию перезаписать** параметр появляется, где можно указать действие по умолчанию.  
  
**По умолчанию действие объекта перезаписи**  
Этот параметр появляется при выборе **False** для **Предупреждать перед перезаписью объекты** параметр.  
  
Используйте этот параметр, чтобы указать объект по умолчанию перезаписывать поведение:  
  
-   При выборе **True**, SSMA будут автоматически перезаписаны объекты в метаданные проекта SQL Server, которые имеют одинаковые имена и находятся в той же схеме целевого объекта для преобразования.  
  
-   При выборе **False**, SSMA не перезаписывает метаданные объекта во время преобразования.
