# <a name="whats-new-in-ssma-for-db2-db2tosql"></a>Что нового в SSMA для DB2 (DB2ToSSL)

В этой статье перечислены помощник по миграции серверов (SSMA) для изменения DB2 в каждом выпуске.

## <a name="ssma-v88"></a>SSMA v8.8

V8.8 релиз SSMA для DB2 включает в себя:

* Улучшения синхронизации объектов sL Server
* Повышение производительности GUI во время оценки и преобразования
* Обновленное `ROWID` отокадляное `varbinary(40)` для облегчения миграции данных
* Улучшенная конверсия выписок `SELECT ... FROM NEW/OLD TABLE`
* Новая конверсия заявлений `ALTER` для процедур и функций
* Новое преобразование деструкционтных назначений

## <a name="ssma-v87"></a>SSMA v8.7

V8.7 релиз SSMA для DB2 включает в себя новый dB2 синтаксис парсер, а также незначительные исправления и улучшения производительности в графическом пользовательском интерфейсе.

Кроме того, SSMA для DB2 теперь предоставляет:

* Исправление обнаружения иностранных ключей при миграции из DB2 на LUW.
* Улучшенная конверсия `SELECT ... FOR UPDATE` оператора.
* Улучшенная конверсия для `COUNT` функции в таблицах МЗ.
* Преобразование заявлений. `SAVEPOINT`
* Преобразование для эмуляции поведения `NULL` DB2 для значений в `ORDER BY` клаузуле.
* Поддержка разбора для заявления ASSOCIATE RESULT SET.

> [!IMPORTANT]
> С SSMA v8.5 и более поздним, .NET 4.7.2 является предварительной установкой. Если вам нужно установить эту версию, вы можете скачать файл времени выполнения [здесь](https://dotnet.microsoft.com/download/dotnet-framework/net472).

## <a name="ssma-v86"></a>SSMA v8.6

В дополнение к целевому набору исправлений, предназначенных для улучшения удобства использования и производительности, v8.6 выпуск SSMA для DB2 был улучшен путем добавления параметра, который позволяет пользователям опускать расширенные свойства SSMA в преобразованном коде.

Чтобы использовать эту настройку, в SSMA **Tools** > для DB2, перейдите к**Tools Project Настройки** > **Общего** > **Преобразования**, а затем под **Misc**, обновить значение **настройки Омит Расширенные свойства** **Да**.

![Установка Расширенных свойств Omit](../db2/media/ssma-omit-extended-properties.png)

Кроме того, SSMA для DB2 теперь предоставляет:

* Исправление для преобразования функций, которые используют значения аргумента по умолчанию.
* Улучшенный разбор положения `PARAMETER` для функций.
* Возможность преобразования `LEAVE` оператора.

> [!IMPORTANT]
> С SSMA v8.5 и более поздним, .NET 4.7.2 является предварительной установкой. Если вам нужно установить эту версию, вы можете скачать файл времени выполнения [здесь](https://dotnet.microsoft.com/download/dotnet-framework/net472).

## <a name="ssma-v85"></a>SSMA v8.5

Выпуск v8.5 SSMA для DB2 дополнен поддержкой проверки подлинности Active Directory Azure и базовой поддержкой функций JSON на сервере S'L, а также целевым набором исправлений, предназначенных для улучшения удобства использования и производительности.

Кроме того, SSMA для DB2 был усовершенствован:

* Поддержка добавления `GET DIAGNOSTICS` преобразования `ROW_NUMBER`для оператора с помощью .
* Исправление ошибки, связанной с пробелами в начале несоблюдения имени объекта.

> [!IMPORTANT]
> С SSMA v8.5, .NET 4.7.2 является предварительной установкой. Если вам нужно установить эту версию, вы можете скачать файл времени выполнения [здесь](https://dotnet.microsoft.com/download/dotnet-framework/net472).

## <a name="ssma-v84"></a>SSMA v8.4

Выпуск v8.4 SSMA для DB2 дополнен целевыми исправлениями, которые предназначены для решения проблем доступности и исправления ошибки, связанной с столбиками макс индекса (чтобы позволить 32 вместо 16) для S'L Server 2016 и более поздних версий.

## <a name="ssma-v83"></a>SSMA v8.3

V8.3 релиз SSMA для DB2 усиливается с целевыми исправлениями, которые предназначены для улучшения качества и преобразования метрик. Кроме того, этот релиз SSMA для DB2 содержит исправления, которые:

* Решение проблем доступности.
* Добавьте базовую поддержку для `hierarchyid` ввода в сервере S'L.
* Заменить использование функции TRIM в `RTRIM` / `LTRIM`запросах обнаружения z/OS с помощью .
* Разрешить пользователю указывать сбор пакетов при подключении `NULLID`в "Стандартном режиме" (по умолчанию).
* Добавить `CREATE TABLE AS SELECT`конверсию для .
* Улучшение конверсий для глобальных таблиц временных топов.
* Устраните проблему с целью проверки уникальности, чтобы определить приоритеты таблиц над ограничениями, если имена сталкиваются.
* Устраните проблему с загрузкой `DATE` `TIMESTAMP` значений столбца по умолчанию для z/OS.
* Поддержка символа корма линии `NEL`Unicode (также известный как).
* Устраните проблему с преобразованием курсора с отсутствующим `RETURN TO` положением.
* Добавить поддержку `GOTO`для меток и .

## <a name="ssma-v82"></a>SSMA v8.2

Выпуск v8.2 SSMA для DB2 дополнен для решения проблем, связанных с подключением к базе данных Azure S'L от консольного инструмента SSMA и отсутствующим COUNT_BIG столбца в декларации представлений во время преобразования. Кроме того, эта версия включает в себя целевой набор исправлений, предназначенных для улучшения качества и конверсии метрик, а также исправления для:

* Проблема с неограниченными некластерными индексами после миграции данных.
* Обнаружение рамочного соглашения .NET во время бесшумной установки.
* Периодический сбой, который возникает при загрузке новой версии.

> [!NOTE]
> Известная проблема с автоматическим обновлением может привести к сбою обновления с SSMA v8.1 до v8.2. Если вы столкнулись с этой ошибкой, пожалуйста, загрузите новую версию и установите ее вручную.

## <a name="ssma-v77"></a>SSMA v7.7

V7.7 релиз SSMA для DB2 содержит следующие изменения:

* Целевые исправления, улучшающие показатели качества и конверсии.
* Основываясь на популярном спросе, 32-битная версия SSMA для DB2 возвращается. По сравнению с предыдущей реализацией (до v7.4) есть два пакета установки, но они не могут быть установлены бок о бок. ## <a name="ssma-v71"></a>SSMA v7.1

V7.1 релиз SSMA для DB2 содержит следующие изменения:

* Сервер S'L 2017 на Windows и Linux CTP1 в настоящее время является поддерживаемой целевой платформой для миграции. Эта функция находится в техническом предварительном просмотре и позволяет схема и перемещение данных для целевой серверов S'L.

* SSMA устанавливаемые бинарные файлы теперь доставляются через файлы пакетов Windows Installer (.msi).

Май 2016 релиз SSMA для DB2 содержит следующие изменения:

* Добавлена поддержка для сервера S'L 2016.

* Удаленная проверка установки для .NET 2.0.
* Удален \*ненужный .dll из установки Db2.
* `save-project` Исправлено `open-project` и команды для консоли SSMA.
* Фиксированная `securepassword` команда для консоли SSMA.
