# <a name="whats-new-in-ssma-for-access-accesstosql"></a>Что нового в SSMA для доступа (AccessToS'L)

В этой статье перечислены помощник по миграции серверов (SSMA) для изменения доступа в каждом выпуске.

## <a name="ssma-v88"></a>SSMA v8.8

V8.8 релиз SSMA для доступа включает в себя:

* Улучшения синхронизации объектов sL Server
* Повышение производительности GUI во время оценки и преобразования
* Новый синтаксисный парсер Access для дальнейшего повышения производительности преобразования

## <a name="ssma-v87"></a>SSMA v8.7

V8.7 выпуск SSMA для доступа улучшил `IIF` конверсию для функции в запросах, а также незначительные исправления и улучшения производительности графического пользовательского интерфейса.

> [!IMPORTANT]
> С SSMA v8.5 и более поздним, .NET 4.7.2 является предварительной установкой. Если вам нужно установить эту версию, вы можете скачать файл времени выполнения [здесь](https://dotnet.microsoft.com/download/dotnet-framework/net472).

## <a name="ssma-v86"></a>SSMA v8.6

В дополнение к целевому набору исправлений, предназначенных для улучшения удобства использования и производительности, v8.6 выпуск SSMA для доступа был улучшен путем добавления параметра, который позволяет пользователям опускать расширенные свойства SSMA в преобразованном коде.

Чтобы использовать эту настройку, в SSMA для доступа, перейдите к > **Настройкам объектов** > **общего** >  **преобразования,** а затем под **Misc,** обновите значение настройки **Омит Расширенные свойства** **да**.**Conversion**

![Установка Расширенных свойств Omit](../access/media/ssma-omit-extended-properties.png)

> [!IMPORTANT]
> С SSMA v8.5 и более поздним, .NET 4.7.2 является предварительной установкой. Если вам нужно установить эту версию, вы можете скачать файл времени выполнения [здесь](https://dotnet.microsoft.com/download/dotnet-framework/net472).

## <a name="ssma-v85"></a>SSMA v8.5

Выпуск v8.5 SSMA for Access дополнен поддержкой аутентификации Active Directory Azure и базовой поддержкой функций JSON на сервере S'L, а также целевым набором исправлений, предназначенных для улучшения удобства использования и производительности.

Кроме того, SSMA for Access теперь`ISNULL`поддерживает `IIF`преобразование нескольких стандартных функций (и т.д.).

> [!IMPORTANT]
> С SSMA v8.5, .NET 4.7.2 является предварительной установкой. Если вам нужно установить эту версию, вы можете скачать файл времени выполнения [здесь](https://dotnet.microsoft.com/download/dotnet-framework/net472).
