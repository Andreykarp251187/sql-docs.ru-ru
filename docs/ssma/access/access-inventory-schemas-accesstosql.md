# <a name="access-inventory-schemas-accesstosql"></a>Схемы инвентаризации Access (AccessToSQL)
В следующих разделах описаны таблицы, которые создаются с SSMA при экспорте схемы доступа к [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)].  
  
## <a name="databases"></a>Базы данных  
Экспорте метаданных базы данных **SSMA_Access_InventoryDatabases** таблицы. Эта таблица содержит следующие столбцы:  
  
|Имя столбца|Тип данных|Описание|  
|---------------|-------------|---------------|  
|**DatabaseId**|**uniqueidentifier**|Идентификатор GUID, однозначно определяющий каждую базу данных. Этот столбец также является первичным ключом для таблицы.|  
|**DatabaseName**|**nvarchar(4000)**|Имя базы данных Access.|  
|**ExportTime**|**datetime**|Дата и время создания эти метаданные с SSMA.|  
|**FilePath**|**nvarchar(4000)**|Полный путь и имя базы данных Access.|  
|**Размер файла**|**bigint**|Размер базы данных Access, в КБ.|  
|**FileOwner**|**nvarchar(4000)**|Учетная запись Windows, которая указана в качестве владельца базы данных Access.|  
|**DateCreated**|**datetime**|Дата и время создания базы данных Access.|  
|**DateModified**|**datetime**|Дата и время последнего изменения базы данных Access.|  
|**TablesCount**|**int**|Количество таблиц в базе данных Access.|  
|**QueriesCount**|**int**|Число запросов в базе данных Access.|  
|**FormsCount**|**int**|Число форм в базе данных Access.|  
|**ModulesCount**|**int**|Число модулей в базе данных Access.|  
|**ReportsCount**|**int**|Число отчетов в базе данных Access.|  
|**MacrosCount**|**int**|Количество макросов в базе данных Access.|  
|**AccessVersion**|**nvarchar(4000)**|Версия базы данных Access.|  
|**Параметры сортировки**|**nvarchar(4000)**|Параметры сортировки базы данных Access. Параметры сортировки определяют, каким образом сортирует и сравнивает строки базы данных.|  
|**JetVersion**|**nvarchar(4000)**|Версия ядра базы данных Jet. Доступ к базам данных использовать ядро СУБД Jet.|  
|**IsUpdatable**|**bit**|Указывает, если базы данных могут обновляться. Если значение равно 1, базы данных является обновляемым. Если значение равно 0, базе данных только для чтения.|  
|**QueryTimeout**|**int**|Настроенные ODBC значение времени ожидания запроса к базе данных, в секундах. Значение по умолчанию — 60 секунд.|  
  
## <a name="tables"></a>Таблицы  
Метаданные таблицы экспортируются в **SSMA_Access_InventoryTables** таблицы. Эта таблица содержит следующие столбцы:  
  
|Имя столбца|Тип данных|Описание|  
|---------------|-------------|---------------|  
|**DatabaseId**|**uniqueidentifier**|Определяет базу данных, содержащий этой таблицы.|  
|**TableId**|**uniqueidentifier**|GUID, однозначно идентифицирующее таблицу. Этот столбец также является первичным ключом для таблицы.|  
|**TableName**|**nvarchar(4000)**|Имя таблицы.|  
|**RowsCount**|**int**|Число строк в таблице.|  
|**ValidationRule**|**nvarchar(4000)**|Правило, которое определяет допустимые входные данные для таблицы. Если нет правила проверки на поле будет содержать пустую строку.|  
|**LinkedTable**|**nvarchar(4000)**|Другой таблицы, если таковое имеется, которая связана с таблицей. Связывание таблиц позволяет дополнения, удаления и обновления с другой таблицей с помощью этой таблицы.|  
|**ExternalSource**|**nvarchar(4000)**|Источник данных, если таковое имеется, который связан с таблицей. Если таблица связана, она имеет источник внешних данных, указанное в этом поле.|  
  
## <a name="columns"></a>Столбцы  
Экспорте метаданных столбца **SSMA_Access_InventoryColumns** таблицы. Эта таблица содержит следующие столбцы:  
  
|Имя столбца|Тип данных|Описание|  
|---------------|-------------|---------------|  
|**DatabaseId**|**uniqueidentifier**|Определяет базу данных, который содержит этот столбец.|  
|**TableId**|**uniqueidentifier**|Идентифицирует таблицу, которая содержит этот столбец.|  
|**ColumnId**|**int**|Увеличивающееся целое число, определяющее столбец. **ColumnId** является первичным ключом для таблицы.|  
|**ColumnName**|**nvarchar(4000)**|Имя столбца.|  
|**IsNullable**|**bit**|Указывает, если столбец может содержать значения null. Если значение равно 1, столбец может содержать значения null. Если значение равно 0, столбец не может содержать значения null. Обратите внимание на то, что правило проверки может также использоваться для предотвращения передачи значений null.|  
|**DataType**|**nvarchar(4000)**|Доступ к данным тип столбца, например **текст** или **Long**.|  
|**IsAutoIncrement**|**bit**|Указывает, если столбец автоматически увеличивает целочисленных значений. Если значение равно 1, целые числа с автоматически увеличивающимися значениями.|  
|**Ordinal**|**smallint**|Порядковый номер столбца в таблице, начиная с нуля.|  
|**DefaultValue**|**nvarchar(4000)**|Значение по умолчанию для столбца.|  
|**ValidationRule**|**nvarchar(4000)**|Правило, которое используется для проверки данных при добавлении или обновлении в столбце.|  
  
## <a name="indexes"></a>Индексы  
Экспорте метаданных индекса **SSMA_Access_InventoryIndexes** таблицы. Эта таблица содержит следующие столбцы:  
  
|Имя столбца|Тип данных|Описание|  
|---------------|-------------|---------------|  
|**DatabaseId**|**uniqueidentifier**|Определяет базу данных, который содержит этот индекс.|  
|**TableId**|**uniqueidentifier**|Идентифицирует таблицу, которая содержит этот индекс.|  
|**IndexId**|**int**|Увеличивающееся целое число, которое идентифицирует индекс. Этот столбец является первичным ключом для таблицы.|  
|**IndexName**|**nvarchar(4000)**|Имя индекса.|  
|**ColumnsIncluded**|**nvarchar(4000)**|Список столбцов, включенных в индекс. Имена столбцов разделяются точкой с запятой.|  
|**IsUnique**|**bit**|Указывает, если каждый элемент в индексе должно быть уникальным. В индексе несколькими столбцами сочетания значений должно быть уникальным. Если значение равно 1, индекс включает уникальные значения.|  
|**IsPK**|**bit**|Указывает, если индекс был автоматически создан как часть определения первичного ключа.|  
|**IsClustered**|**bit**|Указывает, кластеризован ли индекс. Кластеризованный индекс переупорядочивает физическое хранилище данных. В таблице может быть только один кластеризованный индекс.|  
  
## <a name="foreign-keys"></a>Внешние ключи  
Метаданные внешнего ключа экспортируется в **SSMA_Access_InventoryForeignKeys** таблицы. Эта таблица содержит следующие столбцы:  
  
|Имя столбца|Тип данных|Описание|  
|---------------|-------------|---------------|  
|**DatabaseId**|**uniqueidentifier**|Определяет базу данных, который содержит этот внешний ключ.|  
|**TableId**|**uniqueidentifier**|Идентифицирует таблицу, содержащую данного внешнего ключа.|  
|**ForeignKeyId**|**int**|Увеличивающееся целое число, определяющее внешний ключ. Этот столбец является первичным ключом для таблицы.|  
|**ForeignKeyName**|**nvarchar(4000)**|Имя индекса.|  
|**ReferencedTableId**|**uniqueidentifier**|Идентифицирует таблицу, содержащую исходные столбцы.|  
|**SourceColumns**|**nvarchar(4000)**|Список внешний ключевой столбец или столбцы.|  
|**ReferencedColumns**|**nvarchar(4000)**|Список первичный ключевой столбец или столбцы, на которые ссылается внешний ключ.|  
|**IsCascadeForUpdate**|**bit**|Указывает, что при обновлении значения первичного ключа все строки, которые ссылаются на значения этого раздела также обновляются.|  
|**IsCascadeForDelete**|**bit**|Указывает, что если значение первичного ключа удаляется, все строки, которые ссылаются на значения этого раздела также удаляются.|  
|**IsEnforced**|**bit**|Указывает, что применяется ограничение внешнего ключа.|
