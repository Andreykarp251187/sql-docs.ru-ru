# <a name="setting-conversion-and-migration-options-accesstosql"></a>Установка параметров преобразования и миграции (AccessToSQL)
Для каждого проекта SSMA можно задать параметры на уровне проекта. Эти параметры указывают способ преобразования объектов, процесс миграции данных и сопоставлении типов данных источника к целевым типам данных. Перед началом преобразования объектов [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] или SQL Azure или переносить данные в [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] или SQL Azure, убедитесь, что параметры конфигурации подходят для проекта.  
  
## <a name="configuration-options-and-modes"></a>Параметры конфигурации и режимы  
SSMA имеет четыре набора параметров конфигурации и четыре режима для настройки этих параметров: По умолчанию, оптимистичный, полный и пользовательские. По умолчанию режим рекомендуется для большинства пользователей. Используйте режим оптимистичного для простейших преобразований. Используйте полный режим, если вы хотите просмотреть все сообщения. В пользовательский режим задать параметры.  
  
Параметры описаны в разделе «Справочник по пользовательскому интерфейсу» данного документа. Дополнительные сведения о параметрах, а также порядок применения параметров в каждом режиме см. в разделах:

-   [Параметры проекта (графический пользовательский интерфейс)](https://msdn.microsoft.com/cf06baf1-8714-48a3-95dc-781f6ca53693)

## <a name="setting-project-options"></a>Настройка параметров проекта  
В SSMA можно настроить параметры по умолчанию для всех проектов. Эти параметры сохраняются в файле конфигурации SSMA и применяется к любым новым проектом, созданный.  
  
**Чтобы задать параметры проекта по умолчанию**  
  
1.  На **средства** меню, выберите **параметры проекта по умолчанию**.  
  
2.  В **параметры проекта по умолчанию** диалоговое окно, выполните одно из следующих:  
  
    -   Выберите тип проекта миграции, для которого требуются параметры просмотра или изменения из **целевой версии миграции** раскрывающийся список, нажмите кнопку **Общие** в нижней части левой панели, а затем выберите **Преобразование или миграцию или SQL Azure**.  
  
        > [!NOTE]  
        > SQL Azure доступна в **Общие** вкладке только в том случае, если тип проекта создан — SQL Azure.  
  
    -   Чтобы выбрать предварительно определенные режима, выберите **по умолчанию**, **Optimistic**, или **полный** в **режим** раскрывающийся список.  
  
    -   Для указания пользовательского режима, выберите **пользовательских** в **режим** выберите параметр в области слева, выберите параметр или значение в области справа и затем выберите или введите новое значение или значение.  
  
3.  Нажмите кнопку **ОК** для сохранения настроек.  
  
Также можно настроить параметры для текущего проекта. Эти параметры сохраняются в файле проекта.  
  
**Чтобы настроить параметры для текущего проекта**  
  
1.  На **средства** меню, выберите **параметры проекта**.  
  
2.  В **параметры проекта** диалоговое окно, выполните одно из следующих:  
  
    -   Чтобы выбрать предварительно определенные режима, выберите **по умолчанию**, **Optimistic**, или **полный** в **режим** раскрывающийся список.  
  
    -   Для указания пользовательского режима, выберите **пользовательских** в **режим** выберите параметр в области слева, выберите параметр или значение в области справа и затем выберите или введите новое значение или значение.  
  
3.  Нажмите кнопку **ОК** для сохранения настроек.
