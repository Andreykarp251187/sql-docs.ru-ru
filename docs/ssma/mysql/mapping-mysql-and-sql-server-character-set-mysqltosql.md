# <a name="mapping-mysql-and-sql-server-character-set-mysqltosql"></a>Сопоставление кодировок MySQL и SQL Server (MySQLToSQL)
Для символьных типов данных MySQL, выражения и литералы можно указать набор символов (Charset).  
  
## <a name="charset-mapping"></a>Сопоставление CharSet  
Сопоставление CharSet определяется для каждого charset MySQL и используется во время преобразования типа данных символа. Он указывает, как преобразовать типы данных символьных строк набора определенный символ:  
  
-   Для символьных типов национальных SQL Server (NCHAR/NVARCHAR) или  
  
-   Для символьных типов регулярных SQL Server (CHAR/VARCHAR)  
  
1.  **Национальный** целевой базы данных символьные типы данных являются:  
  
    1.  **nchar**  
  
    2.  **nvarchar**  
  
2.  **регулярные** целевой базы данных символьные типы данных являются:  
  
    1.  **char**  
  
    2.  **varchar**  
  
3.  Сопоставление типов допускает только сопоставление **национальных** символьные типы данных. После MySQL символьный тип данных преобразуется в соответствии с сопоставлением типов, применяется сопоставление charset.  
  
> [!NOTE]  
> Сопоставление CharSet могут быть определены для каждого уровня узла обозревателя объектов метаданных и представляют всех наборов символов, чтение из MySQL.  
  
## <a name="charset-mapping-on-different-node-levels"></a>Сопоставление CharSet на разных уровнях  
Сопоставление CharSet меняется на разных уровнях, а именно:  
  
1.  На корневом уровне узел метаданных  
  
2.  В базе данных, категории и уровне узлов объектов  
  
> [!NOTE]  
> Вкладка, выбранного для редактирования сопоставление Charset, содержит три кнопки, независимо от сопоставления на разных уровнях.  
>   
> Подробные сведения.  
>   
> 1.  **Примените:** Применяет изменения, сделанные пользователем, включить, только в том случае, если сопоставление charset был изменен и еще не сохранен.  
> 2.  **Отмена:** Отменяет изменения, внесенные пользователем. Эта кнопка возвращает включена, если сопоставление charset изменить, но не сохранены.  
> 3.  **Восстановить значения по умолчанию:** Сбрасывает все сопоставления в значения по умолчанию.  
  
1.  **На корневом уровне узел метаданных:**  Сетка сопоставления CharSet содержит "Сетка" charset с отдельный столбец для каждого charset. Столбцы сетки являются:

> -   В сетке, а значение **унаследованное** имеет суффикс либо '(CHAR/VARCHAR) "или '(NCHAR/NVARCHAR)" в зависимости от того, какое значение было унаследовано от родительского элемента, этот определенный набор символов.
