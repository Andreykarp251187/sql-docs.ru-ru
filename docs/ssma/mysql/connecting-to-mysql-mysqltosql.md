# <a name="connecting-to-mysql-mysqltosql"></a>Подключение к MySQL (MySQLToSQL)
Чтобы перенести базы данных MySQL в SQL Server или SQL Azure, необходимо подключиться к базе данных MySQL, которые требуется перенести. При подключении, SSMA получает метаданные о все схемы MySQL и затем отображается в панели обозревателя метаданных MySQL. SSMA хранит сведения о сервере базы данных, но не хранит пароли.  
  
Подключение к базе данных остается активным, пока не закрыть проект. При повторном открытии проекта, необходимо повторно подключить, если требуется активное подключение к базе данных.  
  
Метаданные о базе данных MySQL не обновляется автоматически. Вместо этого Если вы хотите обновить метаданные в обозревателе метаданных MySQL, необходимо вручную обновить его. Дополнительные сведения см. в разделе «Обновление MySQL метаданные» далее в этом разделе.  
  
## <a name="required-mysql-permissions"></a>MySQL необходимые разрешения  
Учетная запись, которая используется для подключения к базе данных MySQL должен иметь по крайней мере **CONNECT** разрешения. Это позволяет SSMA для получения метаданных из схемы, принадлежащие подключающийся пользователь. Получить метаданные для объектов в другие схемы, а затем преобразовать объекты в этих схемах, учетная запись должна иметь следующие разрешения:  
  
-   Показать права на объекты базы данных  
  
-   УСТАНОВИТЕ права доступа на «Information_schema»  
  
-   ВЫБЕРИТЕ разрешение для mysql (для определяемых пользователем функций)  
  
## <a name="establishing-a-connection-to-mysql"></a>Подключения к MySQL  
При подключении к базе данных, SSMA считывает метаданные базы данных и добавляет эти метаданные в файл проекта. Эти метаданные используется SSMA, когда он преобразует объекты в синтаксис SQL Server или SQL Azure и когда он переносит данные в SQL Server или SQL Azure. Можно просматривать эти метаданные в панели обозревателя метаданных MySQL и просмотрите свойства отдельных объектов базы данных.  
  
> [!IMPORTANT]  
> Перед попыткой подключения, убедитесь, что сервер базы данных запущен и может принимать подключения.  
  
**Для подключения к MySQL**  
  
1.  На **файл** меню, выберите **подключение к MySQL** (этот параметр включается после создания проекта).  
  
    Если ранее вы подключены к MySQL, команда будет называться **повторное подключение к MySQL**.  
  
2.  В **поставщика** выберите драйвер 5.1 ODBC для MySQL (доверенные). Это поставщик по умолчанию в стандартном режиме.  
  
3.  В **режим** выберите **стандартный режим**. Это режим по умолчанию.  
  
    Позволяет указать имя сервера и порт в стандартном режиме.  
  
4.  В **стандартного режима**, введите следующие значения:  
  
    1.  В **имя_сервера** введите имя сервера MySQL. В **порт сервера** введите номер порта, который будет 3306. Это порт по умолчанию.  
  
    2.  В **имя пользователя** введите учетную запись MySQL, которая имеет необходимые разрешения.  
  
    3.  В **пароль** введите пароль для указанного имени пользователя.  
  
5.  **SSL.** Для безопасного подключения к MySQL, внесите использовать из Secure Socket Layer (SSL), установив флажок **SSL** флажок.  
  
6.  **Настройка:** Он предоставляет параметр для настройки подключения к MySQL через Secure Socket Layer (SSL).  
  
    > [!NOTE]  
    > Чтобы включить **Настройка**, должно быть присвоено SSL **True**.  
  
    На кнопку «Настроить», появляется диалоговое окно. Использование шифрования при подключении к базе данных MySQL, путь к следующим файлам три сертификата, в диалоговом окне должен быть определен [конфиденциальности усиленной Mail сертификаты (PEM)]:  
  
    -   **Центр сертификации SSL:** Указывает путь к файлу со списком доверия центров сертификации SSL.  
  
    -   **SSL-сертификат:** Задает имя файла сертификата SSL для установления безопасного соединения.  
  
    -   **КЛЮЧ SSL:** Задает имя файла ключа SSL, используемый для создания безопасного подключения.  
  
    > [!NOTE]  
    > -   **ОК** кнопка включена, когда предоставила необходимые сведения. Если любой из пути к файлам являются недействительными, кнопку «ОК» будет отключен.  
    > -   **Отменить** кнопки закрывает диалоговое окно и **приводит к отключению** параметр SSL из основной формы подключения.  
  
7.  Дополнительные сведения см. в разделе [подключение к MySQL &#40;MySQLToSQL&#41;](../../ssma/mysql/connect-to-mysql-mysqltosql.md)  
  
## <a name="reconnecting-to-mysql"></a>При повторном подключении к MySQL  
Подключение к серверу базы данных остается активным, пока не закрыть проект. При повторном открытии проекта, необходимо повторно подключить, если требуется активное подключение к базе данных. Вы можете работать вне сети до вы хотите обновить метаданные, загрузка объектов базы данных в SQL Server или SQL Azure и переноса данных.  
  
## <a name="refreshing-mysql-metadata"></a>Обновление метаданных MySQL  
Метаданные о базе данных MySQL не обновляется автоматически. Метаданные в обозревателе метаданных MySQL — это снимок метаданные при первом подключении, или время последнего обновления метаданных вручную. Можно вручную обновить метаданные для всех схем, одной схемы или отдельных объектов базы данных.  
  
**Для обновления метаданных**  
  
1.  Убедитесь, что вы подключены к базе данных.  
  
2.  В обозревателе метаданных MySQL установите флажок рядом с каждым объектом схемы или базы данных, который требуется обновить.  
  
3.  Щелкните правой кнопкой мыши **схемы**, или отдельные схемы, или базы данных объекта, а затем выберите **обновление из базы данных**.  
  
    Если у вас нет активного подключения, отобразит SSMA **подключение к MySQL** диалоговом таким образом, можно подключиться.  
  
4.  В обновление из базы данных-диалоговое окно укажите объекты для обновления.  
  
    -   Чтобы обновить объект, нажмите **Active** поле рядом с объектом, пока не появится стрелка.  
  
    -   Чтобы предотвратить объекта к обновлению, нажмите кнопку **Active** поле возле объект, пока не **X** отображается.  
  
    -   Чтобы обновить или отклонить категорию объектов, щелкните **Active** поле возле папка категории.  
  
    -   Чтобы просмотреть определения цветовое выделение синтаксиса, щелкните **условных обозначений** кнопки.  
  
5.  Нажмите кнопку **ОК**.
