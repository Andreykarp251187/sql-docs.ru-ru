# <a name="getting-started-with-ssma-for-mysql-mysqltosql"></a>Начало работы с SSMA для MySQL (MySQLToSQL)
SQL Server Migration Assistant (SSMA) для MySQL позволяет быстро преобразование схем баз данных MySQL в схемы SQL Server или базе данных SQL Azure, отправьте полученный схем в SQL Server или базе данных SQL Azure и переноса данных из MySQL в SQL Server или базе данных SQL Azure.  
  
В этом разделе представлен процесс установки, а затем ознакомиться с пользовательским интерфейсом SSMA.  
  
## <a name="installing-ssma"></a>Установка SSMA  
Чтобы использовать SSMA, сначала необходимо установить SSMA клиентскую программу на компьютере с доступом к базе данных MySQL источника и целевой экземпляр SQL Server или базе данных SQL Azure. Затем установите MySQL поставщиков (драйвер ODBC для MySQL 5.1 (доверенные)) на компьютере, на котором выполняется клиентская программа SSMA. Инструкции по установке см. в разделе [Установка SSMA для MySQL &#40;MySqlToSql&#41;](../../ssma/mysql/installing-ssma-for-mysql-mysqltosql.md)  
  
Для запуска SSMA, нажмите кнопку **запустить**, пункты **все программы**, пункты **SQL Server Migration Assistant для MySQL**, а затем нажмите кнопку **переход на SQL Server Assistant для MySQL**.  
  
## <a name="ssma-for-mysql-user-interface"></a>SSMA для интерфейса пользователя MySQL  
После SSMA установлен и лицензирован, можно использовать SSMA для переноса баз данных MySQL в SQL Server или базе данных SQL Azure. Он позволяет ознакомиться с пользовательским интерфейсом SSMA, прежде чем начать. В примере ниже показан пользовательский интерфейс для SSMA, включая обучающие ресурсы для метаданных, метаданные, панелей инструментов, область вывода и область списка ошибок:  
  
![SSMA для графического интерфейса пользователя MySql](../../ssma/mysql/media/ssmaformysqlgui.gif "SSMA для графического интерфейса пользователя MySql")  
  
Чтобы начать миграцию, необходимо сделать следующее:  
  
1.  Создайте новый проект.  
  
2.  Подключитесь к базе данных MySQL.  
  
3.  После успешного подключения схемы MySQL будет отображаться в обозревателе метаданных MySQL. Объектов щелкните правой кнопкой мыши в обозревателе метаданных MySQL для выполнения задач, таких как создание отчетов, которые оценивают преобразования в SQL Server, Azure SQL DB.  
  
Эти задачи также можно выполнить с помощью панели инструментов и меню.  
  
Кроме того, необходимо подключиться к экземпляру SQL Server. После успешного подключения иерархии баз данных SQL Server будет отображаться в обозревателе метаданных SQL Server. После преобразования схемы MySQL схем SQL Server, выберите преобразованный схемы в обозревателе метаданных SQL Server, а затем синхронизировать схемы с SQL Server.  
  
Необходимо подключиться к базе данных SQL Azure, при выборе базы данных SQL Azure из миграции в раскрывающемся списке в диалоговом окне "новый проект". После успешного подключения иерархию базах данных SQL Azure будет отображаться в обозревателе метаданных базы данных SQL Azure. После преобразования схемы MySQL в схемы базы данных SQL Azure, выберите преобразованный схемы в обозревателе метаданных базы данных SQL Azure и затем синхронизировать схемы с базой данных SQL Azure.  
  
После синхронизации преобразованный схемы с SQL Server или базе данных SQL Azure, можно вернуться в обозреватель метаданных MySQL и переноса данных из MySQL схем в базы данных SQL Server или базе данных SQL Azure.  
  
Дополнительные сведения об этих задачах и порядок их выполнения см. в разделе [Миграция баз данных MySQL в SQL Server — база данных SQL Azure &#40;MySQLToSql&#41;](../../ssma/mysql/migrating-mysql-databases-to-sql-server-azure-sql-db-mysqltosql.md).  
  
Ниже описаны возможности пользовательского интерфейса SSMA.  
  
### <a name="metadata-explorers"></a>Обучающие ресурсы для метаданных  
SSMA содержит два проводников метаданных для просмотра и выполнения действий для баз данных MySQL и SQL Server.  
  
### <a name="mysql-metadata-explorer"></a>Обозреватель метаданных MySQL  
Обозреватель метаданных MySQL отображаются сведения о схемах MySQL. С помощью обозревателя метаданных MySQL, можно выполнять следующие задачи:  
  
-   Просмотрите объекты в каждой схеме.  
  
-   Выберите объекты для преобразования и затем преобразовать объекты в синтаксис SQL Server. Дополнительные сведения см. в разделе [преобразование баз данных MySQL &#40;MySQLToSQL&#41;](../../ssma/mysql/converting-mysql-databases-mysqltosql.md)  
  
-   Выберите таблицы для переноса данных, а затем перенести данные из этих таблиц SQL Server. Дополнительные сведения см. в разделе [переноса данных MySQL в SQL Server — база данных SQL Azure &#40;MySQLToSQL&#41;](../../ssma/mysql/migrating-mysql-data-into-sql-server-azure-sql-db-mysqltosql.md)  
  
### <a name="sql-server-or-azure-sql-db-metadata-explorer"></a>SQL Server или обозреватель метаданных базы данных Azure SQL  
SQL Server или обозреватель метаданных базы данных SQL Azure отображаются сведения об экземпляре SQL Server или базе данных SQL Azure. При подключении к экземпляру SQL Server или базе данных SQL Azure, SSMA извлекает метаданные об этом экземпляре и сохраняет его в файле проекта.  
  
Можно использовать этот обозреватель метаданных для выбора преобразованных объектов базы данных MySQL и затем синхронизировать их с экземпляром SQL Server или базе данных SQL Azure.  
  
Дополнительные сведения см. в разделе [синхронизации (MySQL на SQL Server и базе данных SQL Azure)](https://msdn.microsoft.com/ac993a6d-0283-4823-8793-6b217677dfa3)  
  
### <a name="metadata"></a>Метаданные  
Справа от каждого обозреватель метаданных находятся вкладки, которые описывают выбранного объекта. Например если выбрать таблицу в обозревателе метаданных MySQL, будет отображаться девять вкладок: **Таблица**, **SQL**, **сопоставление типов**, **данных**, **параметры**, **сопоставление Charset**, **Режимы SQL**, **свойства**, и **отчета**. **Отчетов** вкладка содержит сведения только после создания отчета, содержащий выбранный объект. Если выбрать таблицу в обозревателе метаданных SQL Server, отображается три вкладки: **Таблица**, **SQL** и **данных**.  
  
Большинство параметров метаданных доступны только для чтения. Тем не менее можно изменить следующие метаданные:  
  
-   В обозревателе метаданных MySQL можно изменить сопоставления типов, сопоставление Charset, режимы SQL. Чтобы преобразовать измененный тип сопоставления или сопоставление Charset или режимы SQL, внесите изменения, перед началом преобразования схемы.  
  
-   В обозревателе метаданных SQL Server можно изменить свойства таблиц и индексов на вкладке таблицы. Чтобы увидеть эти изменения в SQL Server, внести эти изменения, прежде чем загрузить схемы в SQL Server.  
  
Изменения, внесенные в обозревателе метаданных, отражаются в метаданные проекта, не в исходной или целевой базы данных.  
  
### <a name="toolbars"></a>Панели инструментов  
SSMA имеет две панели: панель инструментов проекта и инструментов миграции.  
  
### <a name="the-project-toolbar"></a>Панели инструментов проекта  
Панель инструментов проекта содержит кнопки для работы с проектами, подключение к MySQL и подключение к SQL Server или базе данных SQL Azure. Эти кнопки команды похожи на **файл** меню.  
  
### <a name="migration-toolbar"></a>Панель инструментов миграции  
В следующей таблице показаны миграции команды панели инструментов:  
  
|||  
|-|-|  
|**Button**|**Компонент**|  
|**Создание отчета**|Преобразует выбранные объекты MySQL к объектам SQL Server или базе данных SQL Azure, а затем создает отчет, показывающий, насколько успешно была преобразование.<br /><br />Эта команда отключает в том случае, если выбраны объекты в обозревателе метаданных MySQL.|  
|**Преобразовать схему**|Преобразует выбранные объекты MySQL объектов SQL Server или базе данных SQL Azure.<br /><br />Эта команда отключает в том случае, если выбраны объекты в обозревателе метаданных MySQL.|  
|**Перенос данных**|Перенос данных из базы данных MySQL в SQL Server или базе данных SQL Azure. Прежде чем выполнять эту команду необходимо преобразовать схемы MySQL в схемы SQL Server или базе данных SQL Azure и затем загрузить объекты в SQL Server или базе данных SQL Azure.<br /><br />Эта команда отключает в том случае, если выбраны объекты в обозревателе метаданных MySQL.|  
|**Остановить**|Останавливает текущий процесс.|

### <a name="output-pane-and-error-list-pane"></a>Область вывода и область списка ошибок  
**Представление** меню доступны команды для переключения видимости области вывода и список ошибок:  
  
-   На панели вывода показывает сообщения о состоянии из SSMA во время преобразования объекта, объект синхронизации и переноса данных.  
  
-   На панели списка ошибок отображаются ошибки, предупреждения и информационные сообщения в список поддерживает сортировку.
