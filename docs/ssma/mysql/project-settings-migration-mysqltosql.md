# <a name="project-settings-migration-mysqltosql"></a>Параметры проекта (миграция) (MySQLToSQL)
На странице миграции **параметры проекта** диалоговое окно содержит настройки, установленные как SSMA переносит данные из MySQL в SQL Server.  
  
Область миграции доступна в **параметры проекта** и **параметры проекта по умолчанию** диалоговым окнам.  
  
-   Для задания параметров для всех проектов SSMA на **средства** меню, выберите **параметры проекта по умолчанию**, выберите тип проекта в **целевой версии миграции** раскрывающийся список, из который вы хотите получить доступ к параметрам, нажмите кнопку **Общие** в нижней части левой панели, а затем нажмите кнопку **миграции**.  
  
-   Для задания параметров для текущего проекта на **средства** меню, выберите **параметры проекта**, нажмите кнопку **Общие** в нижней части левой панели и нажмите кнопку **Миграции**.  
  
## <a name="options"></a>Параметры  
  
### <a name="bulk-copy"></a>Массовое копирование  
  
|Термин|Определение|  
|--------|--------------|  
|**Размер пакета**|Указывает пакет, размер, используемый во время переноса данных.<br /><br />**Режим по умолчанию**:  1000<br /><br />**Режим оптимистичного**:  1000<br /><br />**Полный режим**:  1000|  
|**Проверочные ограничения**|Указывает, должна ли SSMA проверять ограничения при вставке данных в таблицы SQL Server.<br /><br />**Режим по умолчанию**:  False<br /><br />**Режим оптимистичного**:  False<br /><br />**Полный режим**:  False|  
|**Запускать триггеры**|Указывает, является ли SSMA запускать триггеры вставки при добавлении данных в таблицы SQL Server.<br /><br />**Режим по умолчанию**:  False<br /><br />**Режим оптимистичного**:  False<br /><br />**Полный режим**:  False|  
|**Сохранять ИД**|Указывает, сохраняет ли SSMA MySQL значений идентификаторов, при добавлении данных в SQL Server. Значение False приводит значения идентификаторов, могут быть назначены целевым объектом.<br /><br />**Режим по умолчанию**:  True<br /><br />**Режим оптимистичного**:  True<br /><br />**Полный режим**:  True|  
|**Сохранять значения NULL**|Указывает, сохраняет ли SSMA значения null в источнике данных, при добавлении данных в SQL Server, независимо от значения по умолчанию, указанные в SQL Server.<br /><br />**Режим по умолчанию**:  True<br /><br />**Режим оптимистичного**:  True<br /><br />**Полный режим**:  True|  
|**Блокировка таблицы**|Указывает, блокирует ли SSMA таблиц, при добавлении данных к таблицам во время переноса данных. Получает блокировку массовых обновлений в течение операции массового копирования. Если значение равно False, блокировка устанавливается на уровне строк.<br /><br />**Режим по умолчанию**:  False<br /><br />**Режим оптимистичного**:  False<br /><br />**Полный режим**:  False|

### <a name="migration-engine"></a>Модуль миграции  
  
|Термин|Определение|  
|--------|--------------|  
|**Модуль миграции**|Указывает компонент database engine, используемый во время переноса данных. Перенос данных на стороне клиента ссылается на клиента SSMA при извлечении данных из источника, массовой вставке данных в SQL Server. Перенос данных на стороне сервера относится к SSMA миграции подсистема обработки данных (программа массового копирования), выполняющихся на компьютере SQL Server как задание агента SQL Server, извлечение данных из источника и вставка непосредственно в SQL Server, что позволяет избежать дополнительной клиента прыжок (повышает производительность).<br /><br />**Режим по умолчанию**:  Модуль миграции данных на стороне клиента<br /><br />**Режим оптимистичного**:  Модуль миграции данных на стороне клиента<br /><br />**Полный режим**:  Модуль миграции данных на стороне клиента|  
  
> [!IMPORTANT]  
> При **модуль переноса** параметру присваивается **модуль переноса данных на стороне сервера**, создать проект, установка параметра **подсистема обработки миграции данных на стороне сервера использовать 32-разрядных** отображается . Он указывает, используется ли 32- или 64-разрядная версия программы программы массового копирования (BCP) для переноса данных.

### <a name="parallel-data-migration"></a>Перенос параллельных данных  
  
|Термин|Определение|  
|--------|--------------|  
|**Режим миграции параллельных данных**|Указывает режим, используемый для вилки потоков для включения параллельных операций миграции. В режиме Auto SSMA выбирает число потоков (10 по умолчанию), разделенными для переноса данных. В пользовательский режим, пользователь может указать число потоков, разделенными для переноса данных (минимальное значение — 1 и максимальным — 100). В настоящее время только части данных миграции механизм клиента поддерживает параллельных операций миграции.<br /><br />**Режим по умолчанию**:  Auto<br /><br />**Режим оптимистичного**:  Auto<br /><br />**Полный режим**:  Auto|  
  
> [!IMPORTANT]  
> При **параллельном режиме миграции данных** параметру присваивается **Custom**, создать проект, установка параметра **число потоков** отображается. Он указывает количество потоков, используемых для переноса данных.
