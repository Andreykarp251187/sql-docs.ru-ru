# <a name="whats-new-in-ssma-for-oracle-oracletosql"></a>Что нового в SSMA для Oracle (OracleToSS'L)

В этой статье перечислены помощник по миграции серверов (SSMA) для Oracle изменения в каждом выпуске.

## <a name="ssma-v88"></a>SSMA v8.8

V8.8 релиз SSMA для Oracle включает в себя:

* Улучшения синхронизации объектов sL Server
* Повышение производительности GUI во время оценки и преобразования
* Улучшенная конверсия аналитических `OVER PARTITION` оговорок
* Новая конверсия для `LEAD` аналитической функции
* Новая конверсия для субquery факторинговых оговорок
* Новая `REPLICATE` опция дистрибуции для хранилища данных Azure S'L
* Новый синтаксисный парсер Oracle для дальнейшего повышения производительности преобразования

## <a name="ssma-v87"></a>SSMA v8.7

V8.7 релиз SSMA для Oracle имеет незначительные исправления и улучшения производительности в графическом пользовательском интерфейсе.

Кроме того, SSMA для Oracle теперь позволяет фильтровать объекты на основе состояния действительности в диалоге 'Advanced Object Selection'.

> [!IMPORTANT]
> С SSMA v8.5 и более поздним, .NET 4.7.2 является предварительной установкой. Если вам нужно установить эту версию, вы можете скачать файл времени выполнения [здесь](https://dotnet.microsoft.com/download/dotnet-framework/net472).

## <a name="ssma-v86"></a>SSMA v8.6

В дополнение к целевому набору исправлений, предназначенных для улучшения удобства использования и производительности, v8.6 выпуск SSMA для Oracle был улучшен путем добавления параметра, который позволяет пользователям опускать расширенные свойства SSMA в преобразованном коде.

Чтобы использовать эту настройку, в SSMA для Oracle, перейдите к **Настройкам** > Объектов**Общей** > **Конверсии,****Project Settings** > а затем под **Misc,** обновите значение настройки **Omit Extended Properties** до **Да.**

![Установка Расширенных свойств Omit](../oracle/media/ssma-omit-extended-properties.png)

Кроме того, SSMA для Oracle теперь обеспечивает `XMLTABLE` улучшенный разбор положения.

> [!IMPORTANT]
> С SSMA v8.5 и более поздним, .NET 4.7.2 является предварительной установкой. Если вам нужно установить эту версию, вы можете скачать файл времени выполнения [здесь](https://dotnet.microsoft.com/download/dotnet-framework/net472).

## <a name="ssma-v85"></a>SSMA v8.5

Выпуск v8.5 SSMA для Oracle дополнен поддержкой аутентификации Active Directory Azure и базовой поддержкой функций JSON на сервере S'L, а также целевым набором исправлений, предназначенных для улучшения удобства использования и производительности.

Кроме того, SSMA для Oracle была усилена поддержкой:

* Ограничение числа выбранных объектов для обнаружения до 990 (ограничение `WHERE .. IN (..)` положения Oracle составляет 1000 элементов).
* Миграция `RAW` данных из `UNIQUEIDENTIFIER`.
* Разбор `PARALLEL_ENABLE` клаузул.

Наконец, v8.5 релиз SSMA для Oracle теперь обеспечивает:

* Улучшенная производительность преобразованных упакованных констант.
* Обновление для Поставщика данных Oracle для .NET до версии 19.5.0.

> [!IMPORTANT]
> С SSMA v8.5, .NET 4.7.2 является предварительной установкой. Если вам нужно установить эту версию, вы можете скачать файл времени выполнения [здесь](https://dotnet.microsoft.com/download/dotnet-framework/net472).

## <a name="ssma-v84"></a>SSMA v8.4

Выпуск v8.4 SSMA для Oracle дополнен целевыми исправлениями, которые предназначены для решения проблем доступности и исправления ошибки, связанной с столбиками макс индекса (чтобы позволить 32 вместо 16) для S'L Server 2016 и более поздних версий.

Кроме того, этот выпуск SSMA для `SYS_REFCURSOR` Oracle `OUT` добавляет конверсию в качестве параметров сохраненной процедуры.

> [!IMPORTANT]
> С версиями SSMA 7.4 до 8.4, .NET 4.5.2 является предварительной установкой.

## <a name="ssma-v83"></a>SSMA v8.3

V8.3 релиз SSMA для Oracle усиливается с целевыми исправлениями, которые предназначены для улучшения качества и преобразования метрик. Кроме того, этот релиз SSMA для Oracle предоставляет исправления, которые:

* Решение проблем доступности.
* Добавьте базовую поддержку для `hierarchyid` ввода в сервере S'L.
* Устраните проблему с неизвестным типом возврата для функции, называемой через синоним.
* Обновление ODP.NET до v19.3.

## <a name="ssma-v82"></a>SSMA v8.2

V8.2 релиз SSMA для Oracle усиливается до:

* Добавить `DBMS_OUTPUT.ENABLE` / `DISABLE`поддержку для .
* Удалить `CAST AS FLOAT` `BINARY_FLOAT` для `BINARY_DOUBLE` и столбцов в запросе миграции данных по умолчанию.
* Исправление обновления последовательностей, если текущее значение изменилось.
* Исправление ошибки, связанной с`ROWNUM`неправильной интерпретацией псевдоколонок (и т.д.), если существует столбец с тем же именем.
* Исправить сбой, который `FOR` происходит преобразования циклов с неоднозначным нерешенным идентификатором.

Кроме того, эта версия включает в себя целевой набор исправлений, предназначенных для улучшения качества и конверсии метрик, а также исправления для:

* Проблема с неограниченными некластерными индексами после миграции данных.
* Обнаружение рамочного соглашения .NET во время бесшумной установки.
* Периодический сбой, который возникает при загрузке новой версии.

> [!NOTE]
> Известная проблема с автоматическим обновлением может привести к сбою обновления с SSMA v8.1 до v8.2. Если вы столкнулись с этой ошибкой, пожалуйста, загрузите новую версию и установите ее вручную.

V8.1 релиз SSMA для Oracle усиливается с целевыми исправлениями, которые предназначены для улучшения качества и преобразования метрик.

  > ![SSMA для пакета расширения Oracle](../media/ssma-oracle-ext-pack.png)

  Некоторые функции, включая миграцию данных Тестера и сервера, не поддерживаются при таргетинге на управляемую базу данных Azure S'L. Дополнительные сведения см. [здесь](https://blogs.msdn.microsoft.com/datamigration/2019/02/17/migrate-your-oracle-database-to-azure-sql-database-managed-instance-using-ssma-8-0/).

* Пост-конверсия **Fix советник**. Узнайте больше об этом [здесь](https://blogs.msdn.microsoft.com/datamigration/2019/02/17/%20accelerate-your-oracle-migrations-with-new-machine-learning-capabilities-in-ssma/).

## <a name="ssma-v77"></a>SSMA v7.7

V7.7 релиз SSMA для Oracle содержит следующие изменения:

* Добавлена поддержка для сервера S'L 2016.

  >
  > Сервер S'L 2016 не поддерживает несколько функций Oracle Flashback, в том числе:
  >
  >   * Запрос транзакций Oracle Flashback
  >   * `DBMS_FLASHBACK`Пакет
  >   * Флэшбэк транзакции
  >   * Архив данных Флэшбэка
  >   * Таблица флэшбэков
  >   * Флэшбэк Капля
  >   * База данных флэшбэков

* Добавлено преобразование политики Oracle VPD в объекты политики сервера S'L (Row Level Security для Oracle).
* Снижение времени начальной загрузки для Oracle.
* Улучшенный парсер и разрешители.
* Удаленная проверка установки для .NET 2.0.
* Обновленная зависимость пакета расширений от .NET 3.5 до .NET 4.0.
* `save-project` Исправлено `open-project` и команды для консоли SSMA.
* Фиксированная `securepassword` команда для консоли SSMA.
* Фиксированный подсчет объектов для первоначальной загрузки.
* Исправленная конвертация типов данных символов для Oracle.
* Исправлена ошибка в глобальных настройках.
