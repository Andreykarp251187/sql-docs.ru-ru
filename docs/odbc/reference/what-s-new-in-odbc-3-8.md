# <a name="what39s-new-in-odbc-38"></a>Что&#39;возможности ODBC 3.8
Windows 8 включает в себя обновленную версию ODBC 3.8. ODBC 3.8 в Windows 8 включает в себя следующие компоненты:  
  
-   [Организация пулов соединений с учетом драйвера](../../odbc/reference/develop-app/driver-aware-connection-pooling.md)  
  
-   [Асинхронное выполнение (метод уведомления)](../../odbc/reference/develop-app/asynchronous-execution-notification-method.md)  
  
-   [Доступ к данным трассировки (Windows 8)](https://msdn.microsoft.com/library/windows/desktop/hh829624.aspx)  
  
-   Добавлены команды PowerShell для управления ODBC источников, драйверы ODBC, счетчиков производительности ODBC и данные отслеживание доступа к данным из командной строки.  Дополнительные сведения см. в разделе [Windows данные доступа компоненты PowerShell команды](https://msdn.microsoft.com/library/windows/desktop/jj134064.aspx).  
  
 Windows 7 включает в себя обновленную версию ODBC ODBC 3.8. ODBC 3.8 включает следующие компоненты:  
  
-   Асинхронное выполнение операций соединения. Дополнительные сведения см. в разделе [асинхронное выполнение (метод опроса)](../../odbc/reference/develop-app/asynchronous-execution-polling-method.md).  
  
-   Потоковых выходных параметров. Дополнительные сведения см. в разделе [Получение выходных параметров с помощью метода SQLGetData](../../odbc/reference/develop-app/retrieving-output-parameters-using-sqlgetdata.md).  
  
-   Тип данных ODBC C расширяемости. Дополнительные сведения о типах данных см. в разделе [Типы данных C в ODBC](../../odbc/reference/develop-app/c-data-types-in-odbc.md).  
  
 Драйвер записи следует прочитать [обновления драйвера 3.5 до 3.8](../../odbc/reference/develop-driver/upgrading-a-3-5-driver-to-a-3-8-driver.md).  
  
 Операции асинхронного подключения могут использоваться с ODBC 3.x и ODBC 2.x приложений с помощью драйвера ODBC 3.8.  
  
 Дополнительные сведения см. в разделе [Матрица совместимости](../../odbc/reference/develop-app/compatibility-matrix.md).
