# <a name="embedded-sql"></a>Embedded SQL
Первым методом отправки заявлений s'L в DBMS является встроенный S'L. Поскольку S'L не использует переменные и операторы управления потоком, он часто используется в качестве субязыка базы данных, который может быть добавлен в программу, написанную на обычном языке программирования, таких как C или COBOL. Это центральная идея встроенного S'L: размещение инструкций s'L в программе, написанной на языке программирования хоста. Вкратце, для встраиваемости инструкций s'L на языкх-хостах используются следующие методы:  
  
-   Встроенные выписки s'L обрабатываются специальным прекомпилятором S'L. Все операторы S'L начинаются с введения и заканчиваются терминатором, оба из которых помечает заявление S'L для прекомпилятора. Вводик и терминатор различаются в зависимости от языка хозяина. Например, витупором является "EXEC S'L" в C и "&S'L(" в MUMPS, а терминатором является запятая (;) в C и право йога в MUMPS.  
  
-   Переменные из программы приложения, называемые переменными хоста, могут использоваться во встроенных заявлениях S'L там, где разрешены константы. Они могут быть использованы на входных данных, чтобы адаптировать выписку по S'L к конкретной ситуации и на выходе для получения результатов запроса.  
  
-   Запросы, возвращающие одну строку данных, обрабатываются с помощью однотонного оператора SELECT; в этом заявлении оговаривается как запрос, так и переменные хоста, в которых можно вернуть данные.  
  
-   Запросы, возвращающие несколько строк данных, обрабатываются курсорами. Курсор отслеживает текущую строку в наборе результатов. Заявление DECLARE CURSOR определяет запрос, заявление OPEN начинает обработку запроса, заявление FETCH получает последовательные строки данных, а заявление CLOSE завершает обработку запросов.  
  
-   В то время как курсор открыт, позиционированное обновление и позиционированные инструкции по удалению могут быть использованы для обновления или удаления строки, в настоящее время выбранной курсором.
