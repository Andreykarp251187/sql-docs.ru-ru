# <a name="sqlbindcol-cursor-library"></a>SQLBindCol (библиотека курсоров)
> [!IMPORTANT]  
>  Эта функция будет удалена в будущей версии Windows. Избегайте использования этой функции в новых разработках и планируйте модифицировать приложения, использующие эту функцию в настоящее время. Корпорация Майкрософт рекомендует использовать функцию курсора драйвера.  
  
 На этой теме обсуждается использование функции **S'LBindCol** в библиотеке курсоров. Для получения общей информации о **функциях S'LBindCol** [см.](../../../odbc/reference/syntax/sqlbindcol-function.md)  
  
 Приложение выделяет один или несколько буферов для библиотеки курсоров, чтобы вернуть текущий набор строк. Он вызывает **S'LBindCol** один или несколько раз, чтобы связать эти буферы с набором результатов.  
  
 Приложение может вызвать **S'LBindCol,** чтобы перезавязать столбцы набора результатов после того, как оно вызвало **S'LExtendedFetch,** **S'LFetch,** или **S'LFetchScroll,** до тех пор, пока тип данных C, размер столбца и десятичные цифры связанного столбца остаются теми же. Приложению не нужно закрывать курсор для повторного обращения столбцов с разными адресами.  
  
 Библиотека курсора поддерживает установку атрибута SQL_ATTR_ROW_BIND_OFFSET_PTR оператора для использования связывающих смещений. (**S'LBindCol** не должны быть вызваны для этого повторного происходит.) Если библиотека курсора используется с драйвером ODBC *3.x,* смещение связывания не используется при вызове **S'LFetch.** Смещение связывания используется, если **s'LFetch** называется, когда библиотека курсора используется с драйвером ODBC *2.x,* потому что **S'LFetch** затем отображается на **карту s'LExtendedFetch.**  
  
 Библиотека курсора поддерживает вызов **S'LBindCol** для связывания столбца закладок.  
  
 При работе с драйвером ODBC *2.x* библиотека курсора возвращает S'LSTATE HY090 (недействительная длина строки или длина буфера), когда **s'LBindCol** призван установить длину буфера для столбца закладок к значению, не равному 4. При работе с драйвером ODBC *3.x* библиотека курсора позволяет буферу быть любого размера.
