# <a name="sqlsetpos-cursor-library"></a>SQLSetPos (библиотека курсоров)
> [!IMPORTANT]  
>  Эта функция будет удалена в будущей версии Windows. Избегайте использования этой функции в новых разработках и планируйте модифицировать приложения, использующие эту функцию в настоящее время. Корпорация Майкрософт рекомендует использовать функцию курсора драйвера.  
  
 На этой теме обсуждается использование функции **S'LSetPos** в библиотеке курсоров. Для получения общей информации о **S'LSetPos** [см.](../../../odbc/reference/syntax/sqlsetpos-function.md)  
  
 Библиотека курсора поддерживает SQL_POSITION операцию только для аргумента *операции* в **S'LSetPos.** Он поддерживает SQL_LOCK_NO_CHANGE значение только для аргумента *LockType.*  
  
 Если драйвер не поддерживает объемные операции, библиотека курсоров возвращает S'LSTATE HYC00 (Driver not capable), когда **S'LSetPos** вызывается с *RowNumber,* равным 0. Такое поведение драйвера не рекомендуется.  
  
 Библиотека курсоров не поддерживает SQL_UPDATE и SQL_DELETE операции в вызове в **S'LSetPos.** Библиотека курсора реализует позиционированное обновление или удаляет оператору S'L, создавая поисковое обновление или удаляя выписку с оговоркой WHERE, которая перечисляет значения, хранящиеся в кэше для каждого связанного столбца. Для получения дополнительной информации [см.](../../../odbc/reference/appendixes/processing-positioned-update-and-delete-statements.md)  
  
 Если драйвер не поддерживает статические курсоры, приложение, работая с библиотекой курсоров, должно **вызыватьs S'LSetPos** только на строке, извлеченной **S'LExtendedFetch** или **S'LFetchScroll**, а не **s'LFetch.** Библиотека курсоров осуществит **s'LExtendedFetch** и **S'LFetchScroll,** делая в драйвере неоднократные вызовы **S'LFetch** (с размером ряда 1). Библиотека курсора передает звонки в **S'LFetch,** с другой стороны, до водителя. Если **S'LSetPos** вызывается на многорядум наборе, извлеченном **S'LFetch,** когда драйвер не поддерживает статические курсоры, вызов не будет работать, потому что **S'LSetPos** не работает с курсорами только вперед. Это произойдет даже в том случае, если приложение успешно подназваниевало **s'LSetStmtAttr** для установки SQL_ATTR_CURSOR_TYPE SQL_CURSOR_STATIC, которую поддерживает библиотека курсора, даже если драйвер не поддерживает статические курсоры.
