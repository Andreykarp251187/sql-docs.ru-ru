# <a name="string-functions"></a>Строковые функции
В следующей таблице перечислены функции обработки строк. Приложение может определить, какие строковые функции поддерживаются драйвером, вызвав **SQLGetInfo** с *тип сведений* из SQL_STRING_FUNCTIONS.  
  
## <a name="remarks"></a>Примечания  
 Аргументы обозначается как *строковое_выражение* может быть именем столбца, *символ строковых литералов*, ни результатом другой скалярной функцией, где базовый тип данных может быть представлено как SQL_CHAR, SQL_ VARCHAR или SQL_LONGVARCHAR.  
  
 Аргументы обозначается как *character_exp* представляют собой строку знаков переменной длины.  
  
 Аргументы обозначается как *запустить*, *длина*, или *число* может быть *числовой литерал* , ни результатом другой скалярной функцией, где базовый тип данных может быть представлено как SQL_TINYINT, SQL_SMALLINT или SQL_INTEGER.  
  
 Строковые функции, перечисленные здесь начинаются с 1; то есть первый символ в строке является символом 1.  
  
 Функция BIT_LENGTH, CHAR_LENGTH, CHARACTER_LENGTH, OCTET_LENGTH и ПОЗИЦИИ строки скалярные функции были добавлены в ODBC 3.0 в соответствии со стандартом SQL-92.  
  
|Компонент|Описание|  
|--------------|-----------------|  
|**ASCII (** _строковое_выражение_ **)** (ODBC 1.0)|Возвращает код ASCII первого символа указанного *строковое_выражение* как целое число.|  
|**Функция BIT_LENGTH (** _строковое_выражение_ **)** (ODBC 3.0)|Возвращает длину строкового выражения в битах.<br /><br /> Работает только для строковых типов данных, поэтому будет не неявно преобразовать *строковое_выражение* со строкой, но вместо этого возвращается (внутренний) размер любого типа данных, ему присваивается.|  
|**CHAR (** _кода_ **)** (ODBC 1.0)|Возвращает символ, который имеет ASCII кода значение, заданное параметром *кода*. Значение *кода* должно быть в диапазоне от 0 до 255; в противном случае возвращает значение зависит от источника данных.|  
|**CHAR_LENGTH (** _строковое_выражение_ **)** (ODBC 3.0)|Возвращает длину в символах строкового выражения, если строковое выражение имеет символьный тип данных; в противном случае возвращает длину в байтах строкового выражения (наименьшее целое число не меньше, чем количество битов, разделенное на 8). (Эта функция является таким же, как функция CHARACTER_LENGTH.)|  
|**CHARACTER_LENGTH (** _строковое_выражение_ **)** (ODBC 3.0)|Возвращает длину в символах строкового выражения, если строковое выражение имеет символьный тип данных; в противном случае возвращает длину в байтах строкового выражения (наименьшее целое число не меньше, чем количество битов, разделенное на 8). (Эта функция является таким же, как функция CHAR_LENGTH.)|  
|**CONCAT (** _строковое_выражение1_,_строковое_выражение2_ **)** (ODBC 1.0)|Возвращает символьную строку, являющуюся результатом сцепления *строковое_выражение2* для *строковое_выражение1*. Полученная в результате строка зависит от СУБД. Например, если столбец, представленный *строковое_выражение1* содержится значение NULL, DB2 бы возвращаемого значения NULL, но SQL Server возвратит НЕПУСТАЯ строка.|  
|**РАЗНИЦА (** _строковое_выражение1_,_строковое_выражение2_ **)** (ODBC 2.0)|Возвращает целочисленное значение, указывающее разницу между значениями, возвращаемому функцией SOUNDEX для *строковое_выражение1* и *строковое_выражение2*.|  
|**INSERT (** _строковое_выражение1_, *запустить*, *длина*, _строковое_выражение2_ **)** (ODBC 1.0)|Возвращает символ строки, где *длина* символы были удалены из *строковое_выражение1*, начиная с *запустить*, где и когда *строковое_выражение2* была введена в *строковое_выражение,* начиная *запустить*.|  
|**LCASE (** _строковое_выражение_ **)** (ODBC 1.0)|Возвращает строку, в *строковое_выражение*, все прописные буквы алфавита, преобразованную в нижний регистр.|  
|**LEFT (** _строковое_выражение_, _число_ **)** (ODBC 1.0)|Возвращает крайнее левое *число* символов *строковое_выражение*.|  
|**Длина (** _строковое_выражение_ **)** (ODBC 1.0)|Возвращает количество символов в *строковое_выражение,* исключая конечные пробелы.<br /><br /> **Длина** принимает только строки. Таким образом, неявно преобразует *строковое_выражение* в строку и возвращают длина этой строки (не размер внутренний размер типа данных).|  
|**НАЙДИТЕ (** _строковое_выражение1_, *строковое_выражение2*[, *запустить*] **)** (ODBC 1.0)|Возвращает начальную позицию первого вхождения *строковое_выражение1* в *строковое_выражение2*. Поиск первого вхождения *строковое_выражение1* начинается с позиции первого знака в *строковое_выражение2* Если необязательный аргумент, *запустить*, указан. Если *запустить* указывается, поиск начинается с позиции символа значение *запустить*. Позиция первого символа в *строковое_выражение2* указывается значение 1. Если *строковое_выражение1* не найден в *строковое_выражение2*, возвращается значение 0.<br /><br /> Если приложение может вызвать скалярные функции поиска с *строковое_выражение1*, *строковое_выражение2*, и *запустить* аргументы, драйвер возвращает SQL_FN_STR_LOCATE при  **SQLGetInfo** вызывается с *параметр* из SQL_STRING_FUNCTIONS. Если приложение может вызвать скалярной функции найти только с *строковое_выражение1* и *строковое_выражение2* аргументы, драйвер возвращает SQL_FN_STR_LOCATE_2 при **SQLGetInfo**вызывается с *параметр* из SQL_STRING_FUNCTIONS. Драйверы, которые поддерживают, вызвав функцию поиска с двух или трех аргументов возвращают SQL_FN_STR_LOCATE и SQL_FN_STR_LOCATE_2.|  
|**LTRIM (** _строковое_выражение_ **)** (ODBC 1.0)|Возвращает символы *строковое_выражение*, с удалены пробелы в начале.|  
|**Функция OCTET_LENGTH (** _строковое_выражение_ **)** (ODBC 3.0)|Возвращает длину строкового выражения в байтах. Результатом является наименьшее целочисленное значение, не меньшее, чем число битов, разделенное на 8.<br /><br /> Работает только для строковых типов данных, поэтому будет не неявно преобразовать *строковое_выражение* со строкой, но вместо этого возвращается (внутренний) размер любого типа данных, ему присваивается.|  
|**ПОЗИЦИЯ (** _character_exp_ **IN** _character_exp_ **)** (ODBC 3.0)|Возвращает позицию первого символа выражения во втором выражении символ. Результатом является точное числовое значение с точностью, определяемое реализацией и масштабом 0.|  
|**ПОВТОРИТЕ (** _строковое_выражение,_ _число_ **)** (ODBC 1.0)|Возвращает строку символов, состоящую из *строковое_выражение* повторить *число* раз.|  
|**ЗАМЕНИТЕ (** _строковое_выражение1_, *строковое_выражение2*, _string_exp3_ **)** (ODBC 1.0)|Поиск *строковое_выражение1* foroccurrences из *строковое_выражение2*и замените *string_exp3*.|  
|**СПРАВА (** _строковое_выражение_, _число_ **)** (ODBC 1.0)|Возвращает самым правым *число* символов *строковое_выражение*.|  
|**RTRIM (** _строковое_выражение_ **)** (ODBC 1.0)|Возвращает символы *строковое_выражение* с замыкающими пробелами удалены.|  
|**SOUNDEX (** _строковое_выражение_ **)** (ODBC 2.0)|Возвращает строку символов зависит от источника данных, представляющий звук слов в *строковое_выражение*. Например SQL Server возвращает четырехзначный код SOUNDEX; Oracle возвращает фонетическому представлению каждого слова.|  
|**ПРОСТРАНСТВО (** _число_ **)** (ODBC 2.0)|Возвращает строку символов, состоящий из *число* пробелы.|  
|**SUBSTRING (** _строковое_выражение_, *запустить*, длина **)** (ODBC 1.0)|Возвращает строку символов, который является производным от *строковое_выражение*, начиная с позиции, указанной параметром *запустить* для *длина* символов.|  
|**UCASE (** _строковое_выражение_ **)** (ODBC 1.0)|Возвращает строку, в *строковое_выражение*, со всеми нижний регистр, переведенную в верхний регистр.|
