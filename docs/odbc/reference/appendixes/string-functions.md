# <a name="string-functions"></a>Строковые функции
В следующей таблице перечислены функции манипуляции строками. Приложение может определить, какие функции строки поддерживаются драйвером, позвонив в **S'LGetInfo** с *информационным типом* SQL_STRING_FUNCTIONS.  
  
## <a name="remarks"></a>Remarks  
 Аргументы, обозначаемые как *string_exp* могут быть названием столбца, *символом-строкой-буквальным*или результатом другой функции масштабирования, где базовый тип данных может быть представлен как SQL_CHAR, SQL_VARCHAR или SQL_LONGVARCHAR.  
  
 Аргументы, обозначаемые как *character_exp,* являются строкой символов переменной длины.  
  
 Аргументы, обозначаемые как *начало,* *длина*или *подсчет,* могут быть *числовыми буквами* или результатом другой функции масштабирования, где базовый тип данных может быть представлен как SQL_TINYINT, SQL_SMALLINT или SQL_INTEGER.  
  
 Функции строки, перечисленные здесь, основаны на 1; то есть, первый символ в строке является символом 1.  
  
 В функции odBC 3.0 были добавлены функции BIT_LENGTH, CHAR_LENGTH, CHARACTER_LENGTH, OCTET_LENGTH и POSITION, чтобы выровнять их с S'L-92.  
  
|Компонент|Описание|  
|--------------|-----------------|  
|**ASCII** _(string_exp)_ **)** (ODBC 1.0)|Возвращает значение кода ASCII самого левого символа *string_exp* в качестве целых.|  
|**BIT_LENGTH** _(string_exp)_ **)** (ODBC 3.0)|Возвращает длину строкового выражения в битах.<br /><br /> Не работает только для типов строки данных, поэтому не будет неявно конвертировать *string_exp* в строку, но вместо этого вернет (внутренний) размер любого типа данных, который он дает.|  
|**CHAR** _(код)_ **)** (ODBC 1.0)|Возвращает символ с кодовым значением ASCII, указанным *кодом.* Значение *кода* должно быть между 0 и 255; в противном случае значение возврата зависит от источников данных.|  
|**CHAR_LENGTH** _(string_exp)_ **)** (ODBC 3.0)|Возвращает длину в символы выражения строки, если выражение строки имеет тип данных символов; в противном случае возвращает длину в байтах выражения строки (наименьшее множество не меньше количества битов, разделенных на 8). (Эта функция такая же, как CHARACTER_LENGTH функции.)|  
|**CHARACTER_LENGTH** _(string_exp)_ **)** (ODBC 3.0)|Возвращает длину в символы выражения строки, если выражение строки имеет тип данных символов; в противном случае возвращает длину в байтах выражения строки (наименьшее множество не меньше количества битов, разделенных на 8). (Эта функция такая же, как функция CHAR_LENGTH.)|  
|**КОНКАТ** _(string_exp1_,_string_exp2_**)** (ODBC 1.0)|Возвращает строку персонажа, которая является результатом свертывания *string_exp2* *string_exp1.* Полученная в результате строка зависит от СУБД. Например, если столбец, представленный *string_exp1,* содержал значение NULL, DB2 возвращал NULL, но S'L Server возвращал строку, не являющихся NULL.|  
|**ДИЦИЯ** _(string_exp1_,_string_exp2)_**)** (ODBC 2.0)|Возвращает величину, указывающем разницу между значениями, возвращенными функцией SOUNDEX для *string_exp1* и *string_exp2.*|  
|**INSERT** _(string_exp1,_ *старт,* *длина,* _string_exp2)_**)** (ODBC 1.0)|Возвращает строку символов, где *длина* символов были удалены из *string_exp1,* начиная с *начала*, и где *string_exp2* была вставлена в *string_exp,* начиная с *начала*.|  
|**LCASE (string_exp** _string_exp_ **)** (ODBC 1.0)|Возвращает строку, равную той, что в *string_exp,* со всеми символами верхнего регистра преобразуется в нижний регистр.|  
|**LEFT (string_exp** _string_exp_, _кол_**)** (ODBC 1.0)|Возвращает левые *символы кол* *string_exp*.|  
|**LENGTH (** _string_exp_ **)** (ODBC 1.0)|Возвращает количество символов в *string_exp,* исключая задние пробелы.<br /><br /> **LENGTH** принимает только строки. Поэтому неявно преобразовывать *string_exp* строку и возвращать длину этой строки (не внутренний размер типа данных).|  
|**LOCATE (string_exp1** _string_exp1_, *string_exp2*, *старт***)** (ODBC 1.0)|Возвращает исходное положение первого появления *string_exp1* в течение *string_exp2.* Поиск первого появления *string_exp1* начинается с первого положения символов в *string_exp2* если не указан дополнительный аргумент, *не*будет указан. Если *запуск* указан, поиск начинается с позиции персонажа, указанной значением *старта.* Первая позиция символа в *string_exp2* указывается значением 1. Если *string_exp1* не найдено в *пределах string_exp2,* значение 0 возвращается.<br /><br /> Если приложение может вызвать функцию scalar LOCATE с *string_exp1,* *string_exp2*и *начать* аргументы, водитель возвращается SQL_FN_STR_LOCATE, когда **s'LGetInfo** вызывается с *опцией* SQL_STRING_FUNCTIONS. Если приложение может вызвать функцию scalar LOCATE только с *string_exp1* и *string_exp2* аргументами, водитель возвращается SQL_FN_STR_LOCATE_2, когда **s'LGetInfo** вызывается с *опцией* SQL_STRING_FUNCTIONS. Водители, поддерживающие вызов функции LOCATE двумя или тремя аргументами, возвращают ся SQL_FN_STR_LOCATE и SQL_FN_STR_LOCATE_2.|  
|**LTRIM (string_exp** _string_exp_ **)** (ODBC 1.0)|Возвращает символы *string_exp,* с ведущими пробелами удалены.|  
|**OCTET_LENGTH** _(string_exp)_ **)** (ODBC 3.0)|Возвращает длину строкового выражения в байтах. Результатом является наименьшее целочисленное значение, не меньшее, чем число битов, разделенное на 8.<br /><br /> Не работает только для типов строки данных, поэтому не будет неявно конвертировать *string_exp* в строку, но вместо этого вернет (внутренний) размер любого типа данных, который он дает.|  
|**ПОЗИЦИЯ (character_exp** _character_exp_ **в** _character_exp_**)** (ODBC 3.0)|Возвращает положение выражения первого персонажа во втором выражении персонажа. Результатом является точное число с точностью, определяемой реализацией, и шкалой 0.|  
|**REPEAT** _(string_exp,_ _кол)_**)** (ODBC 1.0)|Возвращает строку символов, состоящую из *string_exp* повторного *времени подсчета.*|  
|**REPLACE** _(string_exp1,_ *string_exp2,* _string_exp3)_**)** (ODBC 1.0)|Поиск *string_exp1* для возникновения *string_exp2,* и заменить *string_exp3*.|  
|**RIGHT (string_exp** _string_exp_, _кол)_**)** (ODBC 1.0)|Возвращает *правоколмдляцы* *символов string_exp.*|  
|**RTRIM(** _string_exp_ **)** (ODBC 1.0)|Возвращает символы *string_exp* с удаленными задними пробелами.|  
|**SOUNDEX (string_exp** _string_exp_ **)** (ODBc 2.0)|Возвращает строку, зависящая от источника данных, представляющую звук слов в *string_exp.* Например, сервер S'L возвращает 4-значный код SOUNDEX; Oracle возвращает фонетическое представление каждого слова.|  
|**SPACE** _(счет)_ **)** (ODBC 2.0)|Возвращает строку символов, состоящую из пробелов *подсчета.*|  
|**SUBSTRING (** _string_exp,_ *старт,* длина **)** (ODBC 1.0)|Возвращает строку символов, которая происходит от *string_exp,* начиная с позиции символа, указанной *стартом* для символов *длины.*|  
|**UCASE (string_exp** _string_exp_ **)** (ODBC 1.0)|Возвращает строку, равную той, что в *string_exp,* со всеми символами нижнего регистра преобразуется в верхний регистр.|
