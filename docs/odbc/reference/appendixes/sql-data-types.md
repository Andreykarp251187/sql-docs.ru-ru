# <a name="sql-data-types"></a>Типы данных SQL
Каждый DBMS определяет свои типы S'L. Каждый драйвер ODBC предоставляет только те типы данных, которые определяет связанный DBMS. Информация о том, как драйвер отображает типы DBMS S'L в идентификаторах типа ODBC, определяемых СЗЛ, и о том, как водитель отображает типы DBMS S'L в своих собственных типах типа Драйвера, возвращается через звонок в **S'LGetTypeInfo.** Кроме того, драйвер возвращает типы данных s'L при описании типов и параметров данных через вызовы на **S'LColAttribute,** **S'LКолонки**, **S'LDescribeCol**, **S'LDescribeParam**, **S'LProcedureColumns**, **s'LSpecialColumns**.  
  
> [!NOTE]  
>  Типы данных S'L содержатся в CONCISE_TYPE SQL_DESC_, SQL_DESC_TYPE и SQL_DESC_DATETIME_INTERVAL_CODE полях дескрипторов реализации. Характеристики типов данных S'L содержатся в SQL_DESC_PRECISION, SQL_DESC_SCALE, SQL_DESC_LENGTH и SQL_DESC_OCTET_LENGTH полях дескрипторов реализации. Для получения дополнительной информации в этом приложении смотрите [идентификаторы и дескрипторы типов данных.](../../../odbc/reference/appendixes/data-type-identifiers-and-descriptors.md)  
  
 Данный драйвер и источник данных не обязательно поддерживают все типы данных S'L, которые определены в этом приложении. Поддержка драйвера для типов данных S'L зависит от уровня S-L-92, который соответствует требованиям водителя. Чтобы определить уровень грамматики S'L-92, поддерживаемой драйвером, приложение вызывает **S'LGetInfo** с SQL_SQL_CONFORMANCE типом информации. Кроме того, данный драйвер и источник данных могут поддерживать дополнительные, специфические для драйвера типы данных S'L. Чтобы определить, какие типы данных поддерживает драйвер, приложение вызывает **S'LGetTypeInfo.** Для получения информации о типах данных, специфичных для драйверов, просмотрите документацию водителя. Для получения информации о типах данных в конкретном источнике данных см.

 Это значение, возвращенное в колонке NAME и CREATE PARAMS по вызову в **S'LGetTypeInfo**. Столбец NAME возвращает обозначение, например, CHAR-в то время как столбец CREATE PARAMS возвращает разделенный запятой список параметров создания, таких как точность, масштаб и длина.  
  
 Приложение использует **S'LGetTypeInfo** или **S'LColAttribute** для определения того, является ли определенный тип данных или определенный столбец в наборе результатов неподписанным.  
  
 SQL_DECIMAL и SQL_NUMERIC типы данных отличаются только точностью. Точность DECIMAL *(p*,*s*) является реализации определены десятичной точности, которая не меньше, чем *р*, в то время как точность NUMERIC *(p*,*s*) точно равна *р*.  
  
 В зависимости от реализации точность SQL_FLOAT может быть 24 или 53: если она 24, то тип данных SQL_FLOAT такой же, как SQL_REAL; если это 53, то SQL_FLOAT тип данных такой же, как SQL_DOUBLE.
