# <a name="transfer-octet-length"></a>Длительность октета передачи
Длина октета переноса столбца — это максимальное количество байтов, возвращенных в приложение при передаче данных в тип данных C по умолчанию. Для данных о символах длина переноса октета не включает пространство для символа с нулевым окончанием. Длина октета переноса столбца может отличаться от количества байтов, необходимых для хранения данных об источнике данных.  
  
 Длина переноса октета, определяемая для каждого типа данных ODBC S'L, отображается в следующей таблице.  
  
|Идентификатор типа S'L|Длина|  
|-------------------------|------------|  
|Все типы символов|Определенная или максимальная (для переменного типа) длина столбца в байтах. Это то же самое значение, что и поле дескриптора SQL_DESC_OCTET_LENGTH.|  
|SQL_DECIMAL<br />SQL_NUMERIC|Количество байтов, необходимых для хранения представления символов этих данных, если набор символов является ANSI, и в два раза больше, если набор символов является UNICODE. Это максимальное количество цифр плюс два, потому что данные возвращаются как строка символов и символы необходимы для цифр, знака и десятичной точки. Например, длина переноса столбца, определяемого как NUMERIC (10,3), составляет 12.|  
|SQL_TINYINT|1|  
|SQL_SMALLINT|2|  
|SQL_INTEGER|4|  
|SQL_BIGINT| 8 |  
|SQL_REAL|4|  
|SQL_FLOAT|8|  
|SQL_DOUBLE|8|  
|SQL_BIT|1|  
|Все бинарные типы|Количество байтов, необходимых для удержания определенного (для фиксированных типов) или максимального (для переменных типов) количества символов.|  
|SQL_TYPE_DATE<br />SQL_TYPE_TIME|6 (размер SQL_DATE_STRUCT или SQL_TIME_STRUCT структуры).|  
|SQL_TYPE_TIMESTAMP|16 (размер SQL_TIMESTAMP_STRUCT структуры).|  
|Все типы интервальных данных|34 (размер интервальная структура).|  
|SQL_GUID|16 (размер структуры GUID).|  
| &nbsp; | &nbsp; |

 Если драйвер не может определить столбец или длину параметра для переменных типов, он возвращается SQL_NO_TOTAL.
