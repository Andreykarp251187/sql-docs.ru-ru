# <a name="sqlgetdata-cursor-library"></a>SQLGetData (библиотека курсоров)
> [!IMPORTANT]  
>  Эта функция будет удалена в будущей версии Windows. Избегайте использования этой функции в новых разработках и планируйте модифицировать приложения, использующие эту функцию в настоящее время. Корпорация Майкрософт рекомендует использовать функцию курсора драйвера.  
  
 На этой теме обсуждается использование функции **S'LGetData** в библиотеке курсоров. Для получения общей информации о **s'LGetData,** [см.](../../../odbc/reference/syntax/sqlgetdata-function.md)  
  
 Библиотека курсора реализует **s'LGetData,** сначала построив заявление **SELECT** с оговоркой **WHERE,** в которой перечислены значения, хранящиеся в кэше для каждого связанного столбца в текущем ряду. Затем он выполняет заявление **SELECT** для перевыбора строки и вызывает **s'LGetData** в драйвере для извлечения данных из источника данных (в отличие от кэша).  
  
> [!CAUTION]  
>  Положение **WHERE,** построенное библиотекой курсоров для идентификации текущей строки, может не идентифицировать строки, определить другую строку или определить несколько строк. Для получения дополнительной [информации см.](../../../odbc/reference/appendixes/constructing-searched-statements.md)  
  
 Если атрибут SQL_ATTR_USE_BOOKMARKS оператора установлен на SQL_UB_VARIABLE, **s-LGetData** может быть вызван в столбец 0 для возврата данных закладок.  
  
 На **вызовы на S'LGetData** распространяются следующие ограничения:  
  
-   **Нельзя** вызывать курсоры только для форвардов.  
  
-   **SLGetData** можно вызвать только при соблюдении следующих условий: выписка **SELECT,** генерируемая набором результатов; заявление **SELECT** не содержало присоединения, положения **UNION** или положения **GROUP BY;** и любые столбцы, использовавшие псевдоним или выражение в списке выбора, не были связаны с **S'LBindCol.**  
  
-   Если драйвер поддерживает только одно активное утверждение, библиотека курсора получает остальную часть результата, установленного перед выполнением оператора **SELECT** и вызовом **S'LGetData.**
