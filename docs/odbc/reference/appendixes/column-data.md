# <a name="column-data"></a>Столбец данных
> [!IMPORTANT]  
>  Эта функция будет удалена в будущей версии Windows. Избегайте использования этой функции в новых разработках и планируйте модифицировать приложения, использующие эту функцию в настоящее время. Корпорация Майкрософт рекомендует использовать функцию курсора драйвера.  
  
 Библиотека курсора создает буфер в кэше для каждого буфера данных, привязанного к набору результатов с **s'LBindCol.** Он использует значения в этих буферах для построения положения **WHERE,** когда он эмулирует позиционированное обновление или удаление оператора. Он обновляет эти буферы из буферов строки, когда он получает данные из источника данных и когда он выполняет позиционированные операторы обновления.  
  
 Когда библиотека курсора обновляет свой кэш из буферов строк, она передает данные в соответствии с типом данных C, указанным в **S'LBindCol.** Например, если тип c данных буфера набора строк SQL_C_SLONG, библиотека курсора передает четыре байта данных; если это SQL_C_CHAR и *BufferLength* 10, библиотека курсоров передает 10 байт данных. Библиотека курсора не выполняет проверку типа или преобразования данных, которые она передает.  
  
> [!NOTE]  
>  Библиотека курсора не обновляет свой кэш для столбца, если*StrLen_or_IndPtr* в соответствующем буфере набора строк SQL_DATA_AT_EXEC или результат омователя SQL_LEN_DATA_AT_EXEC.  
  
 При обновлении столбца, источник данных пустые колодки фиксированной длины символов данных и нулевой колодки фиксированной длины двоичных данных по мере необходимости. Например, источник данных хранит "Смит" в столбце CHAR (10) как "Смит". Библиотека курсора не содержит пустых папок или данных с нулевыми прокладками в буферах строк, когда она копирует эти данные в свой кэш после выполнения оператора обновлений. Поэтому, если приложение требует, чтобы значения в кэше библиотеки курсора были пустыми или нулевыми, оно должно сложить или с нулевым инее накладки значения в буферах строки перед выполнением позиционированного оператора обновления.
