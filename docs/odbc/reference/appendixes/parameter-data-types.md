# <a name="parameter-data-types"></a>Типы данных параметров
Несмотря на то, что каждый параметр, указанный в **S'LBindParameter,** определяется с использованием типа данных S'L, параметры в выписке S'L не имеют типов внутренних данных. Таким образом, параметры маркеров могут быть включены в выписку s'L только в том случае, если их типы данных могут быть выведены из другой антеанды в заявлении. Например, в арифметическом выражении, таком как ? - COLUMN1, тип данных параметра можно вывести из типа данных названного столбца, представленного COLUMN1. Приложение не может использовать маркер параметра, если тип данных не может быть определен.  
  
 В следующей таблице описывается, как определяется тип данных для нескольких типов параметров, в соответствии с S'L-92. Для более полной спецификации о выводе типа параметров при использовании других оговорок S'L см.  
  
|Расположение параметра|Предполагаемый тип данных|  
|---------------------------|-----------------------|  
|Одна операндбитарная арифметика или сравнительный оператор|То же, что и другие операнды|  
|Первая неприязавка в пункте **BETWEEN**|То же, что и вторая операнд|  
|Вторая или третья переутня в пункте **BETWEEN**|То же, что и первый операнд|  
|Выражение, используемое с **IN**|То же самое, что и первое значение или итоговый столбец подкомитета|  
|Значение, используемое с **IN**|То же, что выражение или первое значение, если есть параметр маркера в выражении|  
|Значение шаблона, используемое с **LIKE**|VARCHAR|  
|Значение обновления, **используемое** с ОБНОВЛЕНИЕ|То же, что и столбец обновления|
