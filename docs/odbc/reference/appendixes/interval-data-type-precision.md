# <a name="interval-data-type-precision"></a>Точность интервального типа данных
Точность для типа интервальных данных включает точность интервального ведущего, точность интервала и точность секунд.  
  
 Ведущим полем интервала является подписанный числовый. Максимальное количество цифр для ведущего поля определяется количеством, называемым *точностью интервала,* которая является частью декларации типа данных. Например, декларация: INTERVAL HOUR(5) TO MINUTE имеет интервал, ведущий точностью 5; поле HOUR может принимать значения от -99999 до 99999. Точность интервала, ведущая, содержится в SQL_DESC_DATETIME_INTERVAL_PRECISION поле записи дескриптора.  
  
 Список полей, из которых состоит тип интервальных данных, называется *точностью интервала.* Это не числовое значение, как это может означать термин "точность". Например, интервальная точность типа INTERVAL DAY TO SECOND - это список ДНЯ, ДНЯ, ДНЯ, МИНУТА, ВТОРОГО. Нет поля дескриптора, которое бы не соотведомь, которое удерживает это значение; точность интервала всегда может быть определена типом интервальных данных.  
  
 Любой тип интервальных данных, который имеет поле SECOND, имеет *точность секунд.* Это количество десятичных цифр, разрешенных в дробной части значения секунд. Это отличается от других типов данных, где точность указывает количество цифр до десятичной точки. Точность секундного типа данных — это количество цифр после десятичной точки. Например, если точность секунд установлена до 6, число 123456 в поле фракции будет интерпретировано как .123456, а число 1230 будет интерпретироваться как .001230. Для других типов данных это называется шкалой. Точность секунд интервала содержится в SQL_DESC_PRECISION поле дескриптора. Если точность компонента дробных секунд интервального значения S'L больше, чем то, что может быть проведено в структуре интервала C, определяется ли значение дробных секунд в интервале S'L округляется или усечено при преобразовании в интервальную структуру C.  
  
 При установке SQL_DESC_CONCISE_TYPE поля на тип данных интервала SQL_DESC_TYPE устанавливается SQL_INTERVAL и SQL_DESC_DATETIME_INTERVAL_CODE в код для типа данных интервала. Поле SQL_DESC_DATETIME_INTERVAL_PRECISION автоматически устанавливается на точность интервала по умолчанию 2, а SQL_DESC_PRECISION поле автоматически устанавливается на точность секунд интервала по умолчанию 6. Если любое из этих значений не подходит, приложение должно четко установить поле дескриптора через вызов в **S'LSetDescField.**
