# <a name="sql-minimum-grammar"></a>Минимальная грамматика SQL
В этом разделе описывается минимальный синтаксис S'L, который должен поддерживать драйвер ODBC. Синтаксис, описанный в этом разделе, представляет собой подмножество синтаксиса начального уровня S'L-92.  
  
 Приложение может использовать любой синтаксис в этом разделе и быть уверенным, что любой драйвер, совместимый с ODBC, будет поддерживать этот синтаксис. Чтобы определить, поддерживаются ли дополнительные функции S'L-92, не в данном разделе, приложение должно позвонить в **S'LGetInfo** с SQL_SQL_CONFORMANCE типом информации. Даже если драйвер не соответствует какому-либо уровню соответствия S'L-92, приложение все равно может использовать синтаксис, описанный в этом разделе. С другой стороны, если драйвер соответствует уровню S'L-92, он поддерживает весь синтаксис, включенный в этот уровень. Это включает в себя синтаксис в этом разделе, потому что минимальная грамматика, описанная здесь, представляет собой чистое подмножество самого низкого уровня соответствия S'L-92. После того, как приложение узнает об уровне S'L-92, оно может определить, поддерживается ли функция более высокого уровня (если таковая имеется), позвонив в **S'LGetInfo** с индивидуальным типом информации, соответствующим этой функции.  
  
 Драйверы, которые работают только с источниками данных только для чтения, могут не поддерживать те части грамматики, которые включены в этот раздел, которые касаются изменяющихся данных. Приложение может определить, читается ли источник данных только по телефону **s'LGetInfo** с SQL_DATA_SOURCE_READ_ONLY типом информации.
