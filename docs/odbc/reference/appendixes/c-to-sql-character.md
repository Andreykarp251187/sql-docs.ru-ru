 При преобразовании данных символов C в числовые, дата, время или временные метки s-L данные игнорируются.  
  
 Когда данные о символе C преобразуются в двоичные данные S'L, каждые два байта данных о персонажах преобразуются в один байт (8 бит) двоичных данных. Каждые два байта данных о персонажах представляют собой число в гексадецичной форме. Например, "01" преобразуется в двоичный 00000001, а "FF" преобразуется в двоичный 11111111.  
  
 Водитель всегда преобразует пары гексадецимальных цифр в отдельные байты и игнорирует байт с нулевым прекращением. Из-за этого, если длина строки символа нечетна, последний байт строки (за исключением байта с нулевым прекращением, если таковые имеется) не преобразуется.  
  
> [!NOTE]  
>  Разработчики приложений не рекомендуется связывать данные символов C с бинарным типом данных S'L. Это преобразование, как правило, неэффективно и медленно.
