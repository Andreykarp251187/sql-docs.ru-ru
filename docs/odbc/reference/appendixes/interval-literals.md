# <a name="interval-literals"></a>Литералы интервала
ODBC требует, что все драйверы поддерживают преобразование типа данных SQL_CHAR или SQL_VARCHAR ко всем типам данных C интервал. Если базовый источник данных не поддерживает типы данных интервала, тем не менее, драйвер должен знать правильного формата значение в поле SQL_CHAR для поддержки этих преобразований. Аналогичным образом ODBC требует, что любой ODBC C, тип преобразовываться SQL_CHAR или SQL_VARCHAR, поэтому драйвер должен знать, какой формат сохранено в поле символ интервал должен иметь. В этом разделе описывается синтаксис литералов интервала, в которых разработчику драйверов необходимо использовать для проверки поля SQL_CHAR во время преобразования в или из типа данных interval C.  
  
> [!NOTE]  
>  В разделе отображается полный синтаксис BNF литералы интервала [синтаксис литерала интервала](../../../odbc/reference/appendixes/interval-literal-syntax.md) в приложение в: Грамматика SQL.  
  
 Чтобы передать литералы интервала как часть инструкции SQL, синтаксис escape-предложении определяется для литералов интервала. Дополнительные сведения см. в разделе [Timestamp литералы даты, времени и](../../../odbc/reference/develop-app/date-time-and-timestamp-literals.md).  
  
 Интервал литерал имеет вид:  
  
```  
INTERVAL[<sign>] 'value' <interval qualifier>  
```  
  
 где «ИНТЕРВАЛ» Указывает символьный литерал интервала. Может использоваться знак плюс или минус; он находится за пределами строки интервала и является необязательным.  
  
 Квалификатор интервал может быть полем одной даты и времени или может состоять из двух полей даты и времени, в форме: \< *начальные поле*> TO \< *конечные поля*>.  
  
-   Если интервал состоит из одного поля, одно поле может быть не второе поле, которое может сопровождаться необязательно начальные точности в круглые скобки. Один тип поля также может быть второе поле, которое может сопровождаться необязательно начальные точность и точность необязательно доли секунды в круглые скобки. Если начальные точности и точность в долях секунды для поля секунд, они разделяются запятыми. Если в поле секунд имеет точность в долях секунды, ей также необходимо предоставить начальные точности.  
  
-   Если интервал представляет собой начальные и конечные поля, поле начальные является не второе поле, которое может сопровождаться поля точности интервала в круглые скобки. Конечные поля может быть не второго поля или второе поле, которое может сопровождаться точность в долях секунды интервала в круглые скобки.  
  
 Строка интервал в *значение* заключен в одинарные кавычки. Он может быть литералом год месяц или литерал времени дня. Формат строки в *значение* определяется следующими правилами:  
  
-   Строка содержит десятичное значение для каждого поля, заданного \< *интервал* *квалификатор*>.  
  
-   Если точность интервал включает поля, год и месяц, значения этих полей разделяются знак «минус».  
  
-   Если точность интервал включает поля, день и час, значения этих полей разделяются пробелом.  
  
-   Если точность интервал включает поле час и нижнего поля order (минуты и секунды), значения этих полей разделенных двоеточием.  
  
-   Если точность интервал включает ВТОРОГО поля и явных или подразумеваемых точностью имеет ненулевое значение, непосредственно перед первой цифрой дробной части второй символ — это период.  
  
-   Поле не может быть более двух цифр, за исключением:  
  
    -   Значение начальные поля может быть при условии, явных или подразумеваемых главного параметра точности интервала.  
  
    -   Дробная часть параметра во ВТОРОМ поле можно при условии, явных или подразумеваемых точностью.  
  
    -   Конечные поля выполните обычные ограничения григорианского календаря. (См. в разделе [ограничения григорианского календаря](../../../odbc/reference/appendixes/constraints-of-the-gregorian-calendar.md).)

|---------------------------|------------------------|  
|{YEAR(4) ИНТЕРВАЛ "326"}|Указывает интервал измеряется в годах 326. Главного параметра точности интервала равно 4.|  
|{MONTH(3) ИНТЕРВАЛ "326"}|Указывает интервал в месяцах 326. Главного параметра точности интервала равно 3.|  
|{DAY(4) ИНТЕРВАЛ "3261"}|Указывает интервал 3261 дней. Главного параметра точности интервала равно 4.|  
|{HOUR(3) ИНТЕРВАЛ "163"}|Указывает интервал 163 дней. Главного параметра точности интервала равно 3.|  
|{INTERVAL '163' MINUTE(3)}|Указывает интервал 163 минут. Главного параметра точности интервала равно 3.|  
|{INTERVAL '223.16' SECOND(3,2)}|Указывает интервал 223.16 секунд. Интервал начальные точность равна 3, а точность секунды равно 2.|  
|{ИНТЕРВАЛ "163-11' YEAR(3) МЕСЯЦУ}|Указывает интервал 163 лет 11 месяцев. Главного параметра точности интервала равно 3.|  
|{ИНТЕРВАЛ "163 12' DAY(3) ДО ЧАСА}|Указывает интервал 163 дня и 12 часов. Главного параметра точности интервала равно 3.|  
|{ИНТЕРВАЛ "163 12:39" DAY(3) ДО МИНУТЫ}|Указывает интервал 163 дней, 12 часов, 39 минут. Главного параметра точности интервала равно 3.|  
|{DAY(3) "163 12:39:59.163» ИНТЕРВАЛА ДЛЯ SECOND(3)}|Указывает интервал 163 дней, 12 часов, 39 минут и 59.163 секунд. Интервал начальные точность равна 3, а точность секунд — 3.|  
|{ИНТЕРВАЛ «163:39» HOUR(3) ДО МИНУТЫ}|Указывает интервал 163 часа 39 мин. Главного параметра точности интервала равно 3.|  
|{ИНТЕРВАЛ «163:39:59.163» HOUR(3) ДЛЯ SECOND(4)}|Указывает интервал 163 часа 39 минут, и 59.163 секунд. Интервал начальные точность равна 3, а точность секунды равно 4.|  
|{ИНТЕРВАЛ «163:59.163» MINUTE(3) ДЛЯ SECOND(5)}|Указывает интервал 163 минут 59.163 секунд. Интервал начальные точность равна 3, а точность секунды равно 5.|  
|{ИНТЕРВАЛ - ДЕНЬ "16 23:39:56.23» СЕКУНДЕ}|Указывает интервал минус 16 дней, 23 часа 39 минут, и 56.23 секунд. Подразумеваемый начальные точностью 2, и подразумеваемых точностью 6.|
