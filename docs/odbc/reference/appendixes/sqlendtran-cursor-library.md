# <a name="sqlendtran-cursor-library"></a>SQLEndTran (библиотека курсоров)
> [!IMPORTANT]  
>  Этот компонент будет удален в будущих версиях Windows. Избегайте использования этой функции в новых разработках и запланируйте изменение приложений, которые сейчас ее используют. Корпорация Майкрософт рекомендует использовать функциональные возможности драйвера курсора.  
  
 В этом разделе рассматривается использование **SQLEndTran** функции в библиотеку курсоров. Общие сведения о **SQLEndTran**, см. в разделе [функция SQLEndTran](../../../odbc/reference/syntax/sqlendtran-function.md).  
  
 Библиотека курсоров, не поддерживает транзакции и передает вызовы **SQLEndTran** непосредственно к драйверу. Тем не менее библиотека курсоров поддерживает режимы работы курсоров фиксации и отката, возвращенный источник данных, используя SQL_CURSOR_ROLLBACK_BEHAVIOR и SQL_CURSOR_COMMIT_BEHAVIOR типы сведений:  
  
-   Для источников данных, которые сохраняют курсоров по всем транзакциям изменения, которые возвращаются в источнике данных не выполняется откат в кэше библиотеку курсоров. Чтобы сделать кэша, которые соответствуют данным в источнике данных, приложение необходимо закрыть и заново откройте его.  
  
-   Для источников данных, закрытия курсора на границах транзакции библиотека курсоров закрывает курсоры и удаляет кэши для всех инструкций в соединении.  
  
-   Для источников данных, удаление подготовленных инструкций в границы транзакций приложение должно reprepare всех подготовленных инструкций, прежде чем останавливаясь их.
