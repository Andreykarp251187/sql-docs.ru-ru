# <a name="sqlendtran-cursor-library"></a>SQLEndTran (библиотека курсоров)
> [!IMPORTANT]  
>  Эта функция будет удалена в будущей версии Windows. Избегайте использования этой функции в новых разработках и планируйте модифицировать приложения, использующие эту функцию в настоящее время. Корпорация Майкрософт рекомендует использовать функцию курсора драйвера.  
  
 На этой теме обсуждается использование функции **S'LEndTran** в библиотеке курсоров. Для получения общей информации о **s'LendTran**, см. [SQLEndTran Function](../../../odbc/reference/syntax/sqlendtran-function.md)  
  
 Библиотека курсора не поддерживает транзакции и передает вызовы на **S'LEndTran** непосредственно водителю. Тем не менее, библиотека курсора поддерживает поведение коммитора и отката, возвращенное источником данных с SQL_CURSOR_ROLLBACK_BEHAVIOR и SQL_CURSOR_COMMIT_BEHAVIOR типами информации:  
  
-   Для источников данных, сохраняющие курсоры в транзакциях, изменения, откатываемые в источнике данных, не откатываются в кэше библиотеки курсора. Чтобы кэш соответствовал данным в источнике данных, приложение должно закрыть и повторно открыть курсор.  
  
-   Для источников данных, закрывающие курсоры на границах транзакций, библиотека курсора закрывает курсоры и удаляет кэши для всех инструкций по соединению.  
  
-   Для источников данных, которые удаляют подготовленные операторы на границах транзакций, приложение должно переподготовить все подготовленные операторы по подключению, прежде чем реиксектировать их.
