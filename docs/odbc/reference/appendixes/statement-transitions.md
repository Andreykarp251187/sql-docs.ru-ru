# <a name="statement-transitions"></a>Переходы инструкций
Инструкции ODBC имеют следующие состояния.  
  
|Штат|Описание|  
|-----------|-----------------|  
|S0|Нераспределенное инструкция. (Состояние подключения должны быть C4, C5 или C6. Дополнительные сведения см. в разделе [переходы подключения](../../../odbc/reference/appendixes/connection-transitions.md).)|  
|S1|Выделенной инструкции.|  
|S2|Подготовленной инструкции. Результирующий набор не создается.|  
|S3|Подготовленной инструкции. (Возможно, пустой) результирующий набор будет создаваться.|  
|S4|Инструкция выполняется и результирующий набор не создан.|  
|S5|Инструкцией, выполненной и (возможно, пустой) результирующий набор был создан. Курсор находится открыто и находится перед первой строкой результирующего набора.|  
|S6|Курсор будет расположен с **SQLFetch** или **SQLFetchScroll**.|  
|S7|Курсор будет расположен с **SQLExtendedFetch**.|  
|S8|Функция должна данных. **SQLParamData** не был вызван.|  
|S9|Функция должна данных. **SQLPutData** не был вызван.|  
|S10|Функция должна данных. **SQLPutData** был вызван.|  
|S11|Все еще выполняется. Оператор остается в этом состоянии после возврата функции, выполняемый асинхронно SQL_STILL_EXECUTING. Оператор временно — в этом состоянии при любая функция, которая принимает дескриптор инструкции выполняется. Временный проживания в состоянии S11 не содержится ни в каких таблицах состояния, за исключением таблицы состояний для **SQLCancel**. Пока инструкцию временно находится в состоянии S11, функцию можно отменить, вызвав **SQLCancel** из другого потока.|  
|S12|Асинхронное выполнение отменено. В S12 приложение должно вызвать функцию отмены, пока не будет возвращено значение, отличное от SQL_STILL_EXECUTING. Функция была отменена успешно только в том случае, если функция возвращает значение SQL_ERROR и SQLSTATE HY008 (операция отменена). Если возвращается другое значение, такие как SQL_SUCCESS, произошел сбой операции отмены и функция, выполненная обычным образом.|  
  
 Состояния S2 и S3, называются подготовленного состояния, состояний S5 через S7 курсор состояния, состояний S8 через S10 как необходимость данных состояния и состояния S11 и S12 как асинхронные состояния. В каждом из этих групп переходы отображаются отдельно только в том случае, если они различаются для каждого состояния в группе; в большинстве случаев переходы для каждого состояния в каждой группе одинаковы.  
  
 В следующих таблицах показано, как каждая функция ODBC влияет на состояние инструкции.
