# <a name="statement-transitions"></a>Переходы инструкций
Заявления ODBC имеют следующие состояния.  
  
|Состояние|Описание|  
|-----------|-----------------|  
|S0|Нераспределенное заявление. (Состояние соединения должно быть C4, C5 или C6. Для получения дополнительной [информации см. Переходы подключения](../../../odbc/reference/appendixes/connection-transitions.md).)|  
|S1|Выделенное заявление.|  
|S2|Подготовленное заявление. Набор результатов не будет создан.|  
|S3|Подготовленное заявление. Будет создан набор результатов A (возможно, пустой).|  
|S4|Заявление выполнено, и набор результатов не был создан.|  
|S5|Выполнено заявление и создан (возможно, пустой) набор результатов. Курсор открыт и расположен перед первым рядом набора результатов.|  
|S6|Курсор позиционируется с **помощью S'LFetch** или **S'LFetchScroll**.|  
|S7|Курсор позиционируется с **помощью S'LExtendedFetch**.|  
|S8|Функция нуждается в данных. **S'LParamData** не был вызван.|  
|S9|Функция нуждается в данных. **S'LPutData** не был вызван.|  
|S10|Функция нуждается в данных. **Названа s'LPutData.**|  
|S11|Все еще исполняю. Заявление остается в этом состоянии после выполнения функции асинхронно возвращается SQL_STILL_EXECUTING. Заявление временно находится в этом состоянии, в то время как любая функция, которая принимает ручку оператора, исполняется. Временное проживание в состоянии S11 не отображается ни в одной таблице штатов, за исключением таблицы состояния для **S'LCancel.** В то время как выписка временно находится в состоянии S11, функция может быть отменена, позвонив в **s'LCancel** из другого потока.|  
|S12|Асинхронная казнь отменена. В S12 приложение должно вызывать отмененную функцию до тех пор, пока оно не вернет значение, не SQL_STILL_EXECUTING. Функция была успешно отменена только в том случае, если функция возвращается SQL_ERROR и S'LSTATE HY008 (Операция отменена). Если он возвращает любое другое значение, например SQL_SUCCESS, операция отмены сбой и выполнение функции в обычном режиме.|  
  
 Государства S2 и S3 известны как подготовленные государства, говорится S5 через S7 как курсор государств, говорится S8 через S10 как необходимое data государства, и государства S11 и S12 как асинхронные государства. В каждой из этих групп переходы отображаются отдельно только тогда, когда они отличаются для каждого состояния в группе; в большинстве случаев переходы для каждого состояния в каждой группе одинаковы.  
  
 Следующие таблицы показывают, как каждая функция ODBC влияет на состояние оператора.
