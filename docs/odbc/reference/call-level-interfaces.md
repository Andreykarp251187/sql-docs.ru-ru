# <a name="call-level-interfaces"></a>Интерфейсы уровня вызова
Окончательный метод отправки заявлений S'L в DBMS — интерфейс уровня вызова (CLI). Интерфейс уровня вызова предоставляет библиотеку функций DBMS, которые могут быть вызваны программой приложения. Таким образом, вместо того, чтобы пытаться совмещеть S'L с другим языком программирования, интерфейс уровня вызова аналогичен обычным библиотекам, которые большинство программистов привыкли использовать, такие как строка, вито-из- или математические библиотеки в C. Обратите внимание, что DBMS, поддерживающие встроенный S'L, уже имеют интерфейс уровня вызова, вызовы на которые генерируется прекомпилсетом. Однако эти вызовы не документированы и могут быть изменены без предварительного уведомления.  
  
 Интерфейсы уровня вызова обычно используются в архитектуре клиентов/серверов, в которых программа приложения (клиент) находится на одном компьютере, а DBMS (сервер) — на другом компьютере. Приложение вызывает функции CLI в локальной системе, и эти вызовы отправляются по всей сети в DBMS для обработки.  
  
 Интерфейс уровня вызова аналогичен динамическому S'L, так как операторы S'L передаются dBMS для обработки во время выполнения, но он отличается от встроенных S'L в целом тем, что нет встроенных инструкций s'L и не требуется прекомпилайзер.  
  
 Использование интерфейса уровня вызова обычно включает в себя следующие шаги:  
  
1.  Приложение вызывает функцию CLI для подключения к DBMS.  
  
2.  Приложение создает выписку s'L и помещает его в буфер. Затем он вызывает одну или несколько функций CLI для отправки заявления в DBMS для подготовки и выполнения.  
  
3.  Если заявление является заявлением SELECT, приложение вызывает функцию CLI для возврата результатов в буферах приложения. Как правило, эта функция возвращает одну строку или один столбец данных за один раз.  
  
4.  Приложение вызывает функцию CLI для отключения от DBMS.
