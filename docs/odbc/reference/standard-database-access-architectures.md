# <a name="standard-database-access-architectures"></a>Стандартные архитектуры доступа к базе данных
При анализе компонентов доступа к базе данных, описанных в предыдущем разделе, выясняется, что два из них - интерфейсы программирования и протоколы потока данных - являются хорошими кандидатами на стандартизацию. Два других компонента - механизм ИПК и сетевые протоколы - не только находятся на слишком низком уровне, но и в высшей степени зависят от сети и операционной системы. Существует также третий подход - шлюзы, который предоставляет возможности для стандартизации.  
  
 Этот раздел содержит следующие подразделы.  
  
-   [Стандартный интерфейс программирования](../../odbc/reference/standard-programming-interface.md)  
  
-   [Стандартный протокол потока данных](../../odbc/reference/standard-data-stream-protocol.md)  
  
-   [Стандартный шлюз](../../odbc/reference/standard-gateway.md)
