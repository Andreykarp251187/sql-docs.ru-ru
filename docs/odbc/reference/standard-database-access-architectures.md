# <a name="standard-database-access-architectures"></a>Стандартные архитектуры доступа к базе данных
При просмотре компонентов доступа к базе данных, описанных в предыдущем разделе, оказывается, что два из них — интерфейсы программирования и протоколы потока данных — являются хорошим кандидатом для стандартизации. Два других компонента — механизм IPC и сетевые протоколы — не только в низком уровне, но и сильно зависят от сети и операционной системы. Существует также третий подход — шлюзы, обеспечивающие возможности стандартизации.  
  
 Этот раздел содержит следующие подразделы.  
  
-   [Стандартный интерфейс программирования](../../odbc/reference/standard-programming-interface.md)  
  
-   [Стандартный протокол потока данных](../../odbc/reference/standard-data-stream-protocol.md)  
  
-   [Стандартный шлюз](../../odbc/reference/standard-gateway.md)
