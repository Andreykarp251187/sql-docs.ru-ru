# <a name="sql-modules"></a>Модули SQL
Второй способ отправки инструкций SQL в СУБД — через модули. Вкратце, модуль состоит из группы процедур, которые вызываются из языка программирования узла. Каждая процедура содержит одну инструкцию SQL, и данные передаются в процедуру и из нее с помощью параметров.  
  
 Модуль можно рассматривать как библиотеку объектов, связанную с кодом приложения. Но точно так же, как процедуры и остальная часть приложения связаны, зависит от реализации. Например, процедуры могут быть скомпилированы в объектный код и связаны непосредственно с кодом приложения, они могут быть скомпилированы и сохранены в СУБД и вызываться для доступа к идентификаторам плана, размещенным в коде приложения, или интерпретироваться во время выполнения.  
  
 Основное преимущество модулей заключается в том, что они четко разделяют инструкции SQL от языка программирования. Теоретически, можно изменить его, не меняя другой, и просто повторно связать его.
