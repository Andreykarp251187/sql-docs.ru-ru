# <a name="dbms-based-drivers"></a>Драйверы на основе СУБД
Драйверы на основе СУБД используются с источниками данных, например Oracle или SQL Server, которые предоставляют изолированный компонент database engine для использования драйвера. Эти драйверы доступа к данным физических через автономный обработчика. то есть они отправлять инструкции SQL и получения результатов от обработчика.  
  
 Поскольку драйверы на основе СУБД используется существующей базы данных подсистемы, они обычно более удобны, чем драйверы на основе файла. Несмотря на то, что драйверов на основе СУБД можно легко реализовать путем перевода вызовы ODBC на собственные вызовы API, в результате медленнее драйвера. Более эффективный способ реализации драйверов на основе СУБД является использование базового протокола потока данных, как правило, это назначение собственного API. Например драйвер SQL Server следует использовать TDS (поток данных протокола для SQL Server), вместо того чтобы DB-Library (собственного API для SQL Server). Исключением из этого правила является при собственного API ODBC. Например Watcom SQL — это автономный ядра, которая находится на том же компьютере, что и приложение и загружается непосредственно как драйвер.  
  
 Драйверы на основе СУБД выступает в качестве клиента в конфигурации клиента и сервера где источника данных выступает в качестве сервера. В большинстве случаев клиента (драйвер) и сервера (источник данных) находятся на разных компьютерах, несмотря на то, что оба могут располагаться на одном компьютере под управлением многозадачной операционной системе. Третье *шлюза,* которой располагается между драйвером и источника данных. Шлюз — это программное обеспечение, которое вызывает один СУБД, чтобы он выглядел другой. Например приложения, написанные для использования SQL Server можно также доступ к данным DB2 через шлюз DB2 Decisionware Micro; Этот продукт приводит к DB2, чтобы он выглядел SQL Server.  
  
 На следующем рисунке трех разных конфигурациях драйверов на основе СУБД. В первую конфигурацию драйвер и источник данных находятся на одном компьютере. Во втором драйвер и источник данных находятся на разных компьютерах. В третьем драйвер и источник данных находятся на разных компьютерах и шлюз располагается между ними, установленными на еще один компьютер.
