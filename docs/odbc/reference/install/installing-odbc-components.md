# <a name="installing-odbc-components"></a>Установка компонентов ODBC
> [!NOTE]  
>  Начиная с Windows XP и Windows Server 2003, ODBC включен в систему работы Windows. Вы должны только явно установить ODBC на более ранних версиях Windows.  
  
 В этом разделе описывается, как устанавливаются и удаляются компоненты ODBC. Поскольку разработчики драйверов всегда устанавливают компонент ODBC (их драйвер), им необходимо прочитать этот раздел. Разработчики приложений должны читать этот раздел только в том случае, если они будут поставлять компоненты ODBC в свои приложения. Компоненты ODBC включают менеджера драйвера, драйверов, переводчиков, установщика DLL, библиотеку курсоров и любые связанные с ними файлы. Для целей этого раздела приложения ODBC не считаются компонентами ODBC.  
  
> [!NOTE]  
>  Этот раздел специфичен для платформ Microsoft Windows. То, как компоненты ODBC устанавливаются на других платформах, зависит от платформы.  
  
 Компоненты ODBC устанавливаются и удаляются на основе компонента за компонентами, а не на основе файла за файлом. Например, если переводчик состоит из самого переводчика и ряда файлов данных, эти файлы устанавливаются и удаляются как группа; они не должны устанавливаться и удаляться на основе файла по файлам. Причина этого заключается в том, чтобы убедиться, что в системе существуют только полные компоненты.  
  
 Для установки и удаления компонентов, следующие определяются как компоненты ODBC:  
  
-   **Основные компоненты.** Менеджер драйвера, библиотека курсоров, установщик DLL и любые другие связанные файлы составляют основные компоненты и должны быть установлены и удалены как группа.  
  
-   **Драйверы.** Каждый драйвер является отдельным компонентом.  
  
-   **Переводчиков.** Каждый переводчик является отдельным компонентом.  
  
 При поддержке Unicode в ODBC 3.5 и позже, некоторое внимание должно быть уделено использованию компонентов OLE DB с ODBC. 1.1 версия поставщика OLE DB для ODBC была написана для конкретных спецификаций Unicode в рамках ODBC 3.0. Поскольку эти спецификации изменились в ODBC 3.5, необходимо иметь версию 1.5 или более позднюю версию поставщика при использовании ODBC 3.5 и позже. Этот раздел содержит следующие подразделы.
