# <a name="setup-program"></a>Программа установки
> **ПРИМЕЧАНИЕ:** Начиная с Windows XP и Windows Server 2003, **ODBC включен в систему работы Windows.** Вы должны только явно установить ODBC на более ранних версиях Windows.  
  
 Пользователь запускает программу настройки для запуска процесса настройки. Программа настройки написана разработчиком приложения или драйвера. В дополнение к установке компонентов ODBC, он может установить другое программное обеспечение. Например, разработчики приложений могут использовать одну и ту же программу настройки как для установки компонентов ODBC, так и для установки своих приложений.  
  
 Разработчики могут писать программу настройки с нуля, используя microsoft® Windows® утилиты настройки SDK или программное обеспечение для настройки от других поставщиков. Это дает этим разработчикам полный контроль над внешний вид программы настройки. Программа настройки может быть написана для установки дополнительного программного обеспечения, например приложения ODBC. Для получения дополнительной информации о утилитах настройки Windows SDK см.  
  
 Сколько установки на самом деле делается программой установки зависит от того, какие функции она вызывает в установщике DLL. Установщик DLL содержит функции для установки отдельных компонентов ODBC. Программа настройки просто вызывает **S'LInstallDriverManager**, **S'LInstallDriverEx**, или **S'LInstallTranslatorEx** в установщике DLL для получения пути каталога, в котором компонент должен быть установлен, и добавить информацию о компоненте в реестр. Эти функции на самом деле не копировать файлы; программа настройки делает это, используя информацию в аргументах этих функций.  
  
 Установщик DLL также содержит функции для удаления компонентов ODBC. Программа настройки вызывает **S'LRemoveDriver,** **S'LRemoveDriver**, или **S'LRemoveTranslator** в установке DLL для decrement количество использования компонента в реестре и, если новый объем использования компонента падает до 0, удалить всю информацию о компоненте из реестра. Эти функции фактически не удаляют файлы для компонента; программа настройки делает это, если количество новых способов использования падает до 0.
