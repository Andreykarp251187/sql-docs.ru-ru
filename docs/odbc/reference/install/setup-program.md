# <a name="setup-program"></a>Программа установки
> **ПРИМЕЧАНИЕ.** Начиная с Windows XP и Windows Server 2003, **ODBC включен в операционную систему Windows**. ODBC следует только явным образом установить в более ранних версиях Windows.  
  
 Пользователь запускает программу установки, чтобы начать процесс установки. Программа установки записывается разработчиком приложения или драйвера. Помимо установки компонентов ODBC, его можно установить другое программное обеспечение. Предположим, например разработчикам приложений использует же программу установки для установки компонентов ODBC и для установки своих приложений.  
  
 Разработчики могут создавать с нуля, программу установки с помощью программы установки Microsoft® Windows® SDK или установки программного обеспечения от других поставщиков. Это дает этих разработчикам полный контроль над вида программы установки. Для установки дополнительного программного обеспечения, таких как приложения ODBC могут записываться программу установки. Дополнительные сведения о служебных программ установки пакета SDK для Windows см. в документации пакета SDK для Windows.  
  
 Какая часть установки, фактически выполняются программой установки зависит от того, какие функции вызовы в библиотеки DLL установщика. Библиотека DLL установщика содержит функции для установки отдельных компонентов ODBC. Программа установки просто вызывает **SQLInstallDriverManager**, **SQLInstallDriverEx**, или **SQLInstallTranslatorEx** в установщике библиотеки DLL, чтобы получить путь к каталог, в котором компонент должен быть установлен и добавления сведений о компоненте в реестр. Эти функции фактически не копируйте файлы; Программа установки делает это, используя сведения в аргументах этих функций.  
  
 Библиотека DLL установщика также содержит функции для удаления компонентов ODBC. Программа установки вызовет **SQLRemoveDriverManager**, **SQLRemoveDriver**, или **SQLRemoveTranslator** в установщике число DLL для уменьшения использования компонента реестр и, если новый счетчик использования компонента становится равным 0, удалите все сведения о компоненте из реестра. Эти функции фактически не удаляет файлы для компонента; Программа установки делает это, если новый счетчик использования становится равным 0.
