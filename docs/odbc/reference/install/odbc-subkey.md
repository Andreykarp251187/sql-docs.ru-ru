# <a name="odbc-subkey"></a>Подраздел ODBC
Значения в столбце подраздел ODBC укажите параметры трассировки ODBC. Эти параметры задаются «трассировка» диалогового окна Администратор источников данных ODBC, отображаемого элементом **SQLManageDataSources**. Подраздел ODBC является необязательным. Формат этих значений является, как показано в следующей таблице.  
  
|Имя|Тип данных|Данные|  
|----------|---------------|----------|  
|Trace|REG_SZ|**0** &#124; **1**|  
|TraceFile|REG_SZ|*путь файла трассировки*|  
  
 Принимать значения, описанные в следующей таблице.  
  
|Значение|Значение|  
|-----------|-------------|  
|Trace|Если значение трассировки имеет значение 1, если приложение вызывает **SQLAllocHandle** с параметром SQL_HANDLE_ENV включена трассировка для вызывающего приложения.<br /><br /> Если ключевое слово трассировки равным 0, когда приложение вызывает **SQLAllocHandle** с параметром SQL_HANDLE_ENV, трассировка отключена для вызывающего приложения. Это значение по умолчанию.<br /><br /> Приложения можно включить или отключить трассировку с помощью атрибута SQL_ATTR_TRACE соединения. Тем не менее это так не меняет данные для этого значения.|  
|TraceFile|Если трассировка включена, диспетчер драйверов записывает в файл трассировки, определяемого значением TraceFile.<br /><br /> Если файл трассировки не указан, диспетчер драйверов записывает файл Sql.log на текущем диске. Это значение по умолчанию.<br /><br /> Трассировки следует использовать только для одного приложения или каждое приложение должно указать файл различные трассировки. В противном случае два или несколько приложений будет пытаться открыть один и тот же файл трассировки в то же время, что вызывает ошибку.<br /><br /> Приложение может указать новый файл трассировки с атрибутом SQL_ATTR_TRACEFILE подключения. Тем не менее это так не меняет данные для этого значения.|  
  
 Например предположим, что трассировка включена, и файл трассировки является C:\Odbc.log. Значения в столбце подраздел ODBC будет выглядеть следующим образом:  
  
```  
Trace : REG_SZ : 1  
TraceFile : REG_SZ : C:\ODBC.LOG  
  
```
