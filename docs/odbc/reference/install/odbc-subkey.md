# <a name="odbc-subkey"></a>Подраздел ODBC
Значения в подключке ODBC определяют параметры отслеживания ODBC. Эти параметры устанавливаются через вкладку «Отслеживание» в диалоговом окне администратора источников данных ODBC Data Administrator, отображаемый **S'LManageDataSources.** Подключка ODBC сама по себе не является обязательной. Формат этих значений показан в следующей таблице.  
  
|Имя|Тип данных|Данные|  
|----------|---------------|----------|  
|Трассировка|REG_SZ|**0** &#124; **1**|  
|TraceFile|REG_SZ|*tracefile-путь*|  
  
 Значения имеют значения, описанные в следующей таблице.  
  
|Значение|Значение|  
|-----------|-------------|  
|Трассировка|Если значение trace устанавливается до 1, когда приложение вызывает **S'LAllocHandle** с SQL_HANDLE_ENV опцией, отслеживание включено для вызывающего приложения.<br /><br /> Если ключевое слово Trace установлено до 0, когда приложение вызывает **S'LAllocHandle** с SQL_HANDLE_ENV опцией, трассировка отключена для вызывающего приложения. Это значение по умолчанию.<br /><br /> Приложение может включить или отключить отслеживание с SQL_ATTR_TRACE атрибутом соединения. Однако это не меняет данные для этого значения.|  
|TraceFile|Если трассировка включена, менеджер драйвера записывает в файл трассировки, указанный значением TraceFile.<br /><br /> Если файл трассировки не указан, менеджер драйвера записывает в файл sql.log на текущем диске. Это значение по умолчанию.<br /><br /> Отслеживание должно использоваться только для одного приложения, или каждое приложение должно указать другой файл трассировки. В противном случае два или более приложений попытаются открыть один и тот же файл трассировки одновременно, что вызовет ошибку.<br /><br /> Приложение может указать новый файл трассировки с SQL_ATTR_TRACEFILE атрибутом соединения. Однако это не меняет данные для этого значения.|  
  
 Например, предположим, что трассировка включена, а файл трассировки — C:'Odbc.log. Значения подключки ODBC будут следующими:  
  
```  
Trace : REG_SZ : 1  
TraceFile : REG_SZ : C:\ODBC.LOG  
  
```
