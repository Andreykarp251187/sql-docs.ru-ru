# <a name="database-access-architecture"></a>Архитектура доступа к базе данных
Одним из вопросов при разработке ODBC было то, какая часть архитектуры доступа к базе данных должна быть стандартизироваться. Интерфейсы программирования, описанные в предыдущем разделе, - встроенные модули S'L, S-L и CLIs - являются лишь частью этой архитектуры. На самом деле, поскольку ODBC был в первую очередь предназначен для подключения персональных компьютерных приложений к миникомпьютеру и мэйнфрейму DBMS, существует также ряд сетевых компонентов, некоторые из которых могут быть стандартизированы.
