# <a name="sqlparamdata-function"></a>Функция SQLParamData
**Соответствия**  
 Представлена версия: Соответствие стандартам ODBC 1.0: ISO 92  
  
 **Сводка**  
 Для передачи данных параметров во время выполнения выписки, а также с **помощью данных** потокового параметра, используется **SQLPutData** **s'LParamData,** а также для получения данных потокового параметра.  
  
## <a name="syntax"></a>Синтаксис  
  
```cpp  
  
SQLRETURN SQLParamData(  
     SQLHSTMT       StatementHandle,  
     SQLPOINTER *   ValuePtrPtr);  
```  
  
## <a name="arguments"></a>Аргументы  
 *Обработка заявления*  
 (Вход) Ручка оператора.  
  
 *ValuePtrPtr*  
 (Выход) Указатель на буфер, в котором можно вернуть адрес буфера *ParameterValuePtr,* указанный в **S'LBindParameter** (для данных параметров) или адрес буфера *TargetValuePtr,* указанного в **S'LBindCol** (для данных столбца), как это содержится в поле записи SQL_DESC_DATA_PTR дескриптора.  
  
## <a name="returns"></a>Результаты  
 SQL_SUCCESS, SQL_SUCCESS_WITH_INFO, SQL_NEED_DATA, SQL_NO_DATA, SQL_STILL_EXECUTING, SQL_ERROR, SQL_INVALID_HANDLE или SQL_PARAM_DATA_AVAILABLE.

## <a name="comments"></a>Комментарии  
 **SLParamData** можно вызвать для передачи данных по исполнению данных для двух применений: параметрических данных, которые будут использованы при вызове в **S'L'LExecDirect,** или данных столбцов, которые будут использоваться при обновлении или добавлении строки в **S'LBulkOperations** или обновлены по вызову в **S'LSetPos.** **SQLExecDirect** Во время **выполнения, S'LParamData** возвращает в приложение индикатор, который требует сядрайвер.  
  
 Когда приложение вызывает **s'LExecute,** **S'LExecDirect**, **S'LBulkOperations**, или **S'LSetPos**, водитель возвращает SQL_NEED_DATA если ему нужны данные о выполнении. Затем приложение вызывает **S'LParamData,** чтобы определить, какие данные отправить. Если драйверу требуются данные параметров, драйвер возвращает в буфер * \*вывода ValuePtrPtr* значение, которое приложение поместило в буфер строки. Приложение может использовать это значение, чтобы определить, какие данные параметра запрашивает драйвер. Если драйверу требуются данные столбца, драйвер возвращает в буфер * \*ValuePtrPtr* адрес, к которому изначально был привязан столбец, следующим образом:  
  
 *Связанный адрес* + *Связывание Offset* ((Номер*строки* - 1) x Размер *элемента)*  
  
 где переменные определяются, как указано в следующей таблице.  
  
|Переменная|Описание|  
|--------------|-----------------|  
|*Связанный адрес*|Адрес, указанный в аргументе *TargetValuePtr* в **S'LBindCol**.|  
|*Связывание Offset*|Значение, хранящееся по адресу, указанному в атрибуте SQL_ATTR_ROW_BIND_OFFSET_PTR оператора.|  
|*Row Number*|1-основанное число строки в строке. Для однорядных извлечений, которые по умолчанию, это 1.|  
|*Размер элемента*|Значение атрибута SQL_ATTR_ROW_BIND_TYPE оператора как для буферов данных, так и для буферов длины/индикатора.|  
  
 Он также возвращает SQL_NEED_DATA, что является индикатором для приложения, что он должен вызвать **S'LPutData** для отправки данных.  
  
 Приложение вызывает **sLPutData** столько раз, сколько необходимо для отправки данных по исполнению данных для столбца или параметра. После того, как все данные были отправлены для столбца или параметра, приложение снова вызывает **S'LParamData.** Если **S'LParamData** снова возвращает SQL_NEED_DATA, данные должны быть отправлены для другого параметра или столбца. Таким образом, приложение снова вызывает **S'LPutData**. Если все данные по исполнению данных были отправлены по всем параметрам или столбцам, то **s'LParamData** возвращает SQL_SUCCESS или SQL_SUCCESS_WITH_INFO, значение в * \*ValuePtrPtr* не определено, и заявление S'L может быть выполнено или вызов **S'LBulkOperations** или **SLSetPos** может быть обработан.  
  
 Если данные параметров **s'LParamData** поставляются для поискового обновления или удаления оператора, которое не влияет на строки в источнике данных, вызов в **S'LParamData** возвращается SQL_NO_DATA.

Для получения более подробной информации о том, как обновляются или добавляются данные столбца данных о персональных данных, а также в разделе «Использование S'LSetPos» в [S'LSetPos,](../../../odbc/reference/syntax/sqlsetpos-function.md)«Выполнение массовых обновлений с использованием закладок» в [S'LBulkOperations,](../../../odbc/reference/syntax/sqlbulkoperations-function.md)а также [о длинных данных, а также на S'LSetPos и S'LBulkOperations.](../../../odbc/reference/develop-app/long-data-and-sqlsetpos-and-sqlbulkoperations.md)

Когда **S'LMoreResults**, **S'LExecute**, **S'LGetData**, или **S'LExecDirect** возвращается SQL_PARAM_DATA_AVAILABLE, позвоните **в S'LParamData,** чтобы определить, какой параметр имеет значение.
