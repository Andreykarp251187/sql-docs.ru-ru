# <a name="sqldescribecol-function"></a>SQLDescribeCol, функция
**Соответствия**  
 Представлена версия: Соответствие стандартам ODBC 1.0: ISO 92  
  
 **Сводка**  
 **S'LDescribeCol** возвращает дескриптор результата - имя столбца, тип, размер столбца, десятичные цифры и необлагаемость - для одного столбца в наборе результатов. Эта информация также доступна в полях IRD.  
  
## <a name="syntax"></a>Синтаксис  
  
```cpp  
  
SQLRETURN SQLDescribeCol(  
      SQLHSTMT       StatementHandle,  
      SQLUSMALLINT   ColumnNumber,  
      SQLCHAR *      ColumnName,  
      SQLSMALLINT    BufferLength,  
      SQLSMALLINT *  NameLengthPtr,  
      SQLSMALLINT *  DataTypePtr,  
      SQLULEN *      ColumnSizePtr,  
      SQLSMALLINT *  DecimalDigitsPtr,  
      SQLSMALLINT *  NullablePtr);  
```  
  
## <a name="arguments"></a>Аргументы  
 *Обработка заявления*  
 (Вход) Ручка оператора.  
  
 *ColumnNumber*  
 (Вход) Количество столбцов данных результата, упорядоченных последовательно в увеличении порядка столбца, начиная с 1. Аргумент *ColumnNumber* также может быть установлен на 0 для описания столбца закладки.  
  
 *ColumnName*  
 (Выход) Указатель на нулевый буфер, в котором можно вернуть имя столбца. Это значение считывалось из SQL_DESC_NAME поле IRD. Если столбец неназван или имя столбца не может быть определено, драйвер возвращает пустую строку.  
  
 Если *ColumnName* является NULL, *NameLengthPtr* по-прежнему возвращает общее количество символов (за исключением символа нулевого прекращения для данных символов), доступного для возврата в буфере, на который указывает *ColumnName.*  
  
 *BufferLength*  
 (Вход) Длина буфера*ColumnName* в символах.  
  
 *NameLengthPtr*  
 (Выход) Указатель на буфер, в котором можно вернуть общее количество символов (за \*исключением нулевого прекращения), доступного для возврата в *ColumnName*. Если количество символов, доступных для возврата, больше или равно \* *BufferLength,* название столбца в *ColumnName* усечено до *BufferLength* минус длина символа с нулевым прекращением.  
  
 *DataTypePtr*  
 (Выход) Указатель на буфер, в котором можно вернуть тип данных в столбце. Это значение читается из SQL_DESC_CONCISE_TYPE поле IRD. Это будет одно из значений в [типах данных S'L](../../../odbc/reference/appendixes/sql-data-types.md)или тип данных, специфичный для драйверов, с помощью s-L. Если тип данных не может быть определен, водитель возвращается SQL_UNKNOWN_TYPE.  
  
 В ODBC 3. *x,* SQL_TYPE_DATE, SQL_TYPE_TIME или SQL_TYPE_TIMESTAMP возвращается в * \*DataTypePtr* для данных даты, времени или метки времени, соответственно; в ODBC 2. *x,* SQL_DATE, SQL_TIME или SQL_TIMESTAMP возвращается. Менеджер драйвера выполняет необходимые отображения при ODBC 2. *приложение x* работает с ODBC 3. *x* драйвер или при ODBC 3. *приложение x* работает с ODBC 2. *x* драйвер.  
  
 Когда *ColumnNumber* равен 0 (для столбца закладки), SQL_BINARY возвращается в * \*DataTypePtr* для закладок с переменной длиной. (SQL_INTEGER возвращается, если закладки используются ODBC 3. *x* приложение работает с ODBC 2. *x* драйвер или ODBC 2. *x* приложение работает с ODBC 3. *x* водитель.)  
  
 Более подробную информацию об этих типах данных [можно](../../../odbc/reference/appendixes/sql-data-types.md) узнать в приложении D: Типы данных. Для получения информации о типах данных, специфичных для драйверов, просмотрите документацию водителя.  
  
 *ColumnSizePtr*  
 (Выход) Указатель на буфер, в котором можно вернуть размер (в символах) столбца на источнике данных. Если размер столбца не может быть определен, водитель возвращает 0. Для получения дополнительной информации о размере столбца [в](../../../odbc/reference/appendixes/column-size-decimal-digits-transfer-octet-length-and-display-size.md) приложении D: Типы данных см.  
  
 *ДесятичнаяДияпТр*  
 (Выход) Указатель на буфер, в котором можно вернуть число десятичных цифр столбца на источнике данных. Если количество десятичных цифр не может быть определено или не применимо, водитель возвращает 0. Для получения дополнительной информации о десятичных цифрах см. [Размер столбца, десятичные цифры, длина переноса Octet и размер дисплея](../../../odbc/reference/appendixes/column-size-decimal-digits-transfer-octet-length-and-display-size.md) в приложении D: Типы данных.  
  
 *NullablePtr*  
 (Выход) Указатель на буфер, в котором можно вернуть значение, указывавшее, позволяет ли столбец значенияnull. Это значение читается из SQL_DESC_NULLABLE поле IRD. Значение может быть одним из следующих:  
  
 SQL_NO_NULLS: Столбец не позволяет значения NULL.  
  
 SQL_NULLABLE: Столбец позволяет значения NULL.  
  
 SQL_NULLABLE_UNKNOWN: Драйвер не может определить, допускает ли столбец значения NULL.  
  
## <a name="returns"></a>Результаты  
 SQL_SUCCESS, SQL_SUCCESS_WITH_INFO, SQL_STILL_EXECUTING, SQL_ERROR или SQL_INVALID_HANDLE.

 По причинам производительности приложение не должно вызывать **S'LDescribeCol** перед выполнением оператора.  
  
## <a name="comments"></a>Комментарии  
 Приложение, как правило, вызывает **S'LDescribeCol** после звонка в **S'LPrepare** и до или после связанного вызова в **S'LExecute.** Приложение также может позвонить по телефону **s'LDescribeCol** после звонка в **S'LExecDirect**. Для получения дополнительной информации [см.](../../../odbc/reference/develop-app/result-set-metadata.md)  
  
 **SLDescribeCol** получает имя столбца, тип и длину, генерируемые выпиской **SELECT.** Если столбец является выражением, то*columnName* — это либо пустая строка, либо имя, определяемое драйвером.  
  
> [!NOTE]  
>  ODBC поддерживает SQL_NULLABLE_UNKNOWN в качестве расширения, даже несмотря на то, что спецификация уровня вызова Open Group и S'L Группы доступа не указывает опцию для **S'LDescribeCol.**
