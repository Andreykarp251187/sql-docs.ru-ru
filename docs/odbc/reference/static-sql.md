# <a name="static-sql"></a>Статический SQL
Встроенный S'L, показанный в [встроенном примере S'L,](../../odbc/reference/embedded-sql-example.md) известен как статическое S'L. Это называется статическим S'L, потому что операторы S'L в программе статичны; то есть, они не меняются каждый раз, когда программа запущена. Как описано в предыдущем разделе, эти операторы компилируются при компилировании остальной части программы.  
  
 Статический S'L хорошо работает во многих ситуациях и может быть использован в любом приложении, для которого доступ к данным может быть определен во время разработки программы. Например, программа ввода заказов всегда использует одно и то же заявление для вставки нового заказа, а система бронирования авиакомпаний всегда использует одно и то же заявление для изменения статуса места с доступного на зарезервированное. Каждое из этих утверждений будет обобщаться с помощью переменных хоста; различные значения могут быть вставлены в заказ на продажу, и различные места могут быть зарезервированы. Поскольку такие операторы могут быть жестко закодированы в программе, такие программы имеют то преимущество, что операторы должны быть разобрана, проверена и оптимизирована только один раз, во время компиляции. Это приводит к относительно быстрому коду.
