# <a name="static-sql"></a>Статический SQL
Embedded SQL, показанный на [пример Embedded SQL](../../odbc/reference/embedded-sql-example.md) называется статического SQL. Он вызывается статический SQL, поскольку инструкций SQL в программе являются статическими; то есть они меняются при каждом запуске программы. Как описано в предыдущем разделе, эти инструкции, компилируются при компиляции остальные части программы.  
  
 Статический SQL работает также и во многих случаях и может использоваться в любом приложении, для которого доступ к данным можно определить во время разработки программы. Например приложение ввода заказов всегда использует той же инструкции для вставки нового заказа и система бронирования авиабилетов всегда использует той же инструкции, чтобы изменить состояние рабочее место с доступными для зарезервированных. Каждая из этих инструкций будет обобщить при помощи переменных узла. различные значения, которые могут быть вставлены в заказе на продажу и различных рабочих мест может быть зарезервировано. Поскольку такие инструкции могут быть жестко задано в программы, такие программы имеют преимущество инструкций, которые нужно проанализировать, проверки и оптимизированных для операций только один раз во время компиляции. Это приводит к относительно быстрый код.
