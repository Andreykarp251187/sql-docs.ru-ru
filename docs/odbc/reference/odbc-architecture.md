# <a name="odbc-architecture"></a>Архитектура ODBC
Архитектура ODBC имеет четыре компонента:  
  
-   **Заявка** Выполняет обработку и вызывает функции ODBC для отправки выписок s-L и получения результатов.  
  
-   **Менеджер драйвера** Загружает и разгружает драйверы от имени приложения. Процессы функции ODBC вызывает или передает их водителю.  
  
-   **Водитель** Обработка вызовов функции ODBC, отправка запросов в конкретный источник данных и возврат результатов в приложение. При необходимости драйвер изменяет запрос приложения таким образом, чтобы запрос соответствовал синтаксису, поддерживаемому соответствующим DBMS.  
  
-   **Источник данных** Состоит из данных, к которые пользователь хочет получить доступ, и связанной с ним операционной системы, DBMS и сетевой платформы (если таковые имеются), используемых для доступа к DBMS.  
  
 Обратите внимание на следующие моменты архитектуры ODBC. Во-первых, может существовать несколько драйверов и источников данных, что позволяет приложению одновременно получать доступ к данным из нескольких источников данных. Во-вторых, API ODBC используется в двух местах: между приложением и менеджером драйвера, а также между менеджером драйвера и каждым водителем. Интерфейс между менеджером драйверов и драйверами иногда называют *интерфейсом поставщика услуг,* или *SPI.* Для ODBC интерфейс программирования приложений (API) и интерфейс поставщика услуг (SPI) одинаковы; то есть, менеджер драйвера и каждый драйвер имеют один и тот же интерфейс с теми же функциями.
