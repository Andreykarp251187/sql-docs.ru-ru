# <a name="using-data-sources"></a>Использование источников данных
Источники данных обычно создаются конечным пользователем или специалистом с программой под названием *Администратор ODBC.* Администратор ODBC подсказывает пользователю, чтобы водитель использовал его, а затем вызывает его. Драйвер отображает диалоговый ящик, который запрашивает информацию, необходимую для подключения к источнику данных. После того, как пользователь вводит информацию, водитель хранит ее в системе.  
  
 Позже приложение вызывает менеджера драйвера и передает ему имя источника данных машины или путь файла, содержащего источник данных файла. При передаче имени источника данных машины менеджер водителя ищет систему, чтобы найти драйвер, используемый источником данных. Затем он загружает драйвер и передает ему имя источника данных. Драйвер использует имя источника данных, чтобы найти информацию, необходимую для подключения к источнику данных. Наконец, он подключается к источнику данных, что обычно побуждает пользователя к запросу пользователя и паролю, которые обычно не хранятся.  
  
 При передаче источника данных файла менеджер драйвера открывает файл и загружает указанный драйвер. Если файл также содержит строку соединения, он передает это водителю. Используя информацию в строке соединения, драйвер подключается к источнику данных. Если строка соединения не была пройдена, водитель обычно подсказывает пользователю необходимую информацию.
