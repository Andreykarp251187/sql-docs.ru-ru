# <a name="using-data-sources"></a>Использование источников данных
Источники данных обычно создаются конечным пользователем или специалистом с помощью программы, именуемой *администратором ODBC*. Администратор ODBC выдает пользователю запрос на использование драйвера, а затем вызывает этот драйвер. Драйвер отображает диалоговое окно, в котором запрашиваются сведения, необходимые для подключения к источнику данных. После того, как пользователь введет данные, драйвер сохранит его в системе.  
  
 Позже приложение вызывает диспетчер драйверов и передает ему имя источника данных компьютера или путь к файлу, содержащему файловый источник данных. При передаче имени источника данных компьютера диспетчер драйверов выполняет поиск в системе драйвера, используемого источником данных. Затем он загружает драйвер и передает ему имя источника данных. Драйвер использует имя источника данных для поиска сведений, необходимых для подключения к источнику данных. Наконец, он подключается к источнику данных, который обычно запрашивает у пользователя идентификатор пользователя и пароль, которые обычно не сохраняются.  
  
 При передаче файлового источника данных диспетчер драйверов открывает файл и загружает указанный драйвер. Если файл также содержит строку подключения, он передает его драйверу. Используя сведения в строке подключения, драйвер подключается к источнику данных. Если строка подключения не была передана, драйвер, как правило, запрашивает у пользователя необходимые сведения.
