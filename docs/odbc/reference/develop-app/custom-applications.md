# <a name="custom-applications"></a>Пользовательские приложения
Пользовательские приложения обычно выполняют конкретную задачу для нескольких DBMS. Например, приложение может получать данные из одного DBMS и генерировать отчет, или передавать данные между несколькими DBMS. Что эти приложения имеют в общем, что эти DBMS известны до того, как приложение написано и вряд ли изменится в течение всего срока службы приложения.  
  
 Поэтому пользовательское приложение требует практически никакой совместимости. Разработчик приложения может выбрать один драйвер для каждого DBMS и код непосредственно для этих драйверов. Приложение может безопасно содержать код для использования возможностей этих драйверов и даже совершать звонки в API с национальной базой данных для использования функциональности, не поддерживаемой ODBC.  
  
 Основная проблема совместимости большинства пользовательских приложений заключается в том, изменятся ли целевые DBMS в будущем. Если это так, этот процесс можно упростить, написав более совместимый код для начала. Однако такое изменение ДБМС происходит редко и, как правило, влечет за собой большой объем работы. Из-за этого разработчики пользовательских приложений редко выбирают для увеличения совместимости за счет функциональности; они обычно выбирают для перекодирования этой функциональности при изменении DBMS.
