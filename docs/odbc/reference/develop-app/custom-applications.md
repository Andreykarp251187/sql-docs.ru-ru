# <a name="custom-applications"></a>Пользовательские приложения
Пользовательские приложения обычно выполняют определенную задачу для нескольких СУБД. Например приложение может получать данные из одной СУБД и Создание отчета, или он может передавать данные между несколькими СУБД. Что эти приложения имеют общие, что эти СУБД известны, прежде чем написать приложение и вряд ли изменяются на протяжении приложения.  
  
 Таким образом, пользовательское приложение требует взаимодействия с незначительной или нулевой. Разработчик приложения можно выбрать один драйвер для каждого СУБД и код непосредственно в эти драйверы. Приложения безопасно могут содержать код специфические для драйвера для использования возможности этих драйверов и может даже заставить вызовы базы данных в собственном API, чтобы использовать функциональные возможности, не поддерживаемые в ODBC.  
  
 Основные возможности взаимодействия заботиться о большинстве пользовательских приложений является ли целевой СУБД изменится в будущем. Если Да, этот процесс можно упростить, написав код более широкое взаимодействие со. Однако такое изменение СУБД редко и обычно влечет за собой большой объем работы. По этой причине разработчики пользовательских приложений редко решили повысить взаимодействия за счет функции; Обычно они решили recode эту функциональность, при изменении ее СУБД.
