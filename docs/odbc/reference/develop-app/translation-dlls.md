# <a name="translation-dlls"></a>Библиотеки DLL преобразования
Приложение и источник данных часто хранят данные в различных наборах символов. ODBC предоставляет общий механизм, позволяющий водителю переводить данные с одного набора символов в другой. Он состоит из DLL, который реализует функции перевода **S'LDriverToDataSource** и **S'LDataSourceToDriver**, которые вызываются драйвером для перевода всех данных, протекающих между источником данных и драйвером. Этот DLL может быть написан разработчиком приложения, разработчиком драйверов или третьей стороной.  
  
 Перевод DLL для конкретного источника данных может быть указан в системной информации для этого источника данных; для получения дополнительной [Data Source Specification Subkeys](../../../odbc/reference/install/data-source-specification-subkeys.md)информации см. Он также может быть установлен во время выполнения с SQL_ATTR_TRANSLATE_DLL и SQL_ATTR_TRANSLATE_OPTION атрибутами соединения.  
  
 Вариант перевода — это значение, которое может быть истолковано только конкретным DLL-переводом. Например, если перевод DLL переводится между различными страницами кода, опция может дать номера страниц кода, используемых приложением, и источник данных. Для перевода DLL не требуется использовать вариант перевода.  
  
 После того, как был указан DLL перевода, водитель загружает его и вызывает его для перевода всех данных, протекающих между приложением и источником данных. Это включает в себя все операторы и параметры символов, отправляемых в источник данных, а также все результаты символов, метаданные символов, такие как имена столбцов, и сообщения об ошибках, извлеченные из источника данных. Данные о подключении не переводятся, так как перевод DLL загружается только после подключения приложения к источнику данных.
