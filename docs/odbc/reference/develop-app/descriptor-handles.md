# <a name="descriptor-handles"></a>Указатели дескрипторов
*Дескриптор* — это набор метаданных, описывающий параметры оператора S'L или столбцов набора результатов, как это видно по приложению или драйверу (также известному как *реализация).* Таким образом, дескриптор может заполнить любую из четырех ролей:  
  
-   **Дескриптор параметра применения (APD).** Содержит информацию о буферах приложения, привязанных к параметрам в выписке s'L, например, их адреса, длины и типы данных C.  
  
-   **Дескриптор параметра реализации (IPD).** Содержит информацию о параметрах в выписке по S'L, например, о типах данных, длинах и недействительности.  
  
-   **Описатель строки приложения (ARD).** Содержит информацию о буферах приложения, привязанных к столбцам в наборе результатов, таких как их адреса, длины и типы данных C.  
  
-   **Дескриптор строки реализации (IRD).** Содержит информацию о столбцах в наборе результатов, например, их типы данных, длину и необлагаемость.  
  
 Четыре дескриптора (по одному заполнению каждой роли) выделяются автоматически при выделении оператора. Они известны как *автоматически выделенные дескрипторы* и всегда связаны с этим утверждением. Приложения могут также выделять дескрипторы с **помощью S'LAllocHandle.** Они известны как *явно выделенные дескрипторы.* Они распределяются по соединению и могут быть связаны с одним или несколько заявлениями по этому подключению для выполнения роли APD или ARD на этих заявлениях.  
  
 Большинство операций в ODBC могут выполняться без явного использования дескрипторов приложением. Тем не менее, дескрипторы обеспечивают удобный ярлык для некоторых операций. Например, предположим, что приложение хочет вставить данные из двух различных наборов буферов. Чтобы использовать первый набор буферов, он будет неоднократно вызывать **S'LBindParameter,** чтобы привязать их к параметрам в выписке **INSERT,** а затем выполнить заявление. Чтобы использовать второй набор буферов, он будет повторять этот процесс. Кроме того, он может настроить привязки к первому набору буферов в одном дескрипторе и ко второму набору буферов в другом дескрипторе. Чтобы переключаться между наборами привязок, приложение просто вызовет **S'LSetStmtAttr** и связать правильный дескриптор с утверждением как APD.
