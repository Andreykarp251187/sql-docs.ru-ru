# <a name="descriptor-handles"></a>Указатели дескрипторов
*Дескриптор* — это коллекция метаданных, описывающих параметры инструкции SQL или столбцов результирующего набора, которые отображаются в приложении или драйвере (также известном как *Реализация*). Таким образом, дескриптор может заполнять любую из четырех ролей:  
  
-   **Дескриптор параметра приложения (APD).** Содержит сведения о буферах приложений, привязанных к параметрам в инструкции SQL, таким как адреса, длины и типы данных C.  
  
-   **Дескриптор параметра реализации (IPD).** Содержит сведения о параметрах в инструкции SQL, такие как типы данных SQL, Длина и допустимость значений NULL.  
  
-   **Дескриптор строки приложения (АРД).** Содержит сведения о буферах приложений, привязанных к столбцам в результирующем наборе, таких как адреса, длины и типы данных C.  
  
-   **Дескриптор строки реализации (IRD).** Содержит сведения о столбцах в результирующем наборе, такие как типы данных SQL, Длина и допустимость значений NULL.  
  
 Четыре дескриптора (по одному заполнению каждой роли) распределяются автоматически при выделении инструкции. Они называются *автоматически назначаемыми дескрипторами* и всегда связаны с этим оператором. Приложения также могут распределять дескрипторы с помощью **функцию SQLAllocHandle**. Они называются *явно выделенными дескрипторами*. Они выделяются для соединения и могут быть связаны с одной или несколькими инструкциями в этом соединении для выполнения роли APD или АРД в этих инструкциях.  
  
 Большинство операций в ODBC можно выполнять без явного использования дескрипторов приложением. Однако дескрипторы предоставляют удобное сочетание клавиш для некоторых операций. Например, предположим, что приложению требуется вставить данные из двух разных наборов буферов. Чтобы использовать первый набор буферов, он будет многократно вызывать **SQLBindParameter** , чтобы привязать их к параметрам в инструкции **INSERT** , а затем выполнить инструкцию. Чтобы использовать второй набор буферов, этот процесс будет повторен. Кроме того, он может настроить привязки к первому набору буферов в одном дескрипторе и ко второму набору буферов в другом дескрипторе. Для переключения между наборами привязок приложение просто вызовет **SQLSetStmtAttr** и связывает правильный дескриптор с инструкцией в качестве APD.  
  
 Дополнительные сведения о дескрипторах см. в разделе [Типы дескрипторов](../../../odbc/reference/develop-app/types-of-descriptors.md).
