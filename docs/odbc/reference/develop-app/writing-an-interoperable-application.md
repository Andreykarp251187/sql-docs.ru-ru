# <a name="writing-an-interoperable-application"></a>Создание приложений с поддержкой взаимодействия
Всякий раз, когда приложение использует один и тот же код против нескольких драйверов, этот код должен быть совместимым между этими драйверами. В большинстве случаев это простая задача. Например, код для получения строк с курсором только вперед является одинаковым для всех драйверов. В некоторых случаях это может быть сложнее. Например, в коде для построения идентификаторов для использования в заявлениях S'L необходимо учитывать случай идентификатора, цитирование и одну часть, двухсерийные и три части именования.  
  
 Как правило, совместимый код должен справляться с проблемами поддержки функций и изменчивости функций. *Поддержка функций* означает, поддерживается ли определенная функция. Например, не все транзакции DBMS поддерживают транзакции, и совместимый код должен работать правильно, независимо от поддержки транзакций. *Вариабельность функций* относится к изменению способа поддержки конкретной функции. Например, имена каталогов размещаются в начале идентификаторов в некоторых DBMS и в конце идентификаторов в других.  
  
 Приложения могут иметь дело с поддержкой функций и изменчивостью функций во время проектирования или во время выполнения. Чтобы справиться с поддержкой функций и изменчивостью во время проектирования, разработчик смотрит на целевые DBMS и драйверы и гарантирует, что один и тот же код будет совместим между ними. Как правило, именно таким образом приложения с низкой или ограниченной совместимостью решают эти проблемы.  
  
 Например, если разработчик гарантирует, что вертикальное приложение будет работать только с четырьмя конкретными DBMS, и если каждый из этих DBMS поддерживает транзакции, приложению не нужен код для проверки поддержки транзакций во время выполнения. Он всегда может предположить, что транзакции доступны из-за решения о разработке времени использовать только четыре DBMS, каждая из которых поддерживает транзакции.  
  
 Чтобы иметь дело с поддержкой функций и изменчивостью во время выполнения, приложение должно проверить на различные возможности во время выполнения и действовать соответствующим образом. Как правило, таким образом эти проблемы решаются высокосовместимыми приложениями. Для проблем поддержки функций это означает написание кода, который делает функцию факультативной, или написание кода, который имитирует функцию, когда она недоступна. Для проблем, связанных с изменчивостью функционирования, это означает написание кода, поддерживающего все возможные варианты.
