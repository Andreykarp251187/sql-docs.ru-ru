# <a name="types-of-changes"></a>Типы изменений
Три типа изменений вносятся в ODBC *3.x* (и любую версию ODBC). Каждый из них влияет на обратную совместимость по-разному и обрабатывается по-разному. Эти изменения описаны в следующей таблице.  
  
|Тип изменений|Описание|  
|--------------------|-----------------|  
|Новые функции|Эти функции, которые являются новыми для ODBC *3.x*, таких как вне строки связывания или дескрипторы. Они реализуются только тогда, когда приложение и драйвер, а также менеджер драйвера, имеют версию *3.x,* поэтому нет попытки сделать эти обратные совместимыми.|  
|Дублированные функции|Эти функции, которые существуют в ODBC *2.x* и ODBC *3.x,* но реализуются по-разному в каждом. Примером могут быть **функции S'LAllocHandle** и **S'LAllocStmt.** Обратные проблемы совместимости для этих и других дублированных функций в основном обрабатываются с помощью карт в менеджере драйвера.|  
|Изменения в поведении|Эти функции, которые обрабатываются по-разному в ODBC *2.x* и ODBC *3.x*. Примером **может** #define даты. Эти функции обрабатываются драйвером ODBC *3.x* на основе настройки атрибута среды. (См. [Поведенческие изменения](../../../odbc/reference/develop-app/behavioral-changes.md) для получения дополнительной информации.)|
