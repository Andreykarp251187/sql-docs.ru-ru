# <a name="disconnecting-from-a-data-source-or-driver"></a>Отключение от источника данных или драйвера
Когда приложение закончило с использованием источника данных, оно вызывает **s'LDisconnect**. **SLDisconnect** освобождает все операторы, которые выделены на соединение, и отключает драйвер от источника данных. Он возвращает ошибку, если транзакция находится в процессе.  
  
 После отключения приложение может вызвать **s'LFreeHandle,** чтобы освободить соединение. После освобождения соединения использование ручки соединения в вызове функции ODBC является ошибкой программирования приложения; это имеет неопределенные, но, вероятно, фатальные последствия. При вызове **S'LFreeHandle** драйвер выпускает структуру, используемую для хранения информации о подключении.  
  
 Приложение также может повторно использовать соединение, либо для подключения к другому источнику данных, либо для повторного подключения к одному и тому же источнику данных. Решение оставаться на связи, в отличие от отключения и повторного подключения позже, требует, чтобы автор приложения учитывал относительные затраты на каждый вариант; как подключение к источнику данных, так и оставшиеся подключенные могут быть относительно дорогостоящими в зависимости от среды соединения. При правильном компромиссе приложение должно также делать предположения о вероятности и сроках дальнейших операций на том же источнике данных.
