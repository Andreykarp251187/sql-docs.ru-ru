# <a name="disconnecting-from-a-data-source-or-driver"></a>Отключение от источника данных или драйвера
Когда приложение завершило работу с источником данных, оно вызывает **SQLDisconnect**. **SQLDisconnect** освобождает все инструкции, выделенные для соединения, и отключает драйвер от источника данных. Он возвращает ошибку, если транзакция находится в процессе.  
  
 После отключения приложение может вызвать **SQLFreeHandle** , чтобы освободить подключение. После освобождения подключения возникает ошибка программирования приложения для использования маркера соединения в вызове функции ODBC. Это не определено, но, возможно, некритические последствия. При вызове **SQLFreeHandle** драйвер освобождает структуру, используемую для хранения сведений о соединении.  
  
 Приложение также может повторно использовать соединение для подключения к другому источнику данных или повторного подключения к тому же источнику данных. Решение о том, что подключение будет оставаться подключенным, в отличие от последующего отключения и повторного подключения, требует, чтобы модуль записи приложения рассматривал относительную стоимость каждого параметра. подключение к источнику данных и оставшееся подключение может быть сравнительно дорогостоящим в зависимости от носителя подключения. При обеспечении правильного компромисса приложение должно также делать предположения о вероятности и времени дальнейших операций с одним и тем же источником данных.
