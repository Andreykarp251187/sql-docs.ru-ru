# <a name="block-cursors"></a>Блочные курсоры
Многие приложения тратят значительное количество времени на передачу данных по сети. Часть этого времени тратится на фактические передачи данных по сети, а часть тратится на накладные расходы сети, такие как вызов, сделанный водителем, чтобы запросить ряд данных. Последнее время может быть сокращено, если приложение эффективно использует *блок,* или *жир,* *курсоры,* которые могут вернуть ся более одного ряда за один раз.  
  
 Приложение всегда имеет возможность использовать курсор блока. На источниках данных, из которых может быть извлечена только одна строка за раз, в драйвере должны быть смоделированы курсоры блоков. Это можно сделать, выполняя несколько однорядных извлечений. Хотя это вряд ли обеспечит какой-либо прирост производительности, это открывает возможности для приложений. Такие приложения будут испытывать повышение производительности, как DBMSs реализовать блок курсоры напротив и драйверы, связанные с этими DBMSs подвергать их.  
  
 Строки, возвращенные в одном извлечении с помощью блочного курсора, называются *набором строк.* Важно не путать рядсовую установку с набором результатов. Набор результатов поддерживается в источнике данных, в то время как набор строк поддерживается в буферах приложений. В то время как набор результатов фиксируется, строки не - он изменяет положение и содержимое каждый раз, когда получается новый набор строк. Точно так же, как однорядный курсор, такой как традиционный курсор только вперед, указывает на текущую строку, курсор блока указывает на ряд, который можно рассматривать как *текущие строки.*  
  
 Для выполнения операций, работающих на одной строке при извлечении нескольких строк, приложение должно сначала указать, какая строка является текущей строкой. Текущая строка требуется от вызовов на **S'LGetData** и позиционированных обновлений и удалений. Когда курсор блока сначала возвращает ряд, текущий ряд является первым рядом строки. Чтобы изменить текущую строку, приложение вызывает **S'LSetPos** или **S'LBulkOperations** (для обновления закладкой). На следующей иллюстрации показано отношение набора результатов, набора строк, текущего ряда, курсора строк и курсора блока.  Является ли курсор блоком курсора, независимо от того, является ли он прокруткой. Например, большая часть работы в приложении отчета тратится на поиск и печать строк. Из-за этого, он будет работать быстрее всего с вперед только, блок курсор. Он использует курсор только вперед, чтобы избежать расходов прокрутки курсора, и курсор блока, чтобы уменьшить сетевой трафик.
