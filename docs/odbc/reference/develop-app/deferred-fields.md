# <a name="deferred-fields"></a>Отложенные поля
Значения *отложенных полей* не используются при их установке, но драйвер сохраняет адреса переменных для отложенного эффекта. Для дескриптора параметров приложения драйвер использует содержимое переменных во время вызова в **S'LExecDirect** или **S'LExecute.** Для дескриптора строки приложения драйвер использует содержимое переменных во время получения.  
  
 Ниже приведены отложенные поля:  
  
-   SQL_DESC_DATA_PTR и SQL_DESC_INDICATOR_PTR поля дескриптора.  
  
-   Поле SQL_DESC_OCTET_LENGTH_PTR записи дескриптора приложения.  
  
-   В случае многороу-привлечения SQL_DESC_ARRAY_STATUS_PTR и SQL_DESC_ROWS_PROCESSED_PTR поля заголовка дескриптора.  
  
 При выделении дескриптора отложенные поля каждой записи дескриптора изначально имеют нулевое значение. Значение нулевого значения таково:  
  
-   Если SQL_DESC_ARRAY_STATUS_PTR имеет нулевую ценность, многоуровневый извлечение не возвращает этот компонент диагностической информации на строку.  
  
-   Если SQL_DESC_DATA_PTR имеет нулевое значение, запись не связана.  
  
-   Если SQL_DESC_OCTET_LENGTH_PTR поле ARD имеет нулевое значение, водитель не возвращает информацию о длине для этой колонки.  
  
-   Если SQL_DESC_OCTET_LENGTH_PTR поле APD имеет нулевое значение, а параметр является строкой символов, драйвер предполагает, что строка непрекращается. Для динамических параметров вывода нулевое значение в этой области не позволяет водителю возвращать информацию о длине. (Если поле SQL_DESC_TYPE не указывает параметр строки символа, поле SQL_DESC_OCTET_LENGTH_PTR игнорируется.)  
  
 Приложение не должно распределять или отбрасывать переменные, используемые для отложенных полей, между временем, которое оно связывает с полями, и временем, когда водитель читает или пишет их.
