# <a name="describing-parameters"></a>Описание параметров
**У S'LBindParameter** есть аргументы, описывающие параметр: его тип, точность и масштаб. Драйвер использует эту информацию, или *метаданные,* для преобразования значения параметра в тип, необходимый источнику данных. На первый взгляд может показаться, что драйвер находится в лучшем положении, чтобы знать метаданные параметра, чем приложение; в конце концов, водитель может легко обнаружить метаданные для столбца набора результатов. Как оказалось, это не так. Во-первых, большинство источников данных не дают водителю возможности обнаружить метаданные параметров. Во-вторых, большинство приложений уже знают метаданные.  
  
 Если в приложении жестко закодирована выписка s'L, автор приложения уже знает тип каждого параметра. Если выписка по S'L построена приложением во время выполнения, приложение может определить метаданные при построении оператора. Например, когда приложение конструирует положение  
  
```  
WHERE OrderID = ?  
```  
  
 он может вызывать **S'LКолонки** для столбца OrderID.  
  
 Единственная ситуация, в которой приложение не может легко определить метаданные параметра, это когда пользователь вводит параметризированное заявление. В этом случае приложение вызывает **S'LPrepare** для подготовки оператора, **S'LNumParams** для определения количества параметров, а также **S'LDescribeParam** для описания каждого параметра. Однако, как отмечалось ранее, большинство источников данных не предоставляют водителю возможности обнаружить метаданные параметров, поэтому **S'LDescribeParam** не пользуется широкой поддержкой.
