# <a name="events-generated-by-the-odbc-driver-manager"></a>События, создаваемые диспетчером драйверов ODBC
> [!IMPORTANT]  
>  Поддержка Visual Studio Analyzer была удалена с Windows 8 (Visual Studio Analyzer была включена только в старые версии Visual Studio.). Для альтернативного механизма устранения неполадок используйте отслеживание BID.  
  
 События, генерируемые менеджером драйверов ODBC, регистрируются при нажатии кнопки Start Visual Studio Analyzer. Сам инструмент предлагает системно-определяемые события и возможность создавать пользовательские события. Для получения дополнительной информации о событиях, см *Visual Studio Анализатор Справочник Руководство* в Visual Studio набор документации.  
  
|Визуальное событие анализатор студии|Описание|  
|----------------------------------|-----------------|  
|**Вызов**|Генерируется на каждой записи ODBC API.|  
|**Возвращение Исключение**|Генерируется на каждом возврате ODBC API, если код возврата SQL_ERROR.|  
|**ReturnNormal**|Генерируется на каждом возврате ODBC API, если код возврата не SQL_ERROR.|  
|**Начало подключения**|Означает, что соединение началось; генерируется при вызове диспетчера драйверов ODBC aAIs подключения драйвера.|  
|**Подключение Полное**|Означает, что соединение завершено; генерируется при возвращении AIS подключения драйвера к менеджеру драйвера ODBC.|  
|**Отключение Начало**|Генерируется при вызове функции **драйвера sLDisconnect** драйвера.|  
|**Отключение Полное**|Сгенерированный при возврате функции **драйвера S'LDisconnect** к менеджеру драйвера ODBC.|  
|**ЗапросОтправить**|Генерируется, когда менеджер драйверов ODBC вызывает функции **драйвера S'LPrepare,** **S'LExecute,** **S'LExecDirect,** а также функции каталога, такие как **S'LTables** и **S'LColumns.**|  
|**ЗапросРезультат**|Генерируется при возврате результата, установленного диспетчеру драйверов ODBC для функций, связанных с запросами.|  
|**ТранзакцияStart**|Генерируется, когда приложение устанавливает значение SQL_ATTR_AUTOCOMMIT SQL_AUTOCOMMIT_OFF, или после того, как приложение успешно вызывает **S'LEndTran.**|  
|**ТранзакцияОбязательство**|Генерируется при вызове приложения **S'LEndTran** для совершения локальной транзакции.|  
|**TransactionRollback**|Генерируется при вызове приложения **S'LEndTran** для отката локальной транзакции.|  
|**Присоединиться к DTC**|Генерируется при присоединении приложения к Координатору распределенных транзакций (DTC).|  
|**LeaveDTC**|Генерируется при покидает приложение Координатора распределенных транзакций (DTC).|
