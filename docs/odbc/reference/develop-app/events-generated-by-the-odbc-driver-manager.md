# <a name="events-generated-by-the-odbc-driver-manager"></a>События, создаваемые диспетчером драйверов ODBC
> [!IMPORTANT]  
>  Поддержка Visual Studio Analyzer была удалена, начиная с Windows 8 (Visual Studio Analyzer была включена только в более ранних версиях Visual Studio.). Устранение неполадок механизм альтернативные способы используйте трассировка встроенных средств диагностики.  
  
 При нажатии кнопки запуска анализатора, регистрируются события, создаваемые диспетчером драйверов ODBC. Само средство предлагает системные события и возможность создания пользовательских событий. Дополнительные сведения о событиях см. в разделе *Visual Studio Analyzer справочное руководство по* входящие в набор документации по Visual Studio.  
  
|Visual Studio Analyzer событий|Описание|  
|----------------------------------|-----------------|  
|**Вызов**|Создается при каждой операции API-интерфейса ODBC.|  
|**ReturnException**|Создается на каждой ODBC API, если возвращается значение SQL_ERROR.|  
|**ReturnNormal**|Создается на каждой ODBC API, если код возврата не SQL_ERROR.|  
|**Начало подключения**|Указывает, что подключение установлено; создается, когда диспетчер драйверов ODBC вызывает драйвера подключения API-интерфейсы.|  
|**Подключение выполнено**|Указывает, что подключение завершено; создается, когда соединение драйвера интерфейсы API вернет диспетчер драйверов ODBC.|  
|**Отключение запуска**|Создается, когда диспетчер драйверов ODBC вызывает драйвера **SQLDisconnect** функции.|  
|**Отключение завершено**|Создается, когда драйвер **SQLDisconnect** функция возвращает диспетчер драйверов ODBC.|  
|**QuerySend**|Создается, когда диспетчер драйверов ODBC вызывает драйвера **SQLPrepare**, **SQLExecute**, **SQLExecDirect** функции, а также функций каталога, например **SQLTables** и **SQLColumns**.|  
|**QueryResult**|Создается, когда драйвер возвращает результирующий набор, чтобы диспетчер драйверов ODBC для функций, включающих в себя запросы.|  
|**TransactionStart**|Создается, когда приложение задает значение SQL_ATTR_AUTOCOMMIT равным SQL_AUTOCOMMIT_OFF, или после приложение успешно вызывает **SQLEndTran**.|  
|**TransactionCommit**|Создается, когда приложение вызывает **SQLEndTran** для фиксации локальной транзакции.|  
|**TransactionRollback**|Создается, когда приложение вызывает **SQLEndTran** откат локальной транзакции.|  
|**JoinDTC**|Создается, когда приложение присоединяется координатора распределенных транзакций (DTC).|  
|**LeaveDTC**|Создается, когда приложение покидает координатор распределенных транзакций (DTC).|
