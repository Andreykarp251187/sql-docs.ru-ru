# <a name="testing-interoperable-applications"></a>Тестирование приложений с поддержкой взаимодействия
Тестирование совместимых приложений в лучшем случае требует много времени, а в худшем невозможно, потому что новые драйверы постоянно появляются на рынке. Тем не менее, разумная степень тестирования возможно. Приложения с ограниченной или низкой совместимостью должны быть протестированы только против тех водителей, которые они гарантированно поддерживают. Тем не менее, они должны быть полностью протестированы против этих драйверов.  
  
 Высокосовместимые приложения не могут быть проверены практически против всех драйверов. Лучшее, что большинство разработчиков приложений может сделать, это проверить их полностью против небольшого числа драйверов и беглый против еще нескольких. Проверенные драйверы должны включать в себя самые популярные драйверы для самых популярных DBMS s на рынке приложения; если рынок охватывает все DBMS, драйверы для настольных и серверных DBMS должны быть проверены.  
  
 Одной из проблем тестирования приложений ODBC является количество задействованных компонентов: само приложение, менеджер драйвера, драйвер, DBMS и, возможно, сетевое программное обеспечение или шлюзы. Приложения могут упростить отслеживание ошибок, разместив сообщения об ошибках, возвращенные функциями ODBC через **S'LGetDiagField** и **S'LGetDiagRec.** Эти сообщения идентифицируют производителя и компонент, в котором происходят ошибки.
