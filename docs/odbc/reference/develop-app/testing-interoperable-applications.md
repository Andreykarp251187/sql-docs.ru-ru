# <a name="testing-interoperable-applications"></a>Тестирование приложений с поддержкой взаимодействия
Тестирование приложений с возможностью взаимодействия в самом лучшем много бизнеса и в худшем случае невозможно, так как постоянно появляются новые драйверы, представленными на рынке. Тем не менее можно более разумным степень тестирования. Приложения с низкой или ограниченные взаимодействия требуется только проверены те драйверы, которые они гарантированно поддерживают. Тем не менее они должны быть полностью проверены эти драйверы.  
  
 Поддерживает возможность взаимодействия приложений не может быть практически проверены все драйверы. Лучшее, что большинство разработчиков приложений можно сделать — проверить их полностью небольшое количество драйверов и cursorily еще несколько. Протестированные драйверы должны включать наиболее популярных драйверы для самых популярных СУБД на рынке приложения; Если рынке охватывает все СУБД, должны тестироваться драйверы для настольных компьютеров и серверов СУБД.  
  
 Одна из проблем при тестировании приложения ODBC — это количество компонентов, участвующих: само приложение, диспетчер драйверов, драйвер, СУБД и возможно сетевое программное обеспечение или шлюзы. Приложения можно упростить его для отслеживания ошибок, публикуя сообщения об ошибках, возвращаемые функциями ODBC через **SQLGetDiagField** и **SQLGetDiagRec**. Эти сообщения определите производителя и компонентов, в которых возникают ошибки. Дополнительные сведения см. в разделе [диагностики](../../../odbc/reference/develop-app/diagnostics.md).
