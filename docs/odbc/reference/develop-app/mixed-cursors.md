# <a name="mixed-cursors"></a>Смешанные курсоры

Смешанный курсор представляет собой комбинацию курсора, управляемого ключами, и динамического курсора. Он используется, когда набор результатов слишком велик, чтобы разумно сохранять клавиши для всего набора результатов. Смешанные курсоры реализуются путем создания набора ключей, который меньше всего набора результатов, но больше, чем ряд.  
  
 До тех пор, пока приложение прокручивается в клавиатуре, поведение приводится в движение ключей. Когда приложение прокручивается за пределами клавиатуры, поведение является динамическим: курсор получает запрошенные строки и создает новый набор ключей. После создания нового набора ключей поведение возвращается к клавиатуре в этом ключе.  
  
 Например, предположим, что набор результатов имеет 1000 строк и использует смешанный курсор с размером ключа 100 и рядом 10. Когда первый ряд окберется, курсор создает набор ключей, состоящий из ключей для первых 100 строк. Затем он возвращает первые 10 строк, как это требуется.  
  
 Теперь предположим, что другое приложение удаляет строки 11 и 101. Если курсор пытается получить строку 11, он столкнется с пробелом, поскольку у него есть ключ для этой строки, но строка не существует; это поведение, управляемое ключами. Если курсор пытается получить строку 101, курсор не обнаружит, что строка отсутствует, поскольку у него нет ключа для строки. Вместо этого он получит то, что ранее было строкой 102. Это динамическое поведение курсора.  
  
 Смешанный курсор эквивалентен курсору, управляемому клавиатурой, когда размер набора ключей равен размеру набора результата. Смешанный курсор эквивалентен динамическому курсору, когда размер ключа равен 1.
