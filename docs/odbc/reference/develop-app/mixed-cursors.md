# <a name="mixed-cursors"></a>Смешанные курсоры

Смешанный курсор представляет собой сочетание курсора, управляемого набором ключей, и динамического курсора. Он используется, если результирующий набор слишком велик, чтобы хранить ключи для всего результирующего набора. Смешанные курсоры реализуются путем создания набора ключей, который меньше, чем весь результирующий набор, но больше, чем набор строк.  
  
 Пока приложение прокручивается внутри набора ключей, поведение зависит от набора ключей. При прокрутке приложения за пределами набора ключей поведение является динамическим: курсор извлекает запрошенные строки и создает новый набор ключей. После создания нового набора ключей поведение будет восстановлено с помощью набора ключей, управляемого набором ключей.  
  
 Например, предположим, что результирующий набор содержит 1 000 строк и использует смешанный курсор с размером набора ключей 100 и размером набора строк, равным 10. Когда извлекается первый набор строк, курсор создает набор ключей, состоящий из ключей для первых 100 строк. Затем он возвращает первые 10 строк в соответствии с запросом.  
  
 Теперь предположим, что другое приложение удаляет строки 11 и 101. Если курсор пытается получить строку 11, возникает разрыв, так как он содержит ключ для этой строки, но строка не существует. Это поведение, управляемое набором ключей. Если курсор пытается получить строку 101, курсор не обнаружит, что строка отсутствует, так как у нее нет ключа для строки. Вместо этого он будет получать сведения о ранее заданной строке 102. Это поведение динамического курсора.  
  
 Смешанный курсор эквивалентен курсору, управляемому набором ключей, если размер набора ключей равен размеру результирующего набора. Смешанный курсор эквивалентен динамическому курсору, когда размер набора ключей равен 1.
