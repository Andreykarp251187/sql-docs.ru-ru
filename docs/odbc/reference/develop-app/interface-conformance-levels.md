# <a name="interface-conformance-levels"></a>Уровни соответствия интерфейса
Цель выравнивания заключается в том, чтобы сообщить приложению, какие функции доступны ему от водителя. Схема выравнивания, основанная на функциях, не достигает этой цели в достаточной степени. В ODBC 3. *x,* драйверы классифицируются на основе функций, которыми они обладают. Поддержка функции может включать поддержку функции; он также может включать в себя поддержку поля дескриптора, атрибута оператора, значения "Y" для типа информации, возвращенного **S'LGetInfo,** и так далее.  
  
 Чтобы упростить спецификацию соответствия интерфейса, ODBC определяет три уровня соответствия. Чтобы соответствовать определенному уровню соответствия, водитель должен удовлетворять всем требованиям этого уровня соответствия. Соответствие данному уровню подразумевает полное соответствие всем более низким уровням.  
  
 Уровни соответствия не всегда делятся на поддержку определенного списка функций ODBC, но указывают поддерживаемые функции, перечисленные в следующих разделах. Чтобы обеспечить поддержку функции, водитель должен поддерживать некоторые или все формы вызовов для определенных функций ODBC (для получения дополнительной информации, [см. Соответствие функций),](../../../odbc/reference/develop-app/function-conformance.md)установив определенные атрибуты (см. [Attribute Conformance),](../../../odbc/reference/develop-app/attribute-conformance.md)и некоторые поля дескриптора (см. [Соответствие поля дескриптора).](../../../odbc/reference/develop-app/descriptor-field-conformance.md)  
  
 Приложение обнаруживает уровень соответствия интерфейса драйвера, подключившись к источнику данных и позвонив в **SQL_ODBC_INTERFACE_CONFORMANCE** опцию.  
  
 Драйверы могут свободно реализовывать функции, выходящие за рамки уровня, на который они претендуют на полное соответствие. Приложения обнаруживают такие дополнительные возможности, вызывая **S'LGetOther** (чтобы определить, какие функции ODBC присутствуют) и **S'LGetInfo** (для запроса различных других возможностей ODBC).  
  
 Существует три уровня соответствия интерфейса ODBC: Core, Level 1 и Level 2.  
  
> [!NOTE]
>  Эти уровни соответствия имеют различные требования, чем уровни соответствия ODBC API с тем же именем в ODBC 2 *.x*. В частности, все функции, подразумеваемые ODBC 2 *.x* API соответствия Уровень 1 в настоящее время являются частью уровня соответствия интерфейса Core. В результате многие драйверы ODBC могут сообщать о соответствии интерфейса базового уровня.
