# <a name="rowset-size"></a>Размер набора строк
Какой размер набора строк для использования зависит от приложения. На экране приложения обычно следуют одной из двух стратегий. Первый шаг — определение размера набора строк на число строк, отображаемых на экране; Если пользователь изменяет размер экрана, оно соответствующим образом изменяется размер набора строк. Второй — для установки размера набора строк большее значение, например, 100, что уменьшает количество вызовов к источнику данных. Приложение прокручивает локально в наборе строк, если это возможно и извлекает новые строки только в том случае, когда он прокручивается за пределы набора строк.  
  
 Другие приложения, например отчеты, как правило, присвоено максимальное количество строк, приложение способен обработать - размер набора строк с размером набора строк, иногда сокращается сетевых издержек на каждую строку. Точно как большой набор строк может быть зависит от размера каждой строки и объем доступной памяти.  
  
 Размер набора строк устанавливается вызовом **SQLSetStmtAttr** с *атрибут* аргументом атрибута SQL_ATTR_ROW_ARRAY_SIZE. Приложения можно изменить размер набора строк, привязать новый буферы строк (путем вызова **SQLBindCol** или указания смещения привязки) даже в том случае, после выбранных строк, или оба. Последствия изменения размера набора строк зависящие от функции:  
  
-   **SQLFetch** и **SQLFetchScroll** использовать размер набора строк во время вызова, чтобы определить, сколько строк для выборки. Тем не менее **SQLFetchScroll** с *FetchOrientation* SQL_FETCH_NEXT увеличивается курсор на основе набора строк предыдущей выборке и затем выбирает набор строк, в зависимости от текущего размера набора строк.  
  
-   **SQLSetPos** использует размер набора строк, которая действует на предшествующий ему вызов **SQLFetch** или **SQLFetchScroll**, так как **SQLSetPos** работает для набора строк уже задано. **SQLSetPos** также получают новый размер набора строк при **SQLBulkOperations** был вызван после изменения размера набора строк.  
  
-   **SQLBulkOperations** использует размер набора строк в силе во время вызова, так как он выполняет операции с таблицей, независимо от любого извлеченных строк.
