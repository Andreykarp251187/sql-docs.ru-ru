# <a name="connecting-with-sqlbrowseconnect"></a>Подключение с помощью SQLBrowseConnect
**SQLBrowseConnect**, например **SQLDriverConnect**, использует строку подключения. Тем не менее, с помощью **SQLBrowseConnect**, приложение может создать полную строку подключения во время выполнения. Это позволяет приложению:  
  
-   Создавать свои собственные диалоговые окна для запроса этой информации, тем самым сохраняя контроль над «вид и поведение.»  
  
-   просматривать систему в поисках источников данных, которые может использовать конкретный драйвер, возможно, за несколько шагов. Например, пользователь может сначала просмотреть серверы в сети, а после выбора сервера с помощью драйвера просмотреть доступные базы данных этого сервера.  
  
 Приложение вызывает **SQLBrowseConnect** и передает строку подключения, известный как *просмотреть строку подключения для запроса,* , указывающий источник драйвера или данных. Драйвер возвращает строку подключения, известный как *Обзор результирующую строку подключения,* , содержащий ключевые слова, возможные значения (если ключевое слово принимает набор дискретных значений) и понятных имен. Приложение создает диалоговое окно с понятных имен и запрашивает у пользователя значений. Затем создает новую строку подключения Обзор запроса на основе этих значений и возвращает этот драйвер другим вызовом **SQLBrowseConnect**.  
  
 Так как строки подключения передаются вперед и назад, драйвер может предоставлять несколько уровней просмотра, возвращая новую строку соединения, при переходе со старым паролем приложения. Например, в первый раз, приложение вызывает **SQLBrowseConnect**, драйвер может возвращать ключевые слова для запроса пользователя для имени сервера. Если приложение возвращает имя сервера, драйвер может возвращать ключевые слова для запроса пользователя для базы данных. Процесс просмотра веб-страниц может быть решена, после приложение возвращается имя базы данных.  
  
 Каждый раз **SQLBrowseConnect** возвращает новый обзора результат строку подключения, она возвращает SQL_NEED_DATA, что его код возврата. Это сообщает приложению о том что процесс подключения не завершена. Пока **SQLBrowseConnect** возвращает SQL_SUCCESS, соединение находится в состоянии необходимые данные и не может использоваться для других целей, например, чтобы задать атрибут соединения. Приложение может завершить соединения, Обзор процесса путем вызова **SQLDisconnect**.
