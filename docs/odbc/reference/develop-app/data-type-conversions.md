# <a name="data-type-conversions"></a>Преобразования типов данных
Данные могут преобразовываться из одного типа в другой на одном из четыре раза: когда данные передаются из переменной одно приложение на другой (C для C), при отправке данных в переменную приложения к параметру инструкции (C to SQL), при возврате данных в столбец результирующего набора в переменную приложения (из SQL в C), и при передаче данных из столбца источника данных другой (SQL для SQL).  
  
 Любое преобразование, которое происходит, когда данные передаются из переменной одно приложение в другой выходит за рамки данного документа.  
  
 Когда приложение привязывается к параметр столбца или инструкция set результата, приложение неявно задает преобразование типов данных в его Выбор типа данных переменной приложения. Например предположим, что столбец содержит целочисленные данные. Если приложение связывает целочисленной переменной к столбцу, он указывает, что нужно выполнить преобразование; Если приложение связывает символьной переменной к столбцу, он указывает, что данные преобразовать из целого числа в символ.  
  
 ODBC определяет преобразование данных между каждого типа данных SQL и C. По сути ODBC поддерживает все разумные преобразований, например символ, число, а число с плавающей запятой и не поддерживает неправильно преобразования, такие как число с плавающей запятой, до даты. Драйверы необходимы для поддержки всех преобразований для каждого типа данных SQL, которые они поддерживают. Полный список преобразований между типами данных SQL и C, см. в разделе [преобразование данных из SQL в типы данных C](../../../odbc/reference/appendixes/converting-data-from-sql-to-c-data-types.md) и [преобразование данных из C в типы данных SQL](../../../odbc/reference/appendixes/converting-data-from-c-to-sql-data-types.md) в приложение г Типы данных.  
  
 ODBC также определяет скалярную функцию для преобразования данных из одного типа данных SQL в другой. **Преобразовать** скалярная функция сопоставляется с помощью драйвера для базовой скалярной функции или функции, определенные для выполнения преобразований в источнике данных. Так как эта функция сопоставляется функций, относящихся к СУБД, ODBC не определяет как работают эти преобразования или какие-либо преобразования должно поддерживаться. Приложение обнаруживает, какие преобразования поддерживаются определенного источника драйвера и данные по элементам SQL_CONVERT **SQLGetInfo**. Дополнительные сведения о **преобразовать** скалярной функцией, см. в разделе [escape-последовательности в ODBC](../../../odbc/reference/develop-app/escape-sequences-in-odbc.md) и [явные функции преобразования типа данных](../../../odbc/reference/appendixes/explicit-data-type-conversion-function.md).
