# <a name="sql-type-identifiers"></a>Идентификаторы типа SQL
Каждый источник данных определяет свои собственные типы данных S'L. ODBC определяет идентификаторы типов и описывает общие характеристики типов данных S'L, которые могут быть отображены для каждого типа идентификатора. Именно драйверы определяются тем, как каждый тип данных в базовом источнике данных отображается в идентификатор типа S'L ODBC.  
  
 Например, SQL_CHAR — это идентификатор типа для столбца символов с фиксированной длиной, обычно между 1 и 254 символами. Эти характеристики соответствуют типу данных CHAR, обнаруженным во многих источниках данных S'L. Таким образом, когда приложение обнаруживает, что идентификатор типа для столбца SQL_CHAR, он может предположить, что он, вероятно, имеет дело с столбецом CHAR. Тем не менее, он должен все еще проверить длину байта столбца, прежде чем предположить, что это между 1 и 254 символов; например, драйвер для источника данных, не относящегося к S-S'L, может отображать столбец символов с фиксированной длиной 500 символов для SQL_CHAR или SQL_LONGVARCHAR, поскольку ни то, ни другое не является точным совпадением.  
  
 ODBC определяет широкий спектр идентификаторов типа S'L. Однако водитель не обязан использовать все эти идентификаторы. Вместо этого он использует только те идентификаторы, которые необходимы для разоблачения типов данных S'L, поддерживаемых базовым источником данных. Если основной источник данных поддерживает типы данных S'L, к которым не соответствует идентификатор типа, драйвер может определить дополнительные идентификаторы типа. Для получения дополнительной информации [см.](../../../odbc/reference/develop-app/driver-specific-data-types-descriptor-information-diagnostic.md)
