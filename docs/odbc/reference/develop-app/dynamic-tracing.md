# <a name="dynamic-tracing"></a>Динамическая трассировка
Отслеживание может быть включено или отключено в любой момент запуска приложения. Это позволяет приложению отслеживать любое количество вызовов функций.  
  
 Переменная **ODBCSharedTraceFlagFlag** установлена для динамического отслеживания. Эта переменная распределяется между всеми запущенными копиями менеджера драйвера. Если какое-либо приложение устанавливает эту переменную, трассировка включена для всех приложений ODBC, которые в настоящее время работают. Чтобы отключить трассировку при включении динамического отслеживания, приложение вызывает **S'LSetConnectAttr,** чтобы установить SQL_ATTR_TRACE SQL_TRACE_OFF. Этот вызов выключит трассировку только для этого приложения. Приложения, связанные с Odbc32.lib, могут изменить использование этой переменной. Данные трассировки могут отображаться в окне в режиме реального времени, а не в файле трассировки, который должен быть открыт после сеанса ODBC. Элементы управления могут быть добавлены на экран приложения, чтобы включить или выключить трассировку по своей воле.  
  
 Трасса DLL, поставляемая с ODBC 3 *.x,* не является безопасной для потока. Не гарантируется, что файл журнала записывается правильно, если глобальная трассировка включена (установлена переменная **ODBCSharedTraceFlagFlag)** и несколько приложений записываются в файл трассировки одновременно. Это условие не возвращает ошибку.
