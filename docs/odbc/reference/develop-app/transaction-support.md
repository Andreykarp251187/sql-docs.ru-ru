# <a name="transaction-support"></a>Поддержка транзакций
Степень поддержки транзакций определяется драйвером. ODBC предназначен для реализации на однопользовательской или настольной базе данных, которая не имеет необходимости управлять несколькими обновлениями своих данных. Кроме того, некоторые базы данных, поддерживающие транзакции, делают это только для заявлений о языке манипулирования данными (DML) СЗЛ; существуют ограничения или специальная семантика транзакций в отношении использования языка определения данных (DDL) при активной транзакции. То есть может быть поддержка транзакций для нескольких одновременных обновлений в таблицах, но не для изменения числа и определения таблиц во время транзакции.  
  
 Приложение определяет, поддерживаются ли транзакции, может ли DDL быть включен в транзакцию, и любые специальные эффекты включения DDL в транзакцию, позвонив по **s'LGetInfo** с SQL_TXN_CAPABLE опцией. Для получения более [подробной](../../../odbc/reference/syntax/sqlgetinfo-function.md) информации см.  
  
 Если драйвер не поддерживает транзакции, но приложение имеет возможность (с помощью API, кроме ODBC) блокировать и разблокировать данные, приложения могут добиться поддержки транзакций, блокируя и разблокируя записи и таблицы по мере необходимости. Для реализации примера перевода учетной записи приложение блокирует записи для обеих учетных записей, копирует текущие значения, дебетирует первый счет, кредитирует вторую учетную запись и разблокирует записи. В случае сбываний с какими-либо шагами приложение сбросить учетные записи с помощью копий.  
  
 Даже источники данных, поддерживающие транзакции, могут не поддерживать более одной транзакции одновременно в определенной среде. Приложения **называются S'LGetInfo** с SQL_MULTIPLE_ACTIVE_TXN опцией для определения того, может ли источник данных поддерживать одновременные активные транзакции на нескольких соединениях в одной среде. Поскольку на подключение приходится одна транзакция, это интересно только приложениям, которые имеют несколько подключений к одному и тому же источнику данных.
