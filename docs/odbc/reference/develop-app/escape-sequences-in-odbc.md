# <a name="escape-sequences-in-odbc"></a>Escape-последовательности в ODBC
Ряд языковых функций, таких как внешние соединения и вызовы функций масштабирования, обычно реализуются DBMS. Однако синтаксисы для этих функций, как правило, специфичны для DBMS, даже если стандартные синтаксисы определяются различными органами по стандартизации. Из-за этого ODBC определяет последовательности побега, которые содержат стандартные синтаксисы для следующих языковых функций:  
  
-   Дата, время, метка времени и интервал времени в буквальном  
  
-   Функции scalar, такие как функции конверсии числовых, строк и конверсий типа данных  
  
-   LIKE предикатный персонаж побега  
  
-   Внешние соединения  
  
-   Процедурные вызовы  
  
 Последовательность побега, используемая ODBC, заключается в следующем:  
  
```  
  
(extension)  
  
```  
  
## <a name="remarks"></a>Remarks  
 Последовательность побега распознается и разбирается драйверами, которые заменяют последовательности побега грамматикой DBMS. Для получения дополнительной информации о [ODBC Escape Sequences](../../../odbc/reference/appendixes/odbc-escape-sequences.md) синтаксисе последовательности побега см.  
  
> [!NOTE]  
>  В ODBC 2. *x*, это был стандартный синтаксис последовательности побега: **\*--("Поставщик(**_поставщик-имя),_**продукт (название**_продукта)_**)** ** \*расширение)--** _extension_  
>   
>  В дополнение к этому синтаксису, был определен синтаксис с_extension_**}** сокращением: **{**  
>   
>  В ODBC 3. *x,* длинная форма последовательности побега была унитеса, и форма сокращения используется исключительно.  
  
 Поскольку последовательности побега отображаются драйвером в синтезируете DBMS, приложение может использовать либо последовательность побега, либо синтаксис DBMS. Однако приложения, которые используют синтаксис DBMS, не будут совместимы. При использовании последовательности побега приложения должны убедиться, что атрибут SQL_ATTR_NOSCAN оператора выключен, что по умолчанию. В противном случае последовательность побега будет отправлена непосредственно в источник данных, где она обычно вызывает ошибку синтаксиса.  
  
 Драйверы поддерживают только те последовательности побега, которые они могут сопоставить с основными языковыми функциями. Например, если источник данных не поддерживает внешние соединения, то и драйвер не будет. Чтобы определить, какие последовательности побега поддерживаются, приложение вызывает **S'LGetTypeInfo** и **S'LGetInfo**.
