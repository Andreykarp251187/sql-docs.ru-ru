# <a name="driver-specific-connection-information"></a>Сведения о подключении для конкретного драйвера
**SLConnect** предполагает, что имя источника данных, идентификатор пользователя и пароль достаточны для подключения к источнику данных и что вся другая информация о подключении может храниться в системе. Это часто не так. Например, для входа на сервер водителю может потребоваться один идентификатор пользователя и пароль, а для входа в DBMS — другой идентификатор пользователя и пароль. Поскольку **S'LConnect** принимает один идентификатор пользователя и пароль, это означает, что другой идентификатор пользователя и пароль должны храниться с информацией об источнике данных в системе, если для использования **S'LConnect.** Это потенциальное нарушение безопасности и следует избегать, если пароль зашифрован.  
  
 **SLDriverConnect** позволяет водителю определить произвольное количество информации о подключении в парах значения ключевых слов строки соединения. Например, предположим, что водителю требуется имя источника данных, идентификатор пользователя и пароль для сервера, а также идентификатор пользователя и пароль для DBMS. Пользовательская программа, которая всегда использует источник данных XY'Corp, может побудить пользователя к использованию идентиверий и паролей и создать следующий набор пар значения ключевых слов, или *строки соединения,* чтобы перейти к **S'LDriverConnect:**  
  
> [!NOTE]  
>  Если вы подключаетесь к поставщику источников данных, который поддерживает аутентификацию Windows, следует указать `Trusted_Connection=yes` вместо идентификатора пользователя и информацию о паролях в строке соединения.  
  
```  
DSN={MyDataSourceName};UID={MyUserID};PWD={MyServerPassword};UIDDBMS={MyDBMSUserID};PWDDBMS={MyDBMSUserPassword};  
```  
  
 Ключевое слово **DSN** (Имя источника данных) называет источник данных, ключевые слова **UID** и **PWD** указывают идентификатор пользователя и пароль для сервера, а ключевые слова **UIDDBMS** и **PWDDBMS** указывают идентификатор пользователя и пароль для DBMS. Обратите внимание, что окончательная запятая не является обязательной. **СЗЛДрайверКонн** едкает эту строку; использует имя источника данных XY'Corp для получения дополнительной информации о подключении из системы, такой как адрес сервера; и входит на сервер и DBMS с использованием указанных идентионное идентиматизм пользователя и паролей.  
  
 Пары, оцениваемые ключевыми словами в **S'LDriverConnect,** должны следовать определенным правилам синтаксиса. Ключевые слова и их значения не должны содержать в себе **{}();?? \*Персонажи.** Значение ключевого слова **DSN** не может состоять только из пробелов и не должно содержать ведущие пробелы. Из-за грамматики реестра ключевые слова и\\имена источников данных не могут содержать символ backslash ( ) Пространства не допускаются вокруг равного знака в паре значения ключевых слов.  
  
 Ключевое слово **FILEDSN** может быть использовано в вызове в **S'LDriverConnect** для указания имени файла, содержащего информацию об источнике данных (см. [Подключение с использованием источников файлов](../../../odbc/reference/develop-app/connecting-using-file-data-sources.md), позже в этом разделе). Ключевое слово **SAVEFILE** может быть использовано для указания имени файла .dsn, в котором будут сохранены пары значений ключевых слов успешного соединения, сделанного путем вызова **на S'LDriverConnect.** Для получения более подробной информации об источниках файловых данных можно ознакомиться с описанием функции [S'LDriverConnect.](../../../odbc/reference/syntax/sqldriverconnect-function.md)
