# <a name="driver-specific-connection-information"></a>Сведения о подключении для конкретного драйвера
**SQLConnect** предполагается, что имя источника данных, идентификатор пользователя и пароль достаточны для подключения к источнику данных и что все другие сведения о подключении могут храниться в системе. Это часто не так. Например драйвер может потребоваться идентификатор пользователя и пароль для входа на сервер и с другим Идентификатором пользователя и паролем для входа в СУБД. Так как **SQLConnect** принимает имя одного пользователя и пароль, это означает, что идентификатор пользователя и пароль должны быть сохранены вместе с источника данных в системе если **SQLConnect** будет использоваться. Это потенциальных уязвимостях безопасности, следует избегать, если пароль зашифрован.  
  
 **SQLDriverConnect** позволяет драйверу для определения произвольному набору сведений о соединении в парах "ключевое_слово значение" в строке подключения. Например предположим, что драйвер требует имя источника данных, идентификатор пользователя и пароль для сервера и идентификатор пользователя и пароль для СУБД. Программа, которая всегда использует источник данных XYZ Corp может запрашивать для идентификаторов и паролей и построения следующий набор пар "ключевое слово значение", или *строку подключения,* для передачи **SQLDriverConnect**:  
  
> [!NOTE]  
>  Если вы подключаетесь к поставщик источника данных, который поддерживает проверку подлинности Windows, следует указать `Trusted_Connection=yes` вместо идентификатора и пароля пользователя в строке подключения.  
  
```  
DSN={MyDataSourceName};UID={MyUserID};PWD={MyServerPassword};UIDDBMS={MyDBMSUserID};PWDDBMS={MyDBMSUserPassword};  
```  
  
 **DSN** ключевое слово (имя источника данных) фиксирует имя источника данных, **UID** и **PWD** ключевые слова определяют идентификатор пользователя и пароль для сервера и **UIDDBMS**  и **PWDDBMS** ключевые слова определяют идентификатор пользователя и пароль для СУБД. Обратите внимание на то, что последний точка с запятой не является обязательным. **SQLDriverConnect** анализирует эту строку; использует имя источника данных XYZ Corp для извлечения из системы, например адрес сервера; дополнительные данные соединения и входе на сервер и СУБД, с помощью указанного пользователя идентификаторов и паролей.  
  
 Пар «ключевое_слово значение» в **SQLDriverConnect** должны отвечать определенным правилам синтаксиса. Ключевые слова и их значения не должна содержать **[]{}(),? \*=! @** символов. Значение **DSN** ключевое слово не может состоять только из пробелов и не должно содержать начальные пробелы. Из-за грамматики реестра, ключевые слова и имена источников данных не может содержать обратную косую черту (\\) символов. Не допускаются пробелы вокруг знака равенства в паре «ключевое_слово значение».  
  
 **FILEDSN** слово может использоваться в вызове **SQLDriverConnect** для указания имени файла, который содержит сведения об источнике данных (см. в разделе [подключение с использованием файловых источников данных](../../../odbc/reference/develop-app/connecting-using-file-data-sources.md)далее в этом разделе). **SAVEFILE** слово может использоваться для указания имени файла .dsn, в котором выполнены пар "ключевое слово значение", об успешном подключении вызовом **SQLDriverConnect** будут сохранены. Дополнительные сведения о файловых источников данных, см. в разделе [SQLDriverConnect](../../../odbc/reference/syntax/sqldriverconnect-function.md) описание функции.
