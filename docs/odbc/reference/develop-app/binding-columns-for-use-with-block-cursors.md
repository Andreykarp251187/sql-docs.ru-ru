# <a name="binding-columns-for-use-with-block-cursors"></a>Привязка столбцов для использования с блочными курсорами
Поскольку курсоры блоков возвращают несколько строк, приложения, которые их используют, должны связывать массив переменных с каждым столбцом вместо одной переменной. Эти массивы известны как *буферы рядового набора.* Ниже приведены два стиля связывания:  
  
-   Привязать массив к каждой колонке. Это называется *связующим кругом столбцов,* поскольку каждая структура данных (массив) содержит данные для одного столбца.  
  
-   Определите структуру для хранения данных для всей строки и связывайте массив этих структур. Это называется *связыванием по развязке строки,* поскольку каждая структура данных содержит данные для одной строки.  
  
 Наряду с тем, что приложение связывает отдельные переменные с столбцов, оно вызывает **S'LBindCol,** чтобы связать массивы с столбцов. Разница лишь в том, что пройдены адреса являются адресами массива, а не отдельными переменными. Приложение устанавливает атрибут SQL_BIND_BY_COLUMN оператора, чтобы указать, использует ли он связывание с характерю столбца или строкой. Использование связывания с характерю столбцов или строки в основном является вопросом предпочтений приложения. Привязка по верхуню может в большей степени соответствовать планировке данных приложения, и в этом случае она обеспечит более высокую производительность.
