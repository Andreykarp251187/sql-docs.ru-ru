# <a name="is-odbc-the-answer"></a>Есть ли смысл использовать ODBC?
Прежде чем углубиться в вопрос о совместимости, рассмотрим следующий вопрос: Должно ли приложение использовать ODBC вообще? Это может показаться странным вопрос, чтобы спросить в руководстве по ODBC, но это, по сути, законным. ODBC не был разработан для полной замены родных AБИИ базы данных и не был предназначен для обеспечения доступа к базе данных при любых обстоятельствах. Он был разработан, чтобы обеспечить общий интерфейс для баз данных и был предназначен для свободного приложений программистов от необходимости изучать и поддерживать ссылки на несколько баз данных.  
  
 Пользовательские приложения являются основными кандидатами для нативных AIS базы данных. Основная причина заключается в том, что пользовательские приложения часто работают с одним DBMS и не должны быть совместимыми. AIS базы данных коренных народов может сделать лучше, чем ODBC разоблачения возможностей конкретного DBMS и может разоблачить возможности, не подверженные ODBC. Кроме того, поскольку разработчики пользовательских приложений обычно знакомы с aPI базы данных для своих DBMS, нет никаких причин, чтобы узнать ODBC. Тем не менее, интересно отметить, что для некоторых DBMS, ODBC является родной базой данных API.  
  
 Итак, какие заявки являются кандидатами на ODBC? Лучшими кандидатами являются приложения, которые работают с более чем одним DBMS. Это включает практически все общие и вертикальные приложения. Он также включает в себя ряд пользовательских приложений. Например, пользовательские приложения, которые используют несколько различных DBMS, гораздо проще и чище писать с ODBC, чем с несколькими нативными AIS. И пользовательские приложения, написанные с ODBC, гораздо легче мигрировать, поскольку компания перемещается из одного DBMS в другой или развертывает одно и то же приложение против различных DBMS.
