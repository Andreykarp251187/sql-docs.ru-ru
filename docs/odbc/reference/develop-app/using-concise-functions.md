# <a name="using-concise-functions"></a>Использование сокращенных функций
Некоторые функции ODBC получают неявный доступ к дескрипторам. Авторы приложений могут найти их более удобными, чем вызов **s'LSetDescfield** или **S'LGetDescField**. Эти функции называются *краткими* функциями, поскольку они выполняют ряд функций, включая настройку или получение полей дескриптора. Некоторые краткие функции позволяют приложению установить или получить несколько связанных полей дескриптора в одном вызове функции.  
  
 Краткие функции можно вызывать без предварительного извлечения дескриптора для использования в качестве аргумента. Эти функции работают с полями дескриптора, связанными с ручкой оператора, на которую они вызваны.  
  
 Краткие функции **S'LBindCol** и **S'LBindParameter** связывают столбец или параметр, установив полей дескриптора, которые соответствуют их аргументам. Каждая из этих функций выполняет больше задач, чем просто установка дескрипторов. **СЗЛБандерКол** и **СЗЛБандерПарастер** обеспечивают полную спецификацию привязки столбца данных или динамического параметра. Однако приложение может изменить отдельные детали связывания, позвонив по **S'LSetDesccField** или **S'LSetDesccRec,** и может полностью связать столбец или параметр, сделав ряд подходящих вызовов для этих функций.  
  
 Краткие функции **S'LColAttribute**, **S'LDescribeCol**, **S'LDescribeParam**, **S'LNumParams**, и **S'LNumResultCols** получить значения в дескрипторполях.  
  
 **СЗЛСетДескКРеии** и **СЗЛГетДескКРес** являются краткими функциями, которые с одним вызовом устанавливают или получают несколько полей дескриптора, которые влияют на тип данных и хранение столбцов или параметрных данных. **S'LSetDescCRec** является эффективным способом изменения связывания данных столбца или параметра в один шаг.  
  
 **В** некоторых случаях лаконичная функциями **служат** сужаемые функции. (См. [Поля дескриптора](../../../odbc/reference/develop-app/descriptor-fields.md).)
