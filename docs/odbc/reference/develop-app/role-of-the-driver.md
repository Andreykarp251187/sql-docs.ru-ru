# <a name="role-of-the-driver"></a>Роль драйвера
Драйвер проверяет наличие всех ошибок и предупреждений, не проверенных диспетчером драйверов, и создаваемые им записи состояния заказов. (ODBC 2. драйвер *x* не упорядочивает записи о состоянии.) Сюда входят ошибки и предупреждения при усечении данных, преобразовании данных, синтаксисе и некоторых переходах состояния. Драйвер также может проверять ошибки и предупреждения, частично проверенные диспетчером драйверов. Например, несмотря на то, что диспетчер драйверов проверяет, является ли значение *операции* в **SQLSetPos** допустимым, драйвер должен проверить, поддерживается ли он.  
  
 Драйвер также сопоставляет *собственные ошибки* , т. е. ошибки, возвращенные источником данных, в SQLSTATE. Например, драйвер может сопоставлять ряд различных машинных ошибок для недопустимого синтаксиса SQL в SQLSTATE 42000 (синтаксическая ошибка или нарушение прав доступа). Драйвер возвращает номер собственной ошибки в поле SQL_DIAG_NATIVE записи состояния. Документация по драйверу должна показывать, как ошибки и предупреждения сопоставлены из источника данных с аргументами в **SQLGetDiagRec** и **SQLGetDiagField**.
