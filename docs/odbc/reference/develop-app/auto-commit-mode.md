# <a name="auto-commit-mode"></a>Режим автоматической фиксации
*В режиме автоматической фиксации* каждая операция базы данных — это транзакция, которая фиксируется при выполнении. Этот режим подходит для многих реальных транзакций, состоящих из одной инструкции SQL. Не нужно разделять или указывать завершение этих транзакций. В базах данных без поддержки транзакций режим автоматической фиксации является единственным поддерживаемым режимом. В таких базах данных инструкции фиксируются при их выполнении и нет способа их отката. Поэтому они всегда находятся в режиме автоматической фиксации.  
  
 Если базовая СУБД не поддерживает транзакции в режиме автоматической фиксации, драйвер может эмулировать их путем ручной фиксации каждой инструкции SQL при ее выполнении.  
  
 Если пакет инструкций SQL выполняется в режиме автоматической фиксации, это зависит от источника данных, когда инструкции в пакете фиксируются. Их можно зафиксировать при выполнении или в целом после выполнения всего пакета. Некоторые источники данных могут поддерживать оба этих поведения и могут предоставить способ выбора одного или других. В частности, если в середине пакета возникает ошибка, то это зависит от источника данных, являются ли уже выполненные инструкции фиксированными или выполненными. Таким образом, взаимодействующие приложения, использующие пакеты и требующие их фиксации или отката в целом, должны выполнять пакеты только в режиме фиксации вручную.
