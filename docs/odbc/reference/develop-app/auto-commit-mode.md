# <a name="auto-commit-mode"></a>Режим автоматической фиксации
*В режиме автоматического коммита* каждая операция базы данных — это транзакция, которая совершается при выполнении. Этот режим подходит для многих реальных транзакций, которые состоят из одного оператора S'L. Нет необходимости разгранизать или указать завершение этих транзакций. В базах данных без поддержки транзакций единственным поддерживаемым режимом является режим автоматического фиксации. В таких базах данных операторы заносятся, когда они выполняются, и нет никакого способа откатить их; поэтому они всегда находятся в режиме автоматического коммит.  
  
 Если базовый DBMS не поддерживает транзакции в режиме автоматического фиксации, драйвер может подражать им, вручную выполняя каждое заявление S'L по мере его выполнения.  
  
 Если пакет инструкций S'L выполняется в режиме автоматического коммита, при фиксации данных облагаются исходные данные. Они могут быть зафиксированы по мере их выполнения или в целом после выполнения всей партии. Некоторые источники данных могут поддерживать оба этих поведения и могут обеспечить способ выбора одного или другого. В частности, если ошибка происходит в середине пакета, то исходные данные специфичны независимо от того, зафиксированы или откатаны уже выполненные операторы. Таким образом, совместимые приложения, которые используют пакеты и требуют, чтобы они были зафиксированы или откатили в целом, должны выполнять пакеты только в режиме ручного коммита.
