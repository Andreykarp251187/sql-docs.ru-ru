# <a name="identifier-arguments"></a>Аргументы идентификатора
Если строка в аргументе идентификатор заключен в кавычки, драйвер удаляет начальные и завершающие пробелы и рассматривает буквально строка в кавычках. Если строка не заключен в кавычки, драйвер удаляет конечные пробелы и сверток строку в верхний регистр. Значение аргумента идентификатор является пустым указателем возвращает значение SQL_ERROR и SQLSTATE HY009 (недопустимое использование пустого указателя), если аргумент является имя каталога и каталоги не поддерживаются.  
  
 Эти аргументы считаются аргументы идентификатора в том случае, если атрибут инструкции SQL_ATTR_METADATA_ID имеет значение SQL_TRUE. В этом случае символ подчеркивания (_) и процент знак (%) будет рассматриваться как фактический символ, а не как символ шаблона поиска. Эти аргументы рассматриваются как обычные аргумента или аргумента шаблона, в зависимости от аргумента, если этот атрибут имеет значение SQL_FALSE.  
  
 Несмотря на то, что должны быть заключены в кавычки идентификаторы, содержащие специальные символы в инструкциях SQL, они должны не быть заключены в кавычки при передаче в качестве аргументов функции каталога, так как символы кавычек, передаваемый функции работы с каталогами, интерпретируются буквально. Например, предположим, что идентификатор символа кавычек (который специфические для драйвера и возвращаются через **SQLGetInfo**) является знак двойных кавычек ("«). Первый вызов **SQLTables** возвращает результирующий набор, содержащий сведения о таблице Расчеты с поставщиками, а второй вызов возвращает сведения о таблице «Зарплата», который является, вероятно, не предполагалось.  
  
```  
SQLTables(hstmt1, NULL, 0, NULL, 0, "Accounts Payable", SQL_NTS, NULL, 0);  
SQLTables(hstmt2, NULL, 0, NULL, 0, "\"Accounts Payable\"", SQL_NTS, NULL, 0);  
```  
  
 Заключенные в кавычки идентификаторы используются для различения именем true столбца из псевдо столбца с тем же именем, например ROWID в Oracle. Если «ROWID» передается в качестве аргумента для функции каталога, функция будет работать с столбце псевдо ROWID, если он существует. Если псевдо столбец не существует, функция будет работать со столбцом «ROWID». Если ROWID передается в качестве аргумента для функции каталога, функция будет работать со столбцом ROWID.  
  
 Дополнительные сведения о заключенные в кавычки идентификаторы, см. в разделе [идентификаторы в кавычках](../../../odbc/reference/develop-app/quoted-identifiers.md).
