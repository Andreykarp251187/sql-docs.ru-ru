# <a name="identifier-arguments"></a>Аргументы идентификатора
Если строка в аргументе идентификатора приведена, водитель удаляет ведущие и задние заготовки и обрабатывает буквально строку в кавычках. Если строка не указана, драйвер удаляет задние заготовки и складывает строку в верхний регистр. Установка аргумента идентификатора на нулевой указатель возвращает SQL_ERROR и S'LSTATE HY009 (Недействительное использование нулевого указателя), если аргумент является названием каталога и каталоги не поддерживаются.  
  
 Эти аргументы рассматриваются в качестве аргументов идентификатора, если атрибут SQL_ATTR_METADATA_ID оператора установлен на SQL_TRUE. В этом случае подчеркивание (я) и знак процента (%) будет рассматриваться как фактический символ, а не как символ шаблона поиска. Эти аргументы рассматриваются как обычный аргумент или аргумент шаблона, в зависимости от аргумента, если этот атрибут установлен SQL_FALSE.  
  
 Хотя идентификаторы, содержащие специальные символы, должны цитироваться в заявлениях S'L, они не должны цитироваться при переносе в качестве аргументов функции каталога, поскольку символы цитирования, передаваемые функциям каталога, интерпретируются буквально. Например, предположим, что символ цитаты идентификатора (который специфичен для драйвера и возвращается через **S'LGetInfo)** является двойной отметкой цитаты (). Первый вызов в **S'LTables** возвращает набор результатов, содержащий информацию о таблице, поддавшейся оплате счетов, в то время как второй вызов возвращает информацию о таблице "Счета, поддающаяся оплате", что, вероятно, не является тем, что было задумано.  
  
```  
SQLTables(hstmt1, NULL, 0, NULL, 0, "Accounts Payable", SQL_NTS, NULL, 0);  
SQLTables(hstmt2, NULL, 0, NULL, 0, "\"Accounts Payable\"", SQL_NTS, NULL, 0);  
```  
  
 Цитируемые идентификаторы используются для того, чтобы отличить истинное название столбца от псевдоколонки с тем же именем, например ROWID в Oracle. Если "ROWID" передается в аргументе функции каталога, функция будет работать с псевдоколонкой ROWID, если она существует. Если псевдоколонки не существует, функция будет работать с столбцом "ROWID". Если ROWID передается в аргументе функции каталога, функция будет работать с столбцом ROWID.  
  
 Для получения дополнительной информации о цитируемых идентификаторах, [см.](../../../odbc/reference/develop-app/quoted-identifiers.md)
