# <a name="how-is-metadata-used"></a>Как используются метаданные?
Метаданные требуются приложениям для большинства операций с результирующими наборами. Например, приложение использует тип данных столбца, чтобы определить, какую переменную привязывать к этому столбцу. Он использует длину байта столбца символов, чтобы определить, сколько места ему нужно для отображения данных из этого столбца. Способ определения метаданных для столбца зависит от типа приложения.  
  
 Вертикальные приложения работают с предопределенными таблицами и выполняют предопределенные операции на этих таблицах. Поскольку метаданные набора результатов для таких приложений определяются еще до того, как приложение будет написано и контролируется разработчиком приложения, оно может быть жестко закодировано в приложении. Например, если столбец идентификатора заказа определен в источник данных как 4-байтовое целое число, приложение может всегда привязывать 4-байтовые целые числа к этому столбцу. Если метаданные жестко запрограммированы в приложении, изменения в используемых приложением таблицах обычно подразумевают изменения в коде приложения. Это редко проблема, потому что такие изменения, как правило, сделаны как часть нового выпуска приложения.  
  
 Как и вертикальные приложения, пользовательские приложения обычно работают с предопределенными таблицами и выполняют предопределенные операции на этих таблицах. Например, может быть написано приложение для передачи данных между тремя различными источниками данных; данные, которые будут переданы, как правило, известны, когда приложение написано. Таким образом, пользовательские приложения также имеют жестко закодированные метаданные.  
  
 Общие приложения, особенно те, которые поддерживают специальные запросы, почти никогда не знают метаданных создаваемых ими наборов результатов. Таким образом, они должны обнаружить метаданные во время выполнения, используя функции **S'LNumResultCols**, **S'LDescribeCol**, и **S'LColAttribute**, которые описаны в следующем разделе, [S'LDescribeCol и S'LColAttribute](../../../odbc/reference/develop-app/sqldescribecol-and-sqlcolattribute.md).  
  
 Все приложения, независимо от их типа, могут жестко кодировать метаданные для наборов результатов, возвращенных функциями каталога. Эти наборы результатов определены в справочном разделе этого руководства.
