# <a name="transactions-odbc"></a>Транзакции ODBC
*Транзакция* — это единица работы, которая выполняется как единая, атомная операция; то есть, операция удаётся или терпит неудачу в целом. Например, рассмотреть возможность перевода денег с одного банковского счета на другой. Это включает в себя два этапа: снятие денег с первого счета и внесение их на второй. Важно, чтобы оба шага увенчались успехом; недопустимо, чтобы один шаг был успешным, а другой потерпел неудачу. База данных, поддерживающая транзакции, может гарантировать это.  
  
 Транзакции могут быть завершены либо *путем совершения* или *отката.* При совершении транзакции изменения, внесенные в эту транзакцию, становятся постоянными. При откате транзакции затронутые строки возвращаются в состояние, в неходечерезом до начала транзакции. Для расширения примера перевода учетной записи приложение выполняет одну выписку s'L для дебетового первого счета и другую выписку s'L для зачисления второго счета. Если обе операторы удались, приложение совершает транзакцию. Но если любое из этих заявлений по какой-либо причине не удается, приложение отвернет транзакцию. В любом случае приложение гарантирует согласованное состояние в конце транзакции.  
  
 Одна транзакция может включать в себя несколько операций базы данных, которые происходят в разное время. Если другие транзакции имеют полный доступ к промежуточным результатам, транзакции могут вмешиваться друг в друга. Например, предположим, что одна транзакция вставляет строку, вторая транзакция считывает эту строку, а первая транзакция откатывается. Вторая транзакция теперь имеет данные для строки, которая не существует.  
  
 Чтобы решить эту проблему, существуют различные схемы, чтобы изолировать транзакции друг от друга. *Изоляция транзакций* обычно реализуется путем блокировки строк, что исключает использование одной и той же строки в одно и то же время. В некоторых базах данных блокировка строки может также блокировать другие строки.  
  
 С увеличением изоляции транзакций происходит снижение *параллелизма,* или возможность двух транзакций использовать одни и те же данные в то же время. Для получения дополнительной информации [см.](../../../odbc/reference/develop-app/setting-the-transaction-isolation-level.md)
