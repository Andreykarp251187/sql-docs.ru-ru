# <a name="retrieving-bookmarks"></a>Извлечение закладок
Если приложение будет использовать закладки, то перед тем как подготавливать или выполнять инструкцию, необходимо задать атрибуту инструкции SQL_ATTR_USE_BOOKMARKS значение SQL_UB_VARIABLE. Это необходимо потому, что создание и обслуживание закладок может оказаться дорогостоящей операцией, поэтому закладки должны быть включены только в том случае, если приложение может их использовать.  
  
 Закладки возвращаются в виде столбца 0 результирующего набора. Существует три способа их извлечения приложением:  
  
-   Привязать столбец 0 к результирующему набору. **SQLFetch** или **SQLFetchScroll** возвращает закладки для каждой строки в наборе строк вместе с данными для других связанных столбцов.  
  
-   Вызовите функцию **SQLSetPos** , чтобы поместить строку в набор строк, а затем вызовите **SQLGetData** для столбца 0. Если драйвер поддерживает закладки, он всегда должен поддерживать возможность вызова **SQLGetData** для столбца 0, даже если он не позволяет приложениям вызывать **SQLGetData** для других столбцов до последнего привязанного столбца.  
  
-   Вызовите **SQLBulkOperations** , указав для аргумента *операции* значение SQL_ADD и привязанный столбец 0. Курсор вставляет строку и возвращает закладку для строки в связанном буфере.
