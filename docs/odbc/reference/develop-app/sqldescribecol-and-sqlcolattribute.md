# <a name="sqldescribecol-and-sqlcolattribute"></a>SQLDescribeCol и SQLColAttribute
Для извлечения метаданных набора результатов используются **s-LDescribeCol** и **S'LColAttribute.** Разница между этими двумя функциями заключается в том, что **S'LDescribeCol** всегда возвращает одни и те же пять фрагментов информации (имя столбца, тип данных, точность, масштаб и необоснованность), в то время как **S'LColAttribute** возвращает одну часть информации, запрошенной приложением. Тем не менее, **S'LColAttribute** может вернуть гораздо более богатый выбор метаданных, включая чувствительность к корпусу столбца, размер дисплея, повышенную доступность и доступность поиска.  
  
 Многие приложения, особенно те, которые отображают только данные, требуют только метаданных, возвращенных **S'LDescribeCol**. Для этих приложений, это быстрее использовать **S'LDescribeCol,** чем **S'LColAttribute,** потому что информация возвращается в одном вызове. Другие приложения, особенно те, которые обновляют данные, требуют дополнительных метаданных, возвращенных **S'LColAttribute,** и поэтому используют обе функции. Кроме того, **s'LColAttribute** поддерживает метаданные, специфичные для драйверов; для получения дополнительной информации [см.](../../../odbc/reference/develop-app/driver-specific-data-types-descriptor-information-diagnostic.md)  
  
 Приложение может получить метаданные набора результатов в любое время после того, как заявление было подготовлено или выполнено, и до закрытия курсора набора результатов. Очень немногие приложения требуют метаданных набора результатов после подготовки оператора и до его выполнения. Если это возможно, приложения должны ждать, чтобы получить метаданные до тех пор, пока оператор не будет выполнен, потому что некоторые источники данных не могут вернуть метаданные для подготовленных инструкций и эмуляции этой возможности в драйвере часто является медленным процессом. Например, драйвер может создать результат нулевого ряда, установленный, заменив пункт **WHERE** оператора **SELECT** на пункт WHERE 1 **no 2** и выняв итоговую выписку.  
  
 Метаданные часто являются дорогостоящими для извлечения из источника данных. Из-за этого драйверы должны кэшировать любые метаданные, которые они извлекают с сервера, и удерживать их до тех пор, пока курсор над набором результатов открыт. Кроме того, приложения должны запрашивать только те метаданные, которые им абсолютно необходимы.
