# <a name="diagnostic-handling-rules"></a>Правила диагностической обработки
Следующие правила регулируют диагностическую обработку в **S'LGetDiagRec** и **S'LGetDiagField.**  
  
 Для всех компонентов ODBC:  
  
-   Не следует заменять, изменять или маскировать ошибки или предупреждения, полученные от другого компонента ODBC.  
  
-   Может добавить дополнительную запись состояния, когда они получают диагностическое сообщение от другого компонента ODBC. Добавленная запись должна добавить реальную информационную ценность исходного сообщения.  
  
 Для компонента ODBC, который непосредственно взаимосвязан с источником данных:  
  
-   Необходимо префиксировать идентификатор поставщика, идентификатор компонента и идентификатор источника данных к диагностическому сообщению, получаемому из источника данных.  
  
-   Необходимо сохранить родной код ошибки источника данных.  
  
-   Необходимо сохранить диагностическое сообщение источника данных.  
  
 Для любого компонента ODBC, генерируемого ошибкой или предупреждением независимо от источника данных:  
  
-   Необходимо предоставить правильный S'LSTATE для ошибки или предупреждения.  
  
-   Необходимо создать текст диагностического сообщения.  
  
-   Необходимо прикрепить идентификатор поставщика и идентификатор компонента к диагностическому сообщению.  
  
-   Необходимо вернуть родной код ошибки, если он доступен и значимый.  
  
 Для компонента ODBC, который взаимодействует с менеджером драйвера:  
  
-   Необходимо инициализировать аргументы вывода **S'LGetDiagRec** и **S'LGetDiagField**.  
  
-   Необходимо форматировать и возвращать диагностическую информацию в качестве выходных аргументов в качестве аргументов в области вывода **данных s'LGetDiagRec** и **S'LGetDiagField,** когда эта функция называется.  
  
 Для одного компонента ODBC, кроме менеджера драйвера:  
  
-   Необходимо установить S'LSTATE на основе родной ошибки. Для драйверов на основе файлов и драйверов dBMS, которые не используют шлюз, драйвер должен установить S'LSTATE. Для драйверов на основе DBMS, которые используют шлюз, драйвер или шлюз, поддерживающий ODBC, может установить S'LSTATE.
