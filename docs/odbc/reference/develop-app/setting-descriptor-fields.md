# <a name="setting-descriptor-fields"></a>Установка полей дескриптора
Чтобы изменить поля дескриптора, приложение может вызвать **SQLSetDescField**. Некоторые поля доступны только для чтения и не могут быть заданы. (См. Описание функции [SQLSetDescField](../../../odbc/reference/syntax/sqlsetdescfield-function.md) .)  
  
 Поля записи дескрипторов задаются номером записи (*рекнумбер*), равным 1 или выше, тогда как поля заголовка дескриптора задаются с номером записи 0. Номер записи 0 также используется для задания полей закладок в соответствии с соглашением, которое содержит закладки в столбце 0. Это может оставить впечатление, что поля закладок содержатся в заголовке дескриптора, но это не так. Поля закладок отличаются от полей заголовков.  
  
 При задании полей по отдельности приложение должно следовать последовательности, определенной в [SQLSetDescField](../../../odbc/reference/syntax/sqlsetdescfield-function.md). Задание некоторых полей приводит к тому, что драйвер устанавливает другие поля. Это гарантирует, что дескриптор всегда будет готов к использованию после того, как приложение указало тип данных. Когда приложение задает поле SQL_DESC_TYPE, драйвер проверяет, являются ли другие поля, указывающие тип, допустимыми и согласованными.  
  
 Если вызов функции, который бы установил поле дескриптора, завершается ошибкой, содержимое поля дескриптора не определяется после вызова невыполненной функции.
