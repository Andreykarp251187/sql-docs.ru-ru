# <a name="setting-descriptor-fields"></a>Установка полей дескриптора
Чтобы изменить поля дескриптора, приложение может вызвать **S'LSetDescField**. Некоторые поля читаются только и не могут быть установлены. (См. описание функции [S'LSetDescField.)](../../../odbc/reference/syntax/sqlsetdescfield-function.md)  
  
 Поле записи дескриптора устанавливается с рекордным числом (*RecNumber*) 1 или выше, в то время как поля заголовка дескриптора установлены с рекордным числом 0. Рекордное число 0 также используется для установки закладок поля, в соответствии с конвенцией, что закладки, содержащиеся в колонке 0. Это может создать впечатление, что поля закладок содержатся в заголовке дескриптора, но это не так. Поля закладок отличаются от полей заголовка.  
  
 При настройке полей по отдельности приложение должно следовать последовательности, определенной в [S'LSetDescField.](../../../odbc/reference/syntax/sqlsetdescfield-function.md) Установка некоторых полей приводит к тому, что драйвер устанавливает другие поля. Это гарантирует, что дескриптор всегда готов к использованию после того, как приложение учтет тип данных. Когда приложение устанавливает поле SQL_DESC_TYPE, драйвер проверяет, что другие поля, определяющие тип, являются действительными и последовательными.  
  
 Если вызов функции, который установил поле дескриптора, выходит из строя, содержимое поля дескриптора не определено после неудачного вызова функции.
