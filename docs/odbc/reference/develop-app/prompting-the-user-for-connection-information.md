# <a name="prompting-the-user-for-connection-information"></a>Запрос сведений о подключении у пользователя
Если приложение использует **S'LConnect** и должно подсказать пользователю любую информацию о подключении, такую как имя пользователя и пароль, оно должно сделать это самостоятельно. Хотя это позволяет приложению управлять своим "взглядом и чувствовать", это может заставить приложение содержать код конкретного драйвера. Это происходит, когда приложение должно подсказать пользователю информацию о подключении к конкретному водителю. Это создает невозможную ситуацию для генерических приложений, которые предназначены для работы с любыми драйверами, включая драйверы, которые не существуют при написании приложения.  
  
 **SLDriverConnect** может подсказать пользователю информацию о подключении. Например, упомянутая ранее пользовательская программа может передать следующую строку подключения к **S'LDriverConnect:**  
  
```  
DSN=XYZ Corp;  
```  
  
 Затем драйвер может отображать диалоговую коробку, которая подсказывает идентионные данные пользователей и пароли, по аналогии со следующей иллюстрацией.  
  
 ![Диалоговое окно с запросом идентификаторов и паролей пользователей](../../../odbc/reference/develop-app/media/pr18.gif "pr18")  
  
 То, что драйвер может подсказать информацию о подключении, особенно полезно для общих и вертикальных приложений. Эти приложения не должны содержать информацию о конкретной драйвере, и запрос драйвера для получения необходимой информации не позволяет этой информации выйти из приложения. Об этом свидетельствуют два предыдущих примера. Когда приложение передало водителю только имя источника данных, приложение не содержало никакой конкретной информации водителя и поэтому не было привязано к конкретному драйверу. Когда приложение передало полную строку соединения с драйвером, оно было привязано к драйверу, который мог интерпретировать эту строку.  
  
 Общее приложение может сделать еще один шаг вперед и даже не указать источник данных. Когда **S'LDriverConnect** получает пустую строку соединения, менеджер драйвера отображает следующую коробку диалога.  
  
 ![Диалоговое окно выбора источника данных](../../../odbc/reference/develop-app/media/ch06a.gif "CH06A")  
  
 После того, как пользователь выбирает источник данных, менеджер драйвера строит строку соединения, определяющую этот источник данных, и передает его водителю. Затем водитель может подсказать пользователю любую необходимую ему информацию.  
  
 Условия, при которых водитель подсказывает пользователю, управляются флагом *DriverCompletion;* Есть варианты, чтобы всегда подсказывать, подсказывать, если это необходимо, или никогда не подскажут. Для полного описания этого флага, см. [SQLDriverConnect](../../../odbc/reference/syntax/sqldriverconnect-function.md)
