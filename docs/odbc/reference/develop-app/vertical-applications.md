# <a name="vertical-applications"></a>Вертикальные приложения
По вертикали приложения обычно выполняют четко определенную задачу для одной СУБД. Например, приложение для ввода заказа отслеживает заказы в компании. Общие сведения о таких типах приложений — схема базы данных, которая обычно разработана разработчиком приложения, и, в то время как приложение может работать с несколькими разными СУБД, оно работает с одной СУБД для одного клиента.  
  
 Поскольку для вертикальных приложений обычно требуются определенные функции, например прокручиваемые курсоры или транзакции, они редко поддерживают все СУБД. Вместо этого они, как правило, сильно взаимодействуют между ограниченным набором СУБД. Как правило, разработчики приложений по вертикали выбирают поддержку этих СУБД, представляющих большую часть рынка, и игнорируют остальные. Они могут даже выбрать поддержку конкретных драйверов для этих СУБД, чтобы снизить затраты на тестирование и поддержку продукта.  
  
 Поскольку вертикальные приложения могут поддерживать известный набор СУБД, они иногда содержат код, зависящий от драйвера или СУБД. Однако такой код лучше хранить минимум, так как он требует дополнительного времени для обслуживания.
