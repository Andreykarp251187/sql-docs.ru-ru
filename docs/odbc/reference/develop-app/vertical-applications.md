# <a name="vertical-applications"></a>Вертикальные приложения
Вертикальные приложения обычно выполняют четко определенную задачу против одного DBMS. Например, приложение для ввода заказа отслеживает заказы в компании. Что объединяет эти типы приложений, так это то, что схема базы данных обычно разрабатывается разработчиком приложения, и, хотя приложение может работать с рядом различных DBMS, оно работает с одним DBMS для одного клиента.  
  
 Поскольку вертикальные приложения обычно требуют определенных функциональных возможностей, таких как прокрутки курсоров или транзакций, они редко поддерживают все DBMS. Вместо этого, они, как правило, очень совместимы между ограниченным набором DBMSs. Как правило, разработчики вертикальных приложений предпочитают поддерживать dBMS, которые составляют значительную часть рынка и игнорируют остальные. Они могут даже выбрать для поддержки конкретных драйверов для тех DBMSs, чтобы уменьшить их тестирования и расходов на поддержку продукта.  
  
 Поскольку вертикальные приложения могут поддерживать известный набор DBMS, они иногда содержат код, специфичный для драйверов или DBMS. Тем не менее, такой код лучше всего свести к минимуму, поскольку он требует дополнительного времени для поддержания.
