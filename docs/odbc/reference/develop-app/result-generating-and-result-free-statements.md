# <a name="result-generating-and-result-free-statements"></a>Инструкции, возвращающие и не возвращающие результаты
Инструкции SQL можно разделить на следующие пять категорий слабо:  
  
-   **Результат создания набора инструкций** это инструкции SQL, которые создают результирующий набор. Например **ВЫБЕРИТЕ** инструкции.  
  
-   **Строка генерирующими число инструкций** это инструкции SQL, которые создают количество затронутых строк. Например **обновление** или **удалить** инструкции.  
  
-   **Инструкции определения языка DDL данных** это инструкции SQL, которые изменяют структуру базы данных. Например **CREATE TABLE** или **DROP INDEX**.  
  
-   **Изменение контекста инструкций** это инструкции SQL, изменяющие контекст базы данных. Например **используйте** и **ЗАДАТЬ** инструкций в SQL Server.  
  
-   **Административные инструкции** это инструкции SQL, используемый для административных целей в базе данных. Например **GRANT** и **ОТОЗВАТЬ**.  
  
 В первых двух категорий инструкций SQL, совокупно называются *инструкции, возвращающие*. В последнем три категории инструкций SQL, совокупно называются *возвращающие результаты инструкции*. ODBC определяет семантику пакетов, которые включают только возвращающие инструкции. Эту семантику могут сильно различаться и, следовательно, зависящие от источника данных. Например драйвер SQL Server не поддерживает удаление объекта, а затем ссылка на или повторное создание тот же объект, в том же пакете. Таким образом, термин *пакета* как в данном руководстве относится только к возвращающие пакеты инструкций.
