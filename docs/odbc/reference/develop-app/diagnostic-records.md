# <a name="diagnostic-records"></a>Диагностические записи
Связанные с каждой среды, подключения, инструкции и дескриптора являются *диагностические записи*. Эти записи содержат диагностические сведения о последней вызывается функцией, использовании определенного дескриптора. Записи выполняется замена только в том случае, когда будет вызвана другая функция, с помощью данного дескриптора. Нет ограничений на количество диагностических записей, которые могут храниться в любой момент времени.  
  
 Существует два типа диагностических записей: *запись заголовка* и ноль или более *записей состояния*. Запись заголовка является записи 0; Состояние записи являются записями 1 и более поздних версий. Диагностические записи состоят из нескольких отдельных полей, которые отличаются для заголовочной записи и для записей состояния. Кроме того компоненты ODBC можно определить собственные поля диагностических записей.  
  
 Несмотря на то, что диагностические записи может рассматриваться как структуры, не является обязательным для них должны быть структуры; хранение диагностических сведений драйвер специфические для драйвера.  
  
 Поля диагностических записей, извлекаются с помощью **SQLGetDiagField**. SQLSTATE, номер внутренней ошибки и полями диагностического сообщения записей состояния могут быть получены за один вызов с **SQLGetDiagRec**.
