# <a name="uses-of-catalog-data"></a>Использование данных каталога
Приложения используют данные каталога в различными способами. Ниже приведены некоторые распространенные варианты использования.  
  
-   **Конструирование инструкций SQL во время выполнения.** Вертикальные приложения, такие как приложение для ввода заказов, содержащие жестко запрограммированных инструкциях SQL. Таблицы и столбцы, которые используются приложением исправленных заранее, а представляют собой операторы, доступ к этим таблицам. Например, приложение для ввода заказов обычно содержит один параметризованный **вставить** инструкции по добавлению новых заказов в системе.  
  
     Универсальные приложения, такие как программа электронную таблицу, которая использует ODBC для получения данных, часто создания инструкций SQL во время выполнения на основе входных данных от пользователя. Такое приложение может потребовать пользователю вводить имена таблиц и столбцов для использования. Тем не менее было бы проще для пользователя, если приложение отображается список таблиц и столбцов, из которых пользователь может принять выбранные параметры. Для создания этих списков, приложение будет вызывать **SQLTables** и **SQLColumns** функции работы с каталогами.  
  
-   **Конструирование инструкций SQL во время разработки.** Как правило, сред разработки приложений позволяют программисту для создания запросов к базе данных во время разработки программы. Запросы затем жестко запрограммированы в приложении выполняется построение.  
  
     Такие среды также можно использовать **SQLTables** и **SQLColumns** для создания списков, из которых программист может сделать выбор. Эти среды также могут использовать **SQLPrimaryKeys** и **SQLForeignKeys** автоматически определить и Показать связи между выбранными таблицами и использовать **SQLStatistics** для определения и выделите индексированных полей, чтобы программист могут создавать эффективные запросы.  
  
-   **Создав курсоров.** Использовать приложения, драйвер или по промежуточного слоя, который предоставляет механизм Прокручиваемый курсор **SQLSpecialColumns** для определения, какой столбец или столбцы однозначно идентифицировать строку. Программа может собрать *keyset* содержит значения этих столбцов для каждой строки, которая была получена. Когда приложение выполняет прокрутку назад к строке, в его будет использовать эти значения для получения последних данных для строки. Дополнительные сведения о Прокручиваемые курсоры и наборы ключей, см. в разделе [Прокручиваемые курсоры](../../../odbc/reference/develop-app/scrollable-cursors.md).
