# <a name="parameter-binding-offsets"></a>Смещения привязки параметров
Приложение может указать, что смещение добавляется привязан параметр адреса буферов и соответствующие длины и индикатора адреса буфера, когда **SQLExecDirect** или **SQLExecute** вызывается. Результат этих дополнений определяет адреса, используемые в этих операций.  
  
 Смещения привязки позволяют приложению изменять привязки без вызова **SQLBindParameter** для ранее привязанных параметров. Вызов **SQLBindParameter** выполнять повторную привязку параметра изменяет адрес буфера и указатель длины и индикатора. Повторная привязка со смещением, с другой стороны, просто добавляет смещение к существующей адрес буфера привязанного параметра и адрес буфера длины и индикатора. При использовании смещения привязки — это «шаблон» расположение буферы приложения и приложения можно переместить «шаблон» к различным областям памяти, изменение смещения. Новое смещение должно лежать в любое время и всегда добавляется изначально привязанные значения.  
  
 Чтобы указать смещение привязки, приложение устанавливает атрибут инструкции SQL_ATTR_PARAM_BIND_OFFSET_PTR адрес буфера SQLINTEGER. Прежде чем приложение вызывает функцию, которая использует эти привязки, поскольку адрес буфера длины и индикатора, ни адрес буфера параметра равно 0, а привязанного параметра указывается в инструкции SQL она помещает смещение в байтах в этом буфере. Сумма адрес и смещение должно быть допустимым адресом. (Это означает, что один или оба смещение, адрес, к которому добавляется смещение может быть недействительным, до тех пор, пока их сумма является допустимым адресом).  
  
> [!NOTE]  
>  Смещения привязки не поддерживаются для ODBC 2. *x* драйверы.
