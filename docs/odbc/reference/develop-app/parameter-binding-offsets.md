# <a name="parameter-binding-offsets"></a>Смещения привязки параметров
Приложение может указать, что смещение добавляется к связанным парабировым буферным адресам и соответствующим адресам буфера длины/индикатора при вызове **S'LExecDirect** или **S'LExecute.** Результат этих дополнений определяет адреса, используемые в этих операциях.  
  
 Привязные смещения позволяют приложению изменять привязки без вызова **S'LBindParameter** по ранее связанным параметрам. Призыв к **S'LBindParameter** для повторного вручения параметра изменяет адрес буфера и указатель длины/индикатора. С другой стороны, пересвязание с смещением просто добавляет смещение к существующему связанного буферного адреса параметра и адресу буфера длины/индикатора. При использовании смещений привязки являются "шаблоном" того, как выкладываются буферы приложения, и приложение может переместить этот "шаблон" в различные области памяти, изменив смещение. Новая смещение может быть указана в любое время и всегда добавляется к первоначально связанным значениям.  
  
 Чтобы указать смещение связывания, приложение устанавливает атрибут SQL_ATTR_PARAM_BIND_OFFSET_PTR оператора на адрес буфера S'LINTEGER. Перед вызовом приложения функция, использующая привязки, оно помещает смещение в байтах в этом буфере, если ни адрес буфера параметра, ни адрес буфера длины/индикатора не будет 0, а параметр связан в отчете S'L. Сумма адреса и смещения должны быть действительным адресом. (Это означает, что либо, либо смещение, и адрес, к которому добавляется смещение, могут быть недействительными, если их сумма является действительным адресом.)  
  
> [!NOTE]  
>  Обязательные смещения не поддерживаются ODBC 2. *x* драйверов.
