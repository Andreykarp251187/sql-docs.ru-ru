# <a name="bookmark-types"></a>Типы закладок
Все закладки в ODBC *3.x* являются закладками с переменной длиной. Это позволяет использовать в качестве закладки основной ключ или уникальный индекс, связанный с таблицей. Закладка также может быть 32-битное значение, как это было использовано в ODBC *2.x*. Чтобы указать, что закладка используется с курсором, приложение ODBC *3.x* устанавливает атрибут SQL_ATTR_USE_BOOKMARK оператора на SQL_UB_VARIABLE. Автоматики переменной длины.  
  
 Приложение может вызвать **S'LColAttribute** с *аргументом FieldIdentifier,* установленным для SQL_DESC_OCTET_LENGTH для получения длины закладки. Поскольку закладка с переменной длиной может быть длинным значением, приложение не должно связываться с столбецом 0, если оно не будет использовать закладку для многих строк в строке.  
  
 Закладки с фиксированной длиной поддерживаются только для обратной совместимости. Если приложение ODBC *2.x,* работая с драйвером ODBC *3.x,* вызывает **s'LSetStmtOption,** чтобы установить SQL_USE_BOOKMARKS для SQL_UB_ON, оно отображается в менеджере драйвера для SQL_UB_VARIABLE. Используется закладка с переменной длиной, даже если ее заполнено только 32 бита. Если водитель поддерживает закладки с фиксированной длиной, он будет поддерживать закладки с переменной длиной. Если приложение ODBC *3.x,* работая с драйвером ODBC *2.x,* вызывает **S'LSetStmtAttr,** чтобы установить SQL_ATTR_USE_BOOKMARKS SQL_UB_VARIABLE, оно отображается в менеджере драйвера для SQL_UB_ON и используется 32-битная закладка с фиксированной длиной. Атрибут SQL_ATTR_FETCH_BOOKMARK_PTR оператора должен указывать на 32-битную закладку. Если используемые закладки длиннее 32 бита, например, когда первичные клавиши используются в качестве закладок, курсор должен сопоставить фактические значения до 32-битных значений. Он может, например, построить хэш-таблицу из них. Когда приложение ODBC *3.x,* работая с драйвером ODBC *2.x,* связывает закладку, длина буфера должна быть 4.
