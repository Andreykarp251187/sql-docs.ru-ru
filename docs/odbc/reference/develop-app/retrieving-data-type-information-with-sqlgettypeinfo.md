# <a name="retrieving-data-type-information-with-sqlgettypeinfo"></a>Получение сведений о типе данных с помощью SQLGetTypeInfo
Поскольку отображение от базовых типов данных S'L к идентификаторам типа ODBC являются приблизительными, ODBC предоставляет функцию **(S'LGetTypeInfo),** с помощью которой драйвер может полностью описать каждый тип данных в источнике данных. Эта функция возвращает набор результатов, каждая строка которой описывает характеристики одного типа данных, такие как имя, идентификатор типа, точность, масштаб и необыченность.  
  
 Эта информация обычно используется общими приложениями, которые позволяют пользователю создавать и изменять таблицы. Такие приложения вызывают **s'LGetTypeInfo,** чтобы получить информацию о типе данных, а затем представить некоторые или все из них пользователю. Такие приложения должны быть осведомлены о двух вещах:  
  
-   Более одного типа данных S'L может отображаться на идентификаторе одного типа, что может затруднить определение типа данных. Чтобы решить эту проблему, набор результатов упорядочен сначала по идентификатору типа, а второй по близости к определению идентификатора типа. Кроме того, типы данных, определяемые исходным источником, имеют приоритет над типами данных, определяемых пользователем. Например, предположим, что источник данных определяет типы данных INTEGER и COUNTER как одинаковые, за исключением того, что COUNTER является автоматическим увеличением. Предположим также, что пользовательский тип WHOLENUM является синонимом INTEGER. Каждый из этих типов карты, чтобы SQL_INTEGER. В наборе результатов **в комплекте с s'LGetTypeInfo** появляется СНАЧАЛА INTEGER, за ним следует WHOLENUM, а затем COUNTER. WHOLENUM появляется после INTEGER, потому что он определяется пользователем, но до COUNTER, потому что он более тесно соответствует определению идентификатора типа SQL_INTEGER.  
  
-   ODBC не определяет имена типов данных для использования в заявлениях **CREATE TABLE** и **ALTER TABLE.** Вместо этого приложение должно использовать имя, возвращенное в TYPE_NAME столбец набора результатов, возвращенного **S'LGetTypeInfo.** Причина этого заключается в том, что, хотя большинство из S'L не сильно различаются по DBMSs, имена типов данных сильно различаются. Вместо того, чтобы заставлять драйверы анализировать операторы S'L и заменять стандартные имена типов данных именами типов данных, относясь к DBMS, ODBC требует, чтобы приложения использовали имена, в частности DBMS.  
  
 Обратите внимание, что **sLGetTypeInfo** не обязательно описывает все типы данных, с которыми может столкнуться приложение. В частности, наборы результатов могут содержать типы данных, не поддерживаемые непосредственно источником данных. Например, типы данных столбцов в наборах результатов, возвращаювых функциями каталога, определяются ODBC, и эти типы данных могут не поддерживаться источником данных. Для определения характеристик типов данных в наборе результатов приложение вызывает **S'LColAttribute**.
