# <a name="step-4a-fetch-the-results"></a>Шаг 4а. Выборка результатов
Следующим шагом является получение результатов, как показано на следующей иллюстрации.  
  
 ![Выборка результатов в приложении ODBC](../../../odbc/reference/develop-app/media/pr14.gif "pr14")  
  
 Если выписка, выполненная в "Шаг 3: Построить и выполнить выписку по S'L" была выпиской **SELECT** или функцией каталога, приложение сначала вызывает **S'LNumResultCols,** чтобы определить количество столбцов в наборе результатов. Этот шаг не является необходимым, если приложение уже знает количество столбцов набора результатов, например, когда выписка по S'L жестко закодирована в вертикальном или пользовательском приложении.  
  
 Далее приложение получает имя, тип данных, точность и масштаб каждого набора результата с **помощью S'LDescribeCol.** Опять же, это не обязательно для приложений, таких как вертикальные и пользовательские приложения, которые уже знают эту информацию. Приложение передает эту информацию в **S'LBindCol,** который связывает переменную приложения с столбецом в наборе результатов.  
  
 В настоящее время приложение вызывает **s'LFetch** для получения первого ряда данных и размещения данных из этой строки в переменных, связанных с **S'LBindCol.** Если в строке есть какие-либо длинные данные, то для получения этих данных он вызывает **данные.** Приложение продолжает вызывать **S'LFetch** и **S'LGetData** для получения дополнительных данных. После того, как он закончил извлечение данных, он вызывает **S'LCloseCursor,** чтобы закрыть курсор.  
  
 Для полного описания результатов получения [см.](../../../odbc/reference/develop-app/retrieving-results-basic.md) [Retrieving Results (Advanced)](../../../odbc/reference/develop-app/retrieving-results-advanced.md)  
  
 Приложение теперь возвращается в "Шаг 3: Построить и выполнить заявление S'L" для выполнения другого оператора в той же транзакции; или переходит к "Шаг 5: Совершить транзакцию", чтобы совершить или откатить транзакцию.
