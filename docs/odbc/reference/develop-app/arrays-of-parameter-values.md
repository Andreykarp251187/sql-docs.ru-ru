# <a name="arrays-of-parameter-values"></a>Массивы значений параметров
Часто бывает полезно, чтобы приложения передавали массивы параметров. Например, при использовании массивов параметров и параметризованной инструкции **INSERT** приложение может вставить сразу несколько строк. Использование массивов имеет несколько преимуществ. Во-первых, сетевой трафик сокращается, так как данные для многих инструкций отправляются в одном пакете (если источник данных поддерживает массивы параметров в собственном режиме). Во вторых, некоторые источники данных могут выполнять инструкции SQL с использованием массивов быстрее, чем при выполнении одинакового количества отдельных инструкций SQL. Наконец, если данные хранятся в массиве, как это часто бывает в случае с данными экрана, приложение может привязать все строки в определенном столбце с помощью одного вызова **SQLBindParameter** и обновить их, выполнив одну инструкцию.  
  
 К сожалению, не многие источники данных поддерживают массивы параметров. Однако драйвер может эмулировать массивы параметров, выполняя инструкцию SQL один раз для каждого набора значений параметров. Это может привести к увеличению скорости, поскольку драйвер может подготовить инструкцию, которую планируется выполнить один раз для каждого набора параметров. Это также может привести к упрощению кода приложения.
