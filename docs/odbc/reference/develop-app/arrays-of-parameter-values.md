# <a name="arrays-of-parameter-values"></a>Массивы значений параметров
Часто приложениям полезно передавать массивы параметров. Например, используя массивы параметров и параметризированную выписку **INSERT,** приложение может вставить сразу несколько строк. Использование массивов имеет несколько преимуществ. Во-первых, сетевой трафик сокращается, поскольку данные по многим инструкциям отправляются в один пакет (если источник данных поддерживает массивы параметров на месте). Во-вторых, некоторые источники данных могут выполнять операторы S'L с помощью массивов быстрее, чем выполнение такого же количества отдельных инструкций S'L. Наконец, когда данные хранятся в массиве, как это часто бывает в случае с данными экрана, приложение может связать все строки в определенном столбце одним вызовом в **S'LBindParameter** и обновить их, выполнив одно утверждение.  
  
 К сожалению, не многие источники данных поддерживают параметры массивов. Тем не менее, драйвер может эмулировать параметры массивов, выполнив заявление S'L один раз для каждого набора значений параметров. Это может привести к увеличению скорости, поскольку водитель может подготовить заявление, которое он планирует выполнить один раз для каждого набора параметров. Это также может привести к упрощению кода приложения.
