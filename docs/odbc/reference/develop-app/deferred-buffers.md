# <a name="deferred-buffers"></a>Отложенные буферы
*Отложенный буфер* — это буфер, значение которого используется в какое-то время *после того,* как оно указано в вызове функции. Например, для ассоциированного или *связывания* буфера данных с параметром в выписке **S'LBindParameter** используется для ассоциировать или связывать его с параметром. Приложение определяет количество параметра и передает адрес, длину байта и тип буфера. Водитель сохраняет эту информацию, но не исследует содержимое буфера. Позже, когда приложение выполняет выписку, драйвер получает информацию и использует ее для извлечения данных параметра и отправки ее в источник данных. Таким образом, ввод данных в буфер откладывается. Поскольку отложенные буферы указаны в одной функции и используются в другой, это ошибка программирования приложений, чтобы освободить отложенный буфер, в то время как драйвер все еще ожидает его существования; для получения дополнительной информации, [см. Распределение и освобождение буферов](../../../odbc/reference/develop-app/allocating-and-freeing-buffers.md), позже в этом разделе.  
  
 Буферы ввода и вывода могут быть отложены. В следующей таблице кратко излагается использование отложенных буферов. Обратите внимание, что отложенные буферы, привязанные к столбцам набора результатов, указаны с **помощью S'LBindCol,** а отложенные буферы, привязанные к параметрам оператора S'LL, указаны с **s'LBindParameter.**  
  
|Использование буфера|Тип|Указано с|Где используется|  
|----------------|----------|--------------------|-------------|  
|Отправка данных по вхотозатам|Отложенный ввод|**SQLBindParameter**|**SQLExecute**<br /> **SQLExecDirect**|  
|Отправка данных для обновления или вставки строки в набор результатов|Отложенный ввод|**SQLBindCol**|**функция SQLSetPos;**|  
|Возвращение данных по параметрам вывода и ввода/вывода|Отложенный выход|**SQLBindParameter**|**SQLExecute**<br /> **SQLExecDirect**|  
|Возвращение данных набора результатов|Отложенный выход|**SQLBindCol**|**SQLFetch**<br /> **СЗЛУТСПровотС SLSetPos**|
