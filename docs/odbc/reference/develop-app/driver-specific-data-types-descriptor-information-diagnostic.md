# <a name="driver-specific-data-types-descriptor-types-information-types-diagnostic-types-and-attributes"></a>Специальные типы данных для драйвера, типы дескрипторов, типы сведений, диагностические типы и атрибуты
Драйверы могут распределить значения, связанные с драйвером, на следующие:  
  
-   **Индикаторы типа данных S'L** Они используются в *ParameterTypeВ* в **S'LBindParameter** и в *DataType* в **S'LGetTypeInfo** и возвращены **S'LColAttribute**, **S'LКолонки**, **S'LDescribeCol**, **S'LGetTypeInfo**, **S'L-ОполистаПарам**, **S'LProcedureColumns**, и **S'LSpecialColumns**.  
  
-   **Дескриптор поля** Они используются в *полевых идентификаторах* в **S'LColAttribute,** **S'LGetDescField**и **S'LSetDescField.**  
  
-   **Диагностические поля** Они используются в *ДиагИдентор* в **S'LGetDiagField** и **S'LGetDiagRec**.  
  
-   **Типы информации** Они используются в *InfoType* в **S'LGetInfo**.  
  
-   **Атрибуты подключения и выписки** Они используются в *attribute* в **S'LGetConnectAttr**, **S'LGetStmtAttr**, **S'LSetConnectAttr**, и **S'LSetStmtAttr**.  
  
 Для каждого из этих элементов существует два набора значений: значения, зарезервированные для использования ODBC, и значения, зарезервированные для использования драйверами. Перед реализацией значений, связанных с драйвером, автор драйвера должен запросить значение для каждого конкретного типа, поля или атрибута драйвера из Open Group. Для разработки новых драйверов используйте диапазон, описанный в таблице ниже. Менеджер драйвера ODBC 3.8 не генерирует ошибку, если используется неизвестное значение, которое не находится в диапазоне, описанном ниже. Однако более поздние версии менеджера драйверов могут генерировать ошибку, если получены неизвестные значения, которые не находятся в диапазоне.  
  
 При перенаправлении любого из этих значений в функцию ODBC водитель должен проверить, является ли значение действительным. Драйверы возвращают S'LSTATE HYC00 (необязательная функция не реализована) для значений, относящихся к другим драйверам.  
  
 Начиная с ODBC 3.8, авторы драйверов могут распределять атрибуты, характерные для драйверов, в пределах зарезервированного диапазона.  
  
> [!NOTE]  
>  Менеджер драйвера ODBC 3.8 не проверяет и не применяет эти диапазоны для обратной совместимости. Однако будущая версия менеджера драйверов может обеспечить их соблюдение.  
  
|Тип атрибута|Тип данных ODBC|База диапазона, специфичная для водителя|Ограничение диапазона, конкретного водителя|Константа ODBC для базы диапазона значений конкретного значения для водителя|  
|--------------------|--------------------|---------------------------------|----------------------------------|---------------------------------------------------------|  
|Индикаторы типа данных S'L|SQLSMALLINT|0x4000|0x7FFF|SQL_DRIVER_SQL_TYPE_BASE|  
|Описатели поля|SQLSMALLINT|0x4000|0x7FFF|SQL_DRIVER_DESCRIPTOR_BASE|  
|Диагностические поля|SQLSMALLINT|0x4000|0x7FFF|SQL_DRIVER_DIAGNOSTIC_BASE|  
|Типы информации|СЛЮМЛМИНТ|0x4000|0x7FFF|SQL_DRIVER_INFO_TYPE_BASE|  
|Атрибуты подключения|SQLINTEGER|0x00004000|0x00007FFF|SQL_DRIVER_CONNECT_ATTR_BASE|  
|Атрибуты оператора|SQLINTEGER|0x00004000|0x00007FFF|SQL_DRIVER_STATEMENT_ATTR_BASE|  
  
> [!NOTE]  
>  Типы данных, специфические для драйвера, поля дескриптора, диагностические поля, типы информации, атрибуты оператора и атрибуты соединения должны быть описаны в документации драйвера. При перенаправлении любого из этих значений в функцию ODBC водитель должен проверить, является ли значение действительным. Драйверы возвращают S'LSTATE HYC00 (необязательная функция не реализована) для значений, относящихся к другим драйверам.  
  
 Базовые значения определяются для облегчения развития драйвера. Например, конкретные диагностические атрибуты драйвера могут быть определены в следующем формате:  
  
```  
SQL_DRIVER_DIAGNOSTIC_BASE+0, SQL_DRIVER_DIAGNOSTIC_BASE +1  
```
