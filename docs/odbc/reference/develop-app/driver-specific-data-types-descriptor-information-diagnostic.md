# <a name="driver-specific-data-types-descriptor-types-information-types-diagnostic-types-and-attributes"></a>Специальные типы данных для драйвера, типы дескрипторов, типы сведений, диагностические типы и атрибуты
Драйверы можно выделить значения специфические для драйвера в следующих целях:  
  
-   **Индикаторы типов данных SQL** они используются в *ParameterType* в **SQLBindParameter** и в *DataType* в **SQLGetTypeInfo** и возвращенный **SQLColAttribute**, **SQLColumns**, **SQLDescribeCol**, **SQLGetTypeInfo**,  **SQLDescribeParam**, **SQLProcedureColumns**, и **SQLSpecialColumns**.  
  
-   **Поля дескриптора** они используются в *FieldIdentifier* в **SQLColAttribute**, **SQLGetDescField**, и **SQLSetDescField**.  
  
-   **Диагностические поля** они используются в *DiagIdentifier* в **SQLGetDiagField** и **SQLGetDiagRec**.  
  
-   **Типы сведений** они используются в *InfoType* в **SQLGetInfo**.  
  
-   **Инструкция атрибутов соединения и** они используются в *атрибут* в **SQLGetConnectAttr**, **SQLGetStmtAttr**,  **SQLSetConnectAttr**, и **SQLSetStmtAttr**.  
  
 Для каждого из этих элементов, есть два набора значений: значения, зарезервированные для использования ODBC, а значения, зарезервировано для использования с драйверами. Перед реализацией значений, зависящих от драйвера, разработчику драйверов необходимо запросить значение для каждого типа, зависящего от драйвера, поля или атрибута из Open Group. Для разработки новых приложений драйвера используйте диапазон, описанные в следующей таблице. Диспетчер драйверов ODBC 3.8 не приведет к ошибке, если используется неизвестного значения, который не входит в диапазон, описанных ниже. Тем не менее более поздних версиях диспетчера драйверов может возникнуть ошибка, если неизвестные значения предоставляются, не входящие в диапазон.  
  
 При любой из этих значений передается функции ODBC, драйвер необходимо проверить, является ли значение допустимым. Драйверы возвращают SQLSTATE HYC00 (дополнительная возможность не реализована) для значений, зависящих от драйвера, которые применяются для других драйверов.  
  
 Начиная с ODBC 3.8, драйвер записи можно выделить атрибутов драйвера в зарезервированный диапазон.  
  
> [!NOTE]  
>  Диспетчер драйверов ODBC 3.8 не проверяет и не применяет эти диапазоны для обеспечения обратной совместимости. Будущие версии диспетчера драйверов может применять их, тем не менее.  
  
|Тип атрибута|Тип данных ODBC|Базовый диапазон специфические для драйвера|Размер диапазона специфические для драйвера|Константа ODBC для драйвера значение диапазона базовый|  
|--------------------|--------------------|---------------------------------|----------------------------------|---------------------------------------------------------|  
|Индикаторы типов данных SQL|SQLSMALLINT|0x4000|0x7FFF|SQL_DRIVER_SQL_TYPE_BASE|  
|Поля дескриптора|SQLSMALLINT|0x4000|0x7FFF|SQL_DRIVER_DESCRIPTOR_BASE|  
|Диагностические поля|SQLSMALLINT|0x4000|0x7FFF|SQL_DRIVER_DIAGNOSTIC_BASE|  
|Типы сведений|SQLUSMALLINT|0x4000|0x7FFF|SQL_DRIVER_INFO_TYPE_BASE|  
|Атрибуты соединения|SQLINTEGER|0x00004000|0x00007FFF|SQL_DRIVER_CONNECT_ATTR_BASE|  
|Атрибуты инструкции|SQLINTEGER|0x00004000|0x00007FFF|SQL_DRIVER_STATEMENT_ATTR_BASE|  
  
> [!NOTE]  
>  Типы данных, поля дескриптора, диагностические поля, типы информации, атрибуты инструкций и атрибуты соединения должны быть описаны в документации по драйверу. При любой из этих значений передается функции ODBC, драйвер необходимо проверить, является ли значение допустимым. Драйверы возвращают SQLSTATE HYC00 (дополнительная возможность не реализована) для значений, зависящих от драйвера, которые применяются для других драйверов.  
  
 Базовые значения определены для упрощения разработки драйвера. Например можно определить определенные атрибуты диагностики драйвера в следующем формате:  
  
```  
SQL_DRIVER_DIAGNOSTIC_BASE+0, SQL_DRIVER_DIAGNOSTIC_BASE +1  
```
