# <a name="trace-file"></a>Файл трассировки
Приложение определяет файл трассировки либо путем установки ключевого слова **TraceFile** в регистрационном входе Odbc.ini, либо по телефону **s'LSetConnectAttr** с SQL_ATTR_TRACEFILE атрибутом соединения. Если файл не существует при включении трассировки, диспетчер драйвера создаст файл. Каждое приложение должно иметь свой собственный файл выделенного следа, чтобы избежать раздора. Приложение может использовать несколько файлов трассировки; программа настройки приложения может предоставить пользователю выбор файлов трассировки. Если трассировка включена динамически, приложение может также отображать результаты трассировки, а не регистрироваться в файл трассировки.  
  
 Файл трассировки предоставляет журнал каждого вызова функции ODBC с типами данных и значениями всех аргументов. Он регистрирует все функции ввода и регистрирует все возвращенные функции с кодами возврата и состояниями ошибок.  
  
 В ODBC *3.x*параметры функций соединения не предусмотрены для трассировки DLL.
