# <a name="trace-file"></a>Файл трассировки
Приложение указывает файл трассировки, задавая ключевое слово **TraceFile** в записи реестра Odbc.ini или вызывая **SQLSetConnectAttr** с атрибутом соединения SQL_ATTR_TRACEFILE. Если файл не существует при включенной трассировке, диспетчер драйверов создаст файл. Каждое приложение должно иметь собственный выделенный файл трассировки, чтобы избежать конкуренции. Приложение может использовать более одного файла трассировки. Программа установки приложения может предоставить пользователю возможность выбора файлов трассировки. Если трассировка включена динамически, приложение может также отображать результаты трассировки, а не выполнять запись в файл трассировки.  
  
 Файл трассировки содержит журнал каждого вызова функции ODBC с типами данных и значениями всех аргументов. Он регистрирует все входные функции и регистрирует все возвращаемые функции с кодами возврата и состояниями ошибок.  
  
 В ODBC *3. x*параметры функций подключения не предоставляются библиотеке DLL трассировки.
