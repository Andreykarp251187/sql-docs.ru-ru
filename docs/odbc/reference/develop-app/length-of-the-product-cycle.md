# <a name="length-of-the-product-cycle"></a>Продолжительность цикла продукта
Последний вопрос о взаимодействии — время. Разработка взаимодействующего приложения обычно занимает больше времени, чем разработка нонинтероперабле. Причина заключается в том, что приложение должно проверять возможности СУБД, выполнять одни и те же задачи по-разному для разных СУБД, обойти функции, поддерживаемые некоторыми СУБД, но не другими, и т. д.  
  
 Помимо времени разработки, необходимо учитывать время существования продукта. Если приложение предназначено для однократного использования, например приложение, которое передает данные при переносе из одной СУБД в другую, нет никакой точки в обеспечении его взаимодействия. Приложение будет использоваться один раз и отброшено.  
  
 Если приложение будет существовать в течение длительного времени, оно может быть проще в обслуживании в качестве взаимодействующего приложения. Это справедливо даже для пользовательских приложений, которые имеют одну СУБД в качестве целевого объекта. Это связано с тем, что код с возможностью взаимодействия использует ограниченный набор функций базы данных. Драйвер необходим для того, чтобы эти функции были доступны даже в том случае, если изменения вносятся в основную СУБД. Таким же кодом возможность взаимодействия с скопировав может повлечь за собой изменения в СУБД от разработчика приложения до разработчика драйверов.
