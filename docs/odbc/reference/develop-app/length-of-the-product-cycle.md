# <a name="length-of-the-product-cycle"></a>Продолжительность цикла продукта
Последний вопрос о совместимости - это время. Разработка совместимого приложения обычно занимает больше времени, чем разработка несовместимого. Причина в том, что приложение должно проверять возможности DBMS, выполнять одни и те же задачи по-разному для различных DBMS, обойти функциональность, поддерживаемую некоторыми DBMS, но не другие, и так далее.  
  
 В дополнение к времени разработки, срок службы продукта должны быть рассмотрены. Если приложение предназначено для использования один раз, например, приложение, которое передает данные при переходе из одного DBMS в другое, нет смысла делать его совместимым. Приложение будет использоваться один раз и отбрасываются.  
  
 Если приложение будет существовать в течение длительного времени, может быть проще поддерживать в качестве совместимого приложения. Это верно даже для пользовательских приложений, которые имеют один DBMS в качестве мишени. Причина в том, что совместимый код использует ограниченное подмножество функций базы данных. Драйвер обязан держать эти функции доступными, даже в условиях изменений в базовом DBMS. Таким образом, совместимый код может переложить бремя преодоления изменений в DBMS с разработчика приложений на разработчика драйверов.
