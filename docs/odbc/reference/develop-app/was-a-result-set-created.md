# <a name="was-a-result-set-created"></a>Был ли создан результирующий набор?
В большинстве случаев разработчики приложений знать, будет ли создавать результирующий набор инструкции, которые выполняет их приложение. Это происходит, если приложение использует жестко инструкций SQL, созданных программистом. Он обычно имеет место, когда приложение инструкций SQL во время выполнения: Программист может легко включать код, который помечает ли **ВЫБЕРИТЕ** инструкции или **вставить** составляется инструкция. В некоторых случаях программист невозможно знать ли инструкция создаст результирующий набор. Это значение равно true, если приложение позволяет пользователю вводить и выполнять инструкции SQL. Это также значение true, когда приложение создает инструкцию во время выполнения для выполнения процедуры.  
  
 В таком случае приложение вызывает **SQLNumResultCols** для определения числа столбцов в результирующем наборе. Если значение равно 0, инструкция не удалось создать результирующий набор; Если любому другому числу, инструкция была создана результирующий набор.  
  
 Приложение может вызвать **SQLNumResultCols** в любое время после подготовки или выполнения инструкции. Тем не менее, так как некоторые источники данных невозможно легко описать результирующие наборы, которые будут создаваться по подготовленных инструкций, снижение производительности при **SQLNumResultCols** вызывается после завершения подготовки инструкции, но перед его выполнением.  
  
 Некоторые источники данных также поддерживают определение количества строк, возвращаемых в результирующем наборе инструкции SQL. Чтобы сделать это, приложение вызывает **SQLRowCount**. Именно то, что представляет число строк обозначается SQL_DYNAMIC_CURSOR_ATTRIBUTES2, SQL_FORWARD_ONLY_CURSOR_ATTRIBUTES2, SQL_KEYSET_CURSOR_ATTRIBUTES2 или SQL_STATIC_CURSOR_ATTRIBUTES2 параметра (в зависимости от типа курсора) возвращается вызовом **SQLGetInfo**. Этой битовой маске указывает для каждого типа курсора, является ли число строк возвращается точно, приблизительное, либо доступна не для всех. Количество строк для статических ли управляемые набором ключей курсоры влияют изменения, внесенные через **SQLBulkOperations** или **SQLSetPos**, или позиционированного обновления или инструкций delete, зависит от других bits Возвращает те же аргументы параметра, перечисленных выше. Дополнительные сведения см. в разделе [SQLGetInfo](../../../odbc/reference/syntax/sqlgetinfo-function.md) описание функции.
