# <a name="was-a-result-set-created"></a>Был ли создан результирующий набор?
В большинстве случаев программисты приложений знают, создадут ли операторы, выполняемые их приложением, набор результатов. Это происходит в том случае, если приложение использует жестко закодированные заявления s'L, написанные программистом. Обычно это происходит в случае, когда приложение конструирует операторы S'L во время выполнения: программист может легко включить код, который помечает, строится ли оператор **SELECT** или оператор **INSERT.** В некоторых ситуациях программист не может знать, будет ли заявление создавать набор результатов. Это верно, если приложение предоставляет пользователю возможность ввести и выполнить выписку по S'L. Это также верно, когда приложение строит заявление во время выполнения для выполнения процедуры.  
  
 В таких случаях приложение вызывает **S'LNumResultCols,** чтобы определить количество столбцов в наборе результатов. Если это 0, утверждение не создало набор результатов; если это какое-либо другое число, утверждение создало набор результатов.  
  
 Приложение может **вызыватьs s'LNumResultCols** в любое время после того, как заявление будет подготовлено или выполнено. Однако, поскольку некоторые источники данных не могут легко описать наборы результатов, которые будут созданы подготовленными операторами, производительность пострадает, если **s'LNumResultCols** будет вызван после того, как оператор будет подготовлен, но до того, как оно будет выполнено.  
  
 Некоторые источники данных также поддерживают определение количества строк, которые возвращается в наборе результатов оператора S'L. Для этого приложение вызывает **S'LRowCount**. Точно ездание строки указывается на настройку SQL_DYNAMIC_CURSOR_ATTRIBUTES2, SQL_FORWARD_ONLY_CURSOR_ATTRIBUTES2, SQL_KEYSET_CURSOR_ATTRIBUTES2 или SQL_STATIC_CURSOR_ATTRIBUTES2 (в зависимости от типа курсора), возвращенного по вызову в **S'LGetInfo.** Эта битовая маска указывает для каждого типа курсора, является ли возвращенное количество строк точным, приблизительным или вообще недоступно. Зависит ли количество строк для статических или управляемых ключами курсоров, зависит от изменений, внесенных в **S'LBulkOperations** или **S'LSetPos,** или от позиционированных обновлений или инструкций по удалению, зависит от других битов, возвращенных теми же аргументами опции, перечисленными ранее. Для получения более [подробной](../../../odbc/reference/syntax/sqlgetinfo-function.md) информации см.
