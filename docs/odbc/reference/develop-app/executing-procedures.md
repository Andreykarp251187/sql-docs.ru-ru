# <a name="executing-procedures"></a>Выполнение процедур
ODBC определяет стандартную escape-последовательность для исполнения процедур. Синтаксис этой последовательности и пример кода, который ее использует, см. в разделе [вызовы процедур](../../../odbc/reference/develop-app/procedure-calls.md).  
  
 Для выполнения процедуры приложение выполняет следующие действия:  
  
1.  Задает значения любых параметров. Дополнительные сведения см. в подразделе [Параметры инструкции](../../../odbc/reference/develop-app/statement-parameters.md)далее в этом подразделе.  
  
2.  Вызывает **SQLExecDirect** и передает ему строку, содержащую инструкцию SQL, которая выполняет процедуру. Эта инструкция может использовать escape-последовательность, определенную в синтаксисе ODBC или DBMS. инструкции, использующие синтаксис, зависящие от СУБД, не поддерживают взаимодействие.  
  
3.  При вызове **SQLExecDirect** драйвер:  
  
    -   Извлекает значения текущих параметров и преобразует их при необходимости. Дополнительные сведения см. в подразделе [Параметры инструкции](../../../odbc/reference/develop-app/statement-parameters.md)далее в этом подразделе.  
  
    -   Вызывает процедуру в источнике данных и отправляет ей преобразованные значения параметров. Как драйвер вызывает процедуру, зависит от драйвера. Например, можно изменить инструкцию SQL для использования грамматики SQL источника данных и отправить эту инструкцию для выполнения или вызвать процедуру напрямую, используя механизм удаленного вызова процедур (RPC), определенный в протоколе потока данных СУБД.  
  
    -   Возвращает значения всех входных, выходных или выходных параметров или возвращаемого значения процедуры при условии, что процедура выполнена. Эти значения могут быть недоступны до тех пор, пока не будут обработаны все остальные результаты (количество строк и результирующие наборы), сформированные процедурой. Если процедура завершается неудачно, драйвер возвращает ошибки.
