# <a name="executing-procedures"></a>Выполнение процедур
ODBC определяет стандартную последовательность побега для выполнения процедур. Для синтаксиса этой последовательности и примера кода, который использует его, [см.](../../../odbc/reference/develop-app/procedure-calls.md)  
  
 Для выполнения процедуры приложение выполняет следующие действия:  
  
1.  Устанавливает значения любых параметров. Для получения дополнительной [информации см.](../../../odbc/reference/develop-app/statement-parameters.md)  
  
2.  Вызывает **вызов sLExecDirect** и передает ему строку, содержащую выписку s'L, которая выполняет процедуру. Это утверждение может использовать последовательность побега, определяемую синтаксисом ODBC или DBMS; заявления, которые используют синтаксис DBMS, не совместимы.  
  
3.  При вызове **S'LExecDirect** драйвер:  
  
    -   Извлекает текущие значения параметров и преобразует их по мере необходимости. Для получения дополнительной [информации см.](../../../odbc/reference/develop-app/statement-parameters.md)  
  
    -   Вызывает процедуру в источнике данных и отправляет ее преобразованные значения параметров. То, как водитель называет процедуру, специфична для водителя. Например, он может изменить заявление S'L, чтобы использовать грамматику источника данных s'L и представить это заявление для выполнения, или он может вызвать процедуру непосредственно с помощью механизма удаленного вызова процедуры (RPC), который определен в протоколе потока данных DBMS.  
  
    -   Возвращает значения любых входных/выходных или выходных параметров или значения возврата процедуры, при условии успеха процедуры. Эти значения могут быть доступны только после обработки всех других результатов (количество строк и наборов результатов), генерируемых процедурой. Если процедура не удается, водитель возвращает любые ошибки.
