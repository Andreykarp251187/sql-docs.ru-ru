# <a name="pattern-value-arguments"></a>Аргументы значения шаблона
Некоторые аргументы в функциях каталога, такие как аргумент *TableName* в **S'LTables,** принимают шаблоны поиска. Эти аргументы принимают шаблоны поиска, если атрибут SQL_ATTR_METADATA_ID оператора установлен SQL_FALSE; они являются аргументами идентификатора, которые не принимают шаблон поиска, если этот атрибут установлен для SQL_TRUE.  
  
 Символы шаблона поиска:  
  
-   Подчеркивание (я), которое представляет любой отдельный символ.  
  
-   Процент знака (%), который представляет любую последовательность ноль или более символов.  
  
-   Персонаж побега, который специфичен для водителя и используется для включения подчеркиваев, процентных знаков и символа побега в буквальном смысле. Если персонаж побега предшествует неособому персонажу, персонаж побега не имеет особого значения. Если персонаж побега предшествует особому персонажу, он избегает особого персонажа. Например, «За» будет рассматриваться как два\\символа, «и»\\а, но «%» будет рассматриваться как неспециальный одиночный символ «%».  
  
 Персонаж побега извлекается с SQL_SEARCH_PATTERN_ESCAPE опцией в **S'LGetInfo**. Он должен предшествовать любой подчеркивать, процент знак, или избежать характера в аргумент, который принимает поисковые шаблоны, чтобы включить этот символ в буквальном смысле. Примеры приведены в следующей таблице.  
  
|Шаблон поиска|Описание|  
|--------------------|-----------------|  
|%A%|Все идентификаторы, содержащие букву А|  
|ABC_|Все четыре идентификатора символов, начиная с ABC|  
|АВВС\\|Идентификатор ABC_, предполагая, побег\\характер backslash ( )|  
|\\\\%|Все идентификаторы,\\начиная с backslash ( ), предполагая, побег характер backslash|  
  
 Особое устоит позаботиться, чтобы избежать поиска шаблон символов в аргументах, которые принимают шаблоны поиска. Это особенно верно для подчеркивателя характера, который обычно используется в идентификаторах. Распространенной ошибкой в приложениях является получение значения из одной функции каталога и передача этого значения аргументу шаблона поиска в другой функции каталога. Например, предположим, что приложение извлекает имя таблицы MY_TABLE из исходного набора для **S'LTables** и передает его в **S'LColumns** для получения списка столбцов в MY_TABLE. Вместо того, чтобы получать столбцы для MY_TABLE, приложение получит столбцы для всех таблиц, которые соответствуют шаблону поиска MY_TABLE, таким как MY_TABLE, MY1TABLE, MY2TABLE и так далее.  
  
> [!NOTE]
>  ODBC 2. *драйверы x* не поддерживают шаблоны поиска в аргументе *CatalogName* в **S'LTables.** Драйверы ODBC 3 *.x* принимают шаблоны поиска в этом аргументе, если атрибут среды ODBC_VERSION SQL_ATTR_ установлен для SQL_OV_ODBC3; они не принимают шаблоны поиска в этом аргументе, если он установлен на SQL_OV_ODBC2.  
  
 Передача нулевого указателя аргументу шаблона поиска не ограничивает поиск этого аргумента; то есть, нулевая указка и шаблон поиска % (любые символы) являются эквивалентными. Однако шаблон поиска нулевой длины, то есть действительный указатель на строку нулевой длины, соответствует только пустой строке ("").
