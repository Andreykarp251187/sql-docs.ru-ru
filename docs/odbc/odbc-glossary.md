# <a name="odbc-glossary"></a>Глоссарий по интерфейсу ODBC
## <a name="a"></a>Объект  
 **план доступа**  
 План, созданный ядром СУБД для выполнения инструкции SQL. Эквивалент исполняемого кода, скомпилированного на основе третьего языка, такого как C.  
  
 **Агрегатная функция**  
 Функция, которая создает одно значение из группы значений, часто используемое с предложениями **Group By** и **HAVING** . Агрегатные функции включают в себя **AVG**, **Count**, **Max**, **min**и **Sum**. Также известны как *функции набора*. *См. также* скалярная функция.  
  
 **ANSI**  
 Американский национальный институт стандартов (ANSI). API ODBC основан на интерфейсе уровня вызовов ANSI.  
  
 **APD**  
 *См. раздел* дескриптор параметра приложения (APD).  
  
 **API**  
 Прикладной программный интерфейс. Набор подпрограмм, которые приложение использует для запроса и выполнения служб более низкого уровня. API ODBC состоит из функций ODBC.  
  
 **приложение**  
 Исполняемая программа, вызывающая функции в API ODBC.  
  
 **дескриптор параметра приложения (APD)**  
 Дескриптор, описывающий динамические параметры, используемые в инструкции SQL перед любым преобразованием, заданным приложением.  
  
 **Дескриптор строки приложения (АРД)**  
 Дескриптор, представляющий метаданные столбца и данные в буферах приложения, описывающий строку данных после любого преобразования данных, заданного приложением.  
  
 **ард**  
 *См. раздел* дескриптор строки приложения (АРД).  
  
 **режим автоматической фиксации**  
 Режим фиксации транзакции, в котором транзакции фиксируются сразу после выполнения.  
  
## <a name="b"></a>B  
 **Изменение поведения**  
 Изменение некоторых функциональных возможностей ODBC *3. x* на поведение ODBC *2. x* или наоборот. Вызвано изменением атрибута среды SQL_ATTR_ODBC_VERSION.  
  
 **Большой двоичный объект (BLOB)**  
 Любые двоичные данные на определенное число байтов, например 255. Обычно намного дольше. Такие данные обычно отправляются и извлекаются из источника данных в частях. Также называются *длинными данными*.  
  
 **вязывания**  
 В качестве глагола — это операция связывания столбца в результирующем наборе или параметра в инструкции SQL с переменной приложения. Как существительное, Ассоциация.  
  
 **смещение привязки**  
 Значение, добавляемое к адресам буфера данных и адресам буферов длины или индикатора для всех привязанных данных столбца или параметра, создавая новые адреса.  
  
 **блочный курсор**  
 Курсор, способный одновременно получить более одной строки данных.  
  
 **двойной**  
 Часть памяти приложения, используемая для передачи данных между приложением и драйвером. Буферы часто бывают парами: *буфер данных* и *буфер длины данных*.  
  
 **byte**  
 Восемь бит или один октет. *См. также* октет.  
  
## <a name="c"></a>C  
 **Тип данных C**  
 Тип данных переменной в программе на языке C, в данном случае это приложение.  
  
 **каталога**  
 Набор системных таблиц в базе данных, описывающих форму базы данных. Также называется *схемой* или *словарем данных*.  
  
 **Функция Catalog**  
 Функция ODBC, используемая для получения сведений из каталога базы данных.  
  
 **CLI**  
 *См. раздел* API.  
  
 **клиент/сервер**  
 Стратегия доступа к базе данных, в которой один или несколько клиентов обращаются к данным через сервер. Обычно клиенты реализуют пользовательский интерфейс, а сервер управляет доступом к базе данных.  
  
 **рубрик**  
 Контейнер для одного элемента данных в строке. Также называется *полем*.  
  
 **Сохраните**  
 Для внесения постоянных изменений в транзакцию.  
  
 **параллелизма**  
 Возможность одновременного доступа нескольких транзакций к одним и тем же данным.  
  
 **уровень соответствия**  
 Дискретный набор функциональных возможностей, поддерживаемых драйвером или источником данных. ODBC определяет уровни соответствия API и уровни соответствия SQL.  
  
 **connection**;  
 Конкретный экземпляр драйвера и источника данных.  
  
 **Просмотр подключений**  
 Поиск источников данных в сети для подключения к. Просмотр подключений может содержать несколько шагов. Например, пользователь может сначала просмотреть сеть для серверов, а затем просмотреть конкретный сервер для базы данных.  
  
 **маркер подключения**  
 Маркер структуры данных, содержащий сведения о соединении.  
  
 **Текущая строка**  
 Строка, на которую в данный момент указывает курсор. Выполнение позиционированных операций над текущей строкой.  
  
 **курсор**  
 Часть программного обеспечения, возвращающая строки данных в приложение. Возможно, назван после мерцающего курсора на компьютере терминала; точно так же, как курсор указывает на текущую позицию на экране, курсор в результирующем наборе указывает текущую позицию в результирующем наборе.  
  
## <a name="d"></a>D  
 **буфер данных**  
 Буфер, используемый для передачи данных. Часто с буфером данных связаны *буферы длины данных*.  
  
 **словарь данных**  
 *См* . каталог.  
  
 **буфер длины данных**  
 Буфер, используемый для передачи длины значения в соответствующем *буфере данных*. Буфер длины данных также используется для хранения индикаторов, например, является ли значение данных завершающим нулем.  
  
 **Источник данных**  
 Данные, к которым пользователь хочет получить доступ, и связанную с ним операционную систему, СУБД и сетевую платформу (если они есть).  
  
 **тип данных**  
 Тип фрагмента данных. ODBC определяет типы данных C и SQL. *См. также* индикатор типа.  
  
 **столбец "данные в процессе выполнения"**  
 Столбец, для которого отправляются данные после вызова **SQLSetPos** . Таким образом, именованные данные отправляются во время выполнения, а не помещаются в буфер набора строк. Длинные данные обычно отправляются частями во время выполнения.  
  
 **параметр "данные при выполнении"**  
 Параметр, для которого отправляются данные после вызова **SQLExecute** или **SQLExecDirect** . Таким образом, именованные данные отправляются при выполнении инструкции SQL, а не в буфере параметров. Длинные данные обычно отправляются частями во время выполнения.  
  
 **database**  
 Дискретная коллекция данных в СУБД. Также это СУБД.  
  
 **ядро СУБД**  
 Программное обеспечение в СУБД, которое анализирует и выполняет инструкции SQL и обращается к физическим данным.  
  
 **DBMS**  
 Система управления базами данных. Уровень программного обеспечения между физической базой данных и пользователем. СУБД управляет любым доступом к базе данных.  
  
 **Драйвер на основе СУБД**  
 Драйвер, обращающийся к физическим данным через изолированное ядро СУБД.  
  
 **DDL**  
 Язык описания данных. Эти инструкции в SQL, которые определяют, а не управляют данными. Например, **CREATE TABLE**, **создать индекс**, **предоставить**и **отозвать**.  
  
 **идентификатор с разделителем**  
 Идентификатор, заключенный в символы кавычек идентификаторов, чтобы он мог содержать специальные символы или ключевые слова (также называемые заключенным в кавычки идентификатором).  
  
 **Descriptor**  
 Структура данных, содержащая сведения о данных столбца или динамических параметрах. Физическое представление дескриптора не определено; приложения получают прямой доступ к дескриптору только путем обработки его полей путем вызова функций ODBC с помощью дескриптора дескриптора.  
  
 **база данных настольных систем**  
 СУБД, предназначенная для работы на персональном компьютере. Как правило, эти СУБД не предоставляют изолированного ядра СУБД и должны быть доступны через драйвер на основе файлов. Модули в этих драйверах обычно имеют ограниченную поддержку для SQL и транзакций. Например, dBASE, Paradox, Btrieve или Microsoft® FoxPro®.  
  
 **диагностики**  
 Запись, содержащая диагностические сведения о последней функции с именем, которая использовала определенный обработчик. Диагностические записи связаны с дескрипторами среды, соединения, инструкции и дескриптора.  
  
 **DML**  
 Язык обработки данных. Эти инструкции в SQL управляют, а не определяют данные. Например, **вставьте**, **Обновите**, **удалите**и **выберите**.  
  
 **аудиодрайвера**  
 Библиотека подпрограмм, предоставляющая функции API ODBC. Драйверы относятся к одной СУБД.  
  
 **Диспетчер драйверов**  
 Библиотека подпрограмм, которая управляет доступом к драйверам приложения. Диспетчер драйверов загружает и выгружает драйверы (или подключается к ним и отключается от) и передает вызовы функций ODBC соответствующему драйверу.  
  
 **Библиотека DLL настройки драйвера**  
 Библиотека DLL, содержащая функции установки и настройки драйвера.  
  
 **динамический курсор**  
 Прокручиваемый курсор, способный обнаруживать строки, обновленные, удаленные или вставленные в результирующий набор.  
  
 **динамический SQL**  
 Тип внедренного SQL, в котором инструкции SQL создаются и компилируются во время выполнения. *См. также* статический SQL.  
  
## <a name="e"></a>E  
 **внедренный SQL**  
 Инструкции SQL, включенные непосредственно в программу, написанную на другом языке, например COBOL или C. ODBC, не использует внедренный SQL. *См. также* статический SQL *и* динамический SQL.  
  
 **PXE**  
 Глобальный контекст для доступа к данным; связанный с средой — любая глобальная информация по природе, например список всех соединений в этой среде.  
  
 **маркер среды**  
 Маркер структуры данных, содержащий сведения о среде.  
  
 **Escape-предложение**  
 Предложение в инструкции SQL.  
  
 **работать**  
 Для выполнения инструкции SQL.  
  
## <a name="f"></a>F  
 **курсор FAT**  
 *См. раздел* блочный курсор.  
  
 **водит**  
 Извлечение одной или нескольких строк из результирующего набора.  
  
 **полями**  
 *См* . столбец.  
  
 **драйвер на основе файлов**  
 Драйвер, который напрямую обращается к физическим данным. В этом случае драйвер содержит ядро СУБД и действует как драйвер, так и источник данных.  
  
 **файловый источник данных**  
 Источник данных, для которого сведения о соединении хранятся в DSN-файле.  
  
 **внешний ключ**  
 Столбец или столбцы в таблице, которые соответствуют первичному ключу другой таблицы.  
  
 **однонаправленный курсор**  
 Курсор, который может двигаться только вперед по результирующему набору и обычно извлекает только одну строку за раз. Большинство реляционных баз данных поддерживают только однонаправленные курсоры.  
  
## <a name="h"></a>H  
 **справиться**  
 Значение, уникально идентифицирующее нечто вроде файла или структуры данных. Дескрипторы имеют смысл только для программного обеспечения, которое создает и использует их, но передается другим программным обеспечением для обнаружения объектов. ODBC определяет дескрипторы для сред, соединений, инструкций и дескрипторов.  
  
## <a name="i"></a>I  
 **дескриптор параметра реализации (IPD)**  
 Дескриптор, описывающий динамические параметры, используемые в инструкции SQL после любого преобразования, заданного приложением.  
  
 **Дескриптор строки реализации (IRD)**  
 Дескриптор, описывающий строку данных перед любым преобразованием, заданным приложением.  
  
 **Библиотека DLL установщика**  
 Библиотека DLL, которая устанавливает компоненты ODBC и настраивает источники данных.  
  
 **Механизм обеспечения целостности**  
 Подмножество SQL, разработанное для поддержания целостности базы данных.  
  
 **уровень соответствия интерфейса**  
 Уровень интерфейса ODBC 3,7, поддерживаемый драйвером; может быть ядром, уровнем 1 или уровнем 2.  
  
 **взаимодействие**  
 Способность одного приложения использовать один и тот же код при доступе к данным в разных СУБД.  
  
 **IPD**  
 *См. раздел* Дескриптор параметра реализации (IPD).  
  
 **IRD**  
 *См. раздел* Дескриптор строки реализации (IRD).  
  
 **ISO/IEC**  
 Международный стандарт организации/IEC. API ODBC основан на интерфейсе уровня вызовов ISO/IEC.  
  
## <a name="j"></a>J  
 **join**  
 Операция в реляционной базе данных, которая связывает строки в двух или более таблицах, сопоставляя значения в указанных столбцах.  
  
## <a name="k"></a>K  
 **key**  
 Столбец или столбцы, значения которых указывают строку. *См. также* внешний ключ *и* первичный ключ.  
  
 **KEYSET**  
 Набор ключей, используемых смешанным или управляемым набором ключей курсором для повторной выборки строк.  
  
 **курсор, управляемый набором ключей**  
 Прокручиваемый курсор, который обнаруживает обновленные и удаленные строки с помощью набора ключей.  
  
## <a name="l"></a>L  
 **кавычк**  
 Символьное представление фактического значения данных в инструкции SQL.  
  
 **блокировкой**  
 Процесс, с помощью которого СУБД разрешает доступ к строке в многопользовательской среде. СУБД обычно устанавливает бит на строку или физическую страницу, содержащую строку, указывающую на то, что строка или страница заблокированы.  
  
 **данные типа long**  
 Любые двоичные или символьные данные в определенной длине, например 255 байт или символов. Обычно намного дольше. Такие данные обычно отправляются и извлекаются из источника данных в частях. Также известен как *BLOB*-объекты или *CLOB*s.  
  
## <a name="m"></a>M  
 **Источник данных компьютера**  
 Источник данных, для которого сведения о соединении хранятся в системе (например, в реестре).  
  
 **режим фиксации вручную**  
 Режим фиксации транзакции, в котором транзакции должны быть явно зафиксированы путем вызова **SQLTransact**.  
  
 **метаданных**  
 Данные, описывающие параметр в инструкции SQL или в столбце результирующего набора. Например, тип данных, длина в байтах и точность параметра.  
  
 **Многоуровневый драйвер**  
 *См. раздел* Драйвер на основе СУБД.  
  
## <a name="n"></a>Нет  
 **Значение NULL**  
 Без явного присвоения значения. В частности, значение NULL отличается от нуля или пустого значения.  
  
## <a name="o"></a>O  
 **байт**  
 Восемь бит или один байт. *См. также* Byte.  
  
 **Длина октета**  
 Длина в октетах буфера или содержащихся в нем данных.  
  
 **ODBC**  
 Откройте подключение к базе данных. Спецификация для API, определяющая стандартный набор подпрограмм, с помощью которых приложение может получить доступ к данным в источнике данных.  
  
 **Администратор ODBC**  
 Исполняемая программа, вызывающая библиотеку DLL установщика для настройки источников данных.  
  
 Открыть группу  
 Компания, которая публикует стандарты. В частности, он публикует стандарты группы доступа SQL (САГ).  
  
 **оптимистичный параллелизм**  
 Стратегия увеличения параллелизма, в которой строки не блокируются. Вместо этого перед обновлением или удалением курсор проверяет, были ли они изменены с момента последнего чтения. В этом случае происходит сбой обновления или удаления. *См. также* пессимистичный параллелизм.  
  
 **Внешнее соединение**  
 Соединение, в котором возвращаются как совпадающие, так и несовпадающие строки. Значения всех столбцов несопоставленной таблицы в несовпадающих строках имеют значение NULL.  
  
 **владельцев**  
 Владелец таблицы.  
  
## <a name="p"></a>P  
 **параметр**  
 Переменная в инструкции SQL, помеченная маркером параметра или вопросительным знаком (?). Параметры привязаны к переменным приложения и их значениям, полученным при выполнении инструкции.  
  
 **дескриптор параметра**  
 Дескриптор, описывающий параметры времени выполнения, используемые в инструкции SQL, перед любым преобразованием, заданным приложением (дескриптором параметра приложения или APD), или после любого преобразования, заданного приложением (дескриптор параметра реализации или IPD).  
  
 **массив операций с параметрами**  
 Массив, содержащий значения, которые могут быть заданы приложением для указания того, что соответствующий параметр должен игнорироваться в операции **SQLExecDirect** или **SQLExecute** .  
  
 **массив состояния параметра**  
 Массив, содержащий состояние параметра после вызова **SQLExecDirect** или **SQLExecute**.  
  
 **пессимистический параллелизм**  
 Стратегия реализации возможности сериализации, в которой строки блокируются, чтобы другие транзакции не могли их изменить. *См. также* оптимистичный параллелизм *и* возможности сериализации.  
  
 **позиционированная операция**  
 Любая операция, которая работает с текущей строкой. Например, инструкции позиционированного обновления и удаления, **SQLGetData**и **SQLSetPos**.  
  
 **Инструкция позиционированного обновления**  
 Инструкция SQL, используемая для обновления значений в текущей строке.  
  
 **Инструкция позиционированного удаления**  
 Инструкция SQL, используемая для удаления текущей строки.  
  
 **Обеспечьте**  
 Для компиляции инструкции SQL. План доступа создается путем подготовки инструкции SQL.  
  
 **первичный ключ**  
 Столбец или столбцы, которые однозначно идентифицируют строку в таблице.  
  
 **PROCEDURE**  
 Группа из одной или нескольких предварительно скомпилированных инструкций SQL, которые хранятся в виде именованного объекта в базе данных.  
  
 **столбец процедуры**  
 Аргумент в вызове процедуры, значение, возвращаемое процедурой, или столбец в результирующем наборе, созданном процедурой.  
  
## <a name="q"></a>Q  
 **квалификатор**  
 База данных, содержащая одну или несколько таблиц.  
  
 **запрос**  
 Инструкция SQL. Иногда используется для обозначения инструкции **SELECT** .  
  
 **заключенный в кавычки идентификатор**  
 Идентификатор, заключенный в кавычки идентификатора, чтобы он мог содержать специальные символы или ключевые слова Match (также известные в SQL-92 как идентификатор с разделителями).  
  
## <a name="r"></a>R  
 **основание системы счисления**  
 Основание системы счисления. Обычно 2 или 10.  
  
 **записать**  
 *См* . строку.  
  
 **результирующий набор**  
 Набор строк, созданных путем выполнения инструкции **SELECT** .  
  
 **код возврата**  
 Значение, возвращаемое функцией ODBC.  
  
 **откат**  
 Значение, чтобы вернуть значения, измененные транзакцией, в исходное состояние.  
  
 **строки**  
 Набор связанных столбцов, описывающих конкретную сущность. Также называется *записью*.  
  
 **Дескриптор строки**  
 Дескриптор, описывающий столбцы результирующего набора до любого преобразования, заданного приложением (дескриптор строки реализации или IRD), или после любого преобразования, заданного приложением (дескриптор строки приложения или АРД).  
  
 **массив операций со строками**  
 Массив, содержащий значения, которые могут быть заданы приложением для указания того, что соответствующая строка должна игнорироваться в операции **SQLSetPos** .  
  
 **массив состояний строк**  
 Массив, содержащий состояние строки после вызова **SQLFetch**, **SQLFetchScroll**или **SQLSetPos**.  
  
 **набора строк**  
 Набор строк, возвращаемых в одной выборке с помощью блочного курсора.  
  
 **буферы наборов строк**  
 Буферы, привязанные к столбцам результирующего набора и в котором возвращаются данные для всего набора строк.  
  
## <a name="s"></a>S  
 **саг**  
 *См. раздел* Группа доступа SQL (САГ).  
  
 **скалярная функция**  
 Функция, которая создает одно значение из одного значения. Например, функция, которая изменяет регистр символьных данных.  
  
 **схемы**  
 *См* . каталог.  
  
 **прокручиваемый курсор**  
 Курсор, который может перемещаться вперед или назад по результирующему набору.  
  
 **возможности сериализации**  
 Одновременное выполнение двух транзакций приводит к результату, аналогичному последовательному (или последовательному) выполнению этих транзакций. Сериализуемые транзакции необходимы для поддержания целостности базы данных.  
  
 **база данных сервера**  
 СУБД, предназначенная для работы в среде клиента/сервера. Эти СУБД предоставляют изолированное ядро СУБД, обеспечивающее широкую поддержку SQL и транзакций. Доступ к ним осуществляется через драйверы на основе DBMS. Например, Oracle, Informix, DB/2 или Microsoft® SQL Server.  
  
 **Функция Set**  
 *См. раздел* агрегатная функция.  
  
 **Библиотека DLL установки**  
 *См. раздел* библиотека DLL программы установки драйвера *и* библиотека DLL установки переводчиков.  
  
 **одноуровневый драйвер**  
 *См. раздел* драйвер на основе файлов.  
  
 **SQL**  
 язык SQL. Язык, используемый реляционными базами данных для запроса, обновления и управления данными.  
  
 **Группа доступа SQL (САГ)**  
 Отраслевый консорциум компаний, которые связаны с СУБД SQL. Интерфейс уровня вызовов открытой группы основан на работе, изначально выполненной группой доступа SQL.  
  
 **Уровень соответствия SQL**  
 Уровень грамматики SQL-92, поддерживаемый драйвером; может быть введено, Transitional, промежуточный или полный FIPS.  
  
 **Тип данных SQL**  
 Тип данных столбца или параметра, который хранится в источнике данных.  
  
 **SQLSTATE**  
 Значение из пяти символов, указывающее на конкретную ошибку.  
  
 **Инструкция SQL**  
 Полная фраза в SQL, которая начинается с ключевого слова и полностью описывает выполняемое действие. Например, SELECT * FROM Orders. Инструкции SQL не следует путать с операторами.  
  
 **state**  
 Четко определенное условие элемента. Например, соединение имеет семь состояний, включая Нераспределенные, выделенные, подключенные и необходимые данные. Некоторые операции могут выполняться только в том случае, если элемент находится в определенном состоянии. Например, соединение может быть освобождено только в том случае, если оно находится в выделенном состоянии, а не, например, когда оно находится в подключенном состоянии.  
  
 **Смена состояния**  
 Перемещение элемента из одного состояния в другое. ODBC определяет строгие переходы между состояниями для сред, соединений и инструкций.  
  
 **баланс**  
 Контейнер для всей информации, связанной с инструкцией SQL. Не следует путать с инструкциями SQL.  
  
 **маркер инструкции**  
 Маркер структуры данных, содержащий сведения об инструкции.  
  
 **статический курсор**  
 Прокручиваемый курсор, который не может обнаруживать обновления, удаления или вставки в результирующий набор. Обычно реализуется путем создания копии результирующего набора.  
  
 **статический SQL**  
 Тип внедренного SQL, в котором инструкции SQL жестко кодируются и компилируются при компиляции остальной части программы. *См. также* динамический SQL.  
  
 **хранимая процедура**  
 *См* . процедуру.  
  
## <a name="t"></a>T  
 **table**  
 Коллекция строк.  
  
 **двойное**  
 Преобразование 16-битных адресов в 32-разрядные адреса (или наоборот), когда 16-разрядные приложения используются с 32-разрядными драйверами ODBC.  
  
 **операции**  
 Атомарная единица работы. Работа в рамках транзакции должна быть выполнена целиком; сбой любой части транзакции ведет к сбою всей транзакции.  
  
 **изоляция транзакций**  
 Процесс изоляции одной транзакции от воздействия всех других транзакций.  
  
 **уровень изоляции транзакции**  
 Мера того, насколько хорошо транзакция изолирована. Существует пять уровней изоляции транзакций: чтение незафиксированного, фиксируемое чтение, повторяемое чтение, сериализуемый и управление версиями.  
  
 **Библиотека DLL переводчика**  
 Библиотека DLL, используемая для преобразования данных из одного набора символов в другой.  
  
 **Библиотека DLL установки транслятора**  
 Библиотека DLL, содержащая функции установки и настройки для отдельных переводчиков.  
  
 **двухфазная фиксация**  
 Процесс фиксации распределенной транзакции в два этапа. На первом этапе обработчик транзакций проверяет, можно ли зафиксировать все части транзакции. На втором этапе фиксируются все части транзакции. Если какая бы то ни было часть транзакции указывает на первый этап, который невозможно зафиксировать, второй этап не происходит. ODBC не поддерживает двухфазной фиксации.  
  
 **индикатор типа**  
 Целочисленное значение, передаваемое или возвращаемое из функции ODBC для указания типа данных переменной приложения, параметра или столбца. ODBC определяет индикаторы типов для типов данных C и SQL.  
  
## <a name="v"></a>V  
 **режиме**  
 Альтернативный способ просмотра данных в одной или нескольких таблицах. Представление обычно создается как подмножество столбцов из одной или нескольких таблиц. В ODBC представления обычно эквивалентны таблицам.
