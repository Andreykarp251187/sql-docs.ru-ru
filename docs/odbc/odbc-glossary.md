# <a name="odbc-glossary"></a>Глоссарий по интерфейсу ODBC
## <a name="a"></a>Объект  
 **план доступа**  
 План, созданный компонентом database engine для выполнения инструкции SQL. Эквивалентно исполняемого кода, скомпилированного из третьего поколения языке, как C.  
  
 **Агрегатная функция**  
 Функция, которая создает одно значение из группы значений, часто используется с **GROUP BY** и **HAVING** предложения. Агрегатные функции включают **AVG**, **число**, **MAX**, **MIN**, и **сумма**. Также называется *функции наборов*. *См. также* скалярную функцию.  
  
 **ANSI**  
 Американский национальный институт стандартов. API-Интерфейс ODBC основан на интерфейс уровня вызова ANSI.  
  
 **APD**  
 *См. в разделе* дескрипторе параметра приложения (APD).  
  
 **API**  
 Интерфейс MAPI. Набор процедур, используемых приложением для запроса и выполнения служб более низкого уровня. API-Интерфейс ODBC представляет собой функции ODBC.  
  
 **Приложения**  
 Исполняемая программа, вызывающего функции в API-Интерфейс ODBC.  
  
 **Дескрипторе параметра приложения (APD)**  
 Дескриптор, который описывает динамические параметры, используемые в инструкции SQL, прежде чем какие-либо преобразования, заданный приложением объект.  
  
 **Дескриптор строки приложения (Отменить)**  
 Дескриптор, представляющий метаданные столбца и данные в буферах приложения, описывающий строку данных после преобразования данных определяется приложением.  
  
 **ОТМЕНИТЬ**  
 *См. в разделе* дескриптора строки приложения (Отменить).  
  
 **режим автоматической фиксации**  
 Режим фиксации транзакции, в котором транзакции фиксируются немедленно после их выполнения.  
  
## <a name="b"></a>С  
 **Изменение в поведении**  
 Изменения в некоторые функциональные возможности из ODBC *3.x* поведение ODBC *2.x* поведение, или наоборот. Из-за изменения атрибута среды SQL_ATTR_ODBC_VERSION.  
  
 **большой двоичный объект (BLOB)**  
 Любые двоичные данные через определенное количество байтов, например 255. Обычно гораздо больше. Такие данные обычно отправляются и извлечь из источника данных в частях. Также называется *данные большой длины*.  
  
 **Привязки**  
 Как команда act сопоставления столбцов в результирующем наборе или параметра в инструкции SQL с переменной приложения. Этот термин ассоциации.  
  
 **Смещение привязки**  
 Значение, добавляемое к адреса буферов данных и длины и индикатора адреса буферов, для всех привязанного столбца или параметра данных, создания новых адресов.  
  
 **блочный курсор**  
 Курсор может выборка более одной строки данных за раз.  
  
 **буфер**  
 Блок памяти приложения, используемый для передачи данных между приложением и драйвером. Буферы часто возникают попарно: *буфер данных* и *буфер длины данных*.  
  
 **byte**  
 Восемь бит или один октет. *См. также* октет.  
  
## <a name="c"></a>В  
 **Тип данных C**  
 Тип данных переменной в программе на языке C, в данном случае это приложение.  
  
 **catalog**  
 Набор системных таблиц в базе данных, которые описывают форму базы данных. Также называется *схемы* или *словарь данных*.  
  
 **функции каталога**  
 Функция ODBC используется для извлечения сведений из базы данных каталога.  
  
 **ИНТЕРФЕЙС КОМАНДНОЙ СТРОКИ**  
 *См. в разделе* API.  
  
 **клиент сервер**  
 Стратегии доступа к базе данных, в котором один или несколько клиентов доступ к данным через сервер. Клиенты обычно реализуют пользовательский интерфейс, хотя доступ к базам данных серверных элементов управления.  
  
 **column**  
 Контейнер для одного элемента данных в строке. Также называется *поле*.  
  
 **commit**  
 Для внесения постоянных изменений транзакции.  
  
 **concurrency**  
 Возможность более одной операции доступа к те же данные в то же время.  
  
 **Уровень соответствия**  
 Дискретный набор функциональных возможностей, поддерживаемых драйвером или источником данных. ODBC определяет уровни соответствия API и уровни соответствия SQL.  
  
 **connection**  
 Конкретный экземпляр драйвера и источник данных.  
  
 **Просмотр подключения**  
 Поиск в сети для источников данных для подключения к. Просмотр подключения может включать несколько шагов. Например, пользователь может сначала просматривать серверов в сети и затем конкретного сервера для базы данных.  
  
 **дескриптор подключения**  
 Дескриптор к структуре данных, содержащий сведения о соединении.  
  
 **Текущая строка**  
 Строка, в данный момент указывает курсор. Позиционированных операций действуют на текущей строке.  
  
 **курсор**  
 Часть программного обеспечения, которое возвращает строки данных в приложение. Возможно, с именем мерцающего курсора на компьютере терминалов; так же, как этот курсор указывает текущее положение на экране, курсора на результирующем наборе указывает текущее положение в результирующем наборе.  
  
## <a name="d"></a>D  
 **буфер данных**  
 Буфер, используемый для передачи данных. Часто ассоциируемый с данными — буфера *буфер длины данных*.  
  
 **словарь данных**  
 *См. в разделе* каталога.  
  
 **Длина буфера данных**  
 Буфер, используемый для передачи длина значения в соответствующей *буфер данных*. Длина буфера данных также используется для хранения эффективности, например является ли значение заканчивается нулевым байтом.  
  
 **Источник данных**  
 Данные, которому требуется предоставить доступ и его связан с операционной системой, СУБД и платформы сети (если таковые имеются).  
  
 **Тип данных**  
 Тип фрагмента данных. ODBC определяет типы данных C и SQL. *См. также* индикатор типа.  
  
 **столбец данных во время выполнения**  
 Столбец, для которого данные отправляются после **SQLSetPos** вызывается. Поэтому именованные, так как данные отправляются во время выполнения, а не помещается в буфер строк. Длинные данные обычно передается в части во время выполнения.  
  
 **параметр данных во время выполнения**  
 Параметр, для которого данные отправляются после **SQLExecute** или **SQLExecDirect** вызывается. Поэтому именованные, так как данные отправляются в том случае, если выполнить инструкцию SQL, а помещается в буфер параметров. Длинные данные обычно передается в части во время выполнения.  
  
 **базой данных**  
 Набор дискретных данных в СУБД. Также СУБД.  
  
 **Компонент Database engine**  
 Программное обеспечение в СУБД, анализирует и выполняет инструкции SQL и обращается к физическими данными.  
  
 **СУБД**  
 Система управления базами данных. Уровень программного обеспечения между физической базой данных и пользователем. СУБД управляет любым доступом к базе данных.  
  
 **Драйверов на основе СУБД**  
 Драйвер, который обращается к физическим перемещением данных через изолированный компонент database engine.  
  
 **DDL**  
 Язык описания данных DDL. Эти инструкции в SQL, определяющие, в отличие от для работы с данными. Например **CREATE TABLE**, **CREATE INDEX**, **GRANT**, и **ОТОЗВАТЬ**.  
  
 **Идентификатор с разделителем**  
 Идентификатор, заключенный в кавычки идентификатор, чтобы он может содержать специальные символы или совпадать с ключевыми словами (также известный как заключенные в кавычки идентификатор).  
  
 **Дескриптор**  
 Структура данных, которая содержит сведения о данных в столбце или динамических параметров. Физическое представление дескриптора не определена. приложениям получать непосредственный доступ к дескриптор только управляя его поля путем вызова функции ODBC с помощью дескриптора.  
  
 **для настольных баз данных**  
 СУБД, разработанный для работы на персональных компьютерах. Как правило эти СУБД не предоставляют изолированный компонент database engine и должен осуществляться через драйверов на основе файла. Обычно модули в эти драйверы снижена поддержку SQL и транзакций. Например dBASE, Paradox, Btrieve или FoxPro Microsoft®.  
  
 **Диагностика**  
 Запись, содержащую диагностические сведения о последней функция, вызываемая, использовании определенного дескриптора. Диагностические записи связаны с среды, подключения, инструкции и указатели дескрипторов.  
  
 **DML**  
 Язык обработки данных. Эти инструкции в SQL, которые управляют, а не для определения данных. Например **вставить**, **обновление**, **удалить**, и **ВЫБЕРИТЕ**.  
  
 **Драйвер**  
 Процедуры библиотеки, предоставляющий функций в API-Интерфейс ODBC. Драйверы относятся только к одной СУБД.  
  
 **Диспетчер драйверов**  
 Библиотеку подпрограмму, которая управляет доступом к драйверы для приложения. Диспетчер драйверов загружает и выгружает (или подключается к и отключается от) драйверы и передает вызовы функций ODBC в правильный драйвер.  
  
 **Библиотека DLL программы установки драйвера**  
 Библиотеку DLL, содержащую свойственные конкретному драйверу функции установки и настройки.  
  
 **динамический курсор**  
 Прокручиваемый курсор, для определения строк, обновленных, удалены или вставлены в результирующем наборе.  
  
 **Динамический SQL**  
 Тип embedded SQL в SQL, который создаются и компилируются во время выполнения инструкции. *См. также* статического SQL.  
  
## <a name="e"></a>E  
 **Embedded SQL**  
 Инструкции SQL, которые включаются непосредственно в программе, написанным на другом языке, например COBOL или C. ODBC не использует embedded SQL. *См. также* статический SQL *и* динамического SQL.  
  
 **Среды**  
 Глобальный контекст, в котором для доступа к данным; связанное с данной средой — это данные, имеют глобальную природу, такой как список всех подключений в этой среде.  
  
 **дескриптора среды**  
 Дескриптор к структуре данных, содержащий сведения о среде.  
  
 **предложения escape**  
 Предложение в инструкции SQL.  
  
 **execute**  
 Для выполнения инструкции SQL.  
  
## <a name="f"></a>C  
 **FAT курсора**  
 *См. в разделе* блочный курсор.  
  
 **FETCH**  
 Чтобы получить одну или несколько строк из результирующего набора.  
  
 **field**  
 *См. в разделе* столбца.  
  
 **драйвер на основе файлов**  
 Драйвер, который напрямую обращается к физическим перемещением данных. В этом случае драйвер содержит ядро базы данных и выступает в качестве источника данных и драйвера.  
  
 **файл источника данных**  
 Источник данных, какие подключения сведения хранятся в файле .dsn.  
  
 **Внешний ключ**  
 Столбец или столбцы в таблице, которые соответствуют первичного ключа в другой таблице.  
  
 **однопроходный курсор**  
 Курсор, который можно перемещать только вперед из результирующего набора и обычно выбирается только одну строку за раз. Большинство реляционных баз данных поддерживает только однонаправленные курсоры.  
  
## <a name="h"></a>H  
 **Дескриптор**  
 Значение, которое однозначно определяет, что-нибудь, например структуру данных или файлов. Маркеры имеют смысл только в программное обеспечение, которое создает и использует их, но также передаются другому программному обеспечению для идентификации действия. ODBC определяет маркеры для сред, соединений, инструкций и дескрипторов.  
  
## <a name="i"></a>I  
 **Дескриптор параметра реализации (IPD)**  
 Дескриптор, который описывает динамические параметры, используемые в инструкции SQL, после преобразования определяется приложением.  
  
 **Дескриптор строки реализации (IRD)**  
 Дескриптор, описывающий строку данных, прежде чем какие-либо преобразования, заданный приложением объект.  
  
 **Библиотека DLL установщика**  
 Библиотеку DLL, которая устанавливает компоненты ODBC и настраивает источников данных.  
  
 **Расширенный контроль целостности**  
 Подмножество разработана для обеспечения целостности базы данных SQL.  
  
 **уровень соответствия интерфейса**  
 Уровень интерфейса ODBC 3.7, поддерживаемых драйвером; может быть Core уровня 1 и уровень 2.  
  
 **Взаимодействие**  
 Способность одного приложения для использования того же кода при доступе к данным в различных СУБД.  
  
 **IPD**  
 *См. в разделе* дескриптора параметра реализации (IPD).  
  
 **IRD**  
 *См. в разделе* дескриптора строки реализации (IRD).  
  
 **ISO/IEC**  
 Международные стандарты организации и Международная электротехническая комиссия. API-Интерфейс ODBC основан на интерфейс уровня вызова ISO/IEC.  
  
## <a name="j"></a>J  
 **соединения**  
 Операции в реляционной базе данных, которая связывает строки в двух или более таблицах, сопоставляя значения в заданные столбцы.  
  
## <a name="k"></a>K  
 **Ключ**  
 Столбец или столбцы, значения которых идентификации строки. *См. также* внешний ключ *и* первичный ключ.  
  
 **Управляемые набором ключей**  
 Набор ключей, используемых курсора смешанного или нечувствительными к повторному получению строк.  
  
 **курсоры**  
 Прокручиваемый курсор, который обнаруживает обновленные и удаленные строки с помощью набора ключей.  
  
## <a name="l"></a>L  
 **literal**  
 Представление символа конкретного значения в инструкции SQL.  
  
 **locking**  
 Процесс, по которому СУБД ограничивает доступ к строке в многопользовательской среде. СУБД обычно устанавливает для бита на физическую страницу, содержащую одну строку, которая указывает на строку или строки или страницы заблокирован.  
  
 **данные большого формата**  
 Любой двоичных или символьных данных определенной длины, например 255 байт или символов. Обычно гораздо больше. Такие данные обычно отправляются и извлечь из источника данных в частях. Также называется *BLOB-ОБЪЕКТОВ*s или *CLOB*s.  
  
## <a name="m"></a>M  
 **машинные источники данных**  
 Источник данных, какие подключения сведения хранятся в системе (например, реестр).  
  
 **Режим ручной фиксации**  
 Режим фиксации транзакции, в котором транзакции должна быть явно зафиксирована, вызвав **SQLTransact**.  
  
 **Метаданные**  
 Данные, описывающие параметр в инструкции SQL или столбцом в результирующем наборе. Например тип данных, длина в байтах и точность параметра.  
  
 **многоуровневая драйвера**  
 *См. в разделе* драйверов на основе СУБД.  
  
## <a name="n"></a>в  
 **Значение NULL**  
 Наличие значение не присвоено явным образом. В частности значение NULL отличается от нулевое или пустое.  
  
## <a name="o"></a>O  
 **октет**  
 Восемь бит или один байт. *См. также* байтов.  
  
 **Длительность октета**  
 Длина в октетах буфера или данные, содержащиеся в ней.  
  
 **интерфейс ODBC**  
 Откройте подключение к базе данных. Спецификация API, который определяет стандартный набор подпрограмм, с помощью которых приложение может получить доступ к данных в источнике данных.  
  
 **Администратор ODBC**  
 Исполняемая программа, которая вызывает установщик DLL для настройки источников данных.  
  
 Open Group  
 Компании, которая публикует стандартов. В частности он публикует стандартов группы доступа состояние SQL (СПАДА).  
  
 **Оптимистическая блокировка**  
 Стратегия для увеличения параллелизма, в котором строки не блокируются. Вместо этого прежде чем они обновляются или удаляются, курсор проверяет, см. в разделе, если они были изменены с момента последнего чтении. Если Да, update или delete завершается ошибкой. *См. также* пессимистичный параллелизм.  
  
 **внешнее соединение**  
 Соединение, в какие сопоставления и не имеющие соответствия строки возвращать. Значения всех столбцов из несопоставленных таблицы в не имеющие соответствия строк, присваивается значение NULL.  
  
 **Владелец**  
 Владелец таблицы.  
  
## <a name="p"></a>С  
 **Параметр**  
 Переменная в инструкции SQL, помечена как маркер параметра или вопросительный знак (?). Параметры привязаны к переменным приложения и их значения, полученные при выполнении инструкции.  
  
 **Дескриптор параметра**  
 Дескриптор, который описывает параметры выполнения, используемые в инструкции SQL, либо до любого преобразования, который указан приложением (дескриптора параметра приложения или APD) или после любое преобразование, в соответствии с параметрами приложения (который является реализацией Дескриптор параметра или IPD).  
  
 **Массив параметров операции**  
 Массив, содержащий значения, которые приложение может задать для указания, что соответствующий параметр должны пропускаться в **SQLExecDirect** или **SQLExecute** операции.  
  
 **Массив состояний параметров**  
 Массив, содержащий состояние параметра после вызова **SQLExecDirect** или **SQLExecute**.  
  
 **пессимистичный параллелизм**  
 Стратегию для реализации возможности сериализации, в котором строки блокируются, поэтому другие транзакции не может их изменить. *См. также* оптимистичного параллелизма *и* возможности сериализации.  
  
 **позиционированные операции**  
 Любая операция, которая действует в отношении текущей строки. Например, расположен update и delete, **SQLGetData**, и **SQLSetPos**.  
  
 **инструкции позиционированного обновления**  
 Оператор SQL для обновления значений в текущей строке.  
  
 **инструкции позиционированного delete**  
 Инструкция SQL, используемую для удаления текущей строки.  
  
 **prepare**  
 Для компиляции инструкции SQL. В плане доступа создается путем подготовки инструкции SQL.  
  
 **Первичный ключ**  
 Столбец или столбцы, которые однозначно определяют строку в таблице.  
  
 **procedure**  
 Группа из одной или нескольких предкомпилированных инструкций SQL, которые хранятся в виде именованного объекта в базе данных.  
  
 **процедура столбца**  
 Аргумент при вызове процедуры, значения, возвращенного в процедуру или столбца в результирующий набор, созданный с помощью процедуры.  
  
## <a name="q"></a>Q  
 **Квалификатор**  
 База данных, содержащий одну или несколько таблиц.  
  
 **Запрос**  
 Инструкции SQL. Иногда используется для обозначения **ВЫБЕРИТЕ** инструкции.  
  
 **заключенный в кавычки идентификатор**  
 Идентификатор, заключенный в кавычки идентификатор, чтобы он может содержать специальные символы или совпадать с ключевыми словами (также известен в качестве идентификатора с разделителем — SQL-92).  
  
## <a name="r"></a>R  
 **Основание системы счисления**  
 Базовый номер системы. Обычно 2 или 10.  
  
 **Запись**  
 *См. в разделе* строки.  
  
 **результирующий набор**  
 Набор строк, создать, выполнив **ВЫБЕРИТЕ** инструкции.  
  
 **Код возврата**  
 Значение, возвращаемое с помощью функции ODBC.  
  
 **откат**  
 Для возврата значений, изменена транзакцией в исходное состояние.  
  
 **row**  
 Набор связанных столбцов, которые описывают определенной сущности. Также называется *записи*.  
  
 **Дескриптор строки**  
 Дескриптор, который описывает столбцы результирующего набора, либо до любого преобразования, который указан приложением (дескриптор строки реализации или IRD) или после любого преобразования, заданный приложением объект (дескриптор строки приложения или Отменить).  
  
 **массива операций строк**  
 Массив, содержащий значения, которые приложение может задать для указания, что соответствующая строка должны пропускаться в **SQLSetPos** операции.  
  
 **Массив статусов строк**  
 Массив, содержащий состояние строки после вызова **SQLFetch**, **SQLFetchScroll**, или **SQLSetPos**.  
  
 **Набор строк**  
 Набор строк, возвращенных блочного курсора в одну операцию выборки.  
  
 **буферы строк**  
 Буферы, привязанные к столбцам результирующего набора, и в котором возвращаются данные для всего набора строк.  
  
## <a name="s"></a>S  
 **СОСТОЯНИЕ СПАДА**  
 *См. в разделе* группу доступа SQL (состояние СПАДА).  
  
 **Скалярная функция**  
 Функция, которая создает одно значение из одного значения. Например функция, которая меняет регистр символьных данных.  
  
 **schema**  
 *См. в разделе* каталога.  
  
 **Прокручиваемый курсор**  
 Курсор, можно перемещаться вперед и назад из результирующего набора.  
  
 **Возможность сериализации**  
 Результат, двух транзакций, выполняемых одновременно, является ли так же, как последовательный (или последовательных) выполнения этих транзакций. Сериализуемые транзакции должны поддерживать целостность базы данных.  
  
 **База данных сервера**  
 СУБД, предназначен для запуска в среде клиент сервер. Эти СУБД предоставляют механизм автономную базу данных, который предоставляет богатые возможности для SQL и транзакций. Они доступны через драйверы на основе СУБД. Например Oracle, Informix, DB/2 или Microsoft® SQL Server.  
  
 **функция set**  
 *См. в разделе* агрегатной функции.  
  
 **Библиотека DLL программы установки**  
 *См. в разделе* DLL-файлов установки драйвера *и* translator DLL-файлов установки.  
  
 **Одноуровневая драйвера**  
 *См. в разделе* драйверов на основе файла.  
  
 **SQL**  
 Язык структурированных запросов. Это язык, используемый реляционных баз данных для запроса, обновления и управления данными.  
  
 **Группу доступа SQL (состояние СПАДА)**  
 Отраслевой Консорциум из компаний с СУБД SQL. Интерфейс уровня вызова Open Group основан на работы, выполненной в группу доступа SQL изначально.  
  
 **Уровень соответствия SQL**  
 Уровне грамматики SQL-92, поддерживаемых драйвером; может быть запись, FIPS Transitional, промежуточного или Full.  
  
 **Тип данных SQL**  
 Тип данных столбца или параметра, как оно хранится в источнике данных.  
  
 **SQLSTATE**  
 Пяти символов значение, указывающее конкретной ошибки.  
  
 **Инструкция SQL**  
 Полный фразы в SQL, которая начинается с ключевым словом и полностью описывает действия, которые должны выполняться. Например, ВЫБЕРИТЕ * из заказов. Инструкции SQL не следует путать с инструкциями.  
  
 **state**  
 Условие четко определенный элемент. Например соединение имеет семь состояний, включая нераспределенного, выделенное, подключенных и требующая данных. Некоторые операции могут быть выполнены только в том случае, когда элемент находится в определенном состоянии. Например соединение может быть освобожден только в том случае, если это в выделенный состоянии и не, например, когда он находится в подключенном состоянии.  
  
 **переход состояния**  
 Перемещение элемента из одного состояния в другое. ODBC определяет строгих переходов между состояниями для сред, соединений и инструкций.  
  
 **инструкция**  
 Контейнер для всей информации, относящейся к инструкции SQL. Инструкции не следует путать с инструкциями SQL.  
  
 **Дескриптор инструкции**  
 Дескриптор к структуре данных, содержащий сведения об инструкции.  
  
 **статический курсор**  
 Прокручиваемый курсор, который не удается обнаружить обновления, удаляет или вставляет в результирующем наборе. Обычно реализуется путем копирования результирующего набора.  
  
 **Статический SQL**  
 Тип embedded SQL в SQL, который инструкции жестко и компилируется при компиляции остальные части программы. *См. также* динамического SQL.  
  
 **Хранимая процедура**  
 *См. в разделе* процедуры.  
  
## <a name="t"></a>T  
 **table**  
 Коллекция строк.  
  
 **преобразование**  
 Преобразование 16-разрядного адреса для 32-разрядные адреса (или наоборот), при использовании 16-разрядных приложений с 32-разрядные драйверы ODBC.  
  
 **Транзакции**  
 Атомарная единица работы. Работа в рамках транзакции, которые должны выполняться в целом; При сбое любой части транзакции, вся транзакция завершается ошибкой.  
  
 **уровень изоляции транзакции**  
 Act изоляции одна транзакция, от воздействия других транзакций.  
  
 **уровень изоляции транзакции**  
 Мера того, насколько хорошо изолирована транзакция. Существует пять уровней изоляции транзакций: Read Uncommitted, чтение Committed, Repeatable Read, Serializable и управления версиями.  
  
 **библиотеки DLL перевода**  
 Библиотека DLL используется для преобразования данных из одной кодировки в другую.  
  
 **Библиотека DLL программы установки переводчика**  
 Библиотеку DLL, содержащую translator специальные функции для установки и настройки.  
  
 **двухфазная фиксация**  
 Процесс фиксации распределенной транзакции состоит из двух этапов. На первом этапе обработчика транзакций проверяет, что все части транзакции может быть зафиксирован. На втором этапе фиксируются все части транзакции. Если любой части транзакции на первом этапе указывает, что он не может быть зафиксирована, на втором этапе не происходит. ODBC поддерживает двухфазной фиксации.  
  
 **Индикатор типа**  
 Целочисленное значение, передаваемое или возвращается из функции ODBC для указания типа данных с переменной, параметром или столбцом. ODBC определяет тип индикаторы для типов данных C и SQL.  
  
## <a name="v"></a>V  
 **представление**  
 Альтернативный способ просмотра данных в одной или нескольких таблиц. Представления обычно создается как подмножество столбцов из одной или нескольких таблиц. В ODBC представления обычно эквивалентны, в таблицы.
