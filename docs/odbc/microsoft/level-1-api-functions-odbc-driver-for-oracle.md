# <a name="level-1-api-functions-odbc-driver-for-oracle"></a>Функции API уровня 1 (драйвер ODBC для Oracle)
> [!IMPORTANT]  
>  Эта функция будет удалена в следующей версии Windows. Избегайте использования этого компонента в новых разработках и запланируйте изменение существующих приложений, в которых он применяется. Вместо этого используйте драйвер ODBC, предоставляемый Oracle.  
  
 Функции на этом уровне обеспечивают базовое соответствие интерфейсов и дополнительные функциональные возможности, такие как поддержка транзакций.  
  
|API, функция|Примечания|  
|------------------|-----------|  
|**SQLColumns**|Создает результирующий набор для таблицы, которая является списком столбцов для указанной таблицы или таблиц. При запросе столбцов для открытого синонима необходимо задать атрибут соединения СИНОНИМКОЛУМНС и указать пустую строку в качестве аргумента *сзтаблеовнер* . При возврате столбцов для открытых синонимов драйвер присваивает столбцу имени таблицы пустую строку. Результирующий набор содержит дополнительный столбец с порядковым номером в конце каждой строки. Это значение является порядковым номером столбца в таблице.|  
|**SQLDriverConnect**|Подключается к существующему источнику данных. Дополнительные сведения см. в разделе [Формат и атрибуты строки подключения](../../odbc/microsoft/connection-string-format-and-attributes.md).|  
|**SQLGetConnectOption**|Возвращает текущее значение параметра соединения. Эта функция поддерживается частично. Драйвер поддерживает все значения аргумента *параметром fOption* , но не поддерживает некоторые значения *впарам* для аргумента *параметром fOption* [SQL_TXN_ISOLATION](../../odbc/microsoft/connect-options.md). Дополнительные сведения см. в разделе [Параметры подключения](../../odbc/microsoft/connect-options.md).|  
|**SQLGetData**|Получает значение одного поля в текущей записи заданного результирующего набора.|  
|**SQLGetFunctions**|Возвращает значение TRUE для всех поддерживаемых функций. Реализуется диспетчером драйверов.|  
|**SQLGetInfo**|Возвращает сведения, включая СКЛХДБК, СКЛУСМАЛЛИНТ, СКЛПОИНТЕР, SQLSMALLINT и SQLSMALLINT \*, о драйвере ODBC для Oracle и источнике данных, связанном с маркером подключения, *хдбк*.|  
|**SQLGetStmtOption**|Возвращает текущее значение параметра инструкции. Дополнительные сведения см. в разделе [Параметры инструкции](../../odbc/microsoft/statement-options.md).|  
|**SQLGetTypeInfo**|Возвращает сведения о типах данных, поддерживаемых источником данных. Драйвер возвращает сведения в результирующем наборе SQL.|  
|**SQLParamData**|Используется в сочетании с **SQLPutData** для указания данных параметра во время выполнения инструкции.|  
|**SQLPutData**|Позволяет приложению передавать данные для параметра или столбца в драйвер во время выполнения инструкции.|  
|**SQLSetConnectOption**|Предоставляет доступ к параметрам, регулирующим аспекты подключения. Эта функция поддерживается частично: драйвер поддерживает все значения для аргумента *параметром fOption* , но не поддерживает некоторые значения *впарам* для аргумента *параметром fOption* [SQL_TXN_ISOLATION](../../odbc/microsoft/connect-options.md). Дополнительные сведения см. в разделе [Параметры подключения](../../odbc/microsoft/connect-options.md).|  
|**SQLSetStmtOption**|Задает параметры, связанные с маркером инструкции, *хстмт*. Дополнительные сведения см. в разделе [Параметры инструкции](../../odbc/microsoft/statement-options.md).|  
|**SQLSpecialColumns**|Извлекает оптимальный набор столбцов, уникально идентифицирующих строку в таблице.|  
|**SQLStatistics**|Извлекает список статистических данных по одной таблице и индексам или именам тегов, связанным с таблицей. Драйвер возвращает сведения в виде результирующего набора.|  
|**SQLTables**|Возвращает список имен таблиц, указанных параметром в инструкции **SQLTables** . Если параметр не указан, возвращает имена таблиц, хранящихся в текущем источнике данных. Драйвер возвращает сведения в виде результирующего набора.<br /><br /> Вызовы типа перечисления не получат запись результирующего набора для удаленных представлений или локальных параметризованных представлений. Однако при вызове **SQLTables** с описателем уникального имени таблицы будет найдено совпадение для такого представления (если оно есть) с таким именем; Это позволяет API проверять конфликты имен перед созданием новой таблицы.<br /><br /> ОТКРЫТЫЕ синонимы возвращаются с TABLE_OWNER значением "".<br /><br /> ПРЕДСТАВЛЕНИЯ, принадлежащие SYS или SYSTEM, определяются как СИСТЕМное представление.|
