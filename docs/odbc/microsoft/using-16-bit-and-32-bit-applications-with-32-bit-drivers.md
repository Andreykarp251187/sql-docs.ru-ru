# <a name="using-16-bit-and-32-bit-applications-with-32-bit-drivers"></a>Использование 16-разрядных и 32-разрядных приложений с 32-разрядными драйверами
> [!IMPORTANT]  
>  Поддержка 16-разрядных приложений будет поддерживаться в будущих версиях Windows. Избегайте использования этого компонента в новых разработках и запланируйте изменение существующих приложений, в которых он применяется. Вместо разработки 32-разрядная или 64-разрядных приложений.  
  
 С помощью компонента доступа к данным ODBC можно использовать 16-разрядных и 32-разрядных приложений с 32-разрядные драйверы. ОС Microsoft® Windows® 95/98 и Microsoft Windows и Windows 2000 поддерживают следующие сочетания приложения и драйверы:  
  
-   16-разрядных приложений с 32-разрядными драйверами  
  
-   32-разрядных приложений с 32-разрядными драйверами  
  
 Использование 32-разрядное приложение с 16-разрядных драйверов не поддерживается.  
  
> [!NOTE]  
>  Начиная с выпуска ODBC версии 3.0, Windows NT 4.0 поддерживается.  
  
 ODBC включает в себя компоненты ODBC, необходимые для поддержки приведенных выше конфигураций «преобразования» библиотеки динамической компоновки (DLL) для преобразования 16-разрядные адреса в 32-разрядные адреса и наоборот. Программа установки определяет, какая операционная система, вы используете и устанавливает компоненты ODBC, необходимой для этой системы. Можно также установить компоненты ODBC, используемые всех систем.  
  
 В большинстве случаев перенос приложения или драйвера из 16-разрядное в 32-разрядное включает в себя пять типов изменений:  
  
-   Изменения в коде обработки сообщений  
  
-   Изменения, так как целые числа и дескрипторы являются 32 бита  
  
-   Изменения в вызовах Windows прикладного программирования (API)  
  
-   Изменения, чтобы сделать драйвер поточно ориентированными  
  
-   Изменение компонентов ODBC  
  
 С приложение или драйвер программирования точки зрения основное отличие между 16-разрядных и 32-разрядных компонентов ODBC — что они имеют разные имена. С точки зрения системы архитектура каждого подключения, приложение или драйвер, и средства, используемые для управления источниками данных отличаются.
