# <a name="using-16-bit-and-32-bit-applications-with-32-bit-drivers"></a>Использование 16-разрядных и 32-разрядных приложений с 32-разрядными драйверами
> [!IMPORTANT]  
>  16-битная поддержка приложений будет удалена в будущей версии Windows. Избегайте использования этого компонента в новых разработках и запланируйте изменение существующих приложений, в которых он применяется. Вместо этого разработайте 32-разрядные или 64-разрядные приложения.  
  
 С компонентом доступа к данным ODBC можно использовать 16-разрядные и 32-разрядные приложения с 32-битными драйверами. Операционные системы Microsoft®Windows® 95/98 и Microsoft Windows NT®/Windows 2000 поддерживают следующие комбинации приложений и драйверов:  
  
-   16-разрядные приложения с 32-битными драйверами  
  
-   32-разрядные приложения с 32-битными драйверами  
  
 Использование 32-разрядного приложения с 16-разрядным драйвером не поддерживается.  
  
> [!NOTE]  
>  Начиная с выпуска версии ODBC 3.0, Windows NT 4.0 была поддержана.  
  
 ODBC включает в себя компоненты ODBC, необходимые для поддержки вышеуказанных конфигураций, "стуча" библиотекдинамической ссылки (DLL) для преобразования 16-битных адресов в 32-битные адреса и наоборот. Программа настройки определяет, какую операционную систему вы используете, и устанавливает компоненты ODBC, необходимые этой системой. Вы также можете установить компоненты ODBC, используемые всеми системами.  
  
 В большинстве случаев перенос приложения или драйвера с 16-битного на 32-битный включает в себя пять типов изменений:  
  
-   Изменения в коде обработки сообщений  
  
-   Изменения, потому что все это и рукоятки 32 бита  
  
-   Изменения в вызовах к интерфейсам программирования приложений Windows (AA)  
  
-   Изменения, чтобы сделать драйвер безопасности потока  
  
-   Изменения в компонентах ODBC  
  
 С точки зрения программирования приложений или драйверов, основное различие между 16-битными и 32-битными компонентами ODBC заключается в том, что они имеют разные имена файлов. С точки зрения системы архитектура каждого приложения или соединения драйвера отличается, а инструменты, используемые для управления источниками данных, различны.
