# <a name="connecting-to-a-data-source-odbc-driver-for-oracle"></a>Подключение к источнику данных (драйвер ODBC для Oracle)
> [!IMPORTANT]  
>  Эта функция будет удалена в следующей версии Windows. Избегайте использования этого компонента в новых разработках и запланируйте изменение существующих приложений, в которых он применяется. Вместо этого используйте драйвер ODBC, предоставляемый Oracle.  
  
 Приложение ODBC может передавать сведения о соединении несколькими способами. Например, приложение может иметь драйвер, который всегда запрашивает у пользователя сведения о подключении. Или приложение может ожидать строку подключения, указывающую соединение с источником данных. Способ подключения к источнику данных зависит от метода подключения, используемого приложением ODBC.  
  
 Одним из распространенных способов подключения к источнику данных является диалоговое окно «Источник данных». Если приложение ODBC настроено для использования диалогового окна, откроется диалоговое окно с запросом на ввод соответствующих сведений о соединении с источником данных.  
  
 Можно также подключиться к источнику данных, используя [строку подключения](../../odbc/microsoft/connection-string-format-and-attributes.md).  
  
### <a name="to-connect-to-a-data-source-using-a-dialog-box"></a>Подключение к источнику данных с помощью диалогового окна  
  
1.  Когда откроется диалоговое окно Источник данных, выберите источник данных Oracle и нажмите кнопку ОК. Откроется диалоговое окно Подключение.  
  
2.  Заполните соответствующие сведения в диалоговом окне Подключение и нажмите кнопку ОК.  
  
 После проверки сведений о подключении приложение может использовать драйвер ODBC для Oracle для доступа к информации, содержащейся в источнике данных.
