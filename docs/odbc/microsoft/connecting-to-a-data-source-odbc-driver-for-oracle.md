# <a name="connecting-to-a-data-source-odbc-driver-for-oracle"></a>Подключение к источнику данных (драйвер ODBC для Oracle)
> [!IMPORTANT]  
>  Эта функция будет удалена в будущей версии Windows. Избегайте использования этого компонента в новых разработках и запланируйте изменение существующих приложений, в которых он применяется. Вместо этого используйте драйвер ODBC, предоставленный Oracle.  
  
 Приложение ODBC может передавать информацию о связи несколькими способами. Например, в приложении может быть всегда драйвер для получения информации о связи с пользователем. Или приложение может ожидать строку соединения, которая определяет подключение источника данных. Подключение к источнику данных зависит от метода подключения, используемого вашим приложением ODBC.  
  
 Одним из распространенных способов подключения к источнику данных является диалоговый ящик Источника данных. Если приложение ODBC настроено на использование диалогового окна, это диалоговое окно отображается и подсказывает вам соответствующую информацию о подключении источника данных.  
  
 Вы также можете подключиться к источнику данных с помощью [строки соединения.](../../odbc/microsoft/connection-string-format-and-attributes.md)  
  
### <a name="to-connect-to-a-data-source-using-a-dialog-box"></a>Подключение к источнику данных с помощью диалогового окна  
  
1.  При пояснении диалогового окна источника кода данных выберите источник данных Oracle, а затем нажмите OK. Появляется диалоговая коробка Connect.  
  
2.  Заполните соответствующую информацию для диалогового окна Connect, а затем нажмите OK.  
  
 После проверки информации о подключении приложение может использовать драйвер ODBC для Oracle для доступа к информации, содержащейся в источнике данных.
