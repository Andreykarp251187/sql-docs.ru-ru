# <a name="sqlconfigdatasource-access-driver"></a>SQLConfigDataSource (драйвер для Access)
> [!NOTE]  
>  Эта тема содержит информацию о драйверах доступа. Для получения общей информации об этой [ODBC API Reference](../../odbc/reference/syntax/odbc-api-reference.md)функции, см.  
  
 Функция **S'LConfigDataSource,** используемая для добавления, изменения или удаления источника данных, динамически использует следующие ключевые слова.  
  
|Ключевое слово|Описание|  
|-------------|-----------------|  
|КОЛЛАТИВНАЯПОЗА|Последовательность, в которой сортируются поля.<br /><br /> Это устанавливает тот же вариант, что и **сопоставление последовательности** в поле диалога установки.|  
|COMPACT_DB|Выполняет уплотнение данных в файле базы данных. Имеет следующий формат: COMPACT_DB<\<path_name><optionaL_sort_order>необязательный> по ключевому слову ENCRYPT.<br /><br /> При использовании ключевого слова COMPACT_DB в одном и том же заявлении с ключевым словом DSN, этот драйвер игнорирует ключевое слово DSN. Таким образом, уплотнение базы данных и указание DSN является двухэтапным процессом.|  
|CREATE_DB|Создает файл базы данных. Имеет следующий формат: CREATE_DB \<<path_name path_name>><optional_sort><optional_ENCRYPT optional_ENCRYPT ключевого> слова, где имя пути является полным путем к базе данных Microsoft Access. Ошибка будет возвращена, если имя пути определяет существующую базу данных. Порядок сортировки будет настроен в диалоговом окне новой базы данных, отображаемый при нажатии кнопки «Создание» в диалоговом поле Microsoft Access Setup. Если заказ сортировки не указан, используется General.<br /><br /> При использовании ключевого слова CREATE_DB в том же заявлении с ключевым словом DSN, этот драйвер игнорирует ключевое слово DSN. Таким образом, создание базы данных и указание DSN является двухэтапным процессом. При использовании CREATE_DB ключевого слова, если имя пути создаваемых баз данных Microsoft Access содержит одно или несколько пробелов, то все имя пути должно быть заключено двойными кавычками, как показано в следующих примерах:<br /><br /> "C:'PROGRAM FILES -COMMON FILES- MyAccess.mdb"<br /><br /> "C:'PROGRAM FILES-Access2.mdb"<br /><br /> CREATE_DB КК: «ТЕМП-тест.mdb» (не требуется кавычек)|  
|CREATE_SYSDB|Создает файл базы данных системы. Имеет следующий формат:\<CREATE_SYSDB- \<иных>>по маршруту, где имя пути является полным путем к базе данных Microsoft Access. Ошибка будет возвращена, если имя пути определяет существующую базу данных. Порядок сортировки будет настроен в диалоговом окне **новой базы данных,** отображаемый при нажатии кнопки **«Создание»** в диалоговом поле **ODBC Microsoft Setup.** Если заказ сортировки не указан, используется General.|  
|CREATE_V2DB|Создает файл базы данных, совместимый с Microsoft Access 2.0. Имеет следующий формат:\<CREATE_V2DB-имя \<пути>необязательный сортировка-заказ>, где имя пути является полным путем к базе данных Microsoft Access. Ошибка будет возвращена, если имя пути определяет существующую базу данных. Порядок сортировки будет настроен в диалоговом окне новой базы данных, отображаемый при нажатии кнопки «Создание» в диалоговом поле Microsoft Access Setup. Если заказ сортировки не указан, используется General.<br /><br /> При использовании CREATE_V2DB ключевого слова в том же заявлении с ключевым словом DSN, этот драйвер игнорирует ключевое слово DSN. Таким образом, создание базы данных и указание DSN является двухэтапным процессом.<br /><br /> При использовании ключевого слова CREATE_V2DB, если имя пути создаваемых баз данных Microsoft Access содержит одно или несколько пробелов, то все имя пути должно быть заключено двойными кавычками, как показано в следующих примерах:<br /><br /> "C:'PROGRAM FILES -COMMON FILES- MyAccess.mdb"<br /><br /> "C:'PROGRAM FILES-Access2.mdb"<br /><br /> CREATE_V2DB КК: «ТЕМП-тест.mdb» (не требуется кавычек)|  
|Dbq|Имя файла базы данных.<br /><br /> Это устанавливает тот же вариант, что **и база данных** в поле диалога установки.|  
|ПО УМОЛЧАНИДИР|Спецификация пути к файлу базы данных.|  
|DESCRIPTION|Описание данных в источнике данных.<br /><br /> При этом устанавливается та же опция, что и **описание** в поле диалога установки.|  
|DRIVER|Спецификация пути для драйвера DLL.|  
|ВОДИТЕЛЬ|Идентификатор разных чиханий для водителя.  25 (Доступ к Microsoft)|  
|Fil|Тип файла MS Access для Microsoft Access|  
|НЕЯВНАЯКОММИФИКАЦИЯ|Определяет, будет ли драйвер Microsoft Access выполнять внутренние или неявные коммиты асинхронно. Это значение первоначально настроено на "Да", что означает, что драйвер Microsoft Access будет ждать коммитов во внутренней/неявной транзакции, которая будет завершена.<br /><br /> Значение этого варианта не следует менять без тщательного рассмотрения последствий. Для получения дополнительной информации о опции, *см.*<br /><br /> Это устанавливает ту же опцию, что и **ImplicitCommitSync** в поле диалога установки.|  
|MAXBUFFERSIZE|Размер внутреннего буфера в килобайтах, который используется Microsoft Access для передачи данных на диск и с диска. Размер буфера по умолчанию составляет 2048 кБ (отображается как 2048). Можно использовать любое значение целого, делится на 256. Это устанавливает тот же вариант, что **и размер буфера** в поле диалога установки.|  
|МАКССКАНРОУВ|Количество строк, которые необходимо сканировать при настройке типа данных столбца на основе существующих данных.<br /><br /> Для сканирования строк можно ввести число от 1 до 16. Значение по умолчанию до 8; если он установлен до 0, все строки сканируются. (Номер за пределами лимита вернет ошибку.)<br /><br /> При этом устанавливается та же опция, что и **строки для сканирования** в поле диалога установки.|  
|PAGETIME OUT|Упоняет период времени, в миллисекундах, что страница (если не используется) остается в буфере перед удалением. По умолчанию пять десятых секунды (0,5 секунды). Обратите внимание, что эта опция применяется ко всем источникам данных, которые используют драйвер ODBC.<br /><br /> При этом устанавливается та же опция, что и **Тайм-аут Страницы** в поле диалога установки.|  
|PWD|Пароль.|  
|READONLY|TRUE, чтобы сделать файл только читать; FALSE, чтобы файл не только читать.<br /><br /> При этом устанавливается та же опция, что и **Read Only** в поле диалога установки.|  
|REPAIR_DB|Ремонт базы данных, поврежденной сбоем, который происходит во время процесса коммита.<br /><br /> При использовании ключевого слова REPAIR_DB в том же заявлении с ключевым словом DSN, этот драйвер игнорирует ключевое слово DSN. Таким образом, ремонт базы данных и указание DSN является двухэтапным процессом.|  
|SYSTEMDB|Для драйвера Microsoft Access спецификация пути к файлу базы данных системы.<br /><br /> Это устанавливает тот же вариант, что **и system Database** в поле диалога установки.|  
|Потоков|Количество фоновых потоков для использования двигателя. Это значение по умолчанию до 3, но может быть изменено.<br /><br /> При этом устанавливается та же опция, что и потоки в поле диалога **установки.**|  
|ИД пользователя|Для драйвера Microsoft Access имя идентификатора пользователя используется для входа.|  
|USERCOMMITSYNC|Определяет, будет ли драйвер Microsoft Access выполнять транзакции, определяемые пользователями, асинхронно. Это значение первоначально настроено на "Да", что означает, что драйвер Microsoft Access будет ждать фиксаций в транзакции, определяемой пользователем, которая будет завершена.<br /><br /> Значение этого варианта не следует менять без тщательного рассмотрения последствий. Для получения дополнительной информации о опции, *см.*<br /><br /> Это устанавливает ту же опцию, что **и UserCommitSync** в поле диалога установки.|
