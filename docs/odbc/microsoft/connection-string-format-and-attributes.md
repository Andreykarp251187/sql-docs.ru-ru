# <a name="connection-string-format-and-attributes"></a>Формат и атрибуты строки подключения
> [!IMPORTANT]  
>  Эта функция будет удалена в будущей версии Windows. Избегайте использования этого компонента в новых разработках и запланируйте изменение существующих приложений, в которых он применяется. Вместо этого используйте драйвер ODBC, предоставленный Oracle.  
  
 Вместо использования диалогового окна некоторым приложениям может потребоваться строка соединения, которая определяет информацию о подключении источника данных. Строка соединения состоит из ряда атрибутов, которые определяют, как драйвер подключается к источнику данных. Атрибут определяет определенную часть информации, которую водитель должен знать, прежде чем он сможет сделать соответствующее подключение источника данных. Каждый драйвер может иметь другой набор атрибутов, но формат строки соединения всегда одинаков. Строка соединения имеет следующий формат:  
  
```  
"DSN=data-source-name[;SERVER=value] [;PWD=value] [;UID=value] [;<Attribute>=<value>]"  
```  
  
> [!NOTE]  
>  Драйвер Microsoft ODBC для Oracle поддерживает формат строки соединения первой `CONNECTSTRING`версии `SERVER=`драйвера, который использовал вместо .  
  
 Если вы подключаетесь к поставщику источников данных, который поддерживает аутентификацию Windows, следует указать `Trusted_Connection=yes` вместо идентификатора пользователя и информацию о паролях в строке соединения.  
  
 Необходимо указать имя источника данных, если не указаны атрибуты UID, PWD, SERVER (или CONNECTSTRING) и DRIVER. Однако все остальные атрибуты не являются обязательными. Если вы не указали атрибут, то атрибут по умолчанию к тому, что указан в соответствующей вкладке DSN в диалоговом окне **администратора источников данных ODBC.** Значение атрибута может быть чувствительным к случаям.  
  
 Атрибуты строки соединения следующие:  
  
|Атрибут|Описание|Значение по умолчанию|  
|---------------|-----------------|-------------------|  
|DSN|Имя источника данных, указанное во вкладке Драйверы диалогового окна **администратора источников данных ODBC.**|""|  
|PWD|Пароль для Oracle Server, к который вы хотите получить доступ. Этот драйвер поддерживает ограничения, которые Oracle устанавливает на пароли.|""|  
|SERVER|Строка подключения для Oracle Server, к которой вы хотите получить доступ.|""|  
|ИД пользователя|Имя пользователя Oracle Server. В зависимости от вашей системы этот атрибут может быть необязательным, т.е. определенным базам данных и таблицам может потребоваться этот атрибут для целей безопасности.<br /><br /> Используйте "/" для использования аутентификации операционной системы Oracle.|""|  
|БУФЕРИЗЕ|Оптимальный размер буфера, используемый при извлечении столбцов.<br /><br /> Драйвер оптимизирует извлечения таким образом, чтобы один извлеченный из Oracle Server возвращает достаточно строк, чтобы заполнить буфер такого размера. Большие значения, как правило, увеличивают производительность, если вы получаете много данных.|65535|  
|СИНОНИМКОЛОМКОНС|Когда это значение верно (1), S'LColumn () API вызова возвращает информацию столбца. В противном случае, S'LColumn () возвращает только столбцы для таблиц и представлений. Драйвер ODBC для Oracle обеспечивает более быстрый доступ, когда это значение не установлено.|1|  
|ПРИМЕЧАНИЯ|Когда это значение верно (1), драйвер возвращает столбцы Remarks для набора [результатов S'LColumns.](../../odbc/microsoft/level-1-api-functions-odbc-driver-for-oracle.md) Драйвер ODBC для Oracle обеспечивает более быстрый доступ, когда это значение не установлено.|0|  
|StdDayOfWeek|Обеспечивает соблюдение стандарта ODBC для scalar DAYOFWEEK. По умолчанию это включено, но пользователи, которым нужна локализованная версия, могут изменить поведение, чтобы использовать все, что возвращает Oracle.|1|  
|GuessTheColDef|Уточняется, должен ли драйвер возвращать ненулевое значение для аргумента *cbColDef* компании **S'LDescribeCol.** Применяется только к столбивам, где нет шкалы, определяемой Oracle, таких как вычисленные числовые столбцы и столбцы, определяемые как NUMBER без точности или масштаба. Звонок **S'LDescribeCol** возвращает 130 за точность, когда Oracle не предоставляет эту информацию.|0|  
  
 Например, строка подключения к источнику данных MyDataSource с помощью сервера MyOracleServerOracle и Пользователя Oracle MyUserID будет:  
  
```  
"DSN={MyDataSource};UID={MyUserID};PWD={MyPassword};SERVER={MyOracleServer}"  
```  
  
 A connection string that connects to the MyOtherDataSource data source using operating system authentication and the MyOtherOracleServerOracle Server would be:  
  
```  
"DSN=MyOtherDataSource;UID=/;PWD=;SERVER=MyOtherOracleServer"  
```
