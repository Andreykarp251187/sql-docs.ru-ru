# <a name="dbase-indexes"></a>Индексы dBASE
Драйвер ODBC dBASE автоматически открывает и обновляет файлы индекса dBASE IV. Необходимо использовать диалоговый ящик **Select Indexes,** отображаемый через администратора источника данных ODBC, для ассоциации файлов dBASE III .ndx с файлами dBASE.  
  
 При создании индексов dBASE применяются следующие ограничения:  
  
-   Все имена столбцов должны быть действительными.  
  
-   Все колонны должны быть в одинаковом порядке восходящего или убывающего порядка.  
  
-   Длина любого отдельного столбца текста должна быть менее 100 байтов.  
  
-   Если существует несколько столбцов, все столбцы должны быть столбцов текста, а сумма размеров столбцов должна быть менее 100 байтов.  
  
-   Поля памятки не могут быть проиндексированы.  
  
-   Индекс не должен быть указан для текущего набора полей (т.е. дублирующие индексы не допускаются).  
  
-   Название индекса должно соответствовать конвенции о наименовании индекса dBASE. dBASE III требует, чтобы каждый индекс был в отдельном файле, каждый из которых имеет расширение .ndx. В dBASE IV индексы создаются как имена тегов, которые хранятся в одном файле .mdx. Файл .mdx имеет то же базовое имя, что и файл базы данных (например, Emp.mdx — это файл индекса для базы данных Emp.dbf).  
  
-   dBASE определяет уникальный индекс как индекс, в котором к индексу добавляется только одна запись из набора с идентичными значениями ключевых клавиш.
