# <a name="dbase-indexes"></a>Индексы dBASE
Драйвер для dBASE ODBC автоматически открывает и обновляет файлы dBASE IV индекса. Необходимо использовать **Выбор индексов** диалогового окна через ODBC администратора источника данных должен быть сопоставлен dBASE III .ndx файлы dBASE-файлов.  
  
 Для создания индексов dBASE применяются следующие ограничения:  
  
-   Все имена столбцов должны быть допустимыми.  
  
-   Все столбцы должны быть в том же по возрастанию или убыванию.  
  
-   Длина любого отдельного текстового столбца должен быть менее 100 байт.  
  
-   Если существует более одного столбца, все столбцы должны быть текстовые столбцы, и сумму размеров столбца должен быть менее 100 байт.  
  
-   Невозможно индексировать поля типа MEMO.  
  
-   Индекс не должен быть указан для текущего набора полей (то есть повторяющиеся индексы не допускаются).  
  
-   Имя индекса должно соответствовать соглашению об именовании индекс dBASE. dBASE III требуется каждого индекса в отдельном файле, каждая из которых имеет расширение .ndx. Индексы dBASE IV, создается как имена тегов, которые хранятся в одном MDX-файлу. MDX-файла имеет такое же базовое имя файла базы данных (например, Emp.mdx является файл индекса базы данных Emp.dbf).  
  
-   dBASE определяется уникальный индекс как неделя, где добавляется только одной записи из набора с одинаковыми значениями ключа индекса.
