# <a name="create-table---sql-command"></a>CREATE TABLE (команда SQL)
Создает таблицу с указанными полями.  
  
 Visual FoxPro ODBC Driver поддерживает родной визуальный синтаксис языка FoxPro для этой команды. Для получения информации о конкретной для водителя **см.**  
  
## <a name="syntax"></a>Синтаксис  
  
```  
  
CREATE TABLE | DBF TableName1 [NAME LongTableName] [FREE]  
   (FieldName1FieldType [(nFieldWidth [, nPrecision])]  
      [NULL | NOT NULL]   
      [CHECK lExpression1 [ERROR cMessageText1]]  
      [DEFAULT eExpression1]  
      [PRIMARY KEY | UNIQUE]  
      [REFERENCES TableName2 [TAG TagName1]]  
      [NOCPTRANS]  
   [, FieldName2 ...]  
      [, PRIMARY KEY eExpression2 TAG TagName2  
      |, UNIQUE eExpression3 TAG TagName3]  
      [, FOREIGN KEY eExpression4 TAG TagName4 [NODUP]  
            REFERENCES TableName3 [TAG TagName5]]  
      [, CHECK lExpression2 [ERROR cMessageText2]])  
| FROM ARRAY ArrayName  
```  
  
## <a name="arguments"></a>Аргументы  
 CREATE TABLE &#124; DBF *TableName1*  
 Упогоняет название таблицы для создания. Параметры TABLE и DBF идентичны.  
  
 ИМЯ *LongTableName*  
 Уотволяет длинное название для таблицы. Длинное имя таблицы может быть указано только при открытии базы данных, поскольку длинные имена таблиц хранятся в базах данных.  
  
 Длинные имена могут содержать до 128 символов и могут быть использованы вместо коротких имен файлов в базе данных.  
  
 FREE  
 Упомянет, что таблица не будет добавлена в открытую базу данных. БЕСПЛАТНО не требуется, если база данных не открыта.  
  
 *(FieldName1 FieldType* *(nFieldWidth,* *nPrecision)*  
 Определяет название поля, тип поля, ширину поля и точность поля (количество десятичных мест), соответственно.  
  
 *FieldType* — это единая буква, указывающая [тип данных](../../odbc/microsoft/visual-foxpro-field-data-types.md)поля. Некоторые типы полевых данных требуют, чтобы вы указали *nFieldWidth* или *nPrecision* или оба.  
  
 *nFieldWidth* и *nPrecision* игнорируются для типов D, G, I, L, M, P, T и Y. *nPrecision* по умолчанию до нуля (без десятичных мест), если *nPrecision* не включен для типов B, F или N.  
  
 NULL  
 Позволяет нулевые значения в поле.  
  
 NOT NULL  
 Предотвращает нулевую величину в поле.  
  
 Если вы не опускаете NULL и NOT NULL, текущая настройка SET NULL определяет, разрешены ли нулевые значения в поле. Однако, если вы опускаете NULL и NOT NULL и включаете положение PRIMARY KEY или УНИУГВЕ, текущая настройка SET NULL игнорируется, а поле по умолчанию НЕ NULL.  
  
 CHECK *lExpression1*  
 Определяет правило проверки для поля. *lExpression1* может быть пользовательо-определенной функцией. Всякий раз, когда придатится к пустой записи, правило проверки проверяется. Ошибка генерируется, если правило проверки не допускает пустое значение поля в приложеной записи.  
  
 ERROR *cMessageText1*  
 Определяет сообщение об ошибке, отображаемый Visual FoxPro, когда правило поля генерирует ошибку. Сообщение отображается только при изменении данных в окне просмотра или в окне edit.  
  
 ПО *УМОЛЧАНИЯ eExpression1*  
 Определяет значение по умолчанию для поля. Тип данных *eExpression1* должен быть таким же, как и тип данных поля.  
  
 PRIMARY KEY  
 Создает первичный индекс для поля. Тег первичного индекса имеет то же имя, что и поле.  
  
 UNIQUE  
 Создает индекс кандидата для поля. Тег индекса кандидата имеет то же имя, что и поле.  
  
> [!NOTE]  
>  Кандидатские индексы (созданные путем включения опции УНИКУЕ в CREATE TABLE или ALTER TABLE - S'L) не совпадают с индексами, созданными с опцией УНИУГЕВКЕ в команде INDEX. Индекс, созданный с помощью опции УНИКУЕ в команде INDEX, позволяет дублировать клавиши индекса; индексы кандидатов не позволяют дублировать клавиши индекса. [Дополнительную](../../odbc/microsoft/index-command.md) информацию о опции УНИУСИ см.  
  
 Значения null и дубликаты записей не допускаются в поле, используемом для первичного индекса или индекса кандидатов. Однако Visual FoxPro не будет генерировать ошибку, если вы создаете первичный или кандидатский индекс для поля, поддерживающего нулевые значения. Visual FoxPro будет генерировать ошибку, если вы попытаетесь ввести нулевую или дублирующую значение в поле, используемое для первичного или кандидата индекса.  
  
 СПРАВКА *ТаблицаНай2*»TAG *TagName1»*  
 Определяет таблицу родителей, к которой устанавливается постоянные отношения. Если вы не упускаете *TAGTagName1,* отношение устанавливается с помощью основного ключа индекса родительской таблицы. Если родительская таблица не имеет основного индекса, Visual FoxPro генерирует ошибку.  
  
 Включите TAG *TagName1* для установления отношения на основе существующего тега индекса для родительской таблицы. Имена индексных тегов могут содержать до 10 символов.  
  
 Таблица родителей не может быть свободной таблицей.  
  
 НОКПТРАНС  
 Предотвращает перевод на другую страницу кода для полей символов и памяток. Если таблица преобразуется в другую страницу кода, поля, для которых указан NOCPTRANS, не переводятся. NOCPTRANS может быть указан только для символов и памятки полей.  
  
 Следующий пример создает таблицу под названием mytable, содержащую два поля символов и два поля памятки. Второе поле символов, char2, и второе поле памятки, memo2, включают NOCPTRANS для предотвращения перевода.  
  
```  
CREATE TABLE mytable (char1 C(10), char2 C(10) NOCPTRANS,;  
   memo1 M, memo2 M NOCPTRANS)  
```  
  
 PRIMARY KEY *eExpression2* TAG *TagName2*  
 Определяет первичный индекс для создания. *eExpression2* определяет любое поле или комбинацию полей в таблице. TAG *TagName2* определяет имя созданного основного тега индекса. Имена индексных тегов могут содержать до 10 символов.  
  
 Поскольку таблица может содержать только один основной индекс, вы не можете включить это положение, если вы уже создали основной индекс для поля. Visual FoxPro генерирует ошибку, если вы включаете более одного пункта PRIMARY KEY в CREATE TABLE.  
  
 УНИКУЕ *eExpression3*TAG *TagName3*  
 Создает индекс кандидата. *eExpression3* определяет любое поле или комбинацию полей в таблице. Однако, если вы создали первичный индекс с одним из вариантов PRIMARY KEY, вы не можете включить поле, которое было определено для основного индекса. TAG *TagName3* указывает имя тега для созданного тега кандидата. Имена индексных тегов могут содержать до 10 символов.  
  
 Таблица может иметь несколько индексов кандидатов.  
  
 FOREIGN KEY *eExpression4*TAG *TagName4*(NODUP)  
 Создает иностранный (непервичный) индекс и устанавливает связь с родительской таблицей. *eExpression4* определяет иностранное выражение ключа индекса, а *TagName4* определяет название создаваемого тега ключа иностранного индекса. Имена индексных тегов могут содержать до 10 символов. Включите NODUP для создания иностранного индекса кандидата.  
  
 Можно создать несколько иностранных индексов для таблицы, но в иностранных выражениях индекса должны быть указаны различные поля в таблице.  
  
 СПРАВКА *ТаблицаНа3*»TAG *TagName5*»  
 Определяет таблицу родителей, к которой устанавливается постоянные отношения. Включите TAG *TagName5* для установления отношения, основанного на теге индекса для родительской таблицы. Имена индексных тегов могут содержать до 10 символов. По умолчанию, если вы не отключите TAG *TagName5,* отношение устанавливается с помощью основного индексного ключа родительской таблицы.  
  
 CHECK *eExpression2*(ERROR *cMessageText2)*  
 Опознавательный упор на правило проверки таблицы. ERROR *cMessageText2* определяет сообщение об ошибке, отображаемое Visual FoxPro при выполнении правила проверки таблицы. Сообщение отображается только при изменении данных в окне просмотра или изменении.  
  
 ИЗ МАССИВА *ArrayName*  
 Упогоняет имя существующего массива, содержимое которого является именем, типом, точностью и масштабом для каждого поля в таблице. Содержимое массива может быть определено с помощью функции **AFIELDS**()  
  
## <a name="remarks"></a>Remarks  
 Новая таблица открыта в самой низкой доступной рабочей области и может быть доступна под псевдонимом. Новая таблица открыта исключительно, независимо от текущей настройки SET EXCLUSIVE.  
  
 Если база данных открыта и вы не включаете пункт БЕСПЛАТНО, новая таблица добавляется в базу данных. Невозможно создать новую таблицу с тем же именем, что и таблица в базе данных.  
  
 Если база данных открыта, CREATE TABLE - S'L требует исключительного использования базы данных. Чтобы открыть базу данных для эксклюзивного использования, включите EXCLUSIVE в OPEN DATABASE.  
  
 Если база данных не открыта при создании новой таблицы, включая name, CHECK, DEFAULT, FOREIGN KEY, PRIMARY KEY или ссылканационные положения, генерируют ошибку.  
  
> [!NOTE]  
>  СИНтаксис CREATE TABLE использует запятые для разделения определенных параметров CREATE TABLE. Кроме того, NULL, NOT NULL, CHECK, DEFAULT, PRIMARY KEY и уНИУГВЕПункт должны быть помещены в скобки, содержащие определения столбца.  
  
## <a name="driver-remarks"></a>Замечания водителя  
 Когда приложение отправляет заявление ODBC S'L CREATE TABLE в источник данных, драйвер Visual FoxPro ODBC переводит команду в команду Visual FoxProCREATE TABLE с помощью синтаксиса, показанного в следующей таблице.  
  
|Синтаксис ODBC|Визуальный синтаксис FoxPro|  
|-----------------|--------------------------|  
|CREATE TABLE *базового стола-имя*<br /><br /> (тип*данных столбцов-идентификатора*<br /><br /> (НЕ NULL)<br /><br /> Тип*данных столбцов-идентификатора*<br /><br /> (НЕ NULL) ...)|CREATE *TABLETableName1* (ИМЯ *LongTableName)*<br /><br /> (*FieldName1* *FieldType*<br /><br /> *(nFieldWidth,* *nPrecision))*<br /><br /> (НЕ NULL) )|  
  
 При создании таблицы с помощью драйвера драйвер закрывает таблицу сразу после создания, чтобы разрешить доступ к таблице другим пользователям. Это отличается от Visual FoxPro, который оставляет стол открытым исключительно после создания. Однако, если сохраненная процедура на источнике данных, содержащая заявление CREATE TABLE, выполняется, таблица остается открытой.  
  
 Если источником данных является база данных (.dbc файл), Visual FoxPro ODBC Driver создает таблицу под названием *LongTableName* с тем же именем, что и *имя базового стола.*  
  
### <a name="using-data-definition-language-ddl"></a>Использование языка определения данных (DDL)  
 Вы не можете включить DDL в следующих местах:  
  
-   В пакетной выписке по S'L, которая требует транзакции  
  
-   В режиме ручного коммита, после оператора, требующего транзакции, если приложение сначала не вызывает **S'LTransact.**  
  
 Например, если требуется создать временную таблицу, необходимо создать таблицу перед началом оператора, требующего транзакции. Если вы включите выписку CREATE TABLE в пакетную выписку по S'L, которая требует транзакции, водитель возвращает сообщение об ошибке.
