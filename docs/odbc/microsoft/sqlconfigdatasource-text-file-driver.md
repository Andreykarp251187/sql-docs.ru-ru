# <a name="sqlconfigdatasource-text-file-driver"></a>SQLConfigDataSource (драйвер для текстовых файлов)
> [!NOTE]  
>  В этом разделе содержатся сведения, относящиеся к драйверам текстовых файлов. Общие сведения об этой функции см. в соответствующем разделе [справочника по API ODBC](../../odbc/reference/syntax/odbc-api-reference.md).  
  
 Функция **SQLConfigDataSource** , которая используется для динамического добавления, изменения или удаления источника данных, использует следующие ключевые слова.  
  
|Ключевое слово|Описание|  
|-------------|-----------------|  
|CHARACTERSET|Для текстового драйвера OEM или ANSI.|  
|колнамехеадер|Для текстового драйвера указывает, будут ли в первой записи данных указываться имена столбцов. Либо TRUE, либо FALSE.|  
|дефаултдир|Спецификация пути к каталогу.|  
|DESCRIPTION|Описание данных в источнике данных.<br /><br /> Этот параметр устанавливается так же, как и **Описание** в диалоговом окне установки.|  
|DRIVER|Спецификация пути к библиотеке DLL драйвера.|  
|дриверид|Целочисленный идентификатор драйвера. 27 (текст)|  
|РАСШИРЕНИЙ|Выводит список расширений имен файлов текстовых файлов в источнике данных.<br /><br /> Этот параметр задается в диалоговом окне установки того же параметра, что и **список расширений** .|  
|ФАЙ|Текст типа файла|  
|FILETYPE|Тип файла для драйвера текста (текст).|  
|FORMAT|Для драйвера текста может быть FIXEDLENGTH, ТАБДЕЛИМИТЕД, CSVDELIMITED (запятая) или с РАЗДЕЛИТЕЛями () (специальным символом, указанным в круглых скобках). Специальный символ — это один символ в длину, который может быть в символьном, десятичном или шестнадцатеричном формате.|  
|макссканровс|Число строк, просматриваемых при задании типа данных столбца на основе существующих данных.<br /><br /> Для текстового драйвера можно ввести число от 1 до 32767 в качестве числа строк для проверки. Тем не менее значение по умолчанию всегда равно 25. (Число за пределами ограничения будет возвращать ошибку.)<br /><br /> Это задает тот же параметр, что и **строки для проверки** в диалоговом окне программы установки.|  
|READONLY|Значение TRUE, чтобы сделать файл доступным только для чтения; Значение FALSE, чтобы сделать файл недоступным только для чтения.<br /><br /> Этот параметр задается в диалоговом окне установки того же параметра, что и « **только чтение** ».|
