# <a name="core-level-api-functions-odbc-driver-for-oracle"></a>Функции API основного уровня (драйвер ODBC для Oracle)
> [!IMPORTANT]  
>  Эта функция будет удалена в будущей версии Windows. Избегайте использования этого компонента в новых разработках и запланируйте изменение существующих приложений, в которых он применяется. Вместо этого используйте драйвер ODBC, предоставленный Oracle.  
  
 Функции на этом уровне включают минимальный уровень соответствия интерфейса для водителей ODBC.  
  
|Функция API|Примечания|  
|------------------|-----------|  
|**СЗЛАллокКоннект**|Выделяет память для ручки соединения, *HDbc,* в среде, идентифицированной *henv*. Менеджер драйвера обрабатывает этот вызов и вызывает функцию **драйвера S'LAllocConnect** всякий раз, когда называется **s'LConnect,** **S'LBrowseConnect**или **S'LDriverConnect.**|  
|**СЗЛАллокЕнв**|Отображает диалоговую коробку с указанием требования к программному обеспечению Oracle Client, а затем возвращает SQL_NULL_HANDLE. Если программное обеспечение Oracle Client не установлено, эта функция выделяет память для обработки среды, *henv*и инициализирует интерфейс уровня вызова ODBC для использования приложением.|  
|**СЗЛАлокСтмт**|Выделяет память для ручки оператора и связывает ручку оператора с соединением, указанным hdbc. Менеджер драйвера передает этот вызов водителю, который выделяет память для структуры hstmt.|  
|**SQLBindCol**|Назначает место для хранения результата и определяет тип результата.|  
|**SQLCancel**|Отменяет обработку на ручке оператора, hstmt. В некоторых случаях Oracle не допускает отмену оператора, работая. Это означает, что запущенное заявление будет продолжаться до тех пор, пока Oracle не завершит процесс, после чего результаты из инструкций будут аннулированы драйвером ODBC для Oracle.|  
|**СЗЛКолАтрибуты**|Возвращает информацию о дескрипторе для столбца в наборе результатов. Информация о дескрипторе возвращается в виде строки символов, 32-битного значения, зависящей от дескриптора, или в целых значениях.|  
|**SQLConnect**|Подключается к источнику данных. Для использования Oracle операционная система аутентификации, указать "/" в качестве параметра *szUID* и "" в качестве параметра *szAuthStr.*|  
|**SQLDescribeCol**|Возвращает имя, тип, точность, масштаб и недействительность данного столбца результата. **Примечание: Отчеты S'LDescribeCol** вычисляют столбцы как SQL_VARCHAR.|  
|**СЗЛДИСКоннект**|Закрывает соединение. Если объединение соединения включено для общей среды, а приложение вызывает **S-LDisconnect** на соединение в этой среде, соединение возвращается в пул соединения и по-прежнему доступно для других компонентов, использующих ту же общую среду.|  
|**Sqlerror**|Возвращает информацию об ошибке или статусе последней ошибки. Драйвер поддерживает стек или список ошибок, которые могут быть возвращены для *hstmt,* *hdbc*, и *henv* аргументы, в зависимости от того, как вызов **на S'LError** производится. Очередь ошибки смывается после каждой выписки. Обычно получает сообщение об ошибке Oracle и в противном случае пуст.|  
|**SQLExecDirect**|Выполняет новое, неподготовленное заявление S'L. Драйвер использует текущие значения переменных параметра, если в отчете существуют какие-либо параметры. Если в таблице, представлении или поле в названиях содержатся пробелы, приложи имена в задние кавычки. Например, если база данных содержит таблицу под названием *My Table* и поле *My Field,* прикройте каждый элемент идентификатора следующим образом:<br /><br /> SELECT \`Мой\`стол . \`Мое поле1\`,; \`Мой\`стол . \`Мое\` поле2 \`от моего стола\`|  
|**SQLExecute**|Выполняет подготовленное заявление S'L (заявление, уже подготовленное **S'LPrepare).** Драйвер использует текущие значения переменных параметра, если в отчете существуют какие-либо параметры.|  
|**SQLFetch**|Извлекает одну строку из результата, установленного в местах, указанных предыдущими вызовами в **S'LBindCol.** Подготавливает драйвер для вызова в **S'LGetData** для несвязанных столбцов.|  
|**СЗЛФриКоннект**|Выпускает ручку соединения и освобождает всю память, выделенную для ручки.|  
|**СЗЛФриЕнв**|Закрывает драйвер ODBC для Oracle и выпускает всю память, связанную с драйвером.|  
|**Функция SQLFreeStmt**|Останавливает обработку, связанную с определенным hstmt, закрывает все открытые курсоры, связанные с hstmt, отбрасывает ожидающие результатов и дополнительно освобождает все ресурсы, связанные с ручкой оператора.|  
|**SQLGetCursorName**|Возвращает имя курсора, связанного с данным hstmt.|  
|**SQLNumResultCols**|Возвращает количество столбцов в курсоре набора результатов.|  
|**SQLPrepare**|Подготавливает выписку по S'L, планируя, как оптимизировать и выполнить выписку. Заявление о СЗЛ составляется для выполнения по данным **sLExecDirect**.<br /><br /> Если в таблице, представлении или поле в названиях содержатся пробелы, приложи имена в задние кавычки. Например, если база данных содержит таблицу под названием *My Table* и поле *My Field,* приложить каждый элемент идентификатора следующим образом:<br /><br /> SELECT \`Мой\`стол . \`Мое\` поле \`от моего стола\`<br /><br /> Для получения информации об использовании наборов результатов, содержащих массивы в качестве формальных параметров, [см.](../../odbc/microsoft/returning-array-parameters-from-stored-procedures.md)|  
|**SQLRowCount**|Oracle не предоставляет способ определения количества строк в наборе результатов до тех пор, пока вы не заберете последнюю строку, поэтому он не вернет -1.|  
|**СЗЛСетКурсОрНамейм**|Associates имя курсора с активной ручкой оператора, *hstmt*.|  
|**СЗЛСетПарам**|Заменена на S'LBindParameter в ODBC 2. *x*.|  
|**СЗЛТрансакт**|Запрашивает операцию фиксации или отката для всех активных операций на всех ручках оператора (hstmts), связанных с соединением, или для всех соединений, связанных с обработкой среды, *henv.* Если коммит завершается в ручном режиме, транзакция остается активной; вы можете откатить транзакцию или повторить операцию фиксации. Если операция коммита завершается сбой, когда в автоматическом режиме транзакции транзакция откатывается автоматически; транзакция не может быть неактивной.|
