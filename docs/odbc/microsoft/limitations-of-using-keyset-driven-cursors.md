# <a name="limitations-of-using-keyset-driven-cursors"></a>Ограничения на использование курсоров, управляемых ключевым набором
> [!IMPORTANT]  
>  Эта функция будет удалена в будущей версии Windows. Избегайте использования этого компонента в новых разработках и запланируйте изменение существующих приложений, в которых он применяется. Вместо этого используйте драйвер ODBC, предоставленный Oracle.  
  
 Необходимо, чтобы вы могли получить одну колонку ROWID для запрашиваемой таблицы. Курсор, управляемый клавиатурой, не может использоваться в соединениях, запросах или заявлениях, содержащих положения DISTINCT, GROUP BY, UNION, INTERSECT или MINUS.  
  
 Кроме того, если приложение использует псевдонимы таблицы, курсоры на клавишу не будут работать; требуется только вперед или статические типы курсоров. Использование типа курсора клавиатуры с псевдонимами таблицы вызовет следующую ошибку: «Драйвер ODBC для Oracle не может использовать курсор, управляемый Keyset, при соединении, пересекаясь или в минусе или при чтении только набор результатов».  
  
> [!NOTE]  
>  Из-за того, как драйвер обрабатывает выписку по S'L, отправляемую на сервер Oracle, Oracle внутренне возвращает следующее сообщение об ошибке: «ORA-00964: название таблицы не в списке FROM».
