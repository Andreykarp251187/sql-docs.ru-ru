# <a name="create-index-for-paradox"></a>CREATE INDEX для Paradox
Синтаксис инструкции CREATE INDEX для драйвера ODBC Paradox:  
  
 **CREATE** [**уникальный**] Индекс **индекса** *— имя*  
  
 **Для** *Table-name*  
  
 **(** *столбец-Идентификатор* [**ASC**]  
  
 [**,** *столбец-Идентификатор* [**ASC**]...] **)**  
  
 Драйвер ODBC Paradox не поддерживает ключевое слово **DESC** в грамматике ODBC SQL для инструкции CREATE INDEX. Аргумент *Table-name* может указывать полный путь к таблице.  
  
 Если указано ключевое слово **UNIQUE** , драйвер ODBC Paradox создаст уникальный индекс. Первый уникальный индекс создается как первичный индекс. Это файл первичного ключа Paradox с именем *Table-name*. Пиксел. На основные индексы распространяются следующие ограничения.  
  
-   Первичный индекс должен быть создан до добавления строк в таблицу.  
  
-   Первичный индекс должен быть определен по первым столбцам "n" в таблице.  
  
-   Для каждой таблицы допускается только один первичный индекс.  
  
-   Невозможно обновить таблицу с помощью драйвера Paradox, если для таблицы не определен первичный индекс. (Обратите внимание, что это не верно для пустой таблицы, которую можно обновить, даже если для таблицы не определен уникальный индекс).  
  
-   Аргумент *index-Name* для первичного индекса должен совпадать с базовым именем таблицы, как это требуется Paradox.  
  
 Если ключевое слово **UNIQUE** опущено, драйвер ODBC Paradox создаст неуникальный индекс. Это состоит из двух вторичных файлов индекса Paradox с именем *Table-name*. X*nn* и *Table-name*. Y*nn*, где *nn* — номер столбца в таблице. К неуникальным индексам применяются следующие ограничения.  
  
-   Чтобы создать неуникальный индекс для таблицы, для этой таблицы должен существовать первичный индекс.  
  
-   Для Paradox 3. *x*, аргумент *index-Name* для любого индекса, кроме первичного индекса (уникального или неуникального), должен совпадать с именем столбца. Для Paradox 4. *x* и 5. *x*, имя такого индекса может быть, но не должно совпадать с именем столбца.  
  
-   Для неуникального индекса можно указать только один столбец.  
  
 Невозможно добавить столбцы, если для таблицы определен индекс. Если первый столбец списка аргументов инструкции CREATE TABLE создает индекс, второй столбец не может быть добавлен в список аргументов.  
  
 Например, чтобы использовать столбцы "номер заказа на продажу" и "номер строки" в качестве уникального индекса в SO_LINES таблице, используйте инструкцию:  
  
```  
CREATE UNIQUE INDEX SO_LINES  
 ON SO_LINES (SONum, LineNum)  
```  
  
 Чтобы использовать столбец номер детали в качестве неуникального индекса в SO_LINES таблице, используйте инструкцию:  
  
```  
CREATE INDEX PartNum  
 ON SO_LINES (PartNum)  
```  
  
 Обратите внимание, что при выполнении двух инструкций CREATE INDEX первая инструкция всегда создает первичный индекс с тем же именем, что и таблица, а вторая инструкция всегда создает неуникальный индекс с тем же именем, что и у столбца. Эти индексы будут называться таким образом, даже если в инструкциях CREATE INDEX заданы разные имена и даже если во второй инструкции CREATE INDEX индекс имеет метку UNIQUE.  
  
> [!NOTE]  
>  При использовании драйвера Paradox без реализации ядро СУБД Borland разрешены только инструкции Read и Append.
