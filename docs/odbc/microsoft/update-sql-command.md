# <a name="update---sql-command"></a>UPDATE (команда SQL)
Обновление записей в таблице с новыми значениями.  
  
 Visual FoxPro ODBC Driver поддерживает родной визуальный синтаксис языка FoxPro для этой команды. Для получения информации о конкретной для водителя **см.**  
  
## <a name="syntax"></a>Синтаксис  
  
```  
  
UPDATE [DatabaseName1!]TableName1  
SET Column_Name1 = eExpression1  
   [, Column_Name2 = eExpression2 ...]  
   WHERE FilterCondition1 [AND | OR FilterCondition2 ...]  
```  
  
## <a name="arguments"></a>Аргументы  
 ОБНОВЛЕНИЕ - *База данныхName1!* *ТаблицаНай1*  
 Осведивает таблицу, в которой записи обновляются новыми значениями.  
  
 *База данныхНаи1!* определяет название базы данных, кроме базы данных, указанной с источником данных, содержащим таблицу. Необходимо включить имя базы данных, содержащей таблицу, если база данных не является текущей. Включите восклицательный знак (!) делимитетер после имени базы данных и перед именем таблицы.  
  
 SET *Column_Name1*= *eExpression1*, *Column_Name2*= *eExpression2*  
 Определяет обновленные столбцы и их новые значения. Если вы не упускаете пункт WHERE, каждая строка в столбе обновляется с тем же значением.  
  
 ГДЕ *ФильтрУсловие1*»И &#124; ИЛИ *FilterCondition2*...»  
 Определяет записи, обновляемые новыми значениями.  
  
 *FilterCondition* определяет критерии, которым должны соответствовать записи, чтобы быть обновленными с новыми значениями. Вы можете включить столько условий фильтра, сколько хотите, соединяя их с оператором AND или OR. Вы также можете использовать NOT оператора, чтобы изменить значение логического выражения, или вы можете использовать **EMPTY**( ) для проверки пустого поля.  
  
## <a name="remarks"></a>Remarks  
 ОБНОВЛЕНИЕ - S'L может обновлять только записи в одной таблице.  
  
 В отличие от REPLACE, UPDATE - S'L использует блокировку записей при обновлении нескольких записей в таблицах, открытых для общего доступа. Это уменьшает взаимодействие записей в многопользовательских ситуациях, но может снизить производительность. Для максимальной производительности откройте стол для эксклюзивного использования или используйте **FLOCK**() для блокировки стола.  
  
## <a name="driver-remarks"></a>Замечания водителя  
 Когда приложение отправляет заявление ODBC S'L UPDATE в источник данных, водитель Visual FoxPro ODBC преобразует команду в команду Visual FoxProUPDATE без перевода.
