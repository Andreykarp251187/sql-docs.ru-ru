# <a name="set-collate-command"></a>Команда SET COLLATE
Определяет последовательность сопоставления полей символов в последующих операциях по индексации и сортировке.  
  
## <a name="syntax"></a>Синтаксис  
  
```  
  
SET COLLATE TO cSequenceName  
```  
  
## <a name="arguments"></a>Аргументы  
 *cПоследовательностьНам*  
 Определяет последовательность сопоставления. Доступные параметры последовательности коллайда описаны в следующей таблице.  
  
|Параметры|Язык|  
|-------------|--------------|  
|нидерландский|Голландский|  
|GENERAL|Английский, французский, немецкий, современный испанский, португальский и другие западноевропейские языки|  
|немецкий|Немецкий заказ на телефонную книгу (DIN)|  
|Исландия|Исландский|  
|Машина|Машина (последовательность сопоставления по умолчанию для более ранних версий FoxPro)|  
|НОРДАН|Норвежский, датский|  
|испанский|Традиционный испанский|  
|СВЕФИН|Шведский, финский|  
|УНИКВТ|Уникальный вес|  
  
> [!NOTE]  
>  Когда вы указываете вариант SPANISH, *ch* является одной буквой, которая сортирует между *c* и *d,* и *ll* сортов между *л* и *м*.  
  
 Если вы указываете вариант последовательности коллажа в буквальном виде строки символов, не забудьте приложить опцию в кавычки:  
  
```  
SET COLLATE TO "SWEFIN"  
```  
  
 MACHINE — это параметр последовательности коллаций по умолчанию и представляет собой последовательность Xbase, с которую пользователи знакомы. Персонажи упорядочены по мере их появления на текущей странице кода.  
  
 GENERAL может быть предпочтительнее для пользователей США и Западной Европы. Персонажи упорядочены по мере их появления на текущей странице кода. В версиях FoxPro раньше 2.5 индексы могли быть созданы с помощью функций **UPPER**() или **LOWER (LOWER**() для преобразования полей символов в согласованный случай. В версиях FoxPro позже, чем 2.5, вы можете вместо этого указать опцию general коллажа последовательности и опустить **UPPER**( ) преобразования.  
  
 Если вы указываете опцию последовательности коллайда, кроме MACHINE, и если вы создаете файл .idx, всегда создается компактный .idx.  
  
 Используйте SET ("COLLATE"), чтобы вернуть текущую последовательность коллажа.  
  
 Вы можете указать последовательность сопоставления для источника данных с помощью [ODBC Visual FoxPro Настройка Диалог Box](../../odbc/microsoft/odbc-visual-foxpro-setup-dialog-box.md) или с помощью ключевого слова Collate в строке соединения с [S'LDriverConnect.](../../odbc/microsoft/sqldriverconnect-visual-foxpro-odbc-driver.md) Это идентично выдаче следующей команды:  
  
```  
SET COLLATE TO cSequenceName  
```  
  
## <a name="remarks"></a>Remarks  
 SET COLLATE позволяет заказывать таблицы, содержащие акцентированные символы для любого из поддерживаемых языков. Изменение настройки SET COLLATE не влияет на последовательность сопоставления ранее открытых индексов. Visual FoxPro автоматически поддерживает существующие индексы, обеспечивая гибкость для создания множества различных типов индексов, даже для одного и того же поля.  
  
 Например, если индекс создается с SET COLLATE, установленным на GENERAL, а параметр SET COLLATE позже изменен на SPANISH, индекс сохраняет последовательность сопоставления GENERAL.
