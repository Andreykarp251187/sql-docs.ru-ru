# <a name="setting-options-programmatically-for-the-text-file-driver"></a>Программная настройка параметров драйвера текстовых файлов

|Параметр|Описание|Метод|  
|------------|-----------------|------------|  
|Имя базы данных-источника|Имя, идентифицирующее источник данных, например зарплату или персонал.|Чтобы динамически установить этот параметр, используйте ключевое слово **DSN** в вызове [SQLConfigDataSource](../../odbc/microsoft/sqlconfigdatasource-text-file-driver.md).|  
|Определить формат|Отображает диалоговое окно **Определение формата текста** и позволяет указать схему для отдельных таблиц в каталоге источника данных.|Этот параметр нельзя задать динамически при вызове [SQLConfigDataSource](../../odbc/microsoft/sqlconfigdatasource-text-file-driver.md).|  
|Описание|Необязательное описание данных в источнике данных; Например, "Дата найма, история заработной платы и текущий пересмотр всех сотрудников".|Чтобы динамически установить этот параметр, используйте ключевое слово **Description** в вызове [SQLConfigDataSource](../../odbc/microsoft/sqlconfigdatasource-text-file-driver.md).|  
|Каталог|Выбирает целевой каталог.|Чтобы динамически установить этот параметр, используйте ключевое слово **дефаултдир** в вызове [SQLConfigDataSource](../../odbc/microsoft/sqlconfigdatasource-text-file-driver.md).|  
|Список расширений|Выводит список расширений имен файлов текстовых файлов в источнике данных. При использовании текстового драйвера файл без расширения создается при выполнении инструкции CREATE TABLE с именем, которое не имеет расширения. Другие драйверы создают файл с расширением по умолчанию, если расширение не указано. Чтобы создать файл с расширением txt, необходимо добавить расширение в имя. Отображение файлов без расширений в диалоговом окне « **Определение формата текста** », «*» необходимо добавить в список расширений.|Чтобы динамически установить этот параметр, используйте ключевое слово **Extensions** в вызове [SQLConfigDataSource](../../odbc/microsoft/sqlconfigdatasource-text-file-driver.md).|  
|Только для чтения|Определяет базу данных как доступную только для чтения.|Чтобы динамически установить этот параметр, используйте ключевое слово **ReadOnly** в вызове [SQLConfigDataSource](../../odbc/microsoft/sqlconfigdatasource-text-file-driver.md).|  
|Строки для проверки|Число строк для просмотра, по которым определяется тип данных каждого столбца. Тип данных определяется с учетом максимального числа найденных типов данных. Если обнаружены данные, не соответствующие типу данных, предполагаемому для столбца, то тип данных будет возвращен как значение NULL.<br /><br /> Для текстового драйвера можно ввести число от 1 до 32767 в качестве числа строк для проверки. Тем не менее значение по умолчанию всегда равно 25. (Число за пределами ограничения будет возвращать ошибку.)|Чтобы динамически установить этот параметр, используйте ключевое слово **макссканровс** в вызове [SQLConfigDataSource](../../odbc/microsoft/sqlconfigdatasource-text-file-driver.md).|  
|Выбор каталога|Отображает диалоговое окно, в котором можно выбрать каталог, содержащий файлы, к которым требуется получить доступ.<br /><br /> При определении каталога источника данных укажите каталог, в котором находятся наиболее часто используемые файлы. Драйвер ODBC использует этот каталог в качестве каталога по умолчанию. Если они часто используются, скопируйте другие файлы в этот каталог. Кроме того, можно уточнить имена файлов в инструкции SELECT с помощью имени каталога: `SELECT * FROM C:\MYDIR\EMP`<br /><br /> Можно также указать новый каталог по умолчанию с помощью функции **SQLSetConnectOption** с параметром SQL_CURRENT_QUALIFIER.|Чтобы динамически установить этот параметр, используйте ключевое слово **дефаултдир** в вызове [SQLConfigDataSource](../../odbc/microsoft/sqlconfigdatasource-text-file-driver.md).|
