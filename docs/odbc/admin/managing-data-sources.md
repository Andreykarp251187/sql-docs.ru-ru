# <a name="managing-data-sources"></a>Управление источниками данных
После установки драйвера ODBC из программы установки драйвера можно определить один или несколько источников данных для него. Имя источника данных (DSN) должно предоставить уникальное описание данных; например, *платежная ведомость* или *счета, подаемые*под плату. Источники пользовательских и системных данных, определяемые для всех установленных в настоящее время драйверов, перечислены во вкладке **пользователя DSN** или **System DSN** в диалоговом окне **администратора источников данных ODBC.** Источники файловв данной каталога перечислены во вкладке **Файл DSN;** каталог, который будет показан, вводится в поле **Look in** the box во вкладке **Файл DSN.**  
  
> [!NOTE]  
>  Для управления источником данных, который подключается к 32-битной платформе под 64-битной платформой, используйте c: 'sysWOW64'odbcad32.exe. Для управления источником данных, который подключается к 64-битной драйвер, используйте c:'windows'system32-odbcad32.exe. В **административных инструментах** на 64-разрядной операционной системе Windows 8 есть значки как для 32-битного, так и для 64-битного диалогового ящика **ODBC Data Source Administrator.**  
  
 Если вы используете 64-разрядный odbcad32.exe для настройки или удаления DSN, который подключается к 32-битного драйвера, например, **Driver do Microsoft Access (.mdb),\*** вы получите следующее сообщение об ошибке:  
  
```  
The specified DSN contains an architecture mismatch between the Driver and Application  
```  
  
 Чтобы устранить эту ошибку, используйте 32-разрядный odbcad32.exe для настройки или удаления DSN.  
  
 Источник данных связывает определенный драйвер ODBC с данными, к которым вы хотите получить доступ через этот драйвер. Например, можно создать источник данных для использования драйвера ODBC dBASE для доступа к одному или несколько файлам dBASE, найденным в определенном каталоге на жестком диске или сетевом диске. Используя администратора источника данных ODBC, можно добавлять, изменять и удалять источники данных, как описано в следующей таблице.  
  
|Действие|Описание|  
|------------|-----------------|  
|Добавление источников данных|Можно добавить несколько источников данных, каждый из которых связывая драйвер с некоторыми данными, к которым вы хотите получить доступ, используя этот драйвер. Дайте каждому источнику данных имя, которое однозначно идентифицирует этот источник данных. Например, если вы создаете источник данных для набора файлов dBASE, содержащих информацию о клиентах, можно назвать источник данных "Клиенты". Приложения обычно отображают имена источников данных для пользователей на выбор.<br /><br /> Добавление источника данных файла несколько отличается от добавления источников пользовательских или системных данных. Для получения дополнительной информации обратитесь в файл справки от администратора источника данных ODBC.|  
|Изменение источников данных|В зависимости от ваших требований может возникнуть необходимость в перенастройке источников данных. Вы можете сбросить параметры, нажав **на настройку** в любом диалоговом окне установки драйвера.|  
|Удаляние источников данных|Нажмите **Удалить** после выбора источника данных.|  
  
 Для получения дополнительной информации об источниках файлов можно [ознакомиться на сведениях об источниках файлов connecting Using File Data Sources](../../odbc/reference/develop-app/connecting-using-file-data-sources.md) или [функции S'LDriverConnect.](../../odbc/reference/syntax/sqldriverconnect-function.md)
