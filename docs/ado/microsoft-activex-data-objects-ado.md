# <a name="microsoft-activex-data-objects-ado"></a>Объекты данных Microsoft ActiveX (ADO)

ADO используется в программах на C++ для подключения к SQL Server. Само собой она также работает для подключения к базе данных SQL Azure в облаке.

Каждый раздел в этой статье описывает компонент ADO.

> [!NOTE]
> ADO.NET отличается от ADO. Описание ADO.NET и многих других драйверов подключения SQL, а также их языков см. в разделе [Драйверы SQL Server](../connect/sql-connection-libraries.md).

  
## <a name="ado"></a>ADO  
 Объекты данных ActiveX (ADO) включения клиентские приложения для доступа и управления данными из различных источников через поставщик OLE DB. Простота использования, высокой скорости, издержки нехватки памяти и места на диске с небольшой его основных преимущества: ADO поддерживает основные функции для создания клиента и сервера и веб-приложений.  
  
## <a name="ado-md"></a>ADO MD  
 Объекты данных Microsoft ActiveX (многомерные выражения) (многомерные Объекты ADO) обеспечивает простой доступ к многомерным данным из языков, например Microsoft Visual Basic и Microsoft Visual C++. ADO MD расширяет Microsoft ActiveX Data объектов (ADO) для включения объектов, определенных для многомерных данных, таких как объекты CubeDef и набора ячеек. С ADO MD Обзор многомерной схемой, запроса куба и получения результатов.  
  
 Как и ADO ADO MD использует поставщика OLE DB для получения доступа к данным. Для работы с ADO MD, поставщик должен быть поставщик многомерных данных (MDP), согласно спецификации OLE DB для OLAP. MDPs представление данных в многомерных представлений, в отличие от поставщиков табличных данных (за), представляющих данные в табличных представлениях. См. в документации по поставщику OLE DB для OLAP, более подробные сведения о синтаксисе конкретных и поведений, поддерживаемых поставщиком.  
  
## <a name="rds"></a>СЛУЖБА УДАЛЕННЫХ РАБОЧИХ СТОЛОВ  
 Служба Remote Data (RDS) — это функция ADO, с помощью которого можно перемещать данные с сервера клиентское приложение или веб-странице, работы с данными на стороне клиента и возвращать обновления на сервер в рамках одного цикла обработки.  
  
> [!IMPORTANT]
>  Начиная с Windows 8 и Windows Server 2012, серверные компоненты служб удаленных рабочих СТОЛОВ, больше не включаются в операционной системе Windows (см. в разделе Windows 8 и [настольная книга по совместимости Windows Server 2012](https://www.microsoft.com/download/details.aspx?id=27416) для получения дополнительных сведений). Клиентские компоненты служб удаленных рабочих СТОЛОВ будет поддерживаться в будущих версиях Windows. Избегайте использования этого компонента в новых разработках и запланируйте изменение существующих приложений, в которых он применяется. Приложения, использующие служб удаленных рабочих СТОЛОВ, следует перевести [WCF-сервиса данных](https://go.microsoft.com/fwlink/?LinkId=199565).  
  
## <a name="adox"></a>ADOX  
 Объекты расширения данных Microsoft ActiveX для языка описания данных и безопасности (ADOX) — это расширение объектов ADO и модель программирования. ADOX содержит объекты для создания схем и изменения, а также безопасности. Поскольку это объектно ориентированный подход для операций со схемой, можно написать код, который будет работать для различных данных источников независимо от различий в их собственном вариантов синтаксиса.  
  
 ADOX — это вспомогательная Библиотека базовых объектов ADO. Он предоставляет дополнительные объекты для создание, изменение и удаление объектов схемы, например таблицы и процедуры. Она также включает объекты безопасности для обслуживания пользователей и групп, а также для предоставления или отмены разрешения на объекты.
