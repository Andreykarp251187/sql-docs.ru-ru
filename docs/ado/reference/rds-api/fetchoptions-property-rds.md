# <a name="fetchoptions-property-rds"></a>Свойство FetchOptions (служба удаленных рабочих столов)
Указывает тип асинхронной выборки.  
  
> [!IMPORTANT]
>  Начиная с Windows 8 и Windows Server 2012, серверные компоненты служб удаленных рабочих СТОЛОВ, больше не включаются в операционной системе Windows (см. в разделе Windows 8 и [настольная книга по совместимости Windows Server 2012](https://www.microsoft.com/download/details.aspx?id=27416) для получения дополнительных сведений). Клиентские компоненты служб удаленных рабочих СТОЛОВ будет поддерживаться в будущих версиях Windows. Избегайте использования этого компонента в новых разработках и запланируйте изменение существующих приложений, в которых он применяется. Приложения, использующие служб удаленных рабочих СТОЛОВ, следует перевести [WCF-сервиса данных](https://go.microsoft.com/fwlink/?LinkId=199565).  
  
## <a name="setting-and-return-values"></a>Параметр и возвращаемые значения  
 Задает или возвращает одно из следующих значений.  
  
|Константа|Описание|  
|--------------|-----------------|  
|**adcFetchUpFront**|Все записи из [записей](../../../ado/reference/ado-api/recordset-object-ado.md) обрабатываются до возврата управления в приложение. Полный **записей** извлекается, прежде чем приложение сможет делать ничего с ней.|  
|**adcFetchBackground**|Элемент управления может возвращать в приложение сразу же после первого пакета записей были получены. Последующая чтение **записей** попытки доступа к записи не извлечь в первого пакета будут откладываться, пока фактически извлечения искомого записи, в этот момент управление возвращается в приложение.|  
|**adcFetchAsync**|По умолчанию. Управление немедленно возвращается приложению, пока записи получены в фоновом режиме. Если приложение пытается прочитать запись, которая еще не выбраны, ближайший к искомому запись записи будут считываться и управления немедленно возвращает управление, означает, что текущий конец **записей** был достигнут. Например, вызов [MoveLast](../../../ado/reference/rds-api/movefirst-movelast-movenext-and-moveprevious-methods-rds.md) переместит положения текущей записи к последней записи фактически извлеченных, несмотря на то, что большее число записей будет продолжать заполнения **записей**.|  
  
> [!NOTE]
>  Каждый клиентский исполняемый файл, который использует эти константы необходимо предоставить объявления для них. Можно вырезать и вставить объявления констант из файла Adcvbs.inc, расположенный в папке установки по умолчанию для библиотеки служб удаленных рабочих СТОЛОВ.  
  
## <a name="remarks"></a>Примечания  
 В веб-приложении, обычно требуется использовать **adcFetchAsync** (значение по умолчанию), так как он обеспечивает более высокую производительность. В скомпилированный клиентское приложение, обычно требуется использовать **adcFetchBackground**.
