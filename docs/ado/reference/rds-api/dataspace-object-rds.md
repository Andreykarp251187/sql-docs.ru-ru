# <a name="dataspace-object-rds"></a>Объект DataSpace (служба удаленных рабочих столов)
> [!IMPORTANT]
>  Начиная с Windows 8 и Windows Server 2012, серверные компоненты служб удаленных рабочих СТОЛОВ, больше не включаются в операционной системе Windows (см. в разделе Windows 8 и [настольная книга по совместимости Windows Server 2012](https://www.microsoft.com/download/details.aspx?id=27416) для получения дополнительных сведений). Клиентские компоненты служб удаленных рабочих СТОЛОВ будет поддерживаться в будущих версиях Windows. Избегайте использования этого компонента в новых разработках и запланируйте изменение существующих приложений, в которых он применяется. Приложения, использующие служб удаленных рабочих СТОЛОВ, следует перевести [WCF-сервиса данных](https://go.microsoft.com/fwlink/?LinkId=199565).  
  
 Создает клиентский прокси для пользовательских бизнес-объектов, расположенных на среднем уровне.  
  
 Служба Remote Data Service должна бизнеса объект прокси клиентские компоненты могут обмениваться данными с бизнес-объектов, расположенных на среднем уровне. Учетные записи-посредники упрощения упаковки, распаковке и транспорта (маршалинг) приложения [записей](../../../ado/reference/ado-api/recordset-object-ado.md) данных через границы процесса или компьютера.  
  
 Служба Remote Data Service использует **RDS. Пространство данных** объекта [CreateObject](../../../ado/reference/rds-api/createobject-method-rds.md) метод для создания объекта бизнеса учетных записей-посредников. Объект прокси бизнеса динамически создается каждый раз, когда создается экземпляр его аналога среднего уровня бизнес-объекта. Служба Remote Data Service поддерживает следующие протоколы: HTTP, HTTPS (HTTP Secure Sockets), DCOM и в процессе (клиентские компоненты и бизнес-объекта, которые находятся на одном компьютере).  
  
> [!NOTE]
>  Служба удаленных рабочих СТОЛОВ «без отслеживания состояния» функционирует так же при **RDS. Пространство данных** объект использует протокол HTTP или HTTPS. То есть любые внутренние сведения о запросе клиента удаляется после сервер возвращает ответ.  
  
> [!NOTE]
>  Несмотря на то, что бизнес-объект, появляются в течение времени существования объекта бизнеса прокси-сервера, бизнес-объект, фактически существует только в том случае, пока будет отправлен ответ на запрос. Если выдается запрос (то есть метод вызывается в бизнес-объекте), прокси-сервер открывает новое подключение к серверу, и сервер создает новый экземпляр бизнес-объекта. После бизнес-объект ответа на запрос, сервер уничтожает бизнес-объект и закрывает соединение.  
  
> [!NOTE]
>  Это означает, что невозможно передать данные от одного запроса в другой с помощью свойства объекта бизнеса или переменной. Необходимо использовать другой механизм, например файла или аргумента метода, для сохранения данных состояния.  
  
 Идентификатор класса для **RDS. Пространство данных** BD96C556-65A3 - 11 D 0-983A-00C04FC29E36 является объект.  
  
 **DataSpace** объект безопасен для скриптов.
