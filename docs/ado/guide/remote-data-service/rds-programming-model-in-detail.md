# <a name="rds-programming-model-in-detail"></a>Подробные сведения о модели программирования RDS
Ниже приведены ключевые элементы модели программирования служб удаленных рабочих СТОЛОВ.  
  
-   RDS. Пространство данных  
  
-   RDSServer.DataFactory  
  
-   RDS. DataControl  
  
-   Событие  
  
> [!IMPORTANT]
>  Начиная с Windows 8 и Windows Server 2012, серверные компоненты служб удаленных рабочих СТОЛОВ, больше не включаются в операционной системе Windows (см. в разделе Windows 8 и [настольная книга по совместимости Windows Server 2012](https://www.microsoft.com/download/details.aspx?id=27416) для получения дополнительных сведений). Клиентские компоненты служб удаленных рабочих СТОЛОВ будет поддерживаться в будущих версиях Windows. Избегайте использования этого компонента в новых разработках и запланируйте изменение существующих приложений, в которых он применяется. Приложения, использующие служб удаленных рабочих СТОЛОВ, следует перевести [WCF-сервиса данных](https://go.microsoft.com/fwlink/?LinkId=199565).  
  
## <a name="rdsdataspace"></a>RDS. Пространство данных  
 Клиентское приложение необходимо указать сервер и сервер программу, вызываемую. В ответ приложение получает ссылку на программу сервера и можно считать ссылку, как будто самой программы server.  
  
 Объектная модель служб удаленных рабочих СТОЛОВ реализует функциональность, совместно с [RDS. Пространство данных](../../../ado/reference/rds-api/dataspace-object-rds.md) объекта.  
  
 Программа server указан с идентификатором программы или *ProgID*. Сервер использует *ProgID* и реестра на компьютере сервера, чтобы найти сведения о самой программы для запуска.  
  
 Служба удаленных рабочих СТОЛОВ проводится различие внутренне зависимости от того, программа server на удаленном сервере через Интернет или интрасети. на сервер в локальной сети; или не на сервере, но вместо этого на локальную библиотеку динамической компоновки (DLL). Это различие определяет способ сведения передаются между клиентом и сервером и материальные разницы в тип ссылки, возвращается в клиентское приложение. Тем не менее с вашей точки зрения, это различие не имеет специального значения. Все это означает, что вы получили ссылку можно использовать программы.  
  
## <a name="rdsserverdatafactory"></a>RDSServer.DataFactory  
 RDS предоставляет серверной программы по умолчанию, который можно выполнить SQL-запрос к источнику данных и возврат [набор записей](../../../ado/reference/ado-api/recordset-object-ado.md) объекта или воспользоваться **набор записей** объекта и обновления источника данных.  
  
 Объектная модель служб удаленных рабочих СТОЛОВ реализует функциональность, совместно с [RDSServer.DataFactory](../../../ado/reference/rds-api/datafactory-object-rdsserver.md) объекта.  
  
 Кроме того, этот объект имеет метод для создания пустой **записей** объект, который можно заполнить программным образом ([CreateRecordset](../../../ado/reference/rds-api/createrecordset-method-rds.md)) и другой метод для преобразования **набора записей**  объекта в текстовую строку для создания веб-страницы ([ConvertToString](../../../ado/reference/rds-api/converttostring-method-rds.md)).  
  
 С помощью ADO, вы можете переопределить некоторые из стандартное подключение и команду поведение **RDSServer.DataFactory** с **DataFactory** обработчик и файл настройки, содержащий соединение, команды и параметры безопасности.  
  
 Программа server иногда называют *бизнес-объект*. Можно написать собственные пользовательские бизнес-объект, который может выполнять доступ к сложным данным, проверок допустимости и т. д. Даже в том случае, при написании пользовательского бизнес-объекта, можно создать экземпляр **RDSServer.DataFactory** объекта и использовать некоторые из его методов для выполнения своих собственных задач.  
  
## <a name="rdsdatacontrol"></a>RDS. DataControl  
 Служба удаленных рабочих СТОЛОВ позволяет объединить функциональные возможности **RDS. Пространство данных** и **RDSServer.DataFactory**, а также включить визуальные элементы управления легко использовать **записей** объектов, возвращаемых запросом из источника данных. Попыток служб удаленных рабочих СТОЛОВ, наиболее распространенный случай, сделать максимально автоматически получить доступ к данным на сервере, и ее отображение в элементе управления visual.  
  
 Объектная модель служб удаленных рабочих СТОЛОВ реализует функциональность, совместно с [RDS. DataControl](../../../ado/reference/rds-api/datacontrol-object-rds.md) объекта.  
  
 **RDS. DataControl** имеет два аспекта. Один аспект относится к источнику данных. Если выбрать команду, а также с помощью **Connect** и **SQL** свойства **RDS. DataControl**, будут автоматически использовать **RDS. Пространство данных** создать ссылку на значение по умолчанию **RDSServer.DataFactory** объекта. Затем **RDSServer.DataFactory** будет использовать **Connect** значение свойства для подключения к источнику данных, используйте **SQL** значение свойства для получения  **Набор записей** из источника данных и возврата **записей** объект **RDS. DataControl**.  
  
 Второй аспект относится для отображения возвращенного **записей** сведения в элементе управления visual. Вы можете связать визуальному элементу управления с **RDS. DataControl** (в процессе, называемом привязки) и получить доступ к данным в связанном **записей** объекта, отображение результатов запроса на веб-странице в Microsoft® Internet Explorer. Каждый **RDS. DataControl** один привязывается объект **записей** объект, представляющий результаты одного запроса, чтобы один или несколько визуальных элементов управления (например, текстовое поле, поле со списком, элемент управления DataGrid и т. д). Может быть более чем одного **RDS. DataControl** объекта на каждой странице. Каждый **RDS. DataControl** объект можно подключить к другой источник данных и содержит результаты отдельный запрос.  
  
 **RDS. DataControl** объекта также имеет собственные методы для перехода, сортировка и фильтрация строк связанного **записей** объекта. Эти методы похожи, но не таким, как методы в ADO **записей** объекта.  
  
## <a name="events"></a>События  
 Служба удаленных рабочих СТОЛОВ поддерживает две свои собственные события, которые не зависят от модели событий ADO. [OnReadyStateChange](../../../ado/reference/rds-api/onreadystatechange-event-rds.md) событие вызывается каждый раз, когда **RDS. DataControl** [ReadyState](../../../ado/reference/rds-api/readystate-property-rds.md) изменения свойств, тем самым выставленном успешного завершения асинхронной операции, остановлен или произошла ошибка. [OnError](../../../ado/reference/rds-api/onerror-event-rds.md) событий вызывается всякий раз, когда возникает ошибка, даже если произошла ошибка во время асинхронной операции.  
  
> [!NOTE]
>  Microsoft Internet Explorer предоставляет две дополнительные события для служб удаленных рабочих СТОЛОВ: **onDataSetChanged**, что означает, что **записей** функционально, оно не может быть получение строк, по-прежнему и  **onDataSetComplete**, что означает, что **записей** после получения строк.
