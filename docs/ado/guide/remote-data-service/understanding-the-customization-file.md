# <a name="understanding-the-customization-file"></a>Общие сведения о файле настроек
Заголовок каждого раздела в файле настройки состоит из квадратные скобки (**[]**) содержащий типом и параметром. Четыре раздела типы обозначаются строковых литералов **подключения**, **sql**, **userlist**, или **журналы**. Параметр является строковым литералом, значение по умолчанию, пользовательским идентификатором или ничего.  
  
> [!IMPORTANT]
>  Начиная с Windows 8 и Windows Server 2012, серверные компоненты служб удаленных рабочих СТОЛОВ, больше не включаются в операционной системе Windows (см. в разделе Windows 8 и [настольная книга по совместимости Windows Server 2012](https://www.microsoft.com/download/details.aspx?id=27416) для получения дополнительных сведений). Клиентские компоненты служб удаленных рабочих СТОЛОВ будет поддерживаться в будущих версиях Windows. Избегайте использования этого компонента в новых разработках и запланируйте изменение существующих приложений, в которых он применяется. Приложения, использующие служб удаленных рабочих СТОЛОВ, следует перевести [WCF-сервиса данных](https://go.microsoft.com/fwlink/?LinkId=199565).  
  
 Таким образом в каждом разделе помечен с помощью одного из следующих заголовков раздел:  
  
```console
  
[ connect default ] [ connect    
identifier   
] [ sql  default ] [ sql    
identifier   
] [ userlist    
identifier   
] [ logs ]  
  
```  
  
 Заголовки разделов состоять из следующих частей.  
  
|Часть|Описание|  
|----------|-----------------|  
|**connect**|Строковый литерал, который изменяет строку подключения.|  
|**sql**|Строковый литерал, который изменяет строку команды.|  
|**userlist**|Строковый литерал, который изменяет права доступа для конкретного пользователя.|  
|**Журналы**|Строковый литерал, который указывает файл журнала, запись ошибки из-за.|  
|**default**|Строковый литерал, который используется, если идентификатор не указан, либо найти.|  
|*Идентификатор*|Строка, соответствующая строка в **подключения** или **команда** строка.<br /><br /> -Используйте этот раздел, если заголовок раздела содержит **подключения** и идентификатор строка найдена в строке подключения.<br />-Используйте этот раздел, если заголовок раздела содержит **sql** и идентификатор строка найдена в командной строке.<br />-Используйте этот раздел, если заголовок раздела содержит **userlist** и соответствует строка идентификатора **подключения** идентификатор раздела.|  
  
 **DataFactory** вызывает обработчик исключений, передача параметров клиента. Обработчик выполняет поиск полных строк в параметрах клиента, которые соответствуют идентификаторы в заголовках соответствующий раздел. Если соответствие найдено, содержимое этого раздела применяются к параметр клиента.  
  
 Определенного раздела используется в следующих случаях:  
  
-   Объект **подключения** раздел используется в том случае, если ключевое слово строки подключения значение часть клиента "**источник данных =**_значение_«, соответствует **подключения** Идентификатор раздела *.*  
  
-   **Sql** раздел используется в том случае, если строка команды клиента содержит строку, соответствующую **sql** идентификатор раздела.  
  
-   Объект **подключения** или **sql** раздел с параметром по умолчанию используется в том случае, если совпадающий идентификатор.  
  
-   Объект **userlist** раздел используется в том случае, если **userlist** разделе совпадения идентификатор **подключения** идентификатор раздела. Если имеется соответствие, содержимое **userlist** разделе применяются к соединению, регулируется **подключения** раздел.  
  
-   Если строки в строку подключения и команды не совпадает с идентификатором в любом **подключения** или **sql** заголовка, при этом имеется не **подключения** или **sql**  раздел заголовка с параметром по умолчанию, то строка клиента используется без изменений.  
  
-   **Журналы** используются всякий раз, когда **DataFactory** в операции.
