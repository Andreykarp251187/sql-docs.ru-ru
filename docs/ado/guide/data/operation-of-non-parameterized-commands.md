# <a name="operation-of-non-parameterized-commands"></a>Использование непараметризованных команд
Для команд, которые содержат непараметризованные, выполняются все команды поставщика и **наборы записей** создаются во время выполнения команды. Если команда выполняется синхронно, все **наборы записей** будут полностью заполнены. Если был выбран режим проверки асинхронное заполнение, заполненный состояние **наборы записей** будет зависеть от режима заполнения и размера **наборы записей**.  
  
 Например *родительской команды* может вернуть **записей** клиентов компании из таблицы Customers и *дочерняя команда* может вернуть **Записей** заказов для всех клиентов из таблицы Orders.  
  
```  
SHAPE {SELECT * FROM Customers}   
   APPEND ({SELECT * FROM Orders} AS chapOrders   
   RELATE customerID TO customerID)  
```  
  
 Для непараметризованных родительско дочерних отношений, каждой родительской и дочерней **записей** объект должен быть столбец Общие, чтобы связать их. Столбцы имеют имена в предложении RELATE *родительский столбец* первого и затем *дочерний столбец*. Столбцы могут иметь другие имена в соответствующих **записей** объектов, но должен ссылаться на те же сведения, чтобы указать значимые связи. Например **клиентов** и **записей заказов** объектов может иметь поля customerID. Так как членство дочернего элемента **записей** определяется с помощью команды поставщика и дочерние **записей** может содержать появлению потерянных строк. Эти потерянные строки будут недоступны без дополнительных изменения формы.  
  
 Формирование данных добавляет столбец к родительскому **записей**. Значения в столбце Глава — ссылки на строки в дочерней **записей**, который удовлетворяет предложении RELATE. То есть, то же значение находится в *родительский столбец* заданный родительский объект строки в *дочерний столбец* всех строк Глава дочернего элемента. При использовании нескольких предложений TO в одном предложении RELATE они неявно объединяются с помощью оператора AND. Если родительскими столбцами в предложении RELATE не являются ключом к родительскому **записей**, строка один дочерний элемент может иметь несколько родительских строк.  
  
 Когда вы обращаетесь ссылку в столбце Глава, ADO автоматически извлекает **записей** представляемый ссылкой. Обратите внимание, что в непараметризованную команду, несмотря на то что весь дочерний **записей** была извлечена, глава показан только подмножество строк.  
  
 Если добавленный столбец не содержит *Глава alias*, имя будет создаваться для него автоматически. Объект [поле](../../../ado/reference/ado-api/field-object.md) объекта для столбца будут добавляться на **записей** объекта [поля](../../../ado/reference/ado-api/fields-collection-ado.md) коллекции, а тип данных будут показаны **adChapter**.  
  
 Сведения о навигации по иерархической **записей**, см. в разделе [доступ к строкам в иерархических наборах записей](../../../ado/guide/data/accessing-rows-in-a-hierarchical-recordset.md).
