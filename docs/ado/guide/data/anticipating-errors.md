# <a name="anticipating-errors"></a>Прогнозирование ошибок
Предотвращение ошибок по крайней мере столь важным, как обработка ошибок. В заключительном разделе содержит краткий список меры предосторожности, которые приложение может воспользоваться для ошибок, менее вероятно.  
  
 Проверьте состояние объектов путем проверки значения **состояние** свойство, прежде чем пытаться выполнить операцию с помощью этих объектов. Например, если приложение использует глобальный **подключения**, проверьте его **состояние** свойство, чтобы увидеть, если он еще открыт перед вызовом **откройте** метод.  
  
-   Любую программу, которая принимает данные от пользователя должен включать код для проверки данных перед их отправкой в хранилище данных. Нельзя полагаться на хранилище данных, поставщик, ADO или даже язык программирования, к уведомлению о проблемах. Необходимо проверить каждый байт, введенные пользователями, убедившись, что данные правильный тип поля и обязательные поля не являются пустыми.  
  
 Проверка данных, прежде чем пытаться записать данные в хранилище данных. Самый простой способ сделать это — для обработки **события WillMove** событий или **WillUpdateRecordset** событий. Более подробные сведения об обработке событий ADO, см. в разделе [обработка событий ADO](../../../ado/guide/data/handling-ado-events.md).  
  
 Убедитесь, что **записей** объекты не за пределами **записей** перед попыткой перемещения указателя записи. Если вы попытаетесь **MoveNext** при **EOF** имеет значение True или **MovePrev** при **BOF** имеет значение True, возникнет ошибка. Если выполнять какие-либо **переместить** методы при обоих **EOF** и **BOF** имеют значение True, будет сформирована ошибка.  
  
 Также возникнут ошибки при попытке выполнения операций, таких как **Seek** и **найти** для пустого **записей**.
