# <a name="records-and-streams"></a>Записи и потоки
В настоящее время предоставляет ADO [записей](../../../ado/reference/ado-api/recordset-object-ado.md) объект в качестве основного средства доступа к данным из источников данных, например реляционных баз данных. Тем не менее, некоторые поставщики поддерживают [записи](../../../ado/reference/ado-api/record-object-ado.md) и [Stream](../../../ado/reference/ado-api/stream-object-ado.md) объектов как альтернативные или дополнительные объекты, с которыми можно управлять данных от поставщиков. Подробные сведения о **записи** поведение, см. в документации поставщика.  
  
## <a name="records"></a>Число записей  
 **Запись** объектов функционировать как одну строку **записей**s. Тем не менее **записей** обладают ограниченным набором функций по сравнению с **наборы записей** и имеют различные свойства и методы. Источник данных в **записи** объект может быть команда, которая возвращает одну строку данных от поставщика. С помощью **записи** объектов вместо **записей** объектов для получения результатов из запроса, который возвращает одну строку данных к минимуму дополнительные издержки создания тем сложнее **набора записей**  объекта.  
  
 **Запись** объектов можно использовать в других целях, особенно с поставщиками для источников данных, отличный от традиционных реляционных баз данных, таких как [поставщик Microsoft OLE DB для публикаций в Интернете](../../../ado/guide/appendixes/microsoft-ole-db-provider-for-internet-publishing.md). Большая часть информации, которое необходимо обработать, не как таблиц в базах данных, а также как сообщения систем электронной почты и файлы в существует современных файловых систем. **Записи** и **Stream** объектов упрощают доступ к информации, хранящейся в источников, отличных от реляционных баз данных.  
  
 **Записи** объект может представлять и управления данными, таких как каталоги и файлы в файловой системе или папки и сообщения в системе электронной почты. Для этих целей, источником для **записи** может быть текущей строки открытого **записей**, абсолютный URL-адрес или относительный URL-адрес в сочетании с открытым подключением [подключения](../../../ado/reference/ado-api/connection-object-ado.md) объект.  
  
 Как правило **записей** может использоваться для представления контейнера или родителем в иерархии, например папки или каталога. Объект **записи** можно использовать для возвращения специфической информации о один узел в родительском контейнере, например файла или документа. Основная причина **записей** используются для представления данных этого типа является то, что разнородных эти источники данных. Это означает, что каждый **записи** может иметь собственный набор и количество полей. Традиционные **наборы записей** содержащего строки из базы данных являются однородные, что означает, что каждая строка имеет тот же номер и тип поля.  
  
 Дополнительные сведения об использовании **записи** объекта для обработки этого разнородных данных от поставщиков, например службу публикации в Интернете, см. в разделе [использование объектов ADO для публикаций в Интернете](../../../ado/guide/data/using-ado-for-internet-publishing.md).  
  
## <a name="streams"></a>Потоки  
 **Stream** объекта предоставляет средства для чтения, записи и управлять поток байтов. Этот поток байтов может быть текстовыми или двоичными и ограничивается размером только системные ресурсы. Как правило, ADO **Stream** объекты используются для следующих целей:  
  
-   Для хранения данных **записей** сохранен в формате XML. Сохранить эти XML-потоки из **записей**s может использоваться как источник, при открытии нового **записей**. Дополнительные сведения см. в разделе [потоки и сохраняемость](../../../ado/guide/data/streams-and-persistence.md).  
  
-   Должен содержать [CommandStreams](../../../ado/reference/ado-api/commandstream-property-ado.md) для выполнения к поставщику вместо [CommandText](../../../ado/reference/ado-api/commandtext-property-ado.md). Например диаграмм обновления XML можно использовать как источник для команды к поставщику Microsoft OLE DB для SQL Server.  
  
-   Для получения результатов от поставщика в формате, отличное от **записей**, например результатов XML от поставщика Microsoft OLE DB для SQL Server. Дополнительные сведения см. в разделе [извлечение результирующих наборов в потоки](../../../ado/guide/data/retrieving-resultsets-into-streams.md).  
  
-   Чтобы задать текст или байтов, составляющих файла или сообщения, обычно используется с поставщиками, такими как поставщик Microsoft OLE DB для публикаций в Интернете. Дополнительные сведения о такое использование **Stream** объектов, см. в разделе [использование объектов ADO для публикаций в Интернете](../../../ado/guide/data/using-ado-for-internet-publishing.md).  
  
 Объект **Stream** объекта можно открыть на:  
  
-   Простой файл, заданный параметром URL-адрес.  
  
-   Поле **записи** или **записей** содержащий **Stream** объекта.  
  
-   По умолчанию поток **записи** или **записей** объект, представляющий составного файла или каталога.  
  
-   Ресурс поле, содержащее URL-адрес простого файла.  
  
-   Нет определенного источника вообще. В этом случае **Stream** открыть объект в памяти. Данные могут быть записаны в него и затем сохраненные в другом **Stream** или файл.  
  
-   Поле BLOB-ОБЪЕКТОВ в **записей**.
