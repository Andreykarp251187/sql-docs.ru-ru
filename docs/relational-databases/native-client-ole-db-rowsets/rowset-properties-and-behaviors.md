# <a name="rowset-properties-and-behaviors"></a>Свойства и поведение наборов строк
[!INCLUDE[appliesto-ss-asdb-asdw-pdw-md](../../includes/appliesto-ss-asdb-asdw-pdw-md.md)]

  Это свойства [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщика rowset поставщика Native Client OLE DB.  
  
|Идентификатор свойства|Описание|  
|-----------------|-----------------|  
|DBPROP_ABORTPRESERVE|R/W: Читать/писать<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание: этим свойством определяется поведение набора строк после аварийного завершения операции.<br /><br /> VARIANT_FALSE: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщик Native Client OLE DB аннулирует строки после операции. Объект набора строк практически утратил свою функциональность. Он поддерживает только операции **IUnknown**, а также освобождение необработанных строк и обработчики методов доступа.<br /><br /> VARIANT_TRUE: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщик Native Client OLE DB поддерживает действительный набор строк.|  
|DBPROP_ACCESSORDER|R/W: Читать/писать<br /><br /> По умолчанию: DBPROPVAL_AO_RANDOM<br /><br /> Описание: Заказ доступа. Порядок, в котором должен осуществляться доступ к столбцам в наборе строк.<br /><br /> DBPROPVAL_AO_RANDOM: Колонка доступна в любом порядке.<br /><br /> DBPROPVAL_AO_SEQUENTIALSTORAGEOBJECTS: Столбцы, связанные как объекты хранения, могут быть доступны только в последовательном порядке, определяемом порядком столбца.<br /><br /> DBPROPVAL_AO_SEQUENTIAL: доступ ко всем столбцам должен осуществляться последовательно по порядковому номеру столбца.|  
|DBPROP_APPENDONLY|Это свойство rowset не [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] реализуется поставщиком Native Client OLE DB. При попытке считать или записать значение свойства возникает ошибка.|  
|DBPROP_BLOCKINGSTORAGEOBJECTS|R/W: Только для чтения<br /><br /> По умолчанию: VARIANT_TRUE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Объекты хранения поставщика данных Native Client OLE DB блокируются другими методами набора строк.|  
|DBPROP_BOOKMARKS DBPROP_LITERALBOOKMARKS|R/W: Читать/писать<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщик Native Client OLE DB поддерживает закладки для идентификации строки, когда DBPROP_BOOKMARKS или DBPROP_LITERALBOOKMARKS VARIANT_TRUE.<br /><br /> Присвоение любому из этих свойств значения VARIANT_TRUE не включает позиционирование набора строк по закладке. Чтобы создать набор строк, поддерживающий позиционирование по закладке, присвойте свойству DBPROP_IRowsetLocate или DBPROP_IRowsetScroll значение VARIANT_TRUE.<br /><br /> Поставщик [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB использует [!INCLUDE[msCoName](../../includes/msconame-md.md)] [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] курсор для поддержки строки, содержащей закладки. Дополнительные сведения см. в статье [Наборы строк и курсоры SQL Server](../../relational-databases/native-client-ole-db-rowsets/rowsets-and-sql-server-cursors.md).<br /><br /> Примечание: Установка этих свойств [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] в конфликт с другими свойствами, определяющими курсор поставщика Native Client OLE DB, вызывает ошибку. Например, при попытке потребителя открыть набор строк, свойства DBPROP_BOOKMARKS и DBPROP_OTHERINSERT которого имеют значение VARIANT_TRUE, будет выдана ошибка.|  
|DBPROP_BOOKMARKSKIPPED|R/W: Только для чтения<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщик Native Client OLE DB возвращается DB_E_BADBOOKMARK если потребитель указывает недействительную закладку при позиционировании или поиске закладки.|  
|DBPROP_BOOKMARKTYPE|R/W: Только для чтения<br /><br /> По умолчанию: DBPROPVAL_BMK_NUMERIC<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщик Native Client OLE DB реализует только числовые закладки. Закладка [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщика NATIVE Client OLE DB представляет собой 32-битную неподписанную бессвязную, ввод DBTYPE_UI4.|  
|DBPROP_CACHEDEFERRED|Это свойство rowset не [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] реализуется поставщиком Native Client OLE DB. При попытке считать или записать значение свойства возникает ошибка.|  
|DBPROP_CANFETCHBACKWARDS DBPROP_CANSCROLLBACKWARDS|R/W: Читать/писать<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщик Native Client OLE DB поддерживает обратную извлечения и прокрутку в непоследовательных наборах строк. Поставщик [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB создает набор с поддержкой курсора, когда VARIANT_TRUE DBPROP_CANFETCHBACKWARDS или DBPROP_CANSCROLLBACKWARDS. Дополнительные сведения см. в статье [Наборы строк и курсоры SQL Server](../../relational-databases/native-client-ole-db-rowsets/rowsets-and-sql-server-cursors.md).|  
|DBPROP_CANHOLDROWS|R/W: Читать/писать<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание: По умолчанию поставщик [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB возвращается DB_E_ROWSNOTRELEASED если потребитель пытается получить больше строк для строки, пока ожидаются изменения на тех, которые в настоящее время находятся в строке. Это поведение может быть изменено.<br /><br /> Присвоение свойствам DBPROP_CANHOLDROWS и DBPROP_IRowsetChange значения VARIANT_TRUE подразумевает набор строк с закладками. Если оба свойства имеют значение VARIANT_TRUE, то для набора строк доступен интерфейс **IRowsetLocate**, а свойства DBPROP_BOOKMARKS и DBPROP_LITERALBOOKMARKS имеют значение VARIANT_TRUE.<br /><br /> [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]Ряды поставщиков услуг Native Client OLE DB, содержащие закладки, поддерживаются [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] курсорами.|  
|DBPROP_CHANGEINSERTEDROWS|R/W: Читать/писать<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание: это свойство может иметь только значение VARIANT_TRUE, если набор строк использует курсор, управляемый набором ключей.|  
|DBPROP_COLUMNRESTRICT|R/W: Только для чтения<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщик Native Client OLE DB устанавливает свойство для VARIANT_TRUE, когда столбец в строке не может быть изменен потребителем. Другие столбцы в наборе строк можно изменять, а сами строки — удалять.<br /><br /> Если это свойство имеет значение VARIANT_TRUE, то потребитель по элементу *dwFlags* структуры DBCOLUMNINFO определяет, может ли быть записано значение отдельного столбца. Для изменяемых столбцов элемент *dwFlags* принимает значение DBCOLUMNFLAGS_WRITE.|  
|DBPROP_COMMANDTIMEOUT|R/W: Читать/писать<br /><br /> По умолчанию: 0<br /><br /> Описание: По умолчанию поставщик [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB не тайм-аут на **ICommand::Execute** метод.|  
|DBPROP_COMMITPRESERVE|R/W: Читать/писать<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание: это свойство определяет поведение набора строк после выполнения фиксации.<br /><br /> VARIANT_TRUE: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщик Native Client OLE DB поддерживает действительный набор строк.<br /><br /> VARIANT_FALSE: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщик Native Client OLE DB аннулирует строки после операции коммита. Объект набора строк практически утратил свою функциональность. Он поддерживает только операции **IUnknown**, а также освобождение необработанных строк и обработчики методов доступа.|  
|DBPROP_DEFERRED|R/W: Читать/писать<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание: При установке [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] на VARIANT_TRUE поставщик NATIVE Client OLE DB пытается использовать курсор сервера для набора строк. Столбцы типа **text**, **ntext** и **image** не возвращаются с сервера до тех пор, пока приложение не обратится к ним.|  
|DBPROP_DELAYSTORAGEOBJECTS|R/W: Только для чтения<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщик Native Client OLE DB поддерживает режим немедленного обновления объектов хранения данных.<br /><br /> Изменения, примененные к данным в объекте последовательного потока, немедленно передаются [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]. Фиксация изменений производится в соответствии с режимом транзакций набора строк.|  
|DBPROP_HIDDENCOLUMNS|R/W: Только для чтения<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> **Описание:** Скрытая колонна граф<br /><br /> Если DBPROP_UNIQUEROWS имеет значение VARIANT_TRUE, то свойство DBPROP_HIDDENCOLUMNS возвращает число дополнительных «скрытых» столбцов, добавленных поставщиком для уникальной идентификации строк в наборе. Эти столбцы возвращаются методами **IColumnsInfo::GetColumnInfo** и **IColumnsRowset::GetColumnsRowset**. Однако они не включаются в число строк, которое возвращается в аргументе *pcColumns* метода **IColumnsInfo::GetColumnInfo**.<br /><br /> Чтобы выяснить общее число столбцов, представленных в структуре *prgInfo*, которую возвращает метод **IColumnsInfo::GetColumnInfo** (включая скрытые), потребитель должен добавить значение свойства DBPROP_HIDDENCOLUMNS к числу столбцов, возвращаемых методом **IColumnsInfo::GetColumnInfo** в *pcColumns*. Если свойство DBPROP_UNIQUEROWS имеет значение VARIANT_FALSE, то свойство DBPROP_HIDDENCOLUMNS равно нулю.|  
|DBPROP_IAccessor DBPROP_IColumnsInfo DBPROP_IConvertType DBPROP_IRowset DBPROP_IRowsetInfo|R/W: Только для чтения<br /><br /> По умолчанию: VARIANT_TRUE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщик Native Client OLE DB поддерживает эти интерфейсы на всех строках.|  
|DBPROP_IColumnsRowset|R/W: Читать/писать<br /><br /> По умолчанию: VARIANT_TRUE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщик Native Client OLE DB поддерживает интерфейс **IColumnsRowset.**|  
|DBPROP_IConnectionPointContainer|R/W: Читать/писать<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание: IConnectionPointContainer. Если значение равно VARIANT_TRUE, то набор строк поддерживает заданный интерфейс. Если значение равно VARIANT_FALSE, то набор строк не поддерживает заданный интерфейс. Поставщики, поддерживающие тот или иной интерфейс, должны поддерживать связанное с этим интерфейсом свойство, которое возвращает значение VARIANT_TRUE. Эти свойства служат главным образом для запроса интерфейсов с помощью метода ICommandProperties::SetProperties.|  
|DBPROP_IMultipleResults|R/W: Читать/писать<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщик Native Client OLE DB поддерживает интерфейс **IMultipleResults.**|  
|DBPROP_IRowsetChange DBPROP_IRowsetUpdate|R/W: Читать/писать<br /><br /> По умолчанию: VARIANT_FALSE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщик Native Client OLE DB поддерживает интерфейсы **IRowsetChange** и **IRowsetUpdate.**<br /><br /> Набор строк, созданный с DBPROP_IRowsetChange, равным VARIANT_TRUE, отражает режимы немедленного обновления.<br /><br /> Если DBPROP_IRowsetUpdate имеет значение VARIANT_TRUE, то DBPROP_IRowsetChange также имеет значение VARIANT_TRUE. Набор строк характеризуется поведением отложенного обновления.<br /><br /> Поставщик [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB использует [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] курсор для поддержки рядов, разоблачающих либо **IRowsetChange,** либо **IRowsetUpdate.** Дополнительные сведения см. в статье [Наборы строк и курсоры SQL Server](../../relational-databases/native-client-ole-db-rowsets/rowsets-and-sql-server-cursors.md).|  
|DBPROP_IRowsetIdentity|R/W: Читать/писать<br /><br /> По умолчанию: VARIANT_TRUE<br /><br /> Описание: [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Поставщик Native Client OLE DB поддерживает интерфейс **IRowsetIdentity.** Если набор строк поддерживает этот интерфейс, то любые два дескриптора строк, относящихся к одной базовой строке, всегда представляют одни и те же данные и состояние. Потребители могут вызвать **IRowsetIdentity:: Метод IsSameRow** для сравнения двух строк, чтобы увидеть, если они относятся к одному экземпляру строки.|

Поставщик [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB устанавливает DBPROP_BOOKMARKS и DBPROP_LITERALBOOKMARKS VARIANT_TRUE, когда потребитель запрашивает интерфейс.<br /><br /> Поставщик [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] DB использует курсоры для поддержки **IRowsetLocate** и **IRowsetScroll.** Дополнительные сведения см. в статье [Наборы строк и курсоры SQL Server](../../relational-databases/native-client-ole-db-rowsets/rowsets-and-sql-server-cursors.md).<br /><br /> Установка этих свойств в [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] конфликт с другими свойствами, определяющими курсор поставщика Native Client OLE DB, вызывает ошибку. Например, при попытке потребителя открыть набор строк, свойства DBPROP_IRowsetScroll и DBPROP_OTHERINSERT которого имеют значение VARIANT_TRUE, будет выдана ошибка.|

Поставщик [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Native Client OLE DB может выставить интерфейс на любом наборе строк.|

|DBPROP_IStream|Этот интерфейс не реализован [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] поставщиком Native Client OLE DB. При попытке считать или записать свойство возникает ошибка.|  
|DBPROP_IMMOBILEROWS|R/W: Читать/писать<br /><br /> По умолчанию: VARIANT_TRUE<br /><br /> Описание: это свойство имеет значение VARIANT_TRUE только для тех курсоров, которые управляются набором ключей [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]. Для всех остальных курсоров оно имеет значение VARIANT_FALSE.<br /><br /> VARIANT_TRUE: набор строк не будет переупорядочивать вставленные или обновленные строки. Для метода **IRowsetChange::InsertRow** строки появляются в конце набора. Если набор строк не упорядочен, то метод **IRowsetChange::SetData** не меняет положение обновленных строк. Если набор строк упорядочен, то при изменении методом **IRowsetChange::SetData** столбца, который использовался для упорядочивания набора строк, строка не перемещается. Если набор строк построен на наборе ключевых столбцов (обычно это столбцы, свойство DBPROP_OTHERUPDATEDELETE которых имеет значение VARIANT_TRUE, а свойство DBPROP_OTHERINSERT — значение VARIANT_FALSE), то изменение значения ключевого столбца, как правило, эквивалентно удалению текущей строки вставке новой. Поэтому строка может переместиться или даже исчезнуть из набора строк, если свойство DBPROP_OWNINSERT имеет значение VARIANT_FALSE, даже со значением свойства DBPROP_IMMOBILEROWS, равным VARIANT_TRUE.<br /><br /> VARIANT_FALSE: если набор строк упорядочен, то вставленные строки находятся в наборе строк в правильном порядке. Если набор строк не является упорядоченным, то вставленные строки появляются в конце. Если метод **IRowsetChange::SetData** изменяет столбец, который использовался для упорядочивания набора строк, то строка будет перемещена.
