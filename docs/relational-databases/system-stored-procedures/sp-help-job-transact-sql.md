# <a name="sp_help_job-transact-sql"></a>sp_help_job (Transact-SQL)
[!INCLUDE[tsql-appliesto-ss2008-xxxx-xxxx-xxx-md](../../includes/tsql-appliesto-ss2008-xxxx-xxxx-xxx-md.md)]

  Возвращает сведения о заданиях, используемых агентом [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] для выполнения автоматических действий в [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)].  
  
 
 ![Значок ссылки на раздел](../../database-engine/configure-windows/media/topic-link.gif "Значок ссылки на раздел") [Синтаксические обозначения в Transact-SQL](../../t-sql/language-elements/transact-sql-syntax-conventions-transact-sql.md)  
  
## <a name="syntax"></a>Синтаксис  
  
```  
  
sp_help_job { [ @job_id = ] job_id  
[ @job_name = ] 'job_name' }   
     [ , [ @job_aspect = ] 'job_aspect' ]   
     [ , [ @job_type = ] 'job_type' ]   
     [ , [ @owner_login_name = ] 'login_name' ]   
     [ , [ @subsystem = ] 'subsystem' ]   
     [ , [ @category_name = ] 'category' ]   
     [ , [ @enabled = ] enabled ]   
     [ , [ @execution_status = ] status ]   
     [ , [ @date_comparator = ] 'date_comparison' ]   
     [ , [ @date_created = ] date_created ]   
     [ , [ @date_last_modified = ] date_modified ]   
     [ , [ @description = ] 'description_pattern' ]  
```  
  
## <a name="arguments"></a>Аргументы  
`[ @job_id = ] job_id`Идентификационный номер задания. *job_id* является **уникальнымиии,** с дефолтом NULL.  
  
`[ @job_name = ] 'job_name'`Название работы. *job_name* является **sysname,** с дефолтом NULL.  
  
> [!NOTE]  
>  Для просмотра конкретного задания необходимо указать *job_id* или *job_name.*  Опустить как *job_id,* так и *job_name,* чтобы вернуть информацию обо всех рабочих мест.
  
`[ @job_aspect = ] 'job_aspect'`Атрибут выполнения для отображения. *job_aspect* **varchar(9)**, с дефолтом NULL, и может быть одним из этих значений.  
  
|Значение|Описание|  
|-----------|-----------------|  
|**ALL**|Сведения об аспекте задания|  
|**Работу**|Сведения о задании|  
|**Расписание**|Сведения о расписании|  
|**Шаги**|Сведения об шаге задания|  
|**Цели**|Сведения о цели|  
  
`[ @job_type = ] 'job_type'`Тип заданий для включения в отчет. *job_type* **varchar (12)**, с дефолтом NULL. *job_type* может быть **LOCAL** или **MULTI-SERVER**.  
  
`[ @owner_login_name = ] 'login_name'`Имя владельца задания. *login_name* является **sysname**, с дефолтом NULL.  
  
`[ @subsystem = ] 'subsystem'`Название подсистемы. *подсистема* **nvarchar (40)**, с дефолтом NULL.  
  
`[ @category_name = ] 'category'`Название категории. *категория* **sysname**, с дефолтом NULL.  
  
`[ @enabled = ] enabled`Номер, указывающий, отображается ли информация для включенных заданий или неработающих. *включен* **крошечный**, с по умолчанию NULL. **1** указывает на включенные задания, а **0** — на неработающие задания.  
  
`[ @execution_status = ] status`Состояние выполнения заданий. *статус* **int,** с дефолтом NULL, и может быть одним из этих значений.  
  
|Значение|Описание|  
|-----------|-----------------|  
|**0**|Возвращает только те задания, которые не находятся в состоянии бездействия или приостановки.|  
|**1**|Выполняющиеся.|  
|**2**|Ожидающие потока.|  
|**3**|Ожидающие повторной попытки.|  
|**4**|Бездействующие.|  
|**5**|Приостановленные.|  
|**7**|Выполняющие завершающие действия.|  
  
`[ @date_comparator = ] 'date_comparison'`Сравнительный оператор для использования в сравнении *date_created* и *date_modified*. *date_comparison* **является символом (1)** и \<может быть q, или >.  
  
`[ @date_created = ] date_created`Дата создания задания. *date_created*является **дата**, с дефолтом NULL.  
  
`[ @date_last_modified = ] date_modified`Дата последнего изменения задания. *date_modified* является **дата**, с дефолтом NULL.  
  
`[ @description = ] 'description_pattern'`Описание задания. *description_pattern* **nvarchar (512)**, с дефолтом NULL. *description_pattern* могут включать в себя символы подстановочных знаков сервера S'L Server для сопоставления шаблонов.  
  
## <a name="return-code-values"></a>Значения кода возврата  
 **0** (успех) или **1** (неудача)

## <a name="permissions"></a>Разрешения  
 По умолчанию эту хранимую процедуру могут выполнять только члены предопределенной роли сервера **sysadmin** . Другим пользователям должна быть предоставлена одна из следующих предопределенных ролей базы данных агента [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] в базе данных **msdb** :  
  
-   **SQLAgentUserRole**  
  
-   **SQLAgentReaderRole**  
  
-   **SQLAgentOperatorRole**  
  
 Дополнительные сведения о разрешениях этих ролей см. в разделе [Предопределенные роли базы данных агента SQL Server](../../ssms/agent/sql-server-agent-fixed-database-roles.md).  
  
 Участники **группы s'LAgentUserRole** могут просматривать только те задания, которыми они владеют. Участники **sysadmin**, **S'LAgentReaderRole**и **S'LAgentOperatorRole** могут просматривать все локальные и многофункциональные задания.  
  
## <a name="examples"></a>Примеры  
  
### <a name="a-list-information-for-all-jobs"></a>A. Вывод списка сведений обо всех заданиях  
 На следующем примере показано, как процедура `sp_help_job`, выполняемая без параметров, возвращает сведения обо всех заданиях, определенных в данный момент в базе данных `msdb`.  
  
```  
USE msdb ;  
GO  
  
EXEC dbo.sp_help_job ;  
GO  
```  
  
### <a name="b-listing-information-for-jobs-matching-a-specific-criteria"></a>Б. Вывод сведений о заданиях, совпадающих с определенным критерием  
 В следующем примере показано, как выводятся сведения о многосерверных заданиях, владельцем которых является `françoisa`, которые включены и выполняются.  
  
```  
USE msdb ;  
GO  
  
EXEC dbo.sp_help_job   
   @job_type = N'MULTI-SERVER',  
   @owner_login_name = N'françoisa',  
   @enabled = 1,  
   @execution_status = 1 ;  
GO  
```  
  
### <a name="c-listing-all-aspects-of-information-for-a-job"></a>В. Вывод всех аспектов сведений о задании
