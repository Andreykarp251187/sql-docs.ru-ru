# <a name="manage-sql-server-big-data-clusters-with-azure-data-studio-notebooks"></a>Управление SQL Server кластерами больших данных с помощью Azure Data Studio записных книжек

[!INCLUDE[tsql-appliesto-ssver15-xxxx-xxxx-xxx](../includes/tsql-appliesto-ssver15-xxxx-xxxx-xxx.md)]

[!INCLUDE[sql-server-2019](../includes/sssqlv15-md.md)] предоставляет расширение для Azure Data Studio, включающее в себя записные книжки. Записная книжка содержит документацию и код, которые можно использовать в Azure Data Studio для управления кластерами больших данных SQL Server 2019.

Изначально реализована как проект с открытым исходным кодом, [записные книжки](notebooks-guidance.md) были включены в [Azure Data Studio](https://docs.microsoft.com/sql/azure-data-studio/download). Вы можете использовать разметку Markdown для текста в текстовых ячейках и одно из доступных ядер для написания кода в ячейках кода.

С помощью записных книжках можно развертывать кластеры больших данных для [!INCLUDE[sql-server-2019](../includes/sssqlv15-md.md)].

В дополнение к записным книжкам можно просмотреть набор записных книжек, который называется книгой Jupyter. Книга Jupyter содержит оглавление, помогающее перемещаться по коллекции записных книжек, чтобы можно было легко найти нужную записную книжку, нужно ли устранять неполадки SQL Server или просматривать состояние кластера.

## <a name="prerequisites"></a>Предварительные требования

Для открытия записной книжки необходимы следующие компоненты:

* Последняя версия [сборки Azure Data Studio для участников программы предварительной оценки](https://aka.ms/azuredatastudio-rc)
* Расширение [!INCLUDE[sql-server-2019](../includes/sssqlv15-md.md)], установленное в Azure Data Studio

В дополнение к этим предварительным требованиям для развертывания SQL Server 2019 кластеры больших данных вам также потребуется:

* [azdata](deploy-install-azdata.md)
* [kubectl](https://kubernetes.io/docs/tasks/tools/install-kubectl/#install-kubectl-binary-using-native-package-management)
* [Azure CLI](/cli/azure/install-azure-cli)

## <a name="access-troubleshooting-notebooks"></a>Устранение неполадок в записных книжках
Существует три способа доступа к записным книжкам для устранения неполадок.

### <a name="command-palette"></a>Палитра команд
1. Выберите **View** > **Палитра команд**.
2. Введите @no__t книги 0Jupyter: SQL Server 2019 Guide @ no__t-0.

Jupyter книги проводника с книгой Jupyter, содержащей записные книжки для устранения неполадок, связанные с SQL Server кластерами больших данных.

### <a name="sql-master-dashboard"></a>Главная панель мониторинга SQL
1. После установки Azure Data Studio предварительной оценки подключитесь к экземпляру SQL Server кластера больших данных.
2. После подключения к экземпляру щелкните правой кнопкой мыши имя сервера в разделе **подключения** и выберите пункт **Управление**.
3. На панели мониторинга выберите **SQL Server кластер больших данных**. Щелкните **руководство по SQL Server 2019** , чтобы открыть книгу Jupyter, содержащую нужные записные книжки.
    записные книжки @no__t 0Jupyter на панели мониторинга @ no__t-1

1. Выберите записную книжку для задачи, которую необходимо завершить.

### <a name="controller-dashboard"></a>Панель мониторинга контроллера
1. В представлении **подключения** разверните узел **SQL Server кластеры больших данных**.
2. Добавьте сведения о конечной точке контроллера.
3. После подключения к контроллеру щелкните правой кнопкой мыши конечную точку и выберите пункт **Управление**.
4. После загрузки панели мониторинга выберите **Устранение неполадок** , чтобы открыть руководства по устранению неполадок в книге Jupyter.

## <a name="use-troubleshooting-notebooks"></a>Использование для устранения неполадок записных книжек
1. Найдите руководство по устранению неполадок, которое требуется в содержании книги Jupyter.
1. Записные книжки оптимизированы, поэтому нужно просто выбрать **Run Cells (запустить ячейки**). Это действие будет выполнять каждую ячейку в записной книжке по отдельности до завершения записной книжки.
1. При обнаружении ошибки в книге Jupyter предлагается Записная книжка, которую можно запустить для устранения этой ошибки. Выполните рекомендуемые действия и снова запустите записную книжку.
