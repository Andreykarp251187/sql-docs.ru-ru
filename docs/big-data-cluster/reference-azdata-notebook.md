# <a name="azdata-notebook"></a>Записная книжка аздата

[!INCLUDE[tsql-appliesto-ssver15-xxxx-xxxx-xxx](../includes/tsql-appliesto-ssver15-xxxx-xxxx-xxx.md)]

В следующей статье содержится справочник по командам **записной книжки** в средстве **аздата** . Дополнительные сведения о других командах **аздата** см. в разделе [Справочник по аздата](reference-azdata.md) .

## <a name="commands"></a>Команды
|     |     |
| --- | --- |
[представление записной книжки аздата](#azdata-notebook-view) | Просмотр записной книжки.  Параметр, который должен останавливаться при первой ошибке выполнения ячейки.
[Запуск записной книжки аздата](#azdata-notebook-run) | Запустите записную книжку.  Выполнение останавливается при первой ошибке.
## <a name="azdata-notebook-view"></a>представление записной книжки аздата
Эта команда может взять файл записной книжки и преобразовать Markdown, код и выходные данные в формат цветового терминала.
```bash
azdata notebook view --path -p 
                     [--continue-on-error -c]
```
### <a name="examples"></a>Примеры
Просмотр записной книжки.  Отобразятся все ячейки.
```bash
azdata notebook view --path '/home/me/notebooks/demo_notebook.ipynb'
```
Просмотр записной книжки.  В результате будут показаны все ячейки, если не будет обнаружена ячейка с ошибкой в выходных данных.  В этом случае выходные данные останавливаются.
```bash
azdata notebook view --path '/home/me/notebooks/demo_notebook.ipynb' --stop-on-error
```
### <a name="required-parameters"></a>Обязательные параметры
#### `--path -p`
Путь к записной книжке для просмотра.
### <a name="optional-parameters"></a>Необязательные параметры
#### `--continue-on-error -c`
Продолжить отображение дополнительных ячеек без учета ошибок ячеек, найденных в выходных данных записной книжки.  Поведение по умолчанию — останавливаться при ошибке.  Остановка позволяет увидеть первую ячейку, в которой произошла ошибка.
### <a name="global-arguments"></a>Глобальные аргументы
#### `--debug`
Увеличьте уровень детализации ведения журнала, чтобы отобразить все журналы отладки.
#### `--help -h`
Отображение этого справочного сообщения и выход.
#### `--output -o`
Формат вывода.  Допустимые значения: JSON, jsonc, Table, TSV.  По умолчанию: JSON.
#### `--query -q`
Строка запроса JMESPath. Дополнительные [http://jmespath.org/](http://jmespath.org/]) сведения и примеры см. в разделе.
#### `--verbose`
Увеличение уровня детализации ведения журнала. Используйте параметр--Debug для полных журналов отладки.
## <a name="azdata-notebook-run"></a>Запуск записной книжки аздата
Эта команда создает временный каталог и выполняет заданную в ней записную книжку в качестве рабочего каталога.
```bash
azdata notebook run --path -p 
                    [--output-path]  
                    [--output-html]
```
### <a name="examples"></a>Примеры
Запустите записную книжку.
```bash
azdata notebook run --path '/home/me/notebooks/demo_notebook.ipynb'
```
### <a name="required-parameters"></a>Обязательные параметры
#### `--path -p`
Путь к файлу для запуска записной книжки.
### <a name="optional-parameters"></a>Необязательные параметры
#### `--output-path`
Путь к каталогу, используемый для выходных данных записной книжки.  Записная книжка с выходными данными и файлами, созданными записной книжкой, создается относительно этого каталога.
#### `--output-html`
Необязательный флаг. индикатингг, следует ли дополнительно преобразовать выходную записную книжку в формат HTML.  Создает второй выходной файл.
### <a name="global-arguments"></a>Глобальные аргументы
#### `--debug`
Увеличьте уровень детализации ведения журнала, чтобы отобразить все журналы отладки.
#### `--help -h`
Отображение этого справочного сообщения и выход.
#### `--output -o`
Формат вывода.  Допустимые значения: JSON, jsonc, Table, TSV.  По умолчанию: JSON.
#### `--query -q`
Строка запроса JMESPath. Дополнительные [http://jmespath.org/](http://jmespath.org/]) сведения и примеры см. в разделе.
#### `--verbose`
Увеличение уровня детализации ведения журнала. Используйте параметр--Debug для полных журналов отладки.
